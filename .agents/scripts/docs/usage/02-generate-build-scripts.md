# 生成与构建脚本使用说明

本文档描述 `.agents/scripts/` 目录下所有生成（generate）、构建（build）、收尾类脚本的用法。

---

## generate-nav.py

扫描 `docs/` 目录下的所有 `.md` 文件，提取标题和描述，自动生成文档导航表，并更新 `README.md` 和 `docs/README.md` 中 `<!-- NAV_TABLE_START -->` 与 `<!-- NAV_TABLE_END -->` 标记之间的内容。

```bash
# 自动生成并更新导航表
python .agents/scripts/generate-nav.py
```

---

## generate-dashboard.py

扫描 `.trae/specs/` 目录下所有 Spec 的 `tasks.md`，自动聚合各主题和 Spec 的完成状态，更新根 `README.md` 中 `<!-- SPEC_DASHBOARD_START -->` 与 `<!-- SPEC_DASHBOARD_END -->` 标记之间的执行进度看板。

判定规则：
1. 优先读取 TOML/YAML frontmatter 中的 `status` 字段，若为 `completed`/`done`/`finished` 则视为已完成
2. 否则检查是否存在未勾选的复选框（`- [ ]` 或 `## [ ]`），无未勾选项则视为已完成
3. 自动跳过代码块中的复选框（避免误判示例代码）

```bash
# 自动扫描并更新 Spec 执行进度看板
python .agents/scripts/generate-dashboard.py
```

---

## finalize-atomization.py

原子化操作一键收尾脚本。在文档/代码原子化拆分、文件移动、目录重构等操作完成后运行，一键执行断链自动修复、导航表更新、Spec 看板刷新等后处理工作，确保原子化完成后项目状态一致。

执行步骤：
1. **自动断链修复**：调用 link_fixer 扫描全项目，自动修复相对路径层级错误、绝对路径转换、目录尾部斜杠补全等
2. **导航表更新**：运行 generate-nav.py 刷新文档导航表
3. **Spec 看板更新**：运行 generate-dashboard.py 刷新 Spec 执行进度看板

```bash
# 完整后处理（实际执行修复）
python .agents/scripts/finalize-atomization.py

# 预览模式，不修改文件
python .agents/scripts/finalize-atomization.py --dry-run

# 跳过链接修复
python .agents/scripts/finalize-atomization.py --no-links

# 指定目标目录（仅修复特定子树）
python .agents/scripts/finalize-atomization.py --target docs/retrospective/
```

## build-ref-index.py

构建文件引用反向索引。扫描项目中所有 Markdown 文件提取本地相对链接，建立 `{目标文件: [引用文件列表]}` 的反向映射。在文件移动、删除、大规模重构前，可快速查询受影响的引用方，避免断链遗漏。

功能特性：
- 自动跳过代码块内的示例链接
- 支持文件查询、目录查询、孤立文件检测
- 支持 JSON 格式输出（便于与其他工具集成）
- 自动排除 `.git/`、`vendor/`、`node_modules/` 等目录

```bash
# 构建索引并显示统计摘要 + Top 被引用文件
python .agents/scripts/build-ref-index.py

# 查询哪些文件引用了指定文件（移动前查询影响面）
python .agents/scripts/build-ref-index.py --query AGENTS.md

# 批量查询多个文件
python .agents/scripts/build-ref-index.py --query file1.md path/to/file2.md

# 查询哪些文件引用了指定目录下的任意文件
python .agents/scripts/build-ref-index.py --query-dir docs/retrospective/patterns/

# 显示被引用次数最多的 Top 20 文件
python .agents/scripts/build-ref-index.py --top 20

# 列出未被任何文件引用的孤立文件
python .agents/scripts/build-ref-index.py --orphans

# JSON 格式输出（便于工具集成）
python .agents/scripts/build-ref-index.py --query README.md --json
```

## generate-readme.py

目录 README 自动生成与索引增量更新工具。为 `docs/` 下缺失 README.md 的内容聚合目录批量生成标准化入口文档，并支持通过标记区域（marker region）对已有 README 的索引表进行增量更新，保留用户手动编辑的概述、相关资源等内容。

### 核心概念：标记区域机制

脚本生成的 README 采用三区域结构，通过 HTML 注释标记划分自动维护区和用户编辑区：

```markdown
---
(YAML frontmatter — 自动生成，不建议手动修改)
---

# 标题
> 说明文字
（用户自定义概述内容）← 📝 用户可自由编辑，--update 时保留

<!-- README_INDEX_START -->
## 📄 文档索引          ← 🔄 自动维护：当前目录 .md 文件索引表
| 文档 | 说明 | 标签 |
|------|------|------|
| ...  | ...  | ...  |

## 📁 子目录导航        ← 🔄 自动维护：子目录状态导航表
| 子目录 | 文档数 | 说明 |
|--------|--------|------|
| ...    | ...    | ...  |
<!-- README_INDEX_END -->

## 🔗 相关资源        ← 📝 用户可追加内容，基础链接保留
- [🏠 返回上级](../README.md)
- [📚 文档首页](../../README.md)

<!-- generated by generate-readme.py on YYYY-MM-DD -->
```

标记说明：
- `<!-- README_INDEX_START -->` / `<!-- README_INDEX_END -->`：索引区域边界，`--update` 仅替换此区域内内容
- `<!-- generated by generate-readme.py on ... -->`：脚本生成标识页脚，用于识别哪些 README 可被 `--update` 管理，同时记录最后更新日期
- 📋/✅ 标记：子目录导航中，📋 表示子目录无 README，✅ 表示子目录已有 README

### 命令参考

```bash
# ===== 扫描 =====
# 扫描 docs/ 下所有缺失 README 的目录（含至少1个.md文件）
python .agents/scripts/generate-readme.py --scan

# ===== 生成（首次创建 README）=====
# 为指定目录生成 README
python .agents/scripts/generate-readme.py --target-dir docs/knowledge/my-topic

# 批量处理 P1 阶段内容聚合目录
python .agents/scripts/generate-readme.py --p1

# 处理所有缺失 README 的目录
python .agents/scripts/generate-readme.py --all

# 强制覆盖已存在的 README（谨慎使用）
python .agents/scripts/generate-readme.py --target-dir docs/path --force

# ===== 增量更新（已有 README 的索引刷新）=====
# 增量更新所有脚本生成的 README 索引表（最常用！）
python .agents/scripts/generate-readme.py --update

# 更新指定目录的索引
python .agents/scripts/generate-readme.py --update --target-dir docs/knowledge/my-topic

# ===== 预览模式（所有命令均可加 --dry-run）=====
python .agents/scripts/generate-readme.py --update --dry-run
python .agents/scripts/generate-readme.py --target-dir docs/path --dry-run

# ===== 通用选项 =====
# --path <root>     指定项目根目录（默认自动解析）
# --title <title>   自定义目录标题（单目录生成模式）
# --min-files <n>   最小md文件数阈值（仅对 --all/--scan 生效，默认1）
# --dry-run         预览模式，不写入文件
# --force           生成模式下强制覆盖已有 README
```

### 典型工作流

#### 场景1：新增内容目录

当在 `docs/` 下新建一个子目录并放入 `.md` 文件后：

```bash
# 1. 为新目录生成 README
python .agents/scripts/generate-readme.py --target-dir docs/knowledge/new-topic

# 2. 刷新所有父目录的索引表（自动识别新子目录，状态从📋变为✅）
python .agents/scripts/generate-readme.py --update
```

#### 场景2：现有目录新增/删除 .md 文件

当在已有 README 的目录中新增或删除 `.md` 文件后：

```bash
# 一键刷新所有索引表，自动同步文档索引和子目录导航
python .agents/scripts/generate-readme.py --update
```

脚本会自动：
- 将新文件加入「📄 文档索引」表（提取 frontmatter 的 title/summary/tags/maturity）
- 将删除的文件从索引表移除
- 识别新增的子目录（显示为📋）
- 更新子目录的 README 存在状态（📋→✅）
- 不修改标记区域外的用户内容

#### 场景3：全量补全（首次使用/大规模整理后）

```bash
# 1. 先扫描查看缺失情况
python .agents/scripts/generate-readme.py --scan

# 2. 批量生成所有缺失的 README
python .agents/scripts/generate-readme.py --all

# 3. 为旧版无标记 README 升级标记（首次运行会自动升级）
python .agents/scripts/generate-readme.py --update
```

#### 场景4：CI 门禁集成

在 CI 流水线中，可在文档变更后运行检查：

```bash
# 检查是否有目录缺失 README
python .agents/scripts/generate-readme.py --scan
# 若输出数量>0，说明存在未补全 README 的目录，CI 应失败

# 检查索引是否最新（运行 --update 后检查 git diff）
python .agents/scripts/generate-readme.py --update
git diff --exit-code docs/
# 若有差异，说明索引未更新，CI 应失败并提示开发者运行 --update
```

### 中文标题映射

脚本通过 `DIR_TITLE_MAP` 字典将目录名映射为中文友好标题。新增中文目录名时，需在脚本的 `DIR_TITLE_MAP` 中添加映射：

```python
DIR_TITLE_MAP: dict[str, str] = {
    "my-new-topic": "我的新主题",
    # ...
}
```

未添加映射的目录名将通过 `humanize_dirname()` 自动转换（`my-new-topic` → `My New Topic`）。

### 自动升级旧版 README

对于在标记功能上线前生成的 README（有 `<!-- generated by generate-readme.py -->` 页脚但无 START/END 标记），首次运行 `--update` 时会自动升级：
- 在「📄 文档索引」标题前插入 `README_INDEX_START` 标记
- 在「📁 子目录导航」表格后、「🔗 相关资源」前插入 `README_INDEX_END` 标记
- 升级后即可正常使用增量更新功能

### 验证测试步骤

以下步骤用于验证脚本功能正常（维护参考）：

**前置条件**：选择一个已有自动生成 README 的目录作为测试目标（如 `docs/knowledge/learning/04-docs-markup-tooling/`）。

**步骤1 - 新增文件测试**：
```bash
# 在目标目录下创建测试 .md 文件（含 frontmatter）
# 在目标目录下创建 test-auto-update/ 子目录并放入测试 .md
# 运行更新
python .agents/scripts/generate-readme.py --update --target-dir <目标目录>
# 验证：文档索引表新增了文件，子目录导航新增了📋条目
```

**步骤2 - 新目录生成 README 后状态变更测试**：
```bash
# 为新子目录生成 README
python .agents/scripts/generate-readme.py --target-dir <目标目录>/test-auto-update
# 再次运行更新
python .agents/scripts/generate-readme.py --update --target-dir <目标目录>
# 验证：子目录状态从📋变为✅，链接从 dir/ 变为 dir/README.md
```

**步骤3 - 清理与幂等性测试**：
```bash
# 删除测试文件和目录
# 运行更新
python .agents/scripts/generate-readme.py --update --target-dir <目标目录>
# 验证：测试条目从索引表移除，恢复原状
# 再次运行 --update，应显示"索引已是最新"
python .agents/scripts/generate-readme.py --update --target-dir <目标目录>
```

**步骤4 - 用户内容保留测试**：
```bash
# 手动编辑 README 标记区域外的内容（如在 > 说明下添加一段自定义概述）
# 运行 --update
# 验证：自定义内容完整保留，仅索引表更新
```
