# generate-readme.py 维护与验证手册

> 本文档面向脚本维护者，提供 generate-readme.py 修改后的完整验证流程、测试用例、故障排查方法和回归检查清单。每次修改脚本后，必须按本文档执行验证，确保功能正确。

---

## 1. 验证前置条件

### 1.1 环境检查

```bash
# 确认在项目根目录执行
cd d:\spaces\SpecWeave

# 确认脚本可正常运行（无语法错误）
python .agents/scripts/generate-readme.py --help
```

预期：显示帮助信息，包含 `--scan`、`--update`、`--target-dir`、`--dry-run`、`--force`、`--all`、`--p1` 等参数。

### 1.2 测试目标选择

选择一个**已由脚本生成的** README 目录作为测试目标。判断标准：

- README 文件末尾包含 `<!-- generated by generate-readme.py on YYYY-MM-DD -->` 页脚
- README 中包含 `<!-- README_INDEX_START -->` 和 `<!-- README_INDEX_END -->` 标记

**推荐测试目录**：`docs/knowledge/learning/04-docs-markup-tooling/`（3个子目录、2个文档，结构适中，恢复成本低）。

⚠️ **禁止**选择 P0 手动编写的核心入口 README 作为测试目标（无标记无页脚，不支持自动更新）。

### 1.3 备份策略

开始测试前，记录测试目录的原始状态，以便验证后恢复：

```bash
# 记录测试目录原始git状态
git status --short docs/knowledge/learning/04-docs-markup-tooling/

# 若该目录有未提交的修改，先提交或stash，确保测试可干净回滚
git stash  # 如有需要
```

## 2. 冒烟测试（必做，约2分钟）

冒烟测试验证脚本基本功能未被破坏。每次修改后必须执行。

### 2.1 扫描功能

```bash
python .agents/scripts/generate-readme.py --scan
```

✅ **预期**：输出 `扫描结果：共发现 0 个目录缺失 README`（当前docs目录全覆盖状态下）。若返回非0数字，检查是否为P0手动目录或新增未补全目录。

### 2.2 Dry-Run 预览

```bash
python .agents/scripts/generate-readme.py --update --dry-run
```

✅ **预期**：输出预览信息，格式为 `[PASS] <path>: 将更新/索引已是最新（N个文档，M个子目录）`，最终汇总 `预览完成：X 个将更新，Y 个跳过，0 个失败`。不应有失败项。

### 2.3 单目录生成预览

```bash
# 创建临时测试目录
New-Item -ItemType Directory -Path "docs/_test_smoke" -Force
@"
---
title: "冒烟测试文档"
category: "knowledge"
date: "2026-07-09"
tags: ["test"]
maturity: "L1"
summary: "冒烟测试用临时文档"
---
# 冒烟测试
"@ | Out-File -FilePath "docs/_test_smoke/smoke-test.md" -Encoding utf8

# Dry-run 生成
python .agents/scripts/generate-readme.py --target-dir docs/_test_smoke --dry-run
```

✅ **预期**：显示 `[PASS] docs\_test_smoke: 将生成（1个文档，0个子目录）`。

```bash
# 正式生成
python .agents/scripts/generate-readme.py --target-dir docs/_test_smoke

# 验证生成结果
Test-Path docs/_test_smoke/README.md
```

✅ **预期**：返回 `True`，README.md 已生成且包含正确的frontmatter、索引表、页脚标记。

**清理冒烟测试数据**：
```bash
Remove-Item -Path "docs/_test_smoke" -Recurse -Force
```

## 3. 增量更新端到端测试（核心功能验证，约5分钟）

端到端测试验证 `--update` 功能在真实场景下的正确性。包含 5 个测试用例，覆盖新增、状态变更、删除、幂等、内容保留。

### 测试准备

选择测试目录并记录初始状态：

```bash
$TEST_DIR = "docs/knowledge/learning/04-docs-markup-tooling"

# 记录初始文档数和子目录数（从README中查看）
Select-String -Path "$TEST_DIR/README.md" -Pattern "文档索引|子目录导航"
```

### 用例1：新增文件自动识别

验证新增.md文件后，`--update` 能否自动将其加入文档索引表。

**步骤**：
```bash
# 1. 在测试目录中创建新的.md文件（含完整frontmatter）
@"
---
title: "E2E测试文档"
category: "knowledge"
date: "2026-07-09"
tags: ["e2e-test", "verification"]
maturity: "L1"
summary: "端到端测试-验证新增文件自动加入索引"
---
# E2E测试文档
测试文件。
"@ | Out-File -FilePath "$TEST_DIR/e2e-new-doc-test.md" -Encoding utf8

# 2. 创建新子目录并放入.md文件
New-Item -ItemType Directory -Path "$TEST_DIR/e2e-test-subdir" -Force | Out-Null
@"
---
title: "子目录测试文档"
category: "knowledge"
date: "2026-07-09"
tags: ["e2e-test"]
maturity: "L1"
summary: "端到端测试-验证新子目录自动识别"
---
# 子目录测试
"@ | Out-File -FilePath "$TEST_DIR/e2e-test-subdir/inner-doc.md" -Encoding utf8

# 3. Dry-run 预览
python .agents/scripts/generate-readme.py --update --target-dir $TEST_DIR --dry-run
```

**验证点**：
- [ ] Dry-run 显示"将更新"，文档数+1，子目录数+1
- [ ] 正式执行后，README的「📄 文档索引」表新增了 `[E2E测试文档](e2e-new-doc-test.md)` 行
- [ ] README的「📁 子目录导航」表新增了 `📋 [E2E Test Subdir](e2e-test-subdir/)` 行（📋表示无README）
- [ ] 新行显示正确的标签（`e2e-test` `verification`）和摘要

```bash
# 正式执行
python .agents/scripts/generate-readme.py --update --target-dir $TEST_DIR

# 验证
Select-String -Path "$TEST_DIR/README.md" -Pattern "e2e-new-doc|e2e-test-subdir"
```

### 用例2：子目录README生成后状态变更

验证为新子目录生成README后，父目录导航状态从📋变为✅。

**步骤**：
```bash
# 1. 为新子目录生成README
python .agents/scripts/generate-readme.py --target-dir "$TEST_DIR/e2e-test-subdir"

# 2. 再次运行父目录update
python .agents/scripts/generate-readme.py --update --target-dir $TEST_DIR
```

**验证点**：
- [ ] 子目录导航中 `e2e-test-subdir` 的状态从📋变为✅
- [ ] 链接从 `e2e-test-subdir/` 变为 `e2e-test-subdir/README.md`
- [ ] 文档数从1变为2（原1个文档 + 新生成的README.md）

```bash
Select-String -Path "$TEST_DIR/README.md" -Pattern "e2e-test-subdir"
```

✅ **预期输出包含**：`✅ [E2e Test Subdir](e2e-test-subdir/README.md)`（注意：若目录名未在DIR_TITLE_MAP中，标题可能为自动humanize的英文名，属正常行为）。

### 用例3：用户自定义内容保留

验证标记区域外的用户编辑内容在update后不被破坏。

**步骤**：
```bash
# 1. 手动编辑README，在frontmatter之后、标记区域之前添加自定义内容
#    在 > 说明段落下方添加一行测试文字
$content = Get-Content "$TEST_DIR/README.md" -Raw -Encoding utf8
$customMarker = "<!-- E2E_CUSTOM_CONTENT_TEST -->"
if ($content -notmatch [regex]::Escape($customMarker)) {
    $content = $content -replace "(> .+?`r?`n)`r?`n", "`$1`n$customMarker 这是一段用户自定义的测试内容，update后必须保留。`n`n"
    Set-Content -Path "$TEST_DIR/README.md" -Value $content -Encoding utf8 -NoNewline
}

# 2. 运行update
python .agents/scripts/generate-readme.py --update --target-dir $TEST_DIR
```

**验证点**：
- [ ] 自定义测试文字 `<!-- E2E_CUSTOM_CONTENT_TEST -->` 和后续内容仍然存在
- [ ] 标题和frontmatter未被修改
- [ ] 相关资源章节（🔗）保留
- [ ] 索引表正常更新（或跳过，若无变化）

```bash
Select-String -Path "$TEST_DIR/README.md" -Pattern "E2E_CUSTOM_CONTENT_TEST"
```

### 用例4：删除文件后索引自动清理

验证删除测试文件后，索引表正确移除对应条目。

**步骤**：
```bash
# 1. 删除测试文件和子目录
Remove-Item "$TEST_DIR/e2e-new-doc-test.md" -Force
Remove-Item "$TEST_DIR/e2e-test-subdir" -Recurse -Force

# 2. 运行update恢复索引
python .agents/scripts/generate-readme.py --update --target-dir $TEST_DIR
```

**验证点**：
- [ ] `e2e-new-doc-test.md` 从文档索引表中移除
- [ ] `e2e-test-subdir` 从子目录导航中移除
- [ ] 文档数和子目录数恢复到初始值

```bash
Select-String -Path "$TEST_DIR/README.md" -Pattern "e2e-new-doc|e2e-test-subdir"
```

✅ **预期**：无匹配输出（条目已移除）。

### 用例5：幂等性验证

验证无变化时重复运行 `--update` 不产生文件修改。

**步骤**：
```bash
# 1. 记录当前README的hash
$hash1 = (Get-FileHash "$TEST_DIR/README.md" -Algorithm MD5).Hash

# 2. 连续运行两次update（dry-run）
python .agents/scripts/generate-readme.py --update --target-dir $TEST_DIR --dry-run
python .agents/scripts/generate-readme.py --update --target-dir $TEST_DIR --dry-run

# 3. 正式运行一次update
python .agents/scripts/generate-readme.py --update --target-dir $TEST_DIR

# 4. 再次计算hash
$hash2 = (Get-FileHash "$TEST_DIR/README.md" -Algorithm MD5).Hash
```

**验证点**：
- [ ] Dry-run输出显示"索引已是最新"而非"将更新"
- [ ] 正式执行后 `$hash1 -eq $hash2`（文件未被修改）

```powershell
if ($hash1 -eq $hash2) { Write-Output "✅ 幂等性验证通过：文件未被修改" } else { Write-Output "❌ 幂等性验证失败：文件被无意义修改" }
```

### 清理测试残留

```bash
# 恢复README到原始状态（git）
git checkout -- $TEST_DIR/README.md

# 确认清理完成
git status --short $TEST_DIR/
Select-String -Path "$TEST_DIR/README.md" -Pattern "e2e-|E2E_"
```

✅ **预期**：git status无输出（目录干净），Select-String无匹配（测试内容已清除）。

## 5. 故障排查

### 5.1 常见错误及处理

| 错误现象 | 可能原因 | 排查步骤 |
|---------|---------|---------|
| `--update` 提示 "无法更新：缺少README_INDEX_START标记" | README是手动编写的P0文件或标记被手动删除 | 检查README中是否存在`<!-- README_INDEX_START -->`标记；若是P0文件，需手动添加标记和页脚 |
| 子目录显示📋但实际已有README | 子目录README无`generated by`页脚（手动创建） | 在子目录README末尾添加标准页脚注释，或用`--target-dir --force`重新生成 |
| 新文件未出现在索引表中 | .md文件frontmatter解析失败或文件在排除列表中 | 检查文件是否在根目录（非子目录）；检查frontmatter格式是否正确（YAML语法、`---`分隔符） |
| `--target-dir` 和 `--update` 不能同时使用 | argparse参数互斥组配置错误 | 检查main()中argparse的mutually exclusive group设置 |
| Dry-run显示更新但实际执行后文件无变化 | 幂等性判断认为新旧内容相同 | 这是正常行为，脚本比较新旧索引内容后认为无需更新 |
| 中文标题显示为英文（如"Test Auto Update"） | 目录名不在DIR_TITLE_MAP中 | 在脚本的DIR_TITLE_MAP字典中添加中文映射 |
| 索引表格式错乱 | 标记匹配失败导致替换范围错误 | 检查README中是否有重复的START/END标记；运行upgrade_old_readme()重新升级 |

### 5.2 标记损坏恢复

如果README的标记区域被手动编辑损坏：

```bash
# 方法1：强制重新生成（会丢失标记区域内的手动编辑）
python .agents/scripts/generate-readme.py --target-dir <损坏目录> --force

# 方法2：手动修复标记（推荐）
# 确保以下结构完整：
# <!-- README_INDEX_START -->
# ## 📄 文档索引
# （表格内容）
# ## 📁 子目录导航
# （表格内容）
# <!-- README_INDEX_END -->
```

### 5.3 日志与调试

脚本未设计独立的日志系统，调试时可在关键函数中添加print语句：

- `scan_directory()`：查看扫描到的doc_entries和dir_entries
- `update_marker_region()`：查看标记匹配的起止位置
- `update_readme_index()`：查看新旧索引内容对比结果

调试完成后务必移除调试print语句。
