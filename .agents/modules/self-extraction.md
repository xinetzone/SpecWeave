# Self-Extraction（自我萃取）

## Description
从项目实践中萃取可复用的模式、模板与方法论，形成知识资产库。属于认知层模块，负责扫描代码、文档与配置，识别可复用模式，评估成熟度并分类入库建立索引。

## 所属层级
认知层（Cognition Layer）

## 技术架构
实践采集层（扫描代码、文档、配置）+ 特征提取层（识别可复用模式）+ 质量评估层（评估模式成熟度）+ 资产沉淀层（分类入库并建立索引）。

## 关键实现步骤
| 步骤 | 说明 |
|------|------|
| 构建实践采集器 | 扫描代码、文档与配置文件 |
| 实现特征提取算法 | 识别可复用模式与模板 |
| 开发质量评估模型 | 评估模式成熟度与适用性 |
| 建立资产分类体系 | 按类型、领域、成熟度分类入库 |
| 生成资产索引与导航 | 建立可检索的资产索引 |

## 资源需求
1 名架构师 + 1 名开发者，2 个月

## 时间节点
M3-M5

## 预期成果指标
| 指标 | 目标 |
|------|------|
| 模式萃取率 | ≥ 80% |
| 资产复用率 | ≥ 70% |
| 资产索引完整率 | 100% |
| 萃取准确率 | ≥ 85% |

## 交互方式
- 上游：接收自我复盘模块输出的可复用模式候选
- 下游：向自我进化模块提供可复用模式以支持进化决策
- 协作：向自我发展模块输出知识资产以支持能力评估

## 能力范围
- 代码、文档与配置文件的实践扫描
- 可复用模式与模板的特征提取
- 模式成熟度与适用性的质量评估
- 按类型、领域、成熟度的资产分类入库
- 可检索资产索引与导航的建立

## 约束条件
- 不负责模式的实际应用与执行（归对应执行模块）
- 不负责知识资产的战略规划（归自我发展）
- 萃取须基于客观特征，不得主观判定模式价值
- 资产入库须经过质量评估，低成熟度模式不得直接入库
