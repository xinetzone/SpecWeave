# 数据脱敏：概述与基本原则

## 规范说明

## 目的

本规范旨在建立统一的数据脱敏技术标准，确保AI智能体在接入第三方大模型API、处理用户数据、跨系统数据流转过程中，敏感数据得到有效保护，同时保持数据的业务可用性，防范数据泄露、重识别攻击与合规风险。

## 适用范围

本规范适用于TRAE系统全场景的数据脱敏处理，包括但不限于：

- 提示词预处理与敏感信息过滤
- 对话历史存储与归档
- 训练/微调数据集准备
- API请求/响应数据传输
- 日志记录与审计
- RAG检索文档片段处理
- 多智能体协作数据共享
- 第三方模型调用数据出境

## 基本原则

- **不可逆原则**：除FPE加密脱敏与令牌化场景外，脱敏后的数据应无法通过技术手段恢复原始值
- **可用性平衡原则**：脱敏后的数据应保留必要的业务语义与统计特性，满足分析、测试、模型训练等使用需求
- **可审计原则**：所有脱敏操作必须记录审计日志，包括脱敏规则、执行时间、操作主体、数据范围
- **场景适配原则**：根据数据级别、使用场景、流转目的地选择最合适的脱敏技术，避免过度脱敏或脱敏不足

## 脱敏与加密的区别

| 维度 | 数据脱敏 | 数据加密 |
|---|---|---|
| 目的 | 隐藏敏感信息，保留数据可用性 | 保护数据机密性，授权方可解密 |
| 可逆性 | 多数技术不可逆（FPE/令牌化除外） | 使用密钥可逆 |
| 数据格式 | 通常保持与原始数据相似格式 | 格式通常改变（密文） |
| 使用场景 | 数据分析、测试、展示、第三方共享 | 数据存储、传输保密 |
| 密钥依赖 | 掩码/替换等技术无需密钥 | 必须依赖密钥管理 |
| 业务价值 | 脱敏后可直接使用 | 解密后才能使用 |
