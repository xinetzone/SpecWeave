# 数据脱敏：有效性验证方法

## 脱敏有效性验证方法

## 定性验证：人工审查

**方法说明**：由安全人员、数据保护官（DPO）、业务代表组成审查小组，对脱敏后数据进行人工抽样审查，判断是否仍能识别到特定个人或实体。

**审查要点**：
- 抽样比例不低于数据量的1%，至少覆盖100条记录
- 审查者独立判断，不接触原始数据映射表
- 记录每一条疑似可识别记录的识别路径与所需外部信息量
- 对识别出的风险点回溯改进脱敏规则
- 形成人工审查报告，标注识别风险等级

## 定量验证：重识别风险评估

**k-匿名度验证**：
- 计算每个准标识符组合对应的记录数
- 所有等价类大小应≥设定的k阈值（建议k≥10）
- 识别k<阈值的等价类，进行补充泛化或抑制
- 输出k-匿名合规报告

**l-多样性验证**：
- 对每个等价类检查敏感属性的不同取值数量
- 每个等价类敏感属性取值数应≥设定的l阈值（建议l≥2）
- 对不满足l-多样性的等价类进行额外扰动或抑制

**t-接近度验证**：
- 计算每个等价类敏感属性分布与整体分布的地球移动距离（EMD）
- 所有等价类EMD应≤设定的t阈值（建议t≤0.2）
- t值越小，隐私保护强度越高

**重识别风险量化指标**：
| 指标 | 计算方法 | 可接受阈值 |
|---|---|---|
| 检察官风险（Prosecutor Risk） | 攻击者已知目标在数据集中，重识别成功概率 | ≤1% |
| 记者风险（Journalist Risk） | 攻击者不知道目标是否在数据集中，重识别成功概率 | ≤5% |
| 营销者风险（Marketer Risk） | 攻击者从数据集中随机选一条，重识别成功概率 | ≤10% |

## 可用性验证

**方法说明**：验证脱敏后数据仍保留足够的业务价值，满足业务使用需求。

**验证指标**：
- **结构完整性**：数据格式、字段类型、外键关联关系保持完整
- **统计特性保持**：脱敏后数值型数据的均值、方差、分位数与原始数据偏差≤5%
- **业务规则校验**：脱敏后数据通过所有业务格式校验（如身份证校验位、手机号段合法性）
- **NLP模型准确率**：脱敏后文本数据训练/测试的NLP模型任务准确率下降不超过5%
- **数据分析一致性**：基于脱敏数据的统计分析结论与基于原始数据的结论方向一致
- **查询性能影响**：动态脱敏对API响应时间增加≤100ms

**测试方法**：
1. A/B测试：同一分析任务分别在原始数据和脱敏数据上执行，比较结果偏差
2. 业务方验收：数据使用方确认脱敏数据满足业务场景需求
3. 下游系统测试：脱敏后数据在下游系统中正常流转处理，无格式错误

## 自动化检测工具建议

| 工具类型 | 推荐工具/方案 | 用途 |
|---|---|---|
| PII检测 | Presidio（Microsoft）、PIICatcher、Common Regular Expressions | 结构化/非结构化数据中PII自动识别 |
| 隐私度量 | ARX Data Anonymization Tool、Python `anonymeter`库 | k-匿名、l-多样性、t-接近度计算与重识别风险评估 |
| 差分隐私验证 | OpenDP、Google DP Library、IBM Diffprivlib | 差分隐私噪声添加正确性与隐私预算验证 |
| 静态代码扫描 | Semgrep规则、GitGuardian、TruffleHog | 代码与日志中密钥、凭证泄露检测 |
| NER模型 | spaCy NER、HanLP、LAC（百度） | 中文文本中姓名、地址、组织等实体识别 |
| CI/CD集成 | 自定义脱敏检查脚本 | 数据处理流水线中自动化脱敏合规检查 |

## 脱敏有效性验证Checklist

| 验证项 | 验证方法 | 验收标准 | 责任人 | 验证结果 |
|---|---|---|---|---|
| PII召回率 | 使用标注PII测试集进行检测率测试 | 直接标识符召回率≥99%，准标识符召回率≥95% | 安全工程师 | □通过 □不通过 |
| PII误报率 | 无PII的正常文本测试 | 误报率≤5% | 开发工程师 | □通过 □不通过 |
| k-匿名合规性 | ARX工具或自定义脚本计算等价类大小 | 所有等价类大小≥k=10 | 数据工程师 | □通过 □不通过 |
| l-多样性合规性 | 统计每个等价类敏感属性不同取值 | 每个等价类敏感属性取值≥l=2 | 数据工程师 | □通过 □不通过 |
| 重识别风险评估 | 检察官/记者/营销者风险量化 | 检察官风险≤1%，记者风险≤5%，营销者风险≤10% | 安全工程师 | □通过 □不通过 |
| 格式有效性校验 | 业务规则验证脚本 | 100%通过格式校验（手机号、身份证、邮箱等） | 测试工程师 | □通过 □不通过 |
| 统计特性偏差 | 原始数据与脱敏数据统计指标对比 | 均值/方差/分位数偏差≤5% | 数据分析师 | □通过 □不通过 |
| 模型可用性测试 | 脱敏数据训练/测试模型 | NLP/ML任务准确率下降≤5% | 算法工程师 | □通过 □不通过 |
| 不可逆性验证 | 尝试对脱敏后数据进行还原攻击 | 除FPE/令牌化外，其他技术脱敏后无法还原 | 安全工程师 | □通过 □不通过 |
| 审计日志完整性 | 检查脱敏操作日志记录 | 所有脱敏操作有完整日志，包含规则/时间/主体/范围 | 运维工程师 | □通过 □不通过 |
| 多轮一致性验证 | 同一实体在多轮对话/多表中替换 | 同一原始值映射一致率100% | 开发工程师 | □通过 □不通过 |
| 性能影响测试 | 对比脱敏前后响应时间 | 动态脱敏增加延迟≤100ms | 性能测试工程师 | □通过 □不通过 |
