### 流程说明

- **低风险判定标准**：L1公开数据可直接按低风险流程；L2内部数据经评估接收方安全能力达标、签署DPA、落实加密措施后可按低风险流程
- **高风险判定标准**：涉及L3敏感数据、重要数据、超过100万人个人信息出境的，必须申报国家网信部门安全评估
- **禁止出境判定标准**：L4核心数据一律禁止出境；经评估风险不可控的场景一律终止
- **时限说明**：标注时限为一般情况下的最长时限，紧急业务场景可申请加急，但不得跳过必要环节


## 标准合同条款模板

与境外API供应商签署的数据处理协议（DPA）必须包含以下必备条款：

| 条款编号 | 条款名称 | 核心要求 |
|---|---|---|
| DPA-01 | 数据处理目的与范围限制 | 明确约定数据处理的具体目的，仅限于实现约定的AI服务功能；以正面清单方式列明可处理的数据类型、范围，禁止处理超出约定范围的数据；禁止将数据用于约定目的之外的任何用途（包括但不限于模型训练、产品改进等，除非另行明确授权） |
| DPA-02 | 数据留存期限与删除要求 | 明确约定数据留存的最长期限，不得超过实现处理目的所必需的最短时间；约定到期自动删除机制；要求接收方提供数据删除的书面证明（包括备份数据的删除）；用户提出删除请求时，应在7个工作日内完成删除并反馈 |
| DPA-03 | 数据安全保护措施要求 | 明确约定数据传输和存储的加密标准（传输TLS 1.2+，存储AES-256及以上）；要求实施严格的访问控制（最小权限原则、多因素认证、定期权限审计）；要求通过ISO27001、SOC2等国际安全认证；定期进行安全渗透测试和漏洞评估 |
| DPA-04 | 数据再传输限制 | 明确约定未经数据提供方书面同意，接收方不得将数据向任何第三方（包括关联公司、分包商）再传输或披露；确需使用分包商处理的，分包商须承担不低于本协议的安全义务，且接收方对分包商行为承担连带责任；禁止数据被传输至评估范围以外的国家/地区 |
| DPA-05 | 数据主体权利响应机制 | 约定接收方须配合数据提供方响应数据主体（个人信息主体）的权利请求，包括查询、更正、删除、限制处理、数据可携带等；接收方收到数据主体直接请求时，应在3个工作日内转通知数据提供方，不得自行响应；配合提供实现数据主体权利所需的技术支持 |
| DPA-06 | 安全事件通报义务 | 明确约定接收方发生或疑似发生数据泄露、损毁、丢失等安全事件时，必须在24小时内书面通报数据提供方；通报内容应包括事件性质、影响范围、涉及数据类型和数量、已采取的处置措施、后续补救计划；全力配合事件调查和处置，承担相应责任 |
| DPA-07 | 审计配合义务 | 约定数据提供方或其委托的第三方审计机构有权对接收方履行数据保护义务的情况进行年度审计；接收方须提供必要的便利条件（包括日志、记录、文档访问等）；审计发现重大安全隐患时，接收方须在限定期限内完成整改 |
| DPA-08 | 违约责任与赔偿条款 | 明确约定违反数据保护条款的违约金计算方式；因接收方原因导致数据泄露、滥用等安全事件造成损失的，接收方应承担全部赔偿责任（包括直接损失、间接损失、监管罚款、律师费等）；重大违约情况下数据提供方有权立即终止协议 |
| DPA-09 | 法律适用与争议解决 | 约定协议适用中华人民共和国法律（不含冲突法规则）；争议优先通过友好协商解决，协商不成的，提交数据提供方所在地有管辖权的人民法院诉讼解决；明确排除适用接收方所在国可能强制调取数据的法律（如美国Cloud Act）的适用，确需调取时须提前通报并经中方同意 |
| DPA-10 | 合同终止后数据处置要求 | 合同终止或解除后15个工作日内，接收方须彻底删除其持有的所有数据（包括所有备份副本）；向数据提供方出具由授权代表签署的《数据删除确认函》；合同终止后保密义务、安全事件通报义务、审计配合义务等条款继续有效 |
