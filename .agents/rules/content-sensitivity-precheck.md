# 内容敏感度预检规范

## 概述

内容敏感度预检是启动协议步骤 2.3 的详细规范，用于在任务执行前判定分析对象/产出物的内容敏感度级别，选择正确的工作流模式与存储位置，防止私域/敏感内容进入公共规划区域（`.trae/specs/`、`docs/`）。

**核心原则**：就高不就低——不确定时默认按私域处理，或向用户确认。

## 内容级别定义

### 公开内容（Public）

| 特征 | 说明 |
|------|------|
| 访问控制 | 无访问控制，任何人无需特殊权限即可访问 |
| 典型场景 | 公开发布的网页、开源代码仓库、官方文档、公开新闻/文章、公开发布的API文档、技术博客 |
| 工作流模式 | **标准 Spec Mode**：生成完整 spec.md/tasks.md/checklist.md 三件套 |
| Spec 目录 | `.trae/specs/<theme-subdir>/<project-name>/` |
| 最终产出物 | `docs/` 下对应分类目录 |
| 可提交Git | 是 |

### 私域内容（Private）

| 特征 | 说明 |
|------|------|
| 访问控制 | 需要特殊权限、邀请码、token、登录态才能访问 |
| 典型场景 | 内部会议记录/录音、私域分享链接（带code/token参数）、个人笔记、商业培训材料、含个人隐私的内容、含商业秘密的材料、未公开的项目文档、playground/实验性内容 |
| 工作流模式 | **私域直接执行**：跳过 `.trae/specs/` 公共规划区域，在目标目录直接执行 |
| 产出物目录 | `playground/` 下对应用户目录，或用户明确指定的目录 |
| 可提交Git | 默认不提交（遵循 `.gitignore` 中对临时/私域目录的规则） |

## 判定信号清单

满足以下任一信号即判定为**私域内容**：

### URL特征
- [ ] URL路径中含 `share?code=`、`?token=`、`?key=`、`?invite=`、`?pwd=` 等访问控制参数
- [ ] URL含 `internal.`、`corp.`、`intranet.`、`private.`、`admin.` 等内部域名前缀
- [ ] 域名为企业内部应用（如飞书/企微/钉钉内部链接、企业CRM/ERP/OA系统、企业知识库）
- [ ] 需要登录态才能访问的页面（未登录时跳转登录页）

### 用户信号
- [ ] 用户明确标注"私域"、"内部"、"保密"、"个人"、"私人"、"不要外传"等关键词
- [ ] 用户指定目标目录为 `playground/` 或其他非 `docs/`/`.trae/specs/` 目录
- [ ] 用户提供的源文件位于 `.temp/`、`playground/`、个人目录等非公共区域

### 内容主题信号
- [ ] 内部会议记录/纪要/录音转写
- [ ] 商业培训材料/课程笔记/付费内容
- [ ] 含个人身份信息（PII）的文档
- [ ] 商业秘密/财务数据/客户信息
- [ ] 未公开的产品规划/战略文档
- [ ] 个人笔记/随笔/实验性内容

### 不确定场景处理
- [ ] 信号不明确时 → 默认按私域处理
- [ ] 用户请求与个人判断冲突时 → 向用户确认："这个内容属于私域/内部内容吗？我将直接在playground目录下处理"
- [ ] 内容同时包含公开和私域部分 → 整体按私域处理，公开部分若需沉淀可在后续单独提取

## 工作流分流决策树

```
收到任务
  │
  ├─ 步骤 2.0：任务类型预检（vendor资产）
  │
  ├─ 步骤 2.3：内容敏感度预检 ◀── 本规范
  │   │
  │   ├─ 是否命中私域信号？
  │   │   ├─ 是 ──────────────────► 私域工作流
  │   │   │                           · 跳过 .trae/specs/ 规划
  │   │   │                           · 直接在目标目录执行
  │   │   │                           · 产出物存放在 playground/
  │   │   │                           · 不创建公共spec目录
  │   │   │
  │   │   └─ 否 ──────────────────► 公开工作流
  │   │                               · 标准 Spec Mode
  │   │                               · .trae/specs/<theme>/ 规划
  │   │                               · 产出物最终到 docs/
  │   │
  │   └─ 不确定？ → 默认私域 或 询问用户
  │
  └─ 后续步骤（规范读取→自检→执行）
```

## 私域工作流执行规范

判定为私域内容后，执行以下简化流程：

1. **确定目标目录**：优先使用用户指定的目录；未指定时在 `playground/` 下创建语义化目录（如 `playground/reports/<topic>-<date>/`）
2. **跳过Spec规划**：不生成 `.trae/specs/` 下的 spec.md/tasks.md/checklist.md
3. **直接执行**：根据任务类型选择合适的Skill和工具直接执行
4. **产出物管理**：所有产出物直接存放在目标目录，包括分析报告、笔记、截图等
5. **元数据规范**：产出物frontmatter中标注 `type: "private"` 或 `type: "playground"` 便于区分
6. **不提交公共索引**：不将私域内容加入 `docs/` 下的公共索引和导航表
7. **清理**：任务完成后不在 `.trae/specs/` 下留下任何规划文档

## 私域内容转公开流程

私域内容中如有可复用的方法论或模式，经用户同意后可通过以下流程转为公开资产：

1. 提取通用部分（去除私域细节、个人信息、商业秘密）
2. 抽象为可复用模式/方法论
3. 存入 `docs/retrospective/patterns/` 或 `docs/knowledge/`
4. 更新公共索引
5. 在原私域文档中添加"已沉淀公开模式：[链接]"引用

## 典型案例

### 案例1：联想AI妙记私域分享页（本规范来源）

- **URL**：`https://tx.lenovomm.com/web/share.html?code=dc037cdd...`
- **私域信号**：URL含 `share?code=` 参数，内容为商业培训会议录音
- **正确处理**：直接在 `playground/reports/retrospective-yihuakaitian-meeting-20260711/` 执行，不创建 `.trae/specs/` 目录
- **实际问题**：本次执行先走了标准Spec Mode在 `.trae/specs/` 创建了规划文档，事后需要清理 → 本规范即为此问题的预防机制

### 案例2：公开技术博客分析

- **URL**：`https://example.com/blog/post123`
- **判定**：公开发布，无访问控制 → 公开内容
- **正确处理**：标准Spec Mode，spec目录在 `.trae/specs/competitive-analysis/`，最终报告到 `docs/`

### 案例3：飞书内部文档分析

- **信号**：域名为 `feishu.cn` 内部文档链接，需要登录
- **判定**：私域内容
- **正确处理**：直接在 `playground/` 下执行
