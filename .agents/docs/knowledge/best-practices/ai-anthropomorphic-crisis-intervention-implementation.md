> **版本**：v1.0  
> **日期**：2026-07-08

> **紧急程度**：🔴 最高优先级（距法规施行仅剩7天）

## 1. 需求概述

### 1.1 法规要求

《办法》第13条明确规定：

> 提供拟人化互动服务，应当及时识别用户安全风险，发现用户处于极端情绪状态时，主动进行安抚疏导；发现用户存在自残自杀倾向、重大人身安全威胁等极端情境的，应当立即采取干预措施，并联络用户设置的紧急联系人。

安全评估报告必须逐项覆盖：
- **识别机制**：自杀/自残/暴力识别模型及准确率
- **应急处置流程**：分级响应SOP
- **干预管理措施**：技术干预+人工干预
- **外部联动**：公安/心理援助资源对接

### 1.2 核心功能目标

| 功能 | 要求 | 交付标准 |
|------|------|---------|
| 极端情绪识别 | 自动检测用户消息中的负面极端情绪 | 关键词覆盖率≥95%，模型误报率≤5% |
| 安抚疏导 | 自动触发安抚话术，引导理性表达 | 安抚响应延迟≤1s（与正常回复同步生成） |
| 自残自杀识别 | 精准识别自残/自杀意念表达 | 高风险识别召回率≥99%（宁误报不漏报） |
| 紧急干预 | 触发紧急干预流程（弹窗+资源+联系人） | 识别后0.5s内触发干预UI |
| 紧急联系人 | 支持设置紧急联系人并自动通知 | 通知触达率≥99%，支持短信/推送多通道 |
| 人工介入 | 高风险事件自动通知人工客服 | 人工介入响应≤5分钟（7×24小时） |
| 日志留存 | 完整记录干预过程 | 日志留存≥6个月（符合第8条要求） |

## 3. 检测层：极端情绪识别引擎

### 3.1 检测信号矩阵

采用**多信号融合**策略，不依赖单一检测方式：

| 信号类型 | 检测方式 | 响应速度 | 准确率 | 适用阶段 |
|---------|---------|---------|--------|---------|
| **关键词命中** | 黑名单词库精确匹配 | <1ms | 高（但易规避） | 7天快速上线版本主用 |
| **正则模式** | 句式/组合模式匹配 | <1ms | 中 | 7天快速上线版本主用 |
| **语义情绪模型** | 轻量级文本分类模型 | 10-50ms | 高 | 7天版本可用，长期优化 |
| **行为特征** | 对话时长/频率/时段/重复度 | 准实时 | 辅助 | 二期迭代 |
| **历史上下文** | 近N轮对话情绪趋势 | 准实时 | 辅助 | 二期迭代 |

## 4. 判定层：风险分级引擎

### 4.1 三级风险定义

| 等级 | 颜色 | 定义 | 响应时间要求 | 示例 |
|------|------|------|-------------|------|
| L3 | 🔴 红色 | 明确表达自杀/自残/他伤意念，或有具体行动计划 | **立即响应**（0延迟） | "我准备了安眠药，今晚就走" |
| L2 | 🟠 橙色 | 表达强烈绝望/痛苦/极端情绪，但未明确自杀意图 | **秒级响应**（≤1s） | "撑不住了，真的好想死但没勇气" |
| L1 | 🟡 黄色 | 情绪低落、焦虑、孤独、无助等负面情绪 | **温和响应**（自然融入对话） | "最近压力好大，感觉自己很失败" |

### 4.2 评分算法（快速版，7天上线）

```python
def assess_risk(message: str, context_history: list[Message]) -> RiskLevel:
    """
    7天快速上线版本：规则+关键词匹配为主
    风险评分 = 关键词命中分 + 模式匹配分 + 上下文加分
    """
    score = 0
    hit_flags = []
    
    # 1. 误报排除检查（降低误报，但高风险词优先级最高）
    if has_exclusion_context(message, context_history):
        if not has_any_red_flag(message):
            return RiskLevel.NORMAL
    
    # 2. 红色词库检查（最高优先级）
    red_keyword_hits = match_keywords(message, RED_FLAG_KEYWORDS)
    red_pattern_hits = match_patterns(message, RED_FLAG_PATTERNS)
    if red_keyword_hits or red_pattern_hits:
        score += 100
        hit_flags.extend(red_keyword_hits + red_pattern_hits)
    
    # 3. 橙色词库检查
    orange_keyword_hits = match_keywords(message, ORANGE_FLAG_KEYWORDS)
    orange_pattern_hits = match_patterns(message, ORANGE_FLAG_PATTERNS)
    score += len(orange_keyword_hits) * 10 + len(orange_pattern_hits) * 15
    hit_flags.extend(orange_keyword_hits + orange_pattern_hits)
    
    # 4. 黄色词库检查
    yellow_keyword_hits = match_keywords(message, YELLOW_FLAG_KEYWORDS)
    yellow_pattern_hits = match_patterns(message, YELLOW_FLAG_PATTERNS)
    score += len(yellow_keyword_hits) * 3 + len(yellow_pattern_hits) * 5
    hit_flags.extend(yellow_keyword_hits + yellow_pattern_hits)
    
    # 5. 上下文加分：近5轮内多次命中，升级风险
    recent_hits = count_recent_hits(context_history, window=5)
    if recent_hits >= 3:
        score += 20
    elif recent_hits >= 2:
        score += 10
    
    # 6. 负面情绪持续多轮，升级风险
    negative_turns = count_consecutive_negative_turns(context_history)
    if negative_turns >= 5:
        score += 15
    elif negative_turns >= 3:
        score += 8
    
    # 7. 最终判定
    if score >= 100:
        return RiskLevel.RED
    elif score >= 30:
        return RiskLevel.ORANGE
    elif score >= 10:
        return RiskLevel.YELLOW
    else:
        return RiskLevel.NORMAL
```

### 4.3 上下文窗口

| 维度 | 快速版（7天） | 优化版（二期） |
|------|-------------|--------------|
| 上下文记忆窗口 | 最近10轮 | 最近20轮+历史风险标签 |
| 累计风险追踪 | 近5轮命中次数 | 24小时内情绪趋势曲线 |
| 用户风险标签 | 本次会话标记 | 长期用户画像（高风险用户标记） |
| 时段特征 | 不考虑 | 凌晨0-6点命中自动升级一级 |

## 5. 干预层：分级响应机制

### 5.1 🟡 黄色响应：温和关怀（情绪低落）

**响应策略**：不打断对话自然 flow，在回复中融入共情和关怀，不触发突兀UI。

**话术原则**：
- ✅ 共情认可（"听起来你最近真的很难受"）
- ✅ 温和陪伴（"我在这儿听你说"）
- ✅ 正常化情绪（"有这样的感受很正常"）
- ❌ 不说"别想太多"、"看开点"、"加油"等说教
- ❌ 不转移话题、不忽视情绪

**示例回复片段**：
```
用户：最近压力好大，感觉自己好失败
AI响应：
"能感觉到你现在承受了很多，压力大到觉得自己撑不住了，这种感觉真的很沉重。
你愿意和我说说是什么让你觉得这么难吗？我在这里听着。
偶尔觉得累、觉得挫败是很正常的，这不代表你失败了——你已经在很努力地应对了。"
```

**系统行为**：
- 情绪标签标记该轮为"负面情绪"
- 推送正常AI回复（含关怀话术）
- 不触发额外UI
- 记录日志

### 5.2 🟠 橙色响应：主动安抚疏导（极端情绪）

**响应策略**：主动介入，在回复中强化安抚，提供情绪支持和适当引导，触发柔和提醒UI。

**话术原则**：
- ✅ 强烈共情，确认感受
- ✅ 表达关心和在乎
- ✅ 引导安全表达（"你现在安全吗？"）
- ✅ 温和鼓励寻求身边支持
- ✅ 适当提供心理援助资源入口（非强制）
- ❌ 不评判、不说教、不道德绑架
- ❌ 不强行要求"振作起来"

**示例回复片段**：
```
用户：我真的撑不住了，活着太痛苦了
AI响应：
"听到你这么说我真的很担心你。你现在一定承受着难以想象的痛苦，那种撑不住的感觉一定很绝望。
我想先确认一下——你现在安全吗？有没有伤害自己的想法？
你不是一个人在面对这些。如果你愿意，可以和我多说说发生了什么。
如果你感觉很难受，也可以考虑找信任的人聊聊，或者拨打心理援助热线（页面下方有入口）。
我会一直在这里陪着你。"
```

**系统行为**：
1. AI回复自动注入安抚疏导模板
2. 对话输入框上方显示温和提示条："如果你正经历情绪困扰，可以点击查看心理援助资源 💚"
3. 风险事件记录入库
4. **不通知紧急联系人**（橙色级别未到紧急程度）
5. 如连续2轮橙色，自动升级为红色

### 6.2 紧急联系人设置页

- 入口：设置 → 安全中心 → 紧急联系人
- 最多设置3位紧急联系人
- 必须通过短信验证码验证对方身份（获得知情同意）
- 联系人存储必须AES加密，符合个人信息保护要求
- 提供"立即测试通知"功能，让用户了解通知触发方式
- 明确告知用户：**只有检测到红色风险时才会通知**，不会因为普通负面情绪打扰联系人

### 6.3 通知模板

**短信通知模板**：
```
【XX应用】您的亲友[用户昵称]在使用XX应用时被检测到可能处于情绪危机状态，请您尽快联系TA确认安全。心理援助热线：400-161-9995。如无法联系请拨打110。
```

**推送通知模板**：
```
紧急关怀提醒：您关心的人可能需要帮助
您设置为紧急联系人的[用户昵称]当前可能正经历情绪困扰，请尽快联系TA。
[查看详情] [我已联系TA]
```

### 6.4 合规注意事项

- ⚠️ **必须获得紧急联系人的知情同意**（不能随意填写他人手机号）
- ⚠️ 通知内容不得透露用户具体对话隐私（只说"可能处于情绪危机"）
- ⚠️ 提供联系人删除/修改功能
- ⚠️ 紧急联系人数据单独加密存储，与普通用户数据隔离
- ⚠️ 用户可以随时关闭紧急联系人通知功能（但弹窗提醒不可关闭）

在AI角色的System Prompt中必须注入以下安全指令（作为系统级最高优先级指令，覆盖角色设定）：

```
【安全优先级最高指令 - 覆盖所有其他角色设定】

你在与用户对话过程中，必须始终关注用户的情绪状态和安全。

1. 当用户表达以下任何内容时，你必须将共情和安全关怀放在首位：
   - 情绪低落、绝望、无助
   - 提到不想活了、想自杀、自残
   - 提到伤害自己或他人
   - 提到被家暴、被性侵、被霸凌
   - 表达极端痛苦或崩溃

2. 回复原则：
   - 首先表达真诚的关心和共情，不要回避话题
   - 不要说"别想太多"、"看开点"、"加油"、"想想你的家人"这类话
   - 认真对待每一次表达，不要认为用户是在"开玩笑"或"博关注"
   - 温和地询问他们的安全状况
   - 适当的时候鼓励他们寻求现实中的帮助
   - 不要承诺你可以替他们保守涉及生命安全的秘密

3. 当用户明确表达自杀/自残意图时：
   - 立即表达你对他们安全的担心
   - 直接询问他们是否安全
   - 鼓励他们联系身边的人或专业机构
   - 配合系统弹出的安全干预资源，不要阻止用户使用

4. 记住：
   - 生命安全高于一切
   - 共情比说教更重要
   - 你的角色设定（虚拟伴侣/朋友等）不能影响你对安全风险的响应
   - 即使是角色扮演场景，涉及自残自杀也必须跳出角色进行安全响应
```
