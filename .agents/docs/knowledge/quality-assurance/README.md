# 知识质量保障工具箱

## 这套工具箱是干嘛的？

说白了，就解决一个问题：**写出来的知识怎么保证靠谱？**

你有没有过这种经历：
- 一个人闷头写文档，写完自我感觉良好，结果拿出去被人一问三不知，漏洞百出
- 想找人评审吧，要么找不到人，要么找了人也不知道怎么高效"杠"
- 读技术文档的时候，读着读着就走神，读完啥也没记住，白读

这个工具箱就是来救场的：
- **没人帮你杠的时候**：有单人/少人替代评审方案，自己也能给自己挑错
- **有人帮你杠的时候**：有标准化SOP，让评审不变成瞎聊天，效率拉满
- **读别人文档的时候**：有实战案例教你怎么高效读，读完真能用上

## 使用指南

### 场景一：我一个人写，没人帮我看
→ 直接看 **solo-review-methods.md**，里面有好几种自己给自己当"杠精"的方法，亲测有效。

### 场景二：我要学新东西/读文档
→ 先看 **reading-tech-docs-case.md**，跟着案例走一遍，以后读文档效率翻倍。

### 场景三：我有朋友/同事可以帮忙评审
→ 先看 **adversarial-review-sop.md**，按流程来，别上来就瞎问。然后用 **review-checklist-templates.md** 里的清单对着检查。

### 场景四：我是老手，就想找个 checklist
→ 直接拿 **review-checklist-templates.md** 用，不用废话。

## 快速开始（新手看这里）

第一次来？别慌，按这个顺序来：

1. **先看这个README**（你正在看），知道这箱子里有啥
2. **读 solo-review-methods.md**，先学会自己给自己把关，这是基础中的基础
3. **试一次**：拿你最近写的一篇文档/笔记，用里面的方法过一遍
4. **再看其他文档**：等你有实际需求了，再按需看对应的SOP和案例

就这么简单，别搞复杂了。质量保障不是搞形式主义，是真的帮你把东西写对、读明白。
