# 反馈指南

> **来源**: https://www.minitap.ai/docs/mobile-use-sdk/feedback

我们鼓励您在使用 Minitap 产品时提供反馈。您的 Bug 报告和功能建议将帮助塑造我们工具的未来。

## 如何提供反馈

我们根据产品使用不同的平台：

- **Mobile Use SDK** - 开源，直接在 GitHub 上报告问题
- **Pilot Extension** - 使用 Featurebase 报告 Bug 和建议功能

## Mobile Use SDK

在 GitHub 上报告 Bug 或建议功能

## Pilot Extension

报告 Pilot IDE 扩展的 Bug 或建议功能

## Bug 报告应包含的内容

报告 Bug 时，请包含：

1. **Session ID** - 帮助我们快速调试您的问题（见下文）
2. **描述** - 发生了什么 vs. 您期望什么
3. **复现步骤** - 我们如何重现问题？
4. **环境** - 操作系统、Python 版本、设备类型（Android/iOS）
5. **错误消息** - 任何错误日志或堆栈跟踪
6. **截图/录屏** - 如适用的视觉证据

### 如何获取您的 Session ID

- Mobile Use SDK
- Pilot Extension

当您启用匿名跟踪并且发生错误时，错误输出中会显示一个 Session ID：

```text
Session ID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

报告问题时请包含此 Session ID——它使我们能够准确查看发生了什么并更快地解决您的问题。

Session ID 显示在 Pilot 面板中设备屏幕的下方。

**复制方法：** 只需点击 Session ID——它会自动复制到您的剪贴板。

报告问题时始终包含您的 Session ID——它使我们能够在我们这边准确查看发生了什么并更快地解决您的问题。

## 功能建议

建议功能时：

1. **描述问题** - 您想要完成什么？
2. **提出解决方案** - 您设想它如何工作？
3. **解释价值** - 为什么这对其他人有用？
