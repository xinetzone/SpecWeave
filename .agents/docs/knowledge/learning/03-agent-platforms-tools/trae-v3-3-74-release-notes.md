# TRAE v3.3.74 版本发布笔记

## 版本信息

- **版本号**：v3.3.74
- **发布日期**：2026年7月8日
- **适用平台**：Windows / macOS / Linux
- **更新类型**：功能更新 + Bug修复

***

## 更新内容

### 1. 新增 Browser 配置聚合页【仅个人版】

**功能概述**：设置中心新增专门的浏览器配置聚合页面，将所有与浏览器相关的配置项集中展示。

**设计意图**：

- 统一管理：所有浏览器相关配置一目了然
- 便捷操作：无需在多个设置页面间切换
- 配置透明化：用户可以清晰了解所有浏览器相关的可配置项

**可能包含的配置项**：

- 内置浏览器默认打开方式（内置浏览器/系统浏览器/始终询问）
- Live Preview 预览模式配置
- 浏览器自动化 MCP Server 配置
- 网络代理与限流设置
- 安全策略（混合内容/CORS）
- 性能优化（缓存策略/资源压缩）

**访问方式**：设置中心 → Browser（或"浏览器"）

### 2. Windows 接入 MSSDK【仅个人版】

**功能概述**：TRAE IDE 的 Windows 版本集成微软官方软件开发工具包（Microsoft SDK，简称 MSSDK），将底层沙箱与系统能力实现从自研方案迁移到微软官方 SDK 体系。

---

#### 2.1 MSSDK 概念解析

**MSSDK** 是 **Microsoft SDK** 的缩写，指微软官方提供的软件开发工具包集合，主要包含：

| SDK 类型 | 全称 | 核心定位 | 版本 |
|---------|------|---------|------|
| **Windows SDK** | Windows Software Development Kit | 传统 Windows 开发基石，包含平台标头、库、WinRT 元数据、构建工具 | 10.0.28000+ |
| **Windows App SDK** | Windows App Software Development Kit | 现代化 Windows 开发框架，统一的现代 Windows API，解耦于操作系统版本 | 1.8+ |

**Windows SDK 核心组件**：

- **Platform Headers**（平台标头）：定义 Windows API 的 C/C++ 声明
- **Libraries**（库文件）：包含实现代码的 .lib/.dll 文件
- **WinRT Metadata**（WinRT 元数据）：UWP/WinUI 应用使用的 API 元数据
- **Build Tools**（构建工具）：MSBuild、资源编译器、签名工具等
- **Debugging Tools**（调试工具）：WinDbg、性能分析器等

**Windows App SDK 核心特性**：

- **WinUI 3**：现代化 UI 框架，支持桌面和 UWP 应用
- **MSIX Packaging**：新一代应用打包格式，支持自动更新和沙箱部署
- **App Lifecycle**：统一的应用生命周期管理
- **Windowing APIs**：现代化窗口管理 API
- **Resource Management**：统一的资源管理机制

---

#### 2.2 技术背景与迁移意义

**迁移前状态**：

TRAE 在 Windows 平台上基于自研沙箱 SDK 实现命令执行的安全隔离。根据 TRAE [沙箱文档](https://docs.trae.cn/ide_sandbox)描述：

> 基于自研的沙箱 SDK，TRAE 会自动为你创建一个受限的命令执行环境。

**迁移后状态**：

接入 MSSDK 后，TRAE 将利用微软官方提供的沙箱和系统能力接口：

| 技术领域 | 自研方案 | MSSDK 方案 |
|---------|---------|-----------|
| 进程隔离 | 自研进程监控与隔离机制 | Windows Job Objects + AppContainer |
| 文件系统访问控制 | 自研路径拦截与权限校验 | Windows 沙箱文件系统虚拟化 |
| 网络隔离 | 自研网络代理与流量控制 | Windows 沙箱网络隔离 |
| 系统调用限制 | 自研 API Hook | Windows 沙箱系统调用过滤 |

**迁移的核心意义**：

1. **稳定性保障**：微软官方 SDK 经过广泛测试，稳定性远超自研方案
2. **兼容性提升**：官方 SDK 与 Windows 各版本深度适配
3. **安全合规**：符合微软安全标准和最佳实践
4. **功能扩展**：解锁更多 Windows 平台特有能力

---

#### 2.3 对 TRAE IDE 的具体影响

**安全性层面**：

- **更严格的隔离边界**：基于 Windows AppContainer 技术，提供进程级隔离
- **文件系统虚拟化**：沙箱内进程看到的文件系统是虚拟视图，无法直接访问敏感目录
- **网络访问控制**：可精细控制沙箱内进程的网络访问权限
- **系统资源限制**：通过 Windows Job Objects 限制 CPU、内存等资源使用

**性能层面**：

- **原生系统调用**：使用官方 API 而非用户态 Hook，减少性能开销
- **优化的进程创建**：利用 Windows 沙箱的优化路径，加快命令启动速度
- **内存效率**：减少自研隔离层的内存占用

**兼容性层面**：

- **Windows 10/11 全版本支持**：从 Windows 10 1507 到 Windows 11 26H1
- **不同版本 Windows 行为一致**：消除版本差异带来的兼容性问题
- **与其他 Windows 应用协作**：更好地与系统级应用（如浏览器、终端）协同工作

**功能扩展层面**：

- **浏览器深度集成**：利用官方浏览器 API 实现更深度的浏览器控制
- **系统级操作支持**：支持更多系统级操作，如窗口管理、剪贴板访问等
- **MSIX 打包支持**：为未来 TRAE 的 MSIX 分发打下基础
- **WinUI 3 界面**：为未来界面现代化提供技术基础

---

#### 2.4 与 Browser 配置聚合页的关联

MSSDK 接入与 Browser 配置聚合页存在技术关联：

1. **浏览器控制能力增强**：通过 MSSDK 的官方浏览器 API，TRAE 可以更稳定地控制内置浏览器
2. **沙箱内浏览器操作**：浏览器自动化操作可以在沙箱环境内安全执行
3. **配置项扩展**：未来可能通过 Browser 配置聚合页配置更多基于 MSSDK 的浏览器行为

#### 2.6 总结

**Windows 接入 MSSDK** 是 TRAE 在 Windows 平台上的一项底层技术升级，核心价值在于：

| 维度 | 提升内容 |
|------|---------|
| **安全性** | 基于微软官方沙箱技术，提供更可靠的命令执行隔离 |
| **性能** | 利用官方 SDK 的优化实现，减少性能开销 |
| **兼容性** | 更好地支持 Windows 10/11 的各种版本和配置 |
| **扩展性** | 为后续 Windows 平台特定功能（浏览器深度集成、系统级操作等）打下基础 |
| **合规性** | 符合微软安全标准和最佳实践 |
