## 二、前置准备

在开始安装前，请确保环境满足以下要求。

### 2.1 系统要求

MonkeyCode采用控制台与开发环境分离的架构，不同组件有不同的硬件要求：

| 组件 | CPU | 内存 | 磁盘 | 说明 |
|------|-----|------|------|------|
| **控制台** | 2核 | 4GB | 40GB | 用户管理、任务调度、权限控制等管控功能 |
| **开发环境** | 8核 | 16GB | 100GB | 代码编辑、运行环境、AI任务执行 |

> **提示**：如果计划在开发环境本地运行大语言模型（如Ollama或vLLM），需要额外配置GPU资源。模型规模不同，GPU显存要求差异较大（7B模型建议8GB+显存，70B模型建议80GB+显存）。

### 2.2 Docker环境准备

MonkeyCode基于Docker容器化部署，请先完成Docker环境安装：

**行动项1：验证Docker环境**
- [ ] 执行 `docker --version`，确认Docker版本≥20.10
- [ ] 执行 `docker compose version`，确认Docker Compose版本≥2.0
- [ ] 执行 `docker info`，确认Docker服务正常运行

如未安装Docker，可执行以下命令：

```bash
# Ubuntu/Debian 系统
curl -fsSL https://get.docker.com | bash -s docker --mirror Aliyun
systemctl enable --now docker
```

### 2.3 网络要求

- [ ] 服务器可访问互联网（用于拉取Docker镜像和下载安装脚本）
- [ ] 确保以下端口未被占用：80（控制台Web端口）、443（HTTPS端口，可选）
- [ ] 如使用公有云模型，确保服务器可访问对应模型服务商API地址
- [ ] 内网用户可通过企业代理访问外网（需提前配置Docker代理）

## 三、在线安装步骤

MonkeyCode提供一键在线安装脚本，简化部署流程。

**行动项2：执行在线安装**
- [ ] 登录到目标服务器（建议使用root用户或具有sudo权限的用户）
- [ ] 执行官方安装命令：

```bash
bash -c "$(curl -fsSL 'https://monkeycode-ai.com/online/install')"
```

- [ ] 等待脚本自动完成以下操作：
  1. 检查系统环境和Docker依赖
  2. 拉取最新的MonkeyCode镜像
  3. 生成配置文件
  4. 启动容器服务
  5. 输出访问地址和初始管理员信息

**验证安装**
- [ ] 执行 `docker ps`，确认MonkeyCode相关容器状态为`Up`
- [ ] 打开浏览器访问安装脚本输出的地址（默认为 `http://<服务器IP>`）
- [ ] 看到MonkeyCode登录页面即表示安装成功

> **注意**：如服务器有防火墙，请开放80端口（或自定义端口）以允许外部访问。

## 四、初始配置

安装完成后，需要进行系统初始配置。

### 4.1 管理员账号设置

**行动项3：配置管理员账号**
- [ ] 使用安装脚本输出的初始管理员账号登录系统
- [ ] 首次登录后立即修改默认密码
- [ ] 完善管理员邮箱（用于接收系统通知）
- [ ] 启用双因素认证（推荐，提升账号安全性）

密码建议：长度≥12位，包含大小写字母、数字和特殊字符。

### 4.2 内网访问地址配置

**行动项4：配置访问地址**
- [ ] 进入「系统设置」→「基础配置」
- [ ] 配置「系统访问地址」为内网可访问的URL（如 `http://monkeycode.company.internal`）
- [ ] 如需HTTPS，上传SSL证书并配置443端口
- [ ] 配置「Cookie安全域名」，确保跨域访问正常
- [ ] 保存配置后刷新页面，使用新地址验证访问

### 4.3 成员权限管理

**行动项5：配置团队成员**
- [ ] 进入「用户管理」→「成员列表」
- [ ] 点击「添加成员」，输入成员邮箱和角色
- [ ] 配置角色权限：
  - **管理员**：拥有所有权限，可管理系统配置和用户
  - **开发者**：可创建项目、使用AI编码、提交代码
  - **观察者**：仅可查看项目和进度，不可编辑
- [ ] 成员收到邀请邮件后，点击链接完成注册
- [ ] 验证：使用新成员账号登录，确认权限符合预期

## 五、模型配置

MonkeyCode支持三种模型接入方式，可根据企业需求灵活选择。

### 5.1 云端模型配置

接入公有云模型服务（Kimi/DeepSeek/GLM等）是最简单的方式。

**行动项6：配置云端模型**
- [ ] 进入「系统设置」→「模型管理」
- [ ] 点击「添加模型」，选择要接入的模型服务商
- [ ] 填写模型配置信息：
  - 模型名称：自定义显示名称
  - API Endpoint：服务商API地址（通常有默认值）
  - API Key：从服务商控制台获取的密钥
- [ ] 点击「测试连接」，验证API Key有效且网络连通
- [ ] 保存配置，将模型设置为「启用」状态
- [ ] 创建测试项目，验证AI对话和代码生成功能正常

**常用模型服务商获取API Key地址：**
- Kimi（月之暗面）：https://platform.moonshot.cn/
- DeepSeek：https://platform.deepseek.com/
- GLM（智谱AI）：https://open.bigmodel.cn/

> **安全提示**：API Key属于敏感信息，请妥善保管，避免泄露。建议在模型服务商处配置IP白名单，限制只有MonkeyCode服务器可调用。

### 5.2 本地模型配置（Ollama/vLLM）

对于有更高数据安全要求的场景，可部署本地模型。

**行动项7：部署本地模型（以Ollama为例）**
- [ ] 在开发环境服务器（或独立GPU服务器）安装Ollama：

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

- [ ] 拉取所需模型（以Qwen2 7B为例）：

```bash
ollama pull qwen2:7b
```

- [ ] 启动Ollama服务，确认监听端口（默认11434）：

```bash
# 验证Ollama服务正常
curl http://localhost:11434/api/tags
```

- [ ] 配置Ollama允许外部访问（如需从其他服务器调用）：

```bash
# 编辑Ollama服务配置，添加环境变量 OLLAMA_HOST=0.0.0.0
systemctl edit ollama.service
```

**行动项8：接入本地模型到MonkeyCode**
- [ ] 进入「系统设置」→「模型管理」
- [ ] 点击「添加模型」，选择「OpenAI兼容协议」（Ollama/vLLM均兼容OpenAI协议）
- [ ] 填写配置：
  - 模型名称：如「本地Qwen2-7B」
  - API Endpoint：`http://<ollama服务器IP>:11434/v1`
  - API Key：无需填写（或填写任意字符串，Ollama不校验）
  - 模型标识：填写Ollama中的模型名（如 `qwen2:7b`）
- [ ] 点击「测试连接」，验证能正常调用
- [ ] 保存并启用
- [ ] 创建测试项目，验证本地模型可正常生成代码

> **Anthropic协议兼容**：如本地模型服务支持Anthropic协议，可选择「Anthropic兼容协议」类型进行配置，配置方法类似。

### 5.3 内部模型网关接入

对于已建设统一模型平台的企业，可接入内部模型网关。

**行动项9：接入企业模型网关**
- [ ] 确认内部模型网关兼容OpenAI或Anthropic协议
- [ ] 获取网关地址、认证凭证、可用模型列表
- [ ] 进入「系统设置」→「模型管理」
- [ ] 选择对应协议类型，填写网关地址和认证信息
- [ ] 如网关有自定义请求头，在「高级配置」中添加
- [ ] 测试连接，验证通过后保存
- [ ] 在项目中选择该模型，验证端到端流程

## 六、基础使用流程

完成配置后，即可开始使用MonkeyCode进行AI协作开发。

### 6.1 创建项目

**行动项10：创建第一个项目**
- [ ] 登录后点击「新建项目」
- [ ] 填写项目信息：
  - 项目名称：清晰描述项目用途
  - 项目描述：简要说明项目背景和目标
  - 选择模型：选择要使用的AI模型
  - 选择开发环境：配置代码运行环境（Node.js/Python/Go等）
- [ ] 点击「创建」，等待开发环境初始化（约1-2分钟）
- [ ] 进入项目工作区，看到代码编辑器和终端界面即为成功

### 6.2 编写需求

**行动项11：编写清晰的需求文档**
- [ ] 在项目的「需求」面板中，用自然语言描述你要实现的功能
- [ ] 需求建议包含：
  - 功能目标：这个项目/功能要解决什么问题
  - 技术栈：使用什么语言、框架、数据库
  - 功能清单：列出具体要实现的功能点
  - 界面要求：如有UI，描述界面布局和交互
- [ ] 可上传参考图片、设计稿作为附件
- [ ] 点击「提交需求」，AI会开始理解和分析需求

> **技巧**：需求描述越清晰具体，AI生成的代码质量越高。避免模糊表述如"做一个好看的网站"，尽量给出具体参考。

### 6.3 拆分任务

**行动项12：确认任务拆分**
- [ ] AI分析需求后，会自动拆分为多个具体的开发任务
- [ ] 查看任务列表，确认任务拆分合理
- [ ] 如有需要，可手动调整：
  - 拆分过粗的任务可进一步细化
  - 遗漏的任务可手动添加
  - 不需要的任务可删除
- [ ] 调整任务优先级，设置依赖关系
- [ ] 确认任务计划后，开始执行

### 6.4 AI协作开发

**行动项13：与AI协作完成开发**
- [ ] 从第一个任务开始，点击「开始执行」
- [ ] AI会自动进行：
  - 分析任务需求
  - 编写代码
  - 保存到文件
  - 运行和调试
- [ ] 在AI执行过程中，你可以：
  - 实时查看代码变化
  - 在对话中给AI反馈和指导
  - 要求AI修改或优化某个部分
  - 中断执行，调整方向
- [ ] 一个任务完成后，验证功能是否符合预期
- [ ] 确认无误后，标记任务为「完成」，继续下一个任务

### 6.5 代码预览测试

**行动项14：预览和测试项目**
- [ ] 开发过程中，可随时点击「预览」按钮查看运行效果
- [ ] Web项目会自动打开预览窗口，实时查看页面
- [ ] 使用终端面板可执行命令、运行测试
- [ ] 检查代码：
  - 功能是否完整实现
  - 代码是否有明显错误
  - 界面是否符合预期
- [ ] 如需修改，直接在对话框中告诉AI需要调整的地方
- [ ] 全部任务完成后，进行整体测试
- [ ] 测试通过后，可导出代码或推送到Git仓库

## 七、在线版体验

如果你暂时不想部署私有版本，可先使用官方在线版体验功能。

**行动项15：体验在线版**
- [ ] 打开浏览器访问：https://monkeycode-ai.com/console/tasks
- [ ] 使用手机号或邮箱注册账号
- [ ] 登录后即可创建项目，体验完整功能
- [ ] 在线版每天提供3000万免费Token额度，足够个人学习和小型项目使用
- [ ] 体验后如满意，再考虑部署私有化版本

> **注意**：在线版数据存储在公有云，请勿上传敏感项目和代码。企业正式使用建议部署私有化版本。

## 八、移动端使用

MonkeyCode支持移动端浏览器访问，方便随时随地查看和管理项目。

**行动项16：使用移动端**
- [ ] 确保手机可访问MonkeyCode地址（内网需连接企业VPN）
- [ ] 打开手机浏览器，输入MonkeyCode访问地址
- [ ] 使用账号登录
- [ ] 移动端可进行的操作：
  - 查看项目列表和进度
  - 查看任务状态
  - 补充需求描述
  - 给AI发送消息指令
  - 查看代码变更
  - 接收通知提醒
- [ ] 建议将地址添加到手机桌面书签，方便快速访问

> **提示**：复杂的代码编辑操作建议在电脑端完成，移动端更适合查看进度、补充需求、响应紧急问题等场景。
