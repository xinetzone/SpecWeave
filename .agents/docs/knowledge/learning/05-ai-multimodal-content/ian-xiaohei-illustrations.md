# Ian Xiaohei Illustrations（小嘿插图）学习笔记

## 一、项目概述

- **项目名称**：Ian Xiaohei Illustrations（小嘿插图）
- **作者**：Ian（ByteDance 产品设计师）
- **开源协议**：MIT
- **仓库地址**：https://github.com/helloianneo/ian-xiaohei-illustrations
- **核心定位**：为中文文章中的「认知锚点」生成配图，而非简单装饰

## 二、核心差异点

与普通 AI 绘图工具的区别：

| 维度 | 普通绘图 | 小嘿插图 |
|------|---------|---------|
| 目标 | 生成一幅"好看"的图像 | 把文章中的认知动作画出来 |
| 驱动方式 | 自由 prompt | 先识别认知锚点，再选择其一可视化 |
| 产出格式 | 任意风格 | 16:9 白底手绘线稿 |
| 角色定位 | 无角色或装饰性角色 | "小黑"必须承担核心动作 |

## 三、核心 IP：「小黑」角色设计原则

不是吉祥物，而是「认真参与系统运转的荒诞工作者」。

五条设计原则：

1. **小黑不是吉祥物** — 不是贴纸、不是装饰，是参与系统运转的角色
2. **小黑必须承担核心动作** — 如表达「信任需要一块块铺设」时，小黑拿着信任桥木桩在路上走
3. **去掉小黑画面仍然成立 → 说明太装饰了** — 小黑承担主要工作，而非站那儿卖萌
4. **视觉风格克制** — 纯白背景、黑色手绘线稿（细线、微抖）、留白较大（主体占 2/3~3/4）
5. **奇怪、有趣、清新，但不幼稚也不卖萌**

## 四、三种使用模式

| 模式 | 功能 |
|------|------|
| 配图规划模式 | 只输出 shot list，不生成图片 |
| 直接生成模式 | 直接生成 4–8 张 PNG |
| 单图生成模式 | 单个概念生成一张图 |

生成的图片自动保存到 `workspace/assets` 文件夹。

## 五、工作原理

1. 输入文章 → 分析可配图位置 → 输出 shot list
2. 每张图只讲一个认知锚点（判断 / 流程结构 / 状态 / 隐喻）
3. 为每个锚点创建低级、荒诞的物理环境，让小黑在其中工作
4. 生成 16:9 横版白底手绘线稿，配少量红、橙、蓝三色手写批注

核心原则：**一张图只表达一个意思，不做 PPT 信息图、不写说明书。**

## 六、使用方法

```bash
# 克隆仓库
git clone https://github.com/helloianneo/ian-xiaohei-illustrations.git

# 复制 skill 到 Codex skills 目录
mkdir -p "${CODEX_HOME:-$HOME/.codex}/skills"
cp -R ./ian-xiaohei-illustrations "${CODEX_HOME:-$HOME/.codex}/skills/"
```

在 Codex 中输入：

```
Use $ian-xiaohei-illustrations 为这篇中文文章设计并生成 5 张小黑怪诞正文配图。
```

## 七、局限性与注意事项

- 仅适用于**中文文章配图**，不适合商业插画、品牌 KV、精致扁平插画
- 不支持儿童卡通、可爱 IP、表情包风格
- 不支持需要严格可编辑矢量源文件的项目
- 默认不输出 PPTX / PDF / Keynote / SVG / HTML 格式
- **图中中文文字越少越稳定**，AI 图像模型可能出现错误文字、幻觉标签、风格漂移

## 八、核心启发

文章结尾的核心反思：

> 以前找配图，会在 Unsplash 上搜索关键词来获取图片。翻几页，感觉差不多了就把它们放进去。直到看到 Ian Xiaohei Illustrations，才发现自己一直把配图当装饰。

关键洞察：**工具负责生产，判断负责选择。**

- AI 生成内容越来越容易，但「在哪里配上图片、用什么样的图片最恰当」这样的判断仍然很有价值
- 不是给文章「配一张图」，而是让 AI 把文章中一个重要的认知动作画出来
- 很多时候，读者忘了文字，却记得那张图
