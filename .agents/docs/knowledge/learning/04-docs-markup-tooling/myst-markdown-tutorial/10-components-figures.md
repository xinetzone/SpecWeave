# 第10章：扩展组件 - 图片与表格

图片和表格是技术文档中最常用的可视化元素。MyST 在标准 Markdown 基础上提供了更强大的图片控制和多种表格格式，支持自动编号、交叉引用、复杂布局等专业出版需求。

:::{note}
本章配套示例文件：[examples/figures-tables-demo.md](examples/figures-tables-demo.md)。
:::

## 10.1 图片指令（Image Directive）

### 10.1.1 基础语法回顾

标准 Markdown 的图片语法简单直接：

```markdown
![替代文本](图片路径 "可选标题")
```

- 替代文本（alt）：图片无法显示时的替代文字，屏幕阅读器朗读内容
- 图片路径：相对路径、绝对路径或 URL
- 可选标题：鼠标悬停时显示的提示文字

### 10.1.2 `{image}` 指令增强选项

需要精确控制尺寸、对齐等属性时，使用 ` ```{image} ` 指令：

````markdown
```{image} photo.jpg
:alt: 示例图片
:width: 80%
:align: center
:class: rounded shadow
```
````

**常用选项：**

| 选项 | 作用 | 示例值 |
|------|------|--------|
| `:width:`/`:height:` | 尺寸 | `600px`、`80%` |
| `:align:` | 对齐 | `left`/`center`/`right` |
| `:alt:` | 替代文本（无障碍必需） | `系统架构图` |
| `:class:` | 自定义 CSS 类 | `img-fluid`、`border` |
| `:scale:` | 缩放比例（%） | `50`、`75` |

### 10.1.3 对比与选择

| 特性 | `![alt](path)` | `{image}` 指令 | `{figure}` |
|------|---------------|----------------|------------|
| 简单插入 | ✅ | ✅ | ✅ |
| 尺寸/对齐控制 | ❌ | ✅ | ✅ |
| 标题（caption） | ❌ | ❌ | ✅ |
| 自动编号引用 | ❌ | ❌ | ✅ |

**选择建议**：简单插图用基础语法，需尺寸控制用 `{image}`，需要标题引用的正式图表用 `{figure}`。

## 10.2 图（Figure）组件

`{figure}` 是完整的图形对象，包含图片、标题、编号，可通过 `{numref}` 自动交叉引用，是学术/技术出版中"图 X"的标准实现。

### 10.2.1 基本语法

````markdown
```{figure} architecture.png
:label: fig-architecture
:alt: 系统架构图
:width: 90%
:align: center

图：本系统采用微服务架构，分为接入层、业务层、数据层。
```
````

- `:label:`：标签，用于 `{numref}` 引用，建议以 `fig-` 前缀命名
- 空行后的内容是图题（caption），显示在图片下方并自动编号

### 10.2.2 使用 {numref} 自动引用

```markdown
如 {numref}`fig-architecture` 所示，系统分为三层架构。
```

渲染后："如 图 10.1 所示，系统分为三层架构。"（编号根据章节自动生成）

:::{tip}
自定义引用格式：`` {numref}`图 %s <fig-architecture>` `` 可显式指定前缀。
:::

**其他选项**：`:figwidth:`（figure容器宽度）、`:name:`（同:label:）

**使用场景**：论文/报告中的正式图表、需要正文引用的插图、产品截图说明、流程图/架构图。

## 10.3 表格

MyST 支持多种表格格式，从简单到复杂，满足不同场景需求。

### 10.3.1 管道表格（Pipe Tables）

最常用的简单表格，用 `|` 分隔列，`-` 分隔表头：

```markdown
| 姓名 | 年龄 | 职业 |
|------|------|------|
| 张三 | 28 | 工程师 |
| 李四 | 32 | 设计师 |
```

**对齐控制**：分隔线中用 `:` 指定：
- `:---` 左对齐（默认），`:---:` 居中，`---:` 右对齐

```markdown
| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| A | B | C |
```

**适用场景**：简单行列数据、快速编辑。

:::{warning}
管道表格不支持单元格合并，列宽由内容自动决定。
:::

### 10.3.2 网格表格（Grid Tables）

用 `+`、`-`、`|` 绘制，支持单元格跨行/跨列合并：

```markdown
+------------------------+------------+----------+
| 标题行（跨3列）         |            |          |
+============+===========+ 合并单元格 > 右列     |
| 第一列     | 第二列    |            |          |
+            +-----------+------------+----------+
| （跨两行） | 单元格    | 独立单元格 | 单元格   |
+------------+-----------+------------+----------+
```

:::{note}
- 表头分隔用 `=`
- `>` 表示向右合并
- 空左边框表示向下跨行
- `+` 必须严格对齐
:::

**适用场景**：需要合并单元格的复杂表格、表头分层。

### 10.3.3 list-table 指令

用缩进列表定义表格，适合脚本生成或长内容：

````markdown
```{list-table} 列表定义表格
:header-rows: 1
:widths: 20 30 50

* - 列1
  - 列2
  - 列3
* - 行1列1
  - 行1列2
  - 长内容自动换行无需手动对齐
```
````

- `:header-rows: 1` 第一行是表头
- `:widths:` 列宽比例
- `* -` 开头新行，`-` 对应列

**适用场景**：脚本生成表格、单元格内容长、避免手动对齐 `|`。

### 10.3.4 `{table}` 指令

给表格加标题、标签，支持自动编号引用（类似 figure）：

````markdown
```{table} 各版本功能对比
:label: table-feature-compare
:align: center

| 功能 | 社区版 | 专业版 | 企业版 |
|------|:------:|:------:|:------:|
| 基础编辑 | ✅ | ✅ | ✅ |
| 协作功能 | ❌ | ✅ | ✅ |
```
````

引用：`详见 {numref}`table-feature-compare`` → "详见 表 10.1"。

**适用场景**：需要正文引用的正式表格、带标题的数据表。

### 10.3.5 csv-table 指令

从 CSV 数据生成表格，支持内联或外部文件：

````markdown
```{csv-table} 配置项说明
:header: "名称,类型,默认值,说明"
:widths: 15 10 10 40

host,string,"localhost",服务器监听地址
port,integer,8080,服务端口号
debug,boolean,false,调试模式
```
````

从外部文件导入：
````markdown
```{csv-table}
:file: data/config.csv
:header-rows: 1
```
````

**适用场景**：从 Excel/CSV 导出的数据、批量生成表格。

### 10.3.6 表格对齐与宽度

| 需求 | 实现方式 |
|------|---------|
| 表格整体对齐 | `:align: left/center/right` |
| 单元格对齐 | 管道表格用 `:---`/`:---:`/`---:` |
| 列宽比例 | list-table/csv-table 用 `:widths:` |

## 10.4 常见问题与陷阱

:::{warning}
**图片路径问题**：
- 推荐**相对路径**：`./images/arch.png`，避免绝对路径
- 注意文件名大小写（Linux区分大小写）
:::

:::{warning}
**中文路径兼容**：
- 推荐英文命名：`system-architecture.png`
- Windows 路径分隔符用 `/` 而非 `\`
:::

:::{warning}
**表格对齐错误**：
- 网格表格 `+` 必须**严格对齐**，差一个空格都无法渲染
- 渲染异常时先用 list-table 验证数据，再调整格式
:::

:::{tip}
**大表格处理**：过宽表格在移动端会溢出，建议精简列数或拆分小表。
:::

## 10.5 小结

- **图片**：简单插图用 `![alt](path)`，尺寸控制用 `{image}`，正式引用用 `{figure}`
- **管道表格**：最简单快速，适合简单数据
- **网格表格**：支持单元格合并，适合复杂布局
- **list-table**：列表定义，适合代码生成/长内容
- **{table}**：带标题标签，支持 {numref} 引用
- **csv-table**：从 CSV 数据生成，适合批量导入
