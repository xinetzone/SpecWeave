# 第2章：基础语法（上）- 文本与格式

本章讲解 MyST Markdown 的基础文本格式语法，覆盖标题、段落、文本强调、水平线、转义字符与 HTML 支持，这些是构建文档最基础的元素。

---

## 2.1 标题（Headings）

标题是文档结构的骨架，MyST 使用标准 Markdown 的 ATX 标题语法，支持 1-6 级标题。

### 语法与示例

在标题文本前添加对应数量的 `#` 符号，`#` 与文本之间保留一个空格：

```markdown
# 一级标题（H1）
## 二级标题（H2）
### 三级标题（H3）
#### 四级标题（H4）
##### 五级标题（H5）
###### 六级标题（H6）
```

### 锚点自动生成规则（Slugify）

MyST 自动为每个标题生成 URL 友好的锚点 ID：
- 中文保留原样，英文转小写、空格替换为 `-`
- 标点符号自动移除，重复标题自动加数字后缀

| 标题文本 | 生成的锚点 |
|----------|-----------|
| `## 快速开始` | `#快速开始` |
| `## Installation Guide` | `#installation-guide` |

:::{tip}
**MyST 增强：标题引用**

MyST 支持通过 `{ref}` 角色引用标题，无需手动维护锚点：

```markdown
(section-install)=
## 安装指南

请参阅 {ref}`section-install` 了解详细步骤。
```
:::

### 使用场景建议

- 每篇文档只使用一个一级标题（H1）
- 遵循层级递进，不要跳级（如 H2 下直接用 H4）
- 建议最多使用到四级标题（H4），过深层级增加阅读负担
- 需要交叉引用的章节，建议添加显式标签 `(label)=`

## 2.2 段落与换行

段落是文本的基本组织单元，MyST 遵循 CommonMark 规则。

### 标准段落

连续文本行构成一个段落，段落之间通过**空行**分隔：

```markdown
这是第一个段落。段落可以包含多行文本，
只要中间没有空行，这些行会被视为同一段落，
渲染时自动合并为连续文本。

这是第二个段落。空行之后开始新段落。
```

- 同一段落内的换行被视为空格，多行自动合并
- 段落间空行产生段落间距，行首/行尾空白自动忽略

### 硬换行（强制换行）

在行尾添加**两个空格**然后换行，可在同一段落内产生强制换行（`<br>`）：

```markdown
第一行文本（行尾有两个空格）␠␠
第二行文本，与上一行同属一个段落但强制换行。
```

:::{note}
**MyST 段落处理特点**

- 混合缩进（tab + 空格）不会意外触发代码块
- 正确处理中英文混排间距，中文标点无需额外空格
:::

### 使用场景建议

- 优先用空行分隔段落，这是最清晰的写法
- 硬换行适用于诗歌、地址、列表项内多行文本等场景
- 避免过度使用硬换行，让渲染器自动处理文本流
- 中英文混排时，建议在中英文间添加一个空格提升可读性

## 2.3 文本强调

MyST 支持多种文本强调样式，用于标记重点内容。

### 语法与示例

| 样式 | 语法 | 代码示例 | 显示效果 |
|------|------|---------|---------|
| 粗体 | `**text**` / `__text__` | `这是**粗体文本**` | **粗体文本** |
| 斜体 | `*text*` / `_text_` | `这是*斜体文本*` | *斜体文本* |
| 粗斜体 | `***text***` | `这是***粗斜体***` | ***粗斜体*** |
| 删除线 | `~~text~~` | `~~已删除文本~~` | ~~已删除文本~~ |
| 行内代码 | `` `code` `` | `` 使用 `print()` `` | `print()` |
| 高亮 | `==text==` | `==重点内容==` | ==重点内容== |

:::{tip}
**MyST 注意事项**

- 删除线 `~~text~~` 和高亮 `==text==` 属于 GFM 扩展语法，MyST 原生支持
- 高亮标记在部分解析器中可能需要启用 `markdown-it-mark` 扩展
- 行内代码含反引号时，可用双反引号包裹：``` ``code with ` backtick`` ```
:::

### 使用场景建议

| 样式 | 适用场景 |
|------|----------|
| **粗体** | 核心概念、关键步骤、重要警告 |
| *斜体* | 新术语首次出现、书名、变量名 |
| ~~删除线~~ | 内容变更、价格对比、已废弃功能 |
| `` `行内代码` `` | 代码片段、命令、变量名、配置键 |
| ==高亮== | 重点提示、关键时间节点 |

## 2.4 水平线（Horizontal Rule）

水平线用于分隔文档的不同主题或章节，产生视觉分隔线。

### 语法与示例

在单独一行使用三个或更多的 `---`、`***` 或 `___`，中间可有空格：

```markdown
上面的内容

---

下面的内容
```

三种写法渲染效果相同，都是一条横贯页面的水平线。

:::{note}
**注意：连字符歧义问题**

使用 `---` 作为水平线时，前面必须有空行！否则会被解析为 Setext 二级标题：

```markdown
这会变成二级标题
---
```
:::

### 使用场景建议

- 优先用标题层级组织结构，不要过度使用水平线
- 同一文档内建议统一使用 `---`
- 提示框、代码块等组件前后无需额外添加水平线
- `---` 也用于 YAML frontmatter 的结束标记（文档顶部）

## 2.5 转义字符

转义字符用于显示 Markdown 中有特殊语法含义的字符，使其作为普通文本显示。

### 语法说明

在特殊字符前添加反斜杠 `\`，即可转义该字符，显示为字面字符。

### 可转义特殊字符

| 字符 | 语法用途 | 转义写法 |
|------|---------|---------|
| `\` | 转义字符本身 | `\\` |
| `` ` `` | 行内代码标记 | `` \` `` |
| `*` `_` | 粗体/斜体标记 | `\*` `\_` |
| `{}` `[]` `()` | MyST 指令/链接标记 | `\{` `\}` `\[` `\]` `\(` `\)` |
| `#` `+` `-` `.` `!` | 标题/列表/图片标记 | `\#` `\+` `\-` `\.` `\!` |
| `\|` `:` | 表格/指令分隔符 | `\|` `\:` |

### 代码示例

```markdown
\* 这不是列表项
\# 这不是标题
价格：\$99（$ 默认用于数学公式）
\{name\} 表示占位符
```

:::{tip}
**MyST 新增转义场景**

1. **大括号转义**：普通文本中显示 `{}` 需转义，否则可能被解析为角色导致警告
2. **代码内无需转义**：反引号包裹的代码块/行内代码中，所有字符按字面显示
3. **美元符号**：正文中显示 `$` 建议转义为 `\$`，避免被识别为数学公式
:::

### 使用场景建议

- 优先用代码格式（反引号）包裹技术字符，比转义更易读
- 转义用于句子中偶然出现的特殊字符
- 看到 "Unknown role" 警告时，检查是否有未转义的 `{}`

## 2.6 HTML 支持

MyST 允许直接嵌入原生 HTML，实现 Markdown 无法覆盖的复杂布局。

### 语法与示例

直接在 Markdown 中编写 HTML 标签，渲染时保留并渲染为对应元素：

```markdown
这是普通 Markdown 文本。

<div style="padding: 1rem; background: #f0f7ff; border-radius: 8px;">
  <h3 style="margin-top: 0;">自定义 HTML 区块</h3>
  <p>这是通过原生 HTML 实现的带样式区块。</p>
</div>

行内也可以使用 <span style="color: red;">HTML 标签</span>。

<details>
  <summary>点击展开详情</summary>
  折叠面板内容。
</details>
```

- 块级 HTML（`<div>`/`<table>`/`<details>`）前后需要空行分隔
- 块级 HTML 内部的 Markdown 默认不解析
- 行内 HTML（`<span>`/`<kbd>`/`<sub>`）可直接嵌入段落

:::{warning}
**安全限制说明**

出于安全考虑，MyST 默认限制：
1. 过滤危险标签：`<script>`/`<iframe>`/`<embed>`/`<form>` 等
2. 移除 JavaScript：事件属性和 `javascript:` 链接
3. 如需放宽限制，可在 `myst.yml` 配置 `allowDangerousHtml: true`（仅信任内容时使用）
:::

### 使用场景建议

- **优先使用 MyST 原生语法**：能用 Markdown/指令实现的效果，不要用 HTML
- HTML 仅用于 MyST 无法实现的场景（复杂自定义布局）
- 常见需求都有 MyST 对应方案：折叠面板用 `{dropdown}`、上下标用 `{sub}`/`{sup}`、键盘按键用 `{kbd}`
- 技术文档尽量避免自定义内联样式，保持与主题一致
