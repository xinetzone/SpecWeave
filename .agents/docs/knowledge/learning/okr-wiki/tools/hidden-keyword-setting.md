# 隐藏关键词设置教程

> **来源**: [OKR隐藏关键词设置教程](https://www.dingteam.com/help/653)

---

## 概述

在 OKR 目标管理过程中，企业或团队的 OKR 文本常涉及财务数据、客户隐私、战略规划、销售目标等敏感机密内容。为避免核心信息泄露，确保成员仅查看职责范围内的目标内容，叮当OKR 工具推出「隐藏关键词」功能，通过精准权限管控，实现敏感信息的安全防护。

---

## 如何使用？

### 开启隐藏关键词

1. 在目标编辑状态，鼠标选中需要隐藏的关键词
2. 会弹出隐藏关键词的弹窗
3. 打开开关并对可见范围进行设置
4. 设置后保存 OKR，就完成了隐藏关键词的设置

**效果**:
- 有可见权限的人，可以看到完整的关键词
- 无可见权限的人，看到的关键词会用「\*\*\*」代替

### 修改关键词可见范围

1. 进入 OKR 编辑状态
2. 点击一下关键词
3. 在弹窗内对可见范围进行修改
4. 然后保存 OKR

---

## 可见范围设置

| 范围 | 说明 |
|------|------|
| **直接主管** | 通讯录中的直接主管 |
| **二级主管** | 通讯录中的二级主管 |
| **三级主管** | 通讯录中的三级主管 |
| **目标对齐的OKR负责人** | 当前目标对齐的 OKR 的负责人 |
| **对齐目标的OKR负责人** | 对齐当前目标的 OKR 负责人（细化到 O 或者 KR） |
| **被@的人** | 目标和 KR 中被@的人 |
| **其他人** | 自定义从通讯录中选择 |

---

## 注意事项

### 权限交集原则

关键词与目标的可见范围取交集，即：用户需要同时满足关键词和 OKR 的可见范围条件，才能查看关键词。

### 编辑权限

可以编辑目标的都可以编辑可见范围。

### 不受可见权限影响的角色

以下角色不受可见范围的影响：
- 目标负责人
- 空间管理员（主、子）
- 企业管理员

### 特殊场景权限

| 场景 | 权限说明 |
|------|----------|
| **目标确认人** | 在目标确认时不受可见范围的影响 |
| **总结接收人** | 总结的接收人在总结中有所有任务和总结的可见权限 |
| **评分人** | 评分时不受可见范围的影响 |

## 最佳实践

### Need-to-Know 原则

基于"Need-to-Know"原则（信息安全领域的核心准则之一），仅向必须知晓某类信息的人员提供必要的最小化信息，以平衡效率与安全。

### 分级保密策略

**公司级 OKR**:
- 公开：与战略定位、价值观导向及中长期愿景相关的宏观目标，以及可量化的非敏感成果指标
- 隐藏：涉及财务预测、核心技术进展等敏感数据

**部门级 OKR**:
- 公开：与跨部门协作、资源调配或战略支撑直接相关的目标
- 隐藏：涉及技术细节、成本预算、未公开项目进度等敏感数据

**个人级 OKR**:
- 公开：岗位职责范围内的关键任务目标，以及与团队/部门目标强关联的进度指标
- 隐藏：涉及组织战略敏感点的具体行动计划
