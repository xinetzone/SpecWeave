# OKR制定流程

> **来源**: [叮当OKR落地实操详细指南](https://www.dingteam.com/help/582)

## 制定时间

**OKR周期第一周**

## 两种制定模式

### 模式一：自上而下制定OKR

#### 步骤1：创建公司级OKR

**操作**: 老板/高层/管理员在叮当OKR里创建好公司级的OKR

**要点**:
- 从年度战略拆解
- 明确季度核心方向（1-2个O）
- 设定挑战性目标

#### 步骤2：拟定部门级OKR

**操作**: 部门主管在对齐公司级OKR的基础上，拟定部门的OKR，并组织部门成员围绕目标进行讨论确认

**要点**:
- 承接公司O
- 结合业务分工设定
- 确保纵向对齐

#### 步骤3：设定个人OKR

**操作**: 每个员工根据团队的OKR设定自己的OKR

**要点**:
- 承接团队O
- 突出个人贡献
- 避免与他人重复

#### 步骤4：讨论并确定

**操作**: 部门主管和员工就员工个人OKR讨论并确定

**要点**:
- 确保目标合理
- KR难度适中
- 对齐团队方向

---

### 模式二：自下而上制定OKR

#### 步骤1：设置个人OKR

**操作**: 每个员工在周期初设置个人的OKR

**要点**:
- 基于个人职责和重点工作
- 设定有挑战性的目标

#### 步骤2：一对一沟通调整

**操作**: 在员工设置目标后一周内，主管一对一的跟员工沟通、调整OKR

**要点**:
- 确保与部门目标对齐
- 调整目标难度
- 提供指导和反馈

#### 步骤3：讨论确定部门OKR

**操作**: 在部门会议中讨论确定部门OKR

**要点**:
- 汇总个人目标
- 提炼部门重点
- 确保与公司战略对齐

#### 步骤4：传达公司OKR

**操作**: 在全员会议向每位员工传达公司OKR

**要点**:
- 明确公司方向
- 解释战略意图
- 激发团队动力

#### 步骤5：系统设置与对齐

**操作**: 所有的OKR通过叮当OKR系统进行设置、跟踪和管理，并且与上级目标对齐

**要点**:
- 使用OKR系统管理
- 建立对齐关系
- 确保全员可见

---

## OKR制定需注意的问题

### 注意事项1：明确公司使命

**操作**: 在目标设定之前，先明确公司的使命

**目的**: 确保目标没有偏离公司的使命

### 注意事项2：控制目标数量

**建议**: 一个公司一个周期内的目标不要超过3个

**目的**: 确保聚焦重点，避免目标过多导致分散精力

### 注意事项3：KR必须可衡量

**要求**: KR一定是可衡量的，符合SMART原则

**SMART原则**:
- **S**pecific: 明确性
- **M**easurable: 可衡量性
- **A**chievable: 可实现性
- **R**elevant: 相关性
- **T**ime-bound: 时限性
