# OKR与KPI的区别

> **来源**: [最全OKR制定指南（2种思路+7类方法）](https://www.dingteam.com/help/584)

## 九维度对比

| 维度 | OKR | KPI |
|------|-----|-----|
| **导向** | 目标驱动，主动挑战 | 指标驱动，被动考核 |
| **设定方式** | 团队自主设定，上下对齐 | 上级下达，侧重达标 |
| **目标难度** | 挑战性（60%-70%达成为优） | 可达成性（100%达成为合格） |
| **评分与薪酬** | 不直接挂钩 | 强相关 |
| **适用场景** | 创新、研发、项目型团队 | 生产、运营、销售等稳定流程 |
| **数量** | 少而精（每个O配2-4个KR） | 多而全，覆盖多维度考核 |
| **调整频率** | 季度调整 | 年度固定 |
| **透明度** | 全员公开 | 有限公开 |
| **聚焦点** | 结果导向 | 过程监控 |

## 核心差异

### 1. 思维模式不同

- **OKR**: 关注"我们要去哪里？"——目标导向，鼓励突破
- **KPI**: 关注"我们要达成什么指标？"——指标导向，侧重达标

### 2. 设定方式不同

- **OKR**: 自下而上与自上而下结合，团队共同参与设定
- **KPI**: 自上而下，上级下达考核指标

### 3. 目标难度不同

- **OKR**: 有挑战性，60%-70%完成率是理想状态
- **KPI**: 可达成，100%完成是基本要求

### 4. 与绩效的关系不同

- **OKR**: 与绩效脱钩，是成长工具
- **KPI**: 与绩效强相关，是考核工具

## 适用场景

| 场景 | 推荐方法 | 原因 |
|------|----------|------|
| **创新业务** | OKR | 需要鼓励探索和突破 |
| **成熟业务** | KPI | 需要稳定运营和达标 |
| **研发团队** | OKR | 需要创新和挑战 |
| **生产运营** | KPI | 需要稳定和可预测 |
| **市场推广** | OKR | 需要探索和增长 |
| **客户服务** | KPI | 需要稳定质量 |

## OKR与KPI的结合

在实际应用中，很多公司会将OKR与KPI结合使用：

- **OKR**: 用于设定挑战性目标，驱动创新和增长
- **KPI**: 用于监控日常运营，确保基础业务稳定

**示例**:
- 公司OKR: "打造行业领先的产品"
- 部门KPI: "产品合格率达到99%"
- 团队OKR: "完成核心功能迭代"
- 个人KPI: "代码提交及时率达到95%"
