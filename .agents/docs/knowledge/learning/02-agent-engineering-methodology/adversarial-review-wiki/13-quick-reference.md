# 13、快速参考速查表

---

## 说明

本章是对抗性审查速查卡，**打印贴桌上即用**。所有详细内容请参见对应章节。

---

## 1. 对抗性审查快速流程

**知识研究流程**：
```
跨领域扫描 → 来源分级 → 可信度评分 → 五维验证 → 偏差防御 → 异常标记 → 验证日志
```

**代码/AI协作流程**：
```
明确目标 → 四大攻击者并发 → 问题清单分级 → 交叉验证 → 修复 → 回归验证
```

**核心原则**：证伪导向（证明它有问题）> 证实导向（证明它正确）

---

## 2. 四大攻击者角色速查

| 角色 | 标识 | 攻击目标 | 核心检查项 |
|------|------|---------|-----------|
| 🔴 安全攻击者 | 🔴 | 注入/越权/泄露 | 输入验证、权限校验、敏感数据、SQL/XSS注入 |
| 🟡 性能攻击者 | 🟡 | OOM/死循环/DoS | 大输入处理、循环边界、正则回溯、递归终止 |
| 🟢 边界攻击者 | 🟢 | 异常/空值/极端值 | null/undefined、类型转换、时区/时间、特殊字符 |
| 🔵 时序攻击者 | 🔵 | 并发/竞态/重试 | 幂等性、锁粒度、重试退避、缓存一致性 |

---

## 3. 可信度分级速查

| 标记 | 等级 | 判定标准 | 使用场景 |
|------|------|---------|---------|
| 🟢 | A级 | ≥2个独立一级来源交叉验证 | 核心论据，可直接引用 |
| 🔵 | B级 | 单一权威一级来源 | 辅助论据，标注来源 |
| 🟡 | C级 | 二级来源/待验证 | 仅作参考，必须标注待验证 |
| 🔴 | D级 | 存疑/矛盾/利益冲突 | 排除，不纳入核心档案 |

---

## 4. 异常标记速查

| 标记 | 含义 |
|------|------|
| ⚠️ | 待验证：尚未完成多源交叉验证 |
| ❓ | 存疑：与权威来源矛盾或逻辑存疑 |
| ⚖️ | 争议：领域内存在明确学派争议 |
| 🔍 | 利益冲突：来源与结论存在关联 |

---

## 5. 认知偏差10项自检清单（评审前必问）

评审前，对着这10个问题问自己：

1. **我是否只找了支持自己观点的证据？**（确认偏差）
2. **我是否因为是权威说的就没追问证据？**（权威偏差）
3. **我是否被第一个信息/第一印象锚定了？**（锚定效应）
4. **我是否为了团队和谐压制了不同意见？**（群体思维）
5. **我是否只看到了成功案例没看到失败者？**（幸存者偏差）
6. **我是否事后觉得"我早就知道"？**（事后归因）
7. **我是否因为某方面好就认为全都好？**（晕轮效应）
8. **我是否因为容易想到就觉得概率高？**（可得性启发）
9. **我是否因为问题表述方式影响了判断？**（框架效应）
10. **我是否低估了风险，觉得"不会发生在我身上"？**（乐观偏差）

## 6. 工具选型快速决策

```
快速基线扫描（想先跑起来）→ Garak（NVIDIA，分钟级）
工程团队CI/CD集成 → Promptfoo（OpenAI收购，开箱即用）
深度红队研究/复杂攻击 → PyRIT（Microsoft，多轮编排）
多轮对话/聊天机器人 → DeepTeam（对话场景优化）
合规/标准化评估 → Inspect AI（UK AISI）
Llama生态/安全分类器 → Purple Llama（Meta）
```

**ASR参考阈值**：基线目标 < 20%，成熟目标 < 10%，高危漏洞必须 0%

---

## 7. 五维验证流程（知识研究）

| 维度 | 检查内容 |
|------|---------|
| 1. 来源资质 | 发布机构、作者资质、发表渠道、利益冲突 |
| 2. 交叉验证 | 关键事实≥2个独立来源三角验证 |
| 3. 时效性 | 经典/实证/前沿分别评估"证据保质期" |
| 4. 逻辑一致 | 内部矛盾、循环论证、偷换概念、论据充分性 |
| 5. 偏差识别 | 对照10项偏差清单系统检查 |

## 8. 多Agent对抗Prompt核心骨架

```
你现在扮演[角色]攻击者（🔴安全/🟡性能/🟢边界/🔵时序），
你的唯一目标是找到代码/文档/方案中的BUG/漏洞/缺陷。
不要做任何肯定性评价，只报告问题，按严重程度排序。
重点构造极端输入、边界条件、恶意场景来"搞破坏"。
```

## 9. CI/CD Pipeline分层参考

```

    ↓

    ↓

    ↓
生产 → 持续监控+定期演练 → 漏洞闭环
```
