## 一、什么是工具注册表

工具注册表（Tool Registry）是管理Agent"手脚"的地方。如果说模型是大脑，工具就是Agent的手脚——它要能打开日历查时间、打开地图看距离、进入预订系统看有没有包间。工具注册表管的就是这些"手脚"。

## 二、工具注册表要回答的三个问题

1. **有哪些工具能用**——Agent的能力边界由可用工具决定
2. **每个工具需要什么参数**——调用签名、参数类型、必填项
3. **调用失败怎么办**——重试策略、降级方案、错误处理

## 三、生活场景理解

家庭聚餐助理要办事，不能只靠嘴说：
- 能打开日历查时间
- 能打开地图看距离
- 能进入餐厅预订系统看有没有包间

工具注册表管的就是：有哪些工具、每个工具需要什么参数、调用失败怎么办。

## 四、文章Agent中的工具应用

文章Agent的工具集：
- **读取选题池**：知道当前有哪些待写主题
- **检索历史文章**：避免同一个观点反复写
- **保存草稿**：把选题、提纲、正文保存下来方便继续修改

## 五、关键区分：工具注册表 vs 策略引擎

> **工具注册表只回答"有没有这个工具、怎么调用这个工具"，不回答"这次该不该调用"。**

"该不该调用"是策略引擎（Strategy Engine）的职责。这是一个重要的边界：
- 工具注册表 = 能力清单（你有什么手脚）
- 策略引擎 = 使用规则（什么时候能动、什么时候不能动）

## 六、设计原则

- 工具描述要精确：模糊的工具描述是Agent乱调用的主要原因
- 参数校验要前置：在调用前检查参数完整性和类型
- 错误处理要明确：超时、异常、返回值异常都要有处理策略
- 工具权限要最小化：Agent只能调用完成任务所必需的工具

## 七、常见误区

1. **工具过多**：给Agent太多工具会导致"逛超市"问题——面对20个工具不知道用哪个
2. **没有失败处理**：工具调用失败后Agent卡住或无限重试
3. **工具描述模糊**："搜索资料"这种描述不如"搜索内部知识库中关于XX主题的历史文章"
4. **混淆工具和策略**：在工具层面做业务判断（应该由策略引擎负责）
