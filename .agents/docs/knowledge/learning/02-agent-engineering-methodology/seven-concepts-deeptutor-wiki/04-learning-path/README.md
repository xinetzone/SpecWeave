## 模块简介

恭喜你完成了前三个模块的学习！现在你已经了解了七概念的理论基础、DeepTutor的架构设计，以及两者之间的映射关系。本模块的目标是帮助你：

1. **选对路径**：根据你的时间和目标，选择最适合的阅读顺序
2. **动手实践**：通过可操作的练习，把理论知识转化为实际分析能力
3. **自我检验**：用结构化的清单检查自己的掌握程度
4. **持续进阶**：找到深入学习的方向和资源

## 三条学习路径概览

根据你的目标和可用时间，选择一条最适合你的路径：

| 路径 | 适合人群 | 预计耗时 | 核心目标 |
|------|---------|---------|---------|
| **🌱 快速入门** | 时间有限、想快速了解全貌 | 30分钟 | 知道七概念是什么，能看懂DeepTutor的核心设计思路 |
| **👨‍💻 系统学习** | 想完整掌握方法论并会初步应用 | 2小时 | 理解七概念的要素，能识别真实系统中的方法论体现 |
| **🧠 实战拓展** | 想把七概念用到自己的项目中 | 2小时+练习 | 能用七概念独立分析任意系统，指导自己的设计决策 |

### 路径选择建议

- 如果你是第一次接触七概念，从**快速入门**开始
- 如果你是开发者/产品经理，想把方法论用于工作，走**系统学习**
- 如果你已经在项目中尝试用七概念，直接进入**实战拓展**

## 学习建议

### ✅ 推荐做法

1. **先做练习再看答案**：做练习时不要翻前面的章节，先独立完成再对照检查
2. **用自己的项目练手**：第二个练习强烈建议选你正在做的真实项目，收获最大
3. **不要追求完美**：第一次分析不用做到100分，能找到几个对应点就是进步
4. **反复回查**：做练习时如果忘了某个概念的要素，回去理论章节查，不要硬猜

### ❌ 常见误区

- **跳过练习直接看延伸阅读**：七概念是实践方法论，不做练习永远学不会
- **只做轻量练习不做深度分析**：30分钟练习只能检验记忆，60-90分钟的分析才能真正掌握
- **死记硬背公理和要素**：理解背后的逻辑比记住文字更重要
