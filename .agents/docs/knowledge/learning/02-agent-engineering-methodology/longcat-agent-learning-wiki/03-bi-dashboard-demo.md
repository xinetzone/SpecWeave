## 四、BI数据看板项目实战全流程

### 4.1 项目需求

本次测试中，作者向LongCat-2.0下达了以下需求：

> 请从零开发一个BI数据看板网站，要求100%本地运行。用户可以上传Excel数据，系统自动识别字段并生成数据预览、核心指标卡片和可视化分析结果；使用SQLite保存用户上传记录、分析配置和历史结果；使用ECharts展示图表；页面要像专业SaaS工具，包含清晰的工作台、上传入口、数据预览、图表分析和历史记录页面。不需要接第三方登录。

这个需求涵盖了前端、后端、数据库三个层面，是一个完整的SaaS MVP（最小可行产品）系统，比较接近真实编程Agent的工作强度。

### 4.2 开发流程观察

作者在测试中重点观察了三个关键能力：

1. **是否会先拆任务**：Agent能否将复杂需求分解为可执行的子任务
2. **能否持续修改项目文件**：Agent能否在项目目录中持续创建和修改文件
3. **遇到报错后能否根据终端反馈继续修**：这是Agent与普通代码生成器的本质区别

### 4.3 开发过程

LongCat-2.0在Claude Code中的开发过程按以下顺序推进：

**第一步：任务拆解**

Agent首先将需求拆分为多个子任务，包括：项目结构搭建、Excel解析模块、后端API路由、数据分析引擎、前端页面开发、启动脚本等。

**第二步：搭建项目结构**

创建项目目录和基础文件结构，包括Flask后端框架、前端HTML模板、SQLite数据库初始化脚本等。

**第三步：Excel上传和解析**

实现Excel文件上传功能，自动读取表头和行数据。测试中上传了一个4万多行、21列的Excel文件，表头识别和数据展示均正常，说明文件读取链路完整。

**第四步：后端API路由和数据分析**

实现后端API路由，修改自动分析引擎代码。包括数据统计、字段聚合、关键指标计算等功能。

**第五步：前端页面开发**

开发专业SaaS风格的前端页面，包括：
- **工作台首页**：项目概览和快捷入口
- **数据上传页**：拖拽或点击上传Excel文件
- **数据预览页**：展示上传数据的表头和行数据
- **图表分析页**：使用ECharts生成可视化图表（核心指标卡片、字段统计等）
- **历史记录页**：保存和展示历史分析结果

**第六步：启动脚本和测试**

自动生成启动脚本，并测试运行。在测试过程中发现了一个bug：`db`数据库的`conn_close`未定义错误。Agent自动识别并修复了这个问题。

### 4.4 报错修复能力

修复`conn_close`错误的流程展示了Agent的核心能力：

1. 运行测试时发现数据库连接关闭错误
2. Agent自动读取错误信息，定位到相关代码文件
3. 修改数据库连接关闭的代码逻辑
4. 重新运行测试，验证修复是否成功

这一过程展现了Agent的**loop engineering**能力：不是一次性生成完美代码，而是通过"编码→测试→报错→修复"的循环不断迭代完善。

### 4.5 最终成果

项目最终产出了一个完整的BI数据看板网站，包含：

- **前端**：专业SaaS风格的页面，使用ECharts展示图表
- **后端**：Flask API，处理数据上传、分析和历史记录
- **数据库**：SQLite本地存储，保存用户上传记录、分析配置和历史结果
- **数据持久化**：关闭页面后重新打开，之前上传的数据和分析结果仍然存在

向后端API发送请求，所有页面和API均正常响应。整个开发过程约10多分钟。

### 4.6 关键观察

作者在测试中得出了一个重要结论：**LongCat-2.0接入Claude Code后，不只是能写代码片段，而是有一定项目级开发能力**。它的优势不在于"某一段代码写得多漂亮"，而在于能把一个项目从需求到页面、从页面到数据、从数据到图表、从图表到历史记录——这几块串联起来，持续往前推进。
