## 三、洞察：第一性原理本质分析

### 3.1 复盘（Retrospective）
**误区**：复盘=写总结/事后仪式 → **剥离假设**：未经反思的经验是记忆非知识；复盘产出是可证伪洞察而非报告 → **公理**：复盘是对已发生事件的结构化反事实推理，通过"如果X则Y"推演将时序经验转化为因果知识。

### 3.2 洞察（Insight）
**误区**：洞察=金句/感悟 → **剥离**：可复用洞察必须形式化为「条件C→机制M→做A→导致B」四元组；洞察是萃取中间产物 → **公理**：洞察是跨情境可迁移规律，最小形式为「触发条件→因果机制→可操作结论→预期结果」四元组，不能复用的只是感想。

### 3.3 萃取（Extraction）
**误区**：萃取=摘抄/信息压缩 → **剥离**：萃取是知识形态转换（隐性→显性、具体→抽象）；四层漏斗渐进精炼 → **公理**：萃取是知识从隐性经验到显性模式的形式化转换，通过事件→洞察→模式→原则四层漏斗逐级提升抽象层级。

### 3.4 原子提交（Atomic Commit）
**误区**：原子提交=小提交 → **剥离**：原子=逻辑不可分割（单一职责/可独立回滚）；最优粒度由revert意愿决定 → **公理**：原子提交是变更集的不可分割单一职责单元，满足"同因同果、可独立回滚、review无认知跳跃"三条件，本质是因果内聚性保证。

### 3.5 原子化（Atomization）
**误区**：原子化=拆文件/越小越好 → **剥离**：原子化含拆分与合并两方向；最优粒度是U型曲线最低点（认知负荷vs导航成本） → **公理**：原子化是复杂系统向最优信息粒度的收敛过程，在认知负荷与导航成本间找到平衡点，使各单元独立可读、单一职责、链接完整。

### 3.6 第一性原理（First Principles）
**误区**：第一性原理=从零开始/反经验 → **剥离**：质疑假设但不否定经验结论；未知场景中是最快方法（确定性推导vs随机游走） → **公理**：第一性原理是从认知边界内不可证伪公理出发，通过推导链自下而上重构方案的思维方式——剥离类比，回归本质要素，从公理层推导结论。

### 3.7 对抗性审查（Adversarial Review）
**误区**：对抗性审查=挑错/Code Review → **剥离**：确认偏误是认知固有缺陷；对抗思维应嵌入每一步（元审查） → **公理**：对抗性审查是主动寻找证伪证据的认知防御机制，系统性构造"方案何时失败"的反例，在交付前暴露确认偏误盲区，本质是鲁棒性验证。
