# 引用即触发（Reference-as-Trigger）协作模式

> **来源**：从 `retrospective-insight-create-apps-directory-meta-analysis.md#L139` 的引用触发实施案例中萃取
> **模式类型**：方法论模式
> **验证轮次**：1（本次实施）

---

## 一、模式定义

"引用即触发"是一种 AI 协作中的精确指令模式——用户不通过文字描述"要做什么"，而是通过**选中文件中的特定行号范围**来指定执行范围。智能体读取该位置的内容，将其中列出的所有行动项或机会**一次性全部实施**，消除需求描述中的歧义与遗漏。

## 二、适用场景

- 复盘报告或洞察报告中列出了明确的"潜在机会"或"行动项"列表
- 用户希望**全部实施**这些机会，而非逐项挑选
- 机会列表足够结构化（表格或编号列表），智能体可直接解析

## 三、模式结构

```
用户动作：选中文件#L起始行-结束行（或单行 #L行号）
智能体行为：
  1. 读取被引用位置的完整内容
  2. 解析所有列出的机会/行动项
  3. 按可行性排序（高→中→低）
  4. 按独立性分组并行执行
  5. 全部完成后汇总产出
```

## 四、工作原理

### 4.1 为何优于文字描述

| 方式 | 用户输入 | 智能体理解 | 歧义风险 |
|------|---------|-----------|---------|
| 文字描述 | "执行洞察报告第三章的潜在机会" | 需定位章节、解析内容 | 章节引用可能出错 |
| 引用触发 | 选中 `#L139` | 直接读取确切位置 | 几乎为零 |

### 4.2 关键前提

1. **被引用的文件必须存在且格式清晰**：机会列表应使用表格或编号列表
2. **智能体需具备文件读取能力**：能读取被引用文件的确切行范围
3. **上下文连续性**：用户与智能体在同一个会话中，或被引用的文件在当前工作区内可访问

## 五、使用示例

### 场景：洞察报告列出 4 项潜在机会

```
洞察报告 insight-report.md 第 139 行：
## 三、潜在机会

| 序号 | 机会 | 可行性 |
| 1 | 建立模式成熟度分级 | 高 |
| 2 | 开发行动项扫描脚本 | 中 |
| 3 | 跨项目元分析 | 中 |
| 4 | 建立指令模式库 | 低 |
```

**用户操作**：在 IDE 中选中该章节，智能体获得引用 `insight-report.md#L139`

**智能体行为**：
```
→ 读取 #L139 起的完整章节
→ 解析出 4 项机会，可行性为 [高,中,中,低]
→ 按独立性分组：
   - 第 1 轮并行：#1(高) + #2(中) + #4(低)（均为纯创作，无依赖）
   - 第 2 轮并行：#3(中分析型) + 索引同步
→ 4/4 全部实施完成
```

## 六、与传统模式的关系

"引用即触发"是 [短指令模式库](short-command-patterns.md) 的扩展形式——当用户不需要通用指令（如 `复盘+洞察+萃取`），而是需要执行**特定文档中列出的具体任务**时，引用触发比关键词指令更精确。

它同时是 [复盘→洞察→导出知识闭环](../retrospective-knowledge/review-insight-export-loop.md) 的下游执行环节——闭环产出的"潜在机会"，通过"引用即触发"进入实施阶段。

## 七、使用原则

1. **引用目标必须是结构化列表**：表格或编号列表，而非叙述性段落
2. **一次性全量实施**：不要逐项挑选——列表中的所有条目应一视同仁
3. **并行优先**：按条目之间的独立性分组，最大化子代理并行度
4. **索引同步**：实施完成后立即将新资产注册到索引体系
