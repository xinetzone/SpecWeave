# 视觉原子化原则

## 核心概念

一张图只表达一个意思，不做 PPT 信息图、不写说明书。将复杂信息拆解为最小认知单元，每个单元用独立图像表达。

## 跨领域同构验证

原子化思维在文档工程和视觉设计两个看似无关的领域，得出了相同的结论：

| 领域 | SpecWeave 文档原子化 | 视觉原子化（Ian Xiaohei） |
|------|-----------------|-------------------|
| 操作对象 | Markdown 文档 | 认知锚点 |
| 原子单元 | 单一职责的文档片段 | 单一概念的配图 |
| 组合方式 | 文档间交叉引用 | shot list 中的图序列 |
| 反模式 | 大而全的单文件 | PPT 信息图 / 说明书式配图 |
| 核心理念 | 高内聚、低耦合 | 一张图一个意思 |

## 设计原则

1. **单一概念**：每张图只表达一个认知概念
2. **独立可理解**：脱离上下文仍能理解图的含义
3. **无冗余信息**：不包含与核心概念无关的元素
4. **序列组合**：通过图序列表达复杂逻辑关系

## 适用场景

- 技术文档配图
- 教育内容可视化
- 演示文稿概念图
- 产品说明图

## 核心价值

这是一个跨领域的设计原则验证：**原子化思维在文档工程和视觉设计两个看似无关的领域，得出了相同的结论。** 这证明原子化是一种普适性的设计原则。
