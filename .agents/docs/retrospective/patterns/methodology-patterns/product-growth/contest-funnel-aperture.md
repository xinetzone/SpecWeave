# 赛事漏斗孔径设计

## 一、核心问题

赛事漏斗设计中，每一层筛选标准的「松」与「紧」是一个零和博弈——报名层放得越宽，初赛层的压力就越大。如果各层之间的「孔径」衔接不畅，漏斗就会出现「腰部膨胀」风险。

**典型症状**：

```
报名层宽口径（最大化参与）
    → 大量合规但质量参差不齐的作品涌入
        → 初赛层评审压力激增
            → 评审标准一致性下降 / 人力不足 / 流程公正性受损
```

## 二、漏斗层级模型

赛事漏斗通常包含以下层级，每层有其最优「孔径」：

| 层级 | 目标 | 最优孔径 | 孔径过宽风险 | 孔径过窄风险 |
|------|------|---------|-------------|-------------|
| 报名层 | 最大化参与 | **宽** | 初赛评审压力激增 | 参与量不足 |
| 初赛层 | 精准筛选质量 | **准** | 腰部膨胀、评审疲劳 | 优质项目遗漏 |
| 复赛层 | 深度评估潜力 | **深** | 评审成本过高 | 有潜力项目被误杀 |
| 决赛层 | 严苛把关结果 | **严** | 奖项空缺影响公信力 | 争议性项目进入 |

## 三、孔径衔接原则

### 原则 1：逐层收敛

每一层的输出量应显著低于输入量，形成清晰的收敛路径：

```
报名层：1000+ → 初赛层：300 → 复赛层：50 → 决赛层：10
```

### 原则 2：孔径递减

后一层应比前一层有更严格的筛选标准：

- 报名层：仅筛选「合规」（不评判创意好坏）
- 初赛层：筛选「可用」（功能完整、基本体验）
- 复赛层：筛选「优秀」（创新性、完成度、商业潜力）
- 决赛层：筛选「卓越」（综合最优）

### 原则 3：评审维度差异化

每一层应有不同的评审维度，避免重复劳动：

| 层级 | 评审维度 | 评审方式 | 评审时长 |
|------|---------|---------|---------|
| 报名层 | 合规性（格式、字数、原创声明） | 机器预检 + 人工抽查 | <1 分钟/件 |
| 初赛层 | 功能完整性、基本体验 | 评委打分制 | 5-10 分钟/件 |
| 复赛层 | 创新性、商业潜力、社区价值 | 评委 + 导师评审 | 30 分钟/件 |
| 决赛层 | 综合竞争力、现场表现 | 路演 + 评委质询 | 60 分钟/项目 |

### 原则 4：信息透明传递

每个层级的筛选标准应提前向参赛者公开：

```
报名层：「不评判创意好坏，仅检查合规性」
初赛层：「公布评审四维度及权重」
复赛层：「公布晋级标准与评审流程」
决赛层：「公布奖项设置与评选规则」
```

## 四、「腰部膨胀」风险应对

当报名层孔径过宽导致初赛作品量超出预期时，可采取以下应对策略：

### 策略 1：自动分流机制

在初赛评审启动前，通过自动化的方式进行初步筛选：

- 作品完整度自动检测（文件是否齐全）
- 基本功能截图自动校验（是否可运行）
- 社区互动数据作为辅助参考

### 策略 2：评审标准细化

将初赛评审维度进一步细化，通过多维度打分实现精准筛选：

| 维度 | 权重 | 说明 |
|------|------|------|
| 功能完整性 | 30% | 核心功能是否可用 |
| 用户体验 | 25% | 界面设计、交互流畅度 |
| 技术实现 | 20% | 代码质量、性能表现 |
| 创意新颖 | 25% | 与同类作品的差异化 |

### 策略 3：社区辅助评审

引入社区投票作为辅助参考，分担专业评委压力：

```
专业评委（200 件核心筛选）
    + 社区投票（500 件初步分流）
        = 700 件作品的合理评审路径
```

### 策略 4：评审周期延长

预留足够的评审周期，避免仓促评审导致标准一致性下降。

## 五、实施检查清单

- [ ] 每一层是否有明确的筛选目标？
- [ ] 各层之间的孔径是否有递减设计？
- [ ] 评审维度是否实现了层级差异化？
- [ ] 筛选标准是否提前向参赛者公开？
- [ ] 是否有应对「腰部膨胀」的预案？
- [ ] 评审标准的一致性如何保障？
