# 知识库建设三阶段：生成→重组→精确化

## 模式类型
方法论模式（元方法论）

## 成熟度
L2 已验证（59个Wiki教程建设过程验证）

## 问题场景
批量建设知识库/文档体系/Wiki教程集合时，先追求精确还是先追求广度？"先写好每一篇再继续"会导致产出速度过慢，广度无法覆盖；"先快速写完再整理"又可能导致内容混乱、无法复用。需要明确的阶段顺序指导大规模内容生产。

## 核心定义

大规模知识库建设必须严格按三阶段顺序执行，**顺序不可颠倒**：

```mermaid
flowchart LR
    S1["<b>阶段1：批量生成</b><br/>快速覆盖广度<br/>接受不完美"] --> S2["<b>阶段2：分类重组</b><br/>建立结构体系<br/>可导航可发现"]
    S2 --> S3["<b>阶段3：精确化引用</b><br/>提升引用质量<br/>零断链零漂移"]
    style S1 fill:#fff3e0,stroke:#ff9800
    style S2 fill:#ffebee,stroke:#f44336
    style S3 fill:#e8f5e9,stroke:#4caf50
```

| 阶段 | 目标 | 核心问题 | 典型动作 | 速度 |
|------|------|---------|---------|------|
| **批量生成** | 快速覆盖主题广度 | "有还是没有？" | 主题分组并行写作、接受不完美、快速产出 | 快（日均产出多篇） |
| **分类重组** | 建立MECE结构 | "能不能找到？" | 建立分类体系、导航索引、决策树、CATEGORIES.md | 中（结构设计） |
| **精确化** | 提升引用质量和可信度 | "准不准确？" | 泛化引用精确化、修复断链、跨Wiki精确引用、单一数据源 | 慢（精修） |

## 解决方案

### 阶段1：批量生成（Generation）
**目标**：快速覆盖广度，接受不完美。

- **策略**：主题分组并行写作（可用多个sub-agent并行）
- **心态**：完成胜过完美，先有再好
- **接受**：引用可能是泛化的（指向目录首页而非具体章节）、格式可能不完全统一、有少量断链
- **禁忌**：在这个阶段追求引用精确化——每写一篇都精确化引用会导致速度骤降，广度无法覆盖
- **产出标准**：每个主题都有对应的内容文档，核心知识点覆盖完整

### 阶段2：分类重组（Reorganization）
**目标**：建立MECE分类体系和导航结构，让内容可被发现。

- **策略**：
  - 建立主题分类体系（如SpecWeave的8大主题分类）
  - 创建CATEGORIES.md分类说明文档
  - 建立README.md导航入口和决策树
  - 按分类组织目录结构
- **心态**：结构先于精确
- **产出标准**：
  - 从入口出发能<3次点击到达所有内容
  - 新内容有明确的归属分类（决策树可判断）
  - MECE：相互独立、完全穷尽（无重叠、无遗漏）

### 阶段3：精确化（Refinement）
**目标**：提升引用质量，消除断链和事实漂移。

- **策略**：
  - 应用cross-wiki-reference-directory-first模式：先读目标Wiki目录确认章节号，再写精确引用
  - 将泛化引用（如"详见XXX目录"）精确化为具体章节引用
  - 运行链接检查修复所有断链
  - 建立单一数据源消除事实表述漂移
  - 统一格式和术语
- **心态**：慢工出细活
- **产出标准**：零断链、引用精确到章节、术语统一、事实一致

## 支撑证据

**SpecWeave 59个Wiki建设验证**：
- **阶段1（批量生成）**：4个sub-agent并行写作，快速产出59个Wiki
- **阶段2（分类重组）**：按8大主题分类，创建CATEGORIES.md和learning/README.md导航
- **阶段3（精确化）**：应用cross-wiki-reference-directory-first模式，将泛化引用精确化，修复约1943处链接

**反例验证**：尝试在阶段1就精确化引用→产出速度过慢，一天只能完成2-3个Wiki，远低于目标速度。

## 复用场景
- 大规模知识库、文档体系、教程集合建设
- Wiki批量生产
- 技术文档体系从0到1搭建
- 学习笔记系统建设

## 反模式

| 反模式 | 表现 | 后果 |
|--------|------|------|
| **顺序颠倒（先精确再求广）** | 每写一篇都精确化引用、反复修改格式 | 产出速度过慢，广度无法覆盖，项目延期 |
| **永远在生成** | 一直写新内容，从不做分类整理 | 内容成为信息垃圾堆，无法被发现和复用 |
| **跳过重组直接精确化** | 没有分类结构就开始修复引用 | 改了一批引用后目录结构调整，又要重改 |
| **完美主义拖延** | "这篇还不够好，我再改改"，不进入下一阶段 | 整体进度停滞，投入产出比极低 |

## 实施检查清单

### 阶段1完成门
- [ ] 核心主题全部有对应的内容文档
- [ ] 没有为了精确化而放慢生成速度
- [ ] 接受了少量不完美（泛化引用、小断链）

### 阶段2完成门
- [ ] 建立了MECE分类体系（无重叠、无遗漏）
- [ ] 有清晰的导航入口和决策树
- [ ] 从入口出发<3次点击能到达所有内容
- [ ] 新内容有明确归属（决策树可判断）

### 阶段3完成门
- [ ] 所有泛化引用已精确化为章节级引用
- [ ] 链接检查零断链
- [ ] 术语统一，事实表述无漂移
- [ ] 格式规范一致
