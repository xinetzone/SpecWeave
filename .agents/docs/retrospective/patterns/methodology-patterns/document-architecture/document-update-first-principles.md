# 文档更新第一性原理模式（Document Update First-Principles）

## 模式类型
方法论模式（文档架构/知识库维护）

## 成熟度
L2 已验证（1次验证：vibe-coding-prompts-learning-analysis.md v1.1→v1.2更新，成功识别4个本质缺失并完成补全）

## 适用场景

更新文档（或做任何内容改进）时，避免"类比式更新"陷阱，从本质目标出发识别核心缺失。

| 场景 | 适用度 | 说明 |
|------|--------|------|
| 学习文档/知识条目版本更新 | ✅✅✅ 核心场景 | 已有文档需要深化，而非从零创建 |
| 知识库维护/内容优化 | ✅✅✅ 核心场景 | 批量更新或改进现有知识库条目 |
| README/索引文档迭代 | ✅✅ 强烈推荐 | 避免只做格式美化而忽略实质改进 |
| 复盘报告/洞察报告完善 | ✅✅ 推荐 | 补充缺失的验证/证据/链接 |
| 全新文档创建 | ⚠️ 间接适用 | 创建新文档时可用"本质目标五问"辅助结构设计 |
| 纯格式修正（标点/错别字） | ❌ 不适用 | 格式修正不需要第一性原理分析 |

## 问题背景

更新文档时最容易犯的错误是**类比式更新**：
- 看看别人加了什么，我也加什么
- 看看上次改了什么格式，我也改什么格式
- 给文字加加粗、调调标点（格式美化）
- 补充一些细节（信息增量）
- 调整段落顺序（结构优化）

这些都不是"错"，但都没有触及本质——**文档作为"可复用知识资产"的核心缺失是什么？**

类比式更新的根本问题在于：它从"现有内容"出发做增量，而非从"本质目标"出发做校验。如果现有内容本身就缺失了关键维度，类比式更新只会让已有内容更完善，却永远填补不了结构性缺失。

## 核心规则

### 本质目标五问

更新任何文档前，先回答五个问题：

| 问题 | 目的 | 反例（类比式更新不会问） |
|------|------|------------------------|
| 1. 这个文档为谁服务？ | 明确目标读者 | 只看"上次改了什么" |
| 2. 它要解决什么问题？ | 明确文档使命 | 只看"别人加了什么" |
| 3. 当前内容是否真正解决了？ | 识别效果差距 | 默认"写了就解决了" |
| 4. 缺失了什么本质性的东西？ | 识别结构性缺失 | 只找表面缺陷（错别字/格式） |
| 5. 新增内容是否服务于本质目标？ | 防止无关增量 | "加了总比不加好" |

### 类比式更新 vs 第一性原理更新对比

| 维度 | 类比式更新 | 第一性原理更新 |
|------|----------|-------------|
| 出发点 | 现有内容和已有做法 | 文档的本质目标 |
| 思维方式 | 快思考（System 1）：模仿、套用 | 慢思考（System 2）：追问本质 |
| 典型操作 | 格式美化、细节补充、结构微调 | 识别本质缺失、补全核心维度 |
| 风险 | 结构性缺失永远无法填补 | 需要更多思考时间，可能推翻已有结论 |
| 适用时机 | 纯格式修正、错别字修正 | 版本升级、内容深化、知识库维护 |

### 反面案例：vibe-coding学习文档v1.1

v1.1的内容质量并不差——结构完整、信息准确。从类比角度看似乎已经"完成了学习"。但回到本质目标——"建立可复用知识资产，指导未来实践"——立刻发现4个本质缺失：

| 本质目标 | v1.1缺失 | v1.2补充（第一性原理更新） |
|---------|---------|------------------------|
| 指导未来实践 | 只有"卡兹克说"和"SpaceX案例"，没有本项目亲身验证 | 加入2026-07-09反面案例作为L3验证 |
| 可复用 | 只有理论，没有"知道≠做到"的实践警示 | 加入践行鸿沟章节和决策前三查检查点 |
| 知识网络 | 学习文档没有链接到已沉淀的模式文件 | 补全双向链接 |
| 解答真实困惑 | FAQ只有理论问题，没有实践者最关心的"为什么学了还犯错" | 新增Q7-Q10解答践行问题 |

如果用类比思维做这次更新，可能只会：给文字加加粗、补充卡兹克文章细节、调整段落顺序——这些都没有触及本质。

## 实施步骤

### 步骤1：判断是否需要第一性原理更新
- 是版本升级（v1.x→v2.0）还是小修小补？
- 是否涉及内容深化而非纯格式修正？
- 距离上次实质性更新是否已有一段时间，积累了新的实践验证？
- 回答任一"是"→使用第一性原理更新

### 步骤2：执行本质目标五问
- 拿出一张纸或空白文档，写下五个问题的答案
- 特别关注第4问"缺失了什么本质性的东西"——这是最容易被类比思维跳过的
- 如果第4问答不上来，说明还没真正理解文档的本质目标

### 步骤3：识别本质缺失
- 将当前内容与本质目标做差距分析
- 区分"表面缺陷"（错别字、格式、链接失效）和"本质缺失"（缺少验证、缺少实践警示、缺少链接、缺少FAQ）
- 优先补全本质缺失，再处理表面缺陷

### 步骤4：验证新增内容
- 每一项新增内容，用第5问检验："这个内容是否服务于本质目标？"
- 如果答案是"加了总比不加好"，那大概率不该加
- 更新后运行链接验证等自动化检查

## 反模式

| 反模式 | 为什么错误 | 正确做法 |
|--------|----------|---------|
| 一上来就改格式/加粗 | 格式美化是类比思维的典型表现，不触及本质 | 先做五问分析，再动笔改内容 |
| "加点内容总比不加强" | 无关内容增加噪音，稀释核心信息 | 每一项新增必须通过第5问检验 |
| 只看其他文档怎么写的就模仿 | 其他文档可能也有同样的本质缺失 | 从自己的本质目标推导，而非模仿 |
| 更新后不做链接/引用验证 | 新增内容可能引入断链或无效引用 | 运行check-links.py等工具验证 |
| 把"补充更多细节"当更新 | 细节增量不解决结构性缺失 | 先补结构，再补细节 |
