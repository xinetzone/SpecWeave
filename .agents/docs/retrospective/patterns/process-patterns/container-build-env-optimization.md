# 容器化构建环境优化模式

## 触发场景

- 当 Docker 镜像构建涉及网络依赖和复杂环境配置时
- 适用于：需要构建包含大量依赖的开发/生产环境镜像的场景
- 不适用于：简单的单文件镜像构建；无网络依赖的镜像

## 核心做法

1. **镜像源优化**：使用稳定的镜像源（如清华源、阿里源），替换官方默认源
2. **超时重试配置**：增加网络超时时间（60+秒）和重试次数（10+次），应对不稳定网络
3. **命令链拆分**：将 `conda create && conda install` 等长命令链拆分为独立步骤
4. **验证环节添加**：在关键步骤后添加验证命令，确保工具链安装成功


5. **多阶段构建**：使用 Docker 多阶段构建减少最终镜像体积，分离构建环境和运行环境

## 反模式

- ❌ 在单条命令中串联多个安装步骤，无验证环节，失败后难以定位问题
- ❌ 依赖默认官方镜像源，不考虑网络环境差异
- ❌ 构建命令无重试机制，单次失败即中断整个构建流程

## 检验标准

验证关键工具链是否可用（版本号、可执行性），检查镜像体积是否优化，确认构建成功率是否提升。

## 迁移示例

这个模式可应用于 CI/CD 流水线中的容器镜像构建，以及本地开发环境的快速搭建。
