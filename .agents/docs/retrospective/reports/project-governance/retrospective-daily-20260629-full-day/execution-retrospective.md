## 二、关键决策分析

### 决策1：阶段守卫采用四层递进架构

**背景：** B1规范定义完成后，自然延伸出"如何强制执行"的问题。

**决策：** 采用B1(规范)→B2(离线检测)→C1(运行时拦截)→C2(可视化)四层递进，而非一步到位做运行时拦截。

**Why：** 
- 规范层先定义"什么是对的"
- 离线层先验证"能不能检测到错的"
- 运行时层再拦截"正在发生的错误"
- 可视化层最后回答"整体合规状况如何"

### 决策2：vendor/flexloop从只读升级为自有协作

**背景：** flexloop子模块初始作为第三方只读引用，但发现需要深度协同（Skill开发方法论、跨项目角色映射）。

**决策：** 调整为"自有协作模式"，vendor/.agents/由SpecWeave维护元数据层，vendor/flexloop/保持gitlink追踪自治。

**Why：** 完全萃取到SpecWeave主权区会导致重复维护和上游贡献困难；完全只读无法满足深度协同需求。"元数据层在我、内容层自治"的双区模式平衡了控制权和协作效率。

### 决策3：数据安全治理不新增DSO角色

**背景：** AI智能体数据安全治理体系需要明确责任归属。

**决策：** 不新增DSO（数据安全官）角色，采用现有6角色扩展模式，通过RACI矩阵分配数据安全职责。

**Why：** 
- 角色膨胀增加协作复杂度
- 现有角色（orchestrator/architect/developer/reviewer）已覆盖数据安全全链路
- RACI矩阵可明确"谁审批、谁执行、谁咨询、谁知会"
- 与阶段守卫机制天然集成（安全检查作为守卫节点）

### 决策4：命令Skill采用渐进式披露三层架构

**背景：** 5个命令Skill初版存在L1/L2内容重复问题（层断裂反模式）。

**决策：** 采用L0(ONBOARDING<100行)→L1(SKILL.md<500行)→L2(深度文档不限)三层架构，L1只保留触发词+决策树+核心步骤+安全清单，详细内容通过锚点引用L2。

**Why：** 新会话上下文窗口有限，一次性加载全部细节会挤占任务执行空间；渐进式披露让Agent按需加载，认知建立效率提升30%+。

## 三、成功因素

1. **问题驱动的连锁反应效率**：Mermaid bug → 治理闭环 → 安全模板 → 操作指南 → 模式萃取，一个问题触发了完整的治理链条，而非仅做点修复
2. **四层架构的递进式交付**：阶段守卫在2小时内完成B1→B2→C1→C2四层交付，每层建立在前一层验证基础上，降低了大规模代码的返工风险
3. **复盘即时性**：每个模块完成后立即产出复盘报告（阶段守卫、论坛bot、Mermaid、vendor、数据安全各有专项复盘），知识沉淀与开发同步进行
4. **基建-修复-扩展节奏**：三波次（早间基建→午间修复→下午扩展）自然切换，避免了"只基建不修复"或"只修复不扩展"的偏废
5. **原子提交频率高**：71次提交意味着平均每次提交约590行变更（实际大文件集中在少数提交），小步提交降低了集成风险

## 四、瓶颈与问题

### 问题1：提交信息中文编码问题

**现象：** 早期Git日志显示中文乱码（GBK/UTF-8不匹配）。

**根因：** Windows终端默认编码与Git输出编码不一致。

**修复：** 午间波次已修复（4c3faac），CI脚本编码安全完善。

### 问题2：Mermaid治理存在点修复偏误

**现象：** 第一次修复Mermaid渲染问题（5a17eed）仅改了换行符，遗漏了Markdown list解析冲突问题（7f302a0二次修复），随后又暴露了注释安全问题，最终才形成治理闭环。

**根因：** 初期采用"点修复"思维而非"治理闭环"思维，问题修复停留在表面症状。

**改进：** 后来形成了"bug修复→根因分析→安全模板→检测工具→操作指南→模式萃取"的完整治理闭环模型。

### 问题3：82个断链集中爆发

**现象：** vendor目录结构调整和文件移动后，出现82个file:///绝对路径断链。

**根因：** 大规模重构后链接检测滞后，前期使用了file:///本地绝对路径而非规范的相对路径。

**修复：** 修复check-links.py的file:///解析bug，全部转为相对路径引用。

### 问题4：高密度提交下的注释压缩

**现象：** 晚间3个大模块（RACI/5 Skills/方法论模式）在4分钟内提交，提交粒度偏粗。

**根因：** 晚间工作可能是白天思考的集中输出，开发过程在思维中完成，代码一次性生成。

**影响：** 粗粒度提交增加了代码审查和回滚的难度。
