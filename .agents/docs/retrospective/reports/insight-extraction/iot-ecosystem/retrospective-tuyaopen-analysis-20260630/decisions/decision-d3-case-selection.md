# 决策 D3：应用案例选择

**决策背景**：需要选择一个核心应用案例进行深度分析。

**备选方案**：

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| 方案A：MimiClaw | 本地优先 AI 助手 | 最具代表性，包含 AI+IoT 完整能力 | 分析复杂度较高 |
| 方案B：switch_demo | 智能开关 | 简单明了，易于理解 | 功能单一，模式较少 |
| 方案C：your_chat_bot | AI 聊天机器人 | AI 能力突出 | 缺少硬件集成 |

**最终选择**：**方案A：MimiClaw**

**决策依据**：
1. MimiClaw 展示了 TuyaOpen 的完整能力（AI + IoT + 云服务）
2. 包含多个可复用的设计模式（渠道适配器、LLM 适配器、本地优先、工具调用）
3. 是项目中最具创新性和代表性的应用案例

**决策影响**：
- **正面影响**：模式萃取更丰富，分析价值更高
- **负面影响/代价**：分析复杂度较高
- **影响范围**：模式萃取、知识点提炼

**事后评估**：
- **正确性**：✅ 正确
- **经验启示**：选择最具代表性的案例进行深度分析，能获得更多可复用知识
