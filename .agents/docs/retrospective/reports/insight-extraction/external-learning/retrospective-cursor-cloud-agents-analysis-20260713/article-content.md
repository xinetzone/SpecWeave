Cursor 推出了旗下里程碑式重磅更新 —— Cloud Agents。

团队近期分享了一个耐人寻味的细节：内部成员如今早已不再使用 Tab 键。

Tab 自动补全是 Cursor 起家的核心功能，也是无数开发者依赖的实用工具，可亲手打造这项功能的团队，如今却基本不再使用。

这一变化背后，藏着不少值得深思的信号。

---

## 写代码这件事，正在变成"看视频"

Cursor 内部现在的开发方式挺反直觉的。你在 Slack 频道里 @Cursor，它会启动一个 Cloud Agent，跑在独立的云端虚拟机。

半小时后，它交回来的不是一堆代码差异—— 而是一段视频 。

对，视频。

Agent 自己启动服务器、自己装依赖、自己跑测试，然后把 整个过程录下来 给你看。

你先看两秒钟的视频演示，觉得靠谱再去翻代码。如果同时跑四个不同模型，你会收到四段视频。

换作以前，你得审查四份 700 行的代码差异，那简直是噩梦；但现在你扫一眼四个短视频，就能判断哪个值得继续打磨。

他们团队内部有个更狠的用法：拿 Agent 去修 Bug。

Agent 先复现 Bug、录一段复现视频，然后修复、再录一段修复后的视频，证明同样的操作不再出问题。

那种你在本地调试半天都没法确认"到底修没修好"的老大难 Bug，现在可能 90 秒就直接合并了。

---

## 瓶颈已经不在写代码了，而在"敢不敢合"

Cursor 团队观察到一个悖论：AI 让写代码变得极其容易，但把代码推到生产环境反而更难了。

他们内部有个笑话——讨论某个功能的时候，总有人说"我有个 PR"。PR 写出来很快，但你不敢合。

解决了一个瓶颈，立刻冒出三个新的。 代码审查成了新的卡点。

一个百人团队跑几十个并行 Agent，产出的代码量相当于以前千人规模大厂的 DevEx 才扛得住。

CI/CD 系统直接被搞崩过——不是因为 Cursor 本身用户暴增，而是每个用户运行的 Agent 数量翻了十倍。

所以他们的判断是：未来你需要用 AI 来审查 AI，或者干脆采用另一种模式—— 别审代码了，直接把运行结果给我看 。

---

## 不是让水流更快，而是把管道变宽

Cursor 团队对未来一年最核心的判断，用了一个很形象的比喻：之前整个行业都在想办法让单个模型干更多活，像是加大水压让水流更快。

但真正的突破在于把管道变宽—— 并行化 。

他们做了一个实验叫"委员会"：同一个 Prompt 同时丢给多个不同模型，然后用一个合成层去整合结果。

这个合成智能体本身也能写代码，所以它不只是"选最好的"， 而是吸收了多个模型各自的优势，写出新的代码 。

有意思的是，即便未来某个模型暂时领先，" 模型集群 "跑出来的综合效果仍然更好。一加一确实大于二。

他们还在推子智能体的概念——主 Agent 可以自己决定启动五个探索型子 Agent，每个带不同的 Prompt 和目标，协同完成任务。

你甚至不需要事先定义这些子 Agent 是什么，只需输入"调用探索子智能体"，剩下的它自己搞定。

当你看到系统里成百上千个智能体同时运行，每秒输出数千个 Token 时——用他们的话说——你就再也回不去旧时代了。

---

## Agent 在学着"认识自己"

他们近期分享的内容中，有一段讨论令我印象格外深刻。

他们发现 Agent 在处理具体任务时表现越来越好，但有一个致命短板： 它不知道自己能干什么 。

听起来很荒谬，但确实如此。用户问 Agent 能不能做某件事，任务明明就在眼前且完全可行，产品却直接拒绝了。

原因是 Agent 对自己运行在什么系统里、有哪些工具可用、边界在哪里， 缺乏认知 。

Cursor 现在在做的事，是让 Agent 具备" 自我意识 "。

这里说的不是什么哲学意义上的觉醒，而是它清楚自己跑在哪种环境里，知道密钥怎么用、哪些功能开着、哪些功能关了，甚至能主动发现自身功能的信息缺口，创建记忆文件去填补。

更激进的是，他们已经在研究让模型编辑自己的 System Prompt。

这意味着 Agent 不只是在执行任务，它还在根据任务需求调整自己的"操作系统"。

---

## 一个人花一万美元，可能比雇十个人划算

Cursor 的定价正在经历三个阶段的跃迁：

- 第一阶段，Tab 自动补全，每月 20 美元；
- 第二阶段，本地模型交互，每月几百美元；
- 第三阶段，高度并行的云端 Agent 长时间运行在各自的虚拟机里，每月可能要几千甚至上万美元。

这不是贪婪。他们的逻辑是： 如果一个人借助这些工具真的能完成十个人的工作量，那这笔钱就是物有所值。

而且大家会自然而然选择市面上最好的产品。

团队里有人说，自己的野心比以前更大了，比任何时候都忙。

不是因为 AI 让工作变少了，恰恰相反—— 能做的事情变多了 ，你只是在拼命跟上节奏。

说白了，短期内开发者不会失业，因为现在有太多东西需要去构建。

但工作方式的确在发生剧变：你不再是那个敲键盘的人，你是那个看视频、做判断、切换上下文、管理一群 Agent 的人。

未来软件开发的节奏，会从现在的 3 秒级交互，变成 3 分钟、30 分钟甚至 3 小时的任务处理。

你得到的不再是零碎的代码片段，而是直接可以预览的成果。

Cursor 团队把自己想做成的东西叫"思维层面的无服务器架构"。

你不需要关心底层跑的是什么机器、哪个模型、怎么路由—— 你只管说你想要什么，剩下的，交给那群你看不见的 Agent 。
