## 三、增量信息识别

### 3.1 增量信息 1：人气分计算公式

**文档原文**：
> 人气分 = 点赞数 + 评论数 × 2 + 收藏数 + 转发数

**此前状态**：参赛策略分析报告 v12 中仅记录了"≥500 赞门槛"，未记录完整计分公式。

**策略含义**：
- 评论数权重最高（×2），意味着**引发讨论**比单纯获赞更有效
- 收藏数和转发数权重相同（×1），代表内容质量和传播力
- 最低门槛 500 赞是硬性条件，但过了门槛后，评论数是拉开差距的关键

**对 SpecWeave 的影响**：SpecWeave 的"用 AI 给 AI 建规矩"叙事天然容易引发讨论（技术争议性），在评论数维度有优势。抖音内容策略应引导观众评论而非仅点赞。

### 3.2 增量信息 2：Session ID 获取方式

**文档原文**：
> Session ID 是 TRAE 每段对话任务的唯一标识，用来证明作品确实由 TRAE 开发完成。请在创作时保留关键任务的对话记录，发帖时附上对应 Session ID——双击 TRAE 的对话复制出来即可。

**此前状态**：Demo 制作流程报告中仅说"在 TRAE IDE 的对话历史面板中查看"，未确认具体操作方式。

**操作含义**：Session ID 的获取比预期简单——双击对话即可复制，不需要进入特殊面板或菜单。

## 四、工具使用评估

| 工具 | 用途 | 有效性 | 对比飞书文档 |

| WebFetch | 获取论坛页面 | 完全有效（一次获取全文） | 飞书需浏览器兜底，论坛一次成功 |
