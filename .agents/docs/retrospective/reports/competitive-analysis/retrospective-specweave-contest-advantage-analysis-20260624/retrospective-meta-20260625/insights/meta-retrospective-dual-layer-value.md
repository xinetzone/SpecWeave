# 洞察 6：元复盘的价值——"第一次复盘产出策略，第二次复盘产出方法论" ⭐⭐⭐

**现象**：v11 迭答复盘（`retrospective-v11-iteration/`）复盘了 v10→v11 的策略转向过程，产出了 4 条过程洞察。本次元复盘（当前文档集）复盘了整个 v3→v11 的全生命周期，产出了 6 条关于"如何做竞品分析"的方法论洞察。

**规律**：

```
复盘层级       复盘对象            产物性质           对知识库的贡献
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
第一层（初赛   TRAE 大赛 FAQ      → 赛事策略（一次性）    战术级
   FAQ 分析）
  
第二层（竞品   11 轮增量分析      → 竞争优势评估 +       战役级
   分析）                           参赛策略（时效性）
  
第三层（v11     v10→v11 策略转向  → 4 条过程洞察         战斗级
   迭答复盘）                       （可复用）

第四层（元复盘   v3→v11 全生命周期 → 6 条方法论洞察       方法级 ⬅️ 最高知识转化率
   本复盘）                        （高度通用）
```

**操作指南**：重大项目结束后执行两层复盘——第一层复盘"我们做了什么/做对了什么"（战役级），第二层复盘"这个项目揭示的通用方法论规律是什么"（方法级）。第一层产出适合存档，第二层产出适合入库。

> **一句话**：第一次复盘告诉你"这次做得怎么样"，第二次复盘告诉你"以后该怎么做"——只有第二次复盘能让经验跨项目迁移。
