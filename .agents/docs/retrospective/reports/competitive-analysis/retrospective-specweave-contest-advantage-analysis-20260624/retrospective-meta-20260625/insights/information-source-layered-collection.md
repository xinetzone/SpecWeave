# 洞察 1：信息源分层采集——"规则层优先、操作层次之、品牌层佐证" ⭐⭐⭐⭐⭐

**现象**：11 轮迭代中，规则层来源（赛事细则、初赛指南、FAQ）产出的策略影响权重最高——它们直接改变了评审维度理解、晋级通道策略和资源分配。操作层来源（报名指南、抖音表单、教程）提升了执行精度但未改变策略方向。品牌层来源（官网、Live #13）提供了叙事材料但无策略转向。

**规律**：

| 信息层级 | 对策略的影响 | 示例 |
|---------|------------|------|
| **规则层**（赛事章程、细则、指南） | **方向性改变**——推翻假设、建立新约束、打开新通道 | 评审四维度权重、单作品 Best Shot 规则、全人工评审 |
| **操作层**（报名指南、表单、教程） | **精度提升**——优化执行路径、避免踩坑 | 话题精确格式、HTML ZIP 合规性、提交后不可修改 |
| **品牌层**（官网、直播、灵感示例） | **叙事强化**——提供叙事素材、品牌对齐 | 30+ 灵感示例矩阵、"harness engineering" 概念 |
| **事实层**（参赛身份、已通过审核状态） | **根本性重构**——当核心假设被推翻时触发策略全量重写 | 竹简悟道报名帖揭示真实参赛身份 |

**操作指南**：竞品分析的信息采集应遵循「规则层优先、操作层次之、品牌层佐证」的顺序——先建立约束框架，再填充执行细节，最后收集叙事素材。将操作层和品牌层信息用在错误的时间点（如先研究官网叙事再理解评审规则）会导致分析方向建立在沙滩上。

> **一句话**：信息源的类型决定它能影响分析的哪个层面——用操作层的细节去推导策略方向是浪费时间，用规则层的约束去填充执行细节是大材小用。
