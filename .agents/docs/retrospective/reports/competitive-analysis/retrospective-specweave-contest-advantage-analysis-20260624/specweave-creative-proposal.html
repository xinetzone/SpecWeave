DIFFERENTIATION

三大差异化亮点

⟳
自指涉证明
SpecWeave 的内容是"如何更好地与 AI 协作"，而它的诞生过程本身就是这个命题的最佳实践——142 次对话的量化产出就是证据。不需要解释方法论好不好——方法论产生的结果，本身就是答案。
∅
品类独占
大赛官网 30+ 灵感示例均为 C 端应用（AI 砍价助手、论文转短视频等），没有任何一个涉及"如何更好地使用 TRAE 本身"。这是一个被所有人忽略的维度——别人在展示"做了什么"，SpecWeave 在展示"怎么做"。
⚙
TRAE 产品能力的深度延伸
不是"在 TRAE 之外另建一套规则"。AGENTS.md 体系根植于 TRAE 原生支持的 Rules 特性——Community Live #13 中产品经理介绍的 Rules 配置，正是 SpecWeave 四层架构的产品根基。这是产品能力的系统化扩展。
∞
临界质量效应：知识生产的正反馈
当方法论模式超过 6 个后，新模式开始从模式间的交叉组合中自发涌现。16+ 个模式中，至少 5 个来自交叉组合而非单次复盘。知识生产从"线性累积"进入"组合爆发"——你知道得越多，学得越快。
⇄
层级错位：与 OpenClaw / Hermes Agent 的互补
2026 年 Agent 赛道两大头部框架——OpenClaw（36 万星，Skill 扩展生态）和 Hermes Agent（11 万星，自进化闭环）——都是 Agent 运行时框架，解决"如何让 Agent 运行起来"。SpecWeave 是协作规范层，解决"如何让多角色 Agent 在持续协作中保持质量一致"——互补而非竞争，为运行时框架提供工程化治理。
QUICK START

3 种方式，让 AI 立刻进入状态

01 / 克隆即用
零配置，TRAE 自动加载
git clone https://atomgit.com/daoCollective/SpecWeave
将仓库根目录指定为 TRAE 工作目录，AGENTS.md 自动生效——AI 启动即理解项目架构。
02 / 脚手架初始化
为新项目生成精简骨架
python -m agents init --name "MyProject" --type software --lang zh
从模板生成 AGENTS.md + .agents/ 骨架，按需选择角色和协议。
03 / 参考模式
学习四层架构后自建
克隆仓库后，参考四层架构和 34 个方法论模式，在自己的项目中构建定制化规范体系。
