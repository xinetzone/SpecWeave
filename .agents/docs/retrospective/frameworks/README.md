# 复盘决策框架库

## 🎯 决策框架概述

> **决策矩阵**是将底层概念转化为可操作决策工具的"中间层"。概念回答"为什么"，模式回答"怎么做"，而决策矩阵回答"在X场景下应该选A还是B"——它消除了决策中的主观判断，让不同的人在相同情境下能做出一致的选择。

### 框架的价值

| 价值维度 | 说明 |
|---------|------|
| 🎯 消除歧义 | 将模糊的"应该这样做"转化为明确的"当满足X条件时选Y" |
| 🔄 一致性保障 | 团队成员在相同场景下做出相同决策，减少"因人而异" |
| 📚 知识沉淀 | 将资深成员的决策经验固化为表格，新人可直接查表决策 |
| ⚡ 效率提升 | 避免重复讨论同类问题，查表即可快速决策 |

> **核心洞察**：决策矩阵是**概念的场景化应用**——每个矩阵背后都有一个或多个核心概念作为理论支撑。

## 🚀 如何使用决策矩阵

### 三步决策法

1. **Step 1: 识别场景**——判断你面临的问题属于哪个决策矩阵的覆盖范围
   - 文件放哪里？→ 依赖管理矩阵
   - 目录叫什么？→ 目录命名矩阵
   - 这个文档是元文档吗？→ 元文档处理矩阵
   - 阈值设多少？→ 语义匹配阈值矩阵

2. **Step 2: 查表匹配**——在对应矩阵中找到与你的情况最匹配的行
   - 优先匹配显式条件（如"文件类型=第三方库源码"）
   - 无显式匹配时参考"决策原则"行

3. **Step 3: 执行决策**——按照矩阵推荐的方案执行，并通过保障措施验证

> **使用提示**：如果遇到矩阵未覆盖的场景，这是**补充矩阵的信号**——请先按直觉决策，复盘后将新场景补充到对应矩阵中。

### 矩阵扩展原则

| 原则 | 说明 |
|------|------|
| 📌 场景驱动 | 只在遇到真实决策困境时才扩展矩阵，不要预设场景 |
| 🧪 验证先行 | 新条目至少在2个独立场景中验证后再加入矩阵 |
| 📝 记录来源 | 每个矩阵条目都应标注"来源"（从哪个复盘/决策中萃取） |

## 🔗 与概念层的关系

### 概念→矩阵映射表

| 决策矩阵 | 支撑的核心概念 | 概念如何体现在矩阵中 |
|---------|---------------|---------------------|
| 临时依赖管理决策矩阵 | [零依赖原则](../concepts/zero-dependency-principle.md) | 零依赖原则要求基础设施脚本无第三方依赖，而依赖矩阵确保临时依赖不污染版本库 |
| 目录命名决策矩阵 | [语义前缀](../concepts/semantic-prefix.md) | `.agents/`、`docs/`、`vendor/`等语义前缀是目录命名惯例的标准化体现 |
| 元文档处理决策矩阵 | [元文档](../concepts/meta-document.md) + [上下文感知](../concepts/context-awareness.md) | 元文档概念定义了什么是元文档，上下文感知概念要求对不同文档类型采用不同检查策略 |
| 语义匹配阈值决策矩阵 | [上下文感知](../concepts/context-awareness.md) | 上下文感知要求根据场景调整检查行为——阈值矩阵就是"按场景调整阈值"的具体实现 |

### 三层知识体系关系图

```
┌─────────────────────────────────────────────────────────┐
│  概念层（10个核心概念）                                   │
│  回答"为什么"——底层思维模型与元原则                       │
│  例：上下文感知、元文档、零依赖原则                        │
└──────────────────────┬──────────────────────────────────┘
                       │ 概念的场景化应用
                       ▼
┌─────────────────────────────────────────────────────────┐
│  框架层（4个决策矩阵）                                   │
│  回答"选A还是B"——消除主观判断的决策表格                   │
│  例：元文档处理矩阵、语义匹配阈值矩阵                      │
└──────────────────────┬──────────────────────────────────┘
                       │ 矩阵的代码化/模板化
                       ▼
┌─────────────────────────────────────────────────────────┐
│  模式层（200+可复用模式）                                │
│  回答"怎么做"——可直接复制的代码/架构/方法论模板           │
│  例：三层检查工具、元文档检测模式、Git钩子链架构           │
└─────────────────────────────────────────────────────────┘
```
