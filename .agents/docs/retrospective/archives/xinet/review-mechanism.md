# xinet 归档定期回顾与更新机制

> **版本**: 1.0
> **日期**: 2026-06-25

## 概述

本机制旨在确保 xinet 目录归档内容的时效性与准确性，通过定期回顾检查，及时发现并处理过期内容、更新归档索引、优化归档策略。

## 回顾周期

| 周期 | 频率 | 负责人 | 主要任务 |
|------|------|--------|---------|
| 月度回顾 | 每月 1 次 | 归档管理员 | 检查临时归档、清理过期内容 |
| 季度回顾 | 每季度 1 次 | 归档管理员 | 评估核心归档时效性、更新分类标准 |
| 年度回顾 | 每年 1 次 | 归档管理员 + 技术负责人 | 全面审查归档体系、优化存储结构 |

---

## 月度回顾检查清单

### 执行频率：每月最后一个工作日

### 检查项目

- [ ] **临时归档清理**
  - [ ] 检查 `temporary/` 目录中是否有超过 90 天未访问的文件
  - [ ] 确认过期文件是否可删除或需升级为参考归档
  - [ ] 更新归档索引中的"更新记录"字段

- [ ] **敏感信息扫描**
  - [ ] 运行 `security_cleanup_xinet.py` 扫描新增敏感文件
  - [ ] 检查 `.gitignore` 是否覆盖所有敏感文件模式
  - [ ] 确认无新的明文密钥入库

- [ ] **索引同步**
  - [ ] 检查归档索引是否与实际文件状态一致
  - [ ] 更新文件修改时间戳
  - [ ] 清理已删除文件的索引记录

- [ ] **文件完整性**
  - [ ] 随机抽样检查核心归档文件是否可正常打开
  - [ ] 检查文件大小是否异常变化
  - [ ] 验证归档路径是否符合命名规范

### 输出

- 更新后的 `archive_index.csv`
- 月度回顾记录（追加至 `review_log.md`）
- 清理/升级文件清单

## 季度回顾检查清单

### 执行频率：每季度最后一个工作日

### 检查项目

- [ ] **核心归档时效性评估**
  - [ ] 评估 `core/` 目录中内容是否仍具有核心价值
  - [ ] 识别过时的代码、文档、配置
  - [ ] 将过时内容降级为参考归档或移至临时归档

- [ ] **分类标准优化**
  - [ ] 审查当前分类体系是否覆盖所有文件类型
  - [ ] 评估分类准确性，修正误分类项
  - [ ] 更新分类规则文档

- [ ] **价值评估标准更新**
  - [ ] 审查价值评估维度是否合理
  - [ ] 根据实际使用情况调整评估权重
  - [ ] 更新评估标准文档

- [ ] **归档结构优化**
  - [ ] 评估当前三层结构（core/reference/temporary）是否适用
  - [ ] 检查目录层级是否过深或过浅
  - [ ] 提出结构调整建议

- [ ] **技术栈更新**
  - [ ] 检查归档文件使用的技术栈是否过时
  - [ ] 评估是否需要迁移至新版本
  - [ ] 更新技术栈兼容性说明

### 输出

- 更新后的分类规则文档
- 更新后的价值评估标准文档
- 归档结构调整建议
- 季度回顾报告

## 归档更新流程

### 更新触发条件

1. 文件内容发生实质性变化
2. 文件价值等级发生变化
3. 文件分类发生变化
4. 归档规范发生更新
5. 定期回顾发现需要更新的内容

### 更新步骤

1. **创建更新记录**
   - 记录更新原因、更新内容、更新人、更新时间
   - 更新记录格式：`YYYY-MM-DD | [原因] | [变更内容] | [操作人]`

2. **更新归档文件**
   - 如果是内容更新：复制新文件到归档目录，保留旧版本（添加版本后缀）
   - 如果是等级/分类变更：更新归档索引，必要时移动文件

3. **更新归档索引**
   - 更新 `archive_index.csv` 中的对应字段
   - 在"更新记录"字段中追加本次更新记录

4. **通知相关角色**
   - 通过团队沟通渠道通知相关人员
   - 说明更新内容与影响范围

### 更新记录示例

```
2026-06-25 | 内容更新 | WeChat/main.py 新增文章导出功能 | 归档管理员
2026-06-25 | 等级变更 | Dao/README.md 从中等价值升级为高价值 | 归档管理员
2026-06-25 | 分类变更 | test_main.ipynb 从其他分类调整为测试文件 | 归档管理员
```

## 工具支持

### 自动化脚本清单

| 脚本 | 用途 | 执行频率 |
|------|------|---------|
| `scan_xinet.py` | 全量文件扫描与分类 | 按需 / 季度 |
| `evaluate_xinet.py` | 内容价值评估 | 按需 / 季度 |
| `archive_xinet.py` | 归档索引生成与目录创建 | 按需 / 季度 |
| `security_cleanup_xinet.py` | 敏感信息扫描与清理 | 月度 / 按需 |

### 执行命令

```bash
# 月度敏感信息扫描
python .trae/specs/xinet-content-extraction-and-archiving/output/security_cleanup_xinet.py

# 季度全量扫描与评估
python .trae/specs/xinet-content-extraction-and-archiving/output/scan_xinet.py
python .trae/specs/xinet-content-extraction-and-archiving/output/evaluate_xinet.py
python .trae/specs/xinet-content-extraction-and-archiving/output/archive_xinet.py
```

## 权限与责任

### 归档管理员职责

- 执行定期回顾检查
- 更新归档索引与规范文档
- 处理归档文件的增删改
- 报告归档状态与问题

### 技术负责人职责

- 审批归档结构调整方案
- 审核年度回顾报告
- 协调跨团队归档需求

### 访问权限

- 归档目录读写权限：归档管理员、技术负责人
- 归档目录只读权限：团队成员
- 敏感文件访问：仅归档管理员
