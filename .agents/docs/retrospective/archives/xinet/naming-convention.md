# xinet 归档命名规范

> **版本**: 1.0
> **日期**: 2026-06-25

## 规范目的

统一归档文件命名规则，确保文件可追溯、可检索、可管理。

## 命名格式

```
{category-slug}-{sanitized-path}-{timestamp}{extension}
```

### 字段定义

| 字段 | 格式 | 说明 | 示例 |
|------|------|------|------|
| category-slug | 小写英文 | 文件分类简写 | `code`, `doc`, `config` |
| sanitized-path | kebab-case | 原始路径安全版本 | `wechat-main-py` |
| timestamp | YYYYMMDDHHMMSS | 归档时间戳 | `20260625143000` |
| extension | 原始扩展名 | 保留原始文件扩展名 | `.py`, `.md` |

### 分类简写映射

| 分类 | Slug | 说明 |
|------|------|------|
| 代码文件 | code | Python/TypeScript/JavaScript 等代码 |
| 文档文件 | doc | Markdown/TXT 等文档 |
| 配置文件 | config | JSON/YAML/TOML 等配置 |
| 凭证文件 | credential | 含敏感信息的配置 |
| 备份文件 | backup | Git 仓库/备份目录 |
| 测试文件 | test | 测试代码/笔记本 |
| 构建产物 | build | node_modules/dist 等 |
| 其他 | other | 未分类文件 |

## 路径安全规则

### 字符替换表

| 原字符 | 替换为 | 原因 |
|--------|--------|------|
| `\` | `-` | 路径分隔符 |
| `/` | `-` | 路径分隔符 |
| `:` | `-` | Windows 驱动器分隔符 |
| `*` | `-` | 通配符 |
| `?` | `-` | 通配符 |
| `"` | `-` | 引号 |
| `<` | `-` | 特殊字符 |
| `>` | `-` | 特殊字符 |
| `|` | `-` | 特殊字符 |
| ` ` | `-` | 空格 |
| `_` | `-` | 下划线统一 |

### 长度限制

- 文件名最大长度：200 字符
- 超长路径截断处理：保留前 200 字符

## 命名示例

### 示例 1：核心代码文件

```
原始路径: WeChat/main.py
归档路径: core/code-wechat-main-py-20260625143000.py
```

### 示例 2：参考文档

```
原始路径: Dao/README.md
归档路径: reference/doc-dao-readme-md-20260625143000.md
```

### 示例 3：临时构建产物

```
原始路径: AI/boshu-laozi-a2a/node_modules/
归档路径: temporary/build-ai-boshu-laozi-a2a-node-modules-20260625143000
```

## 版本管理

- 同一文件多次归档时，时间戳不同，可区分版本
- 更新记录记录在归档索引的"更新记录"字段中

## 索引规范

归档索引文件 `archive_index.csv` 必须包含以下字段：
- 原始路径：文件在 xinet 目录中的相对路径
- 绝对路径：文件完整路径
- 归档路径：归档后的相对路径
- 价值等级：高/中/低
- 分类标签：文件分类
- 文件大小：字节数
- 修改时间：最后修改时间
- 归档时间：归档操作时间
- 更新记录：变更历史

## 合规检查

### 检查项

- [ ] 文件名是否符合 kebab-case 规范
- [ ] 是否包含非法字符
- [ ] 文件名长度是否 ≤ 200 字符
- [ ] 是否保留原始扩展名
- [ ] 索引清单是否完整

### 工具支持

可使用 `archive_xinet.py` 脚本自动生成符合规范的归档路径。
