# v2.0模板增强任务复盘—洞察萃取（示例）

本次复盘共萃取 1 个可复用洞察/模式。

---

## 💡 洞察1：跨vendor知识融合流程

### 优先级：🟡P1
### 模式类型：流程模式
### 成熟度：L1候选（1次验证）

### 观察到的现象
本次任务需要吸收vendor/flexloop中task-execution-summary skill的优秀设计到我们自有复盘流程中。通过"高层文档优先研究法"快速理解vendor设计，然后采用"融合而非替换"的策略，选择性吸收P0-P4优先级、ROI评估、五维分析等优秀设计，保留我们自有的四文件原子化结构优势，最终产出v2.0模板，整个过程顺畅高效，没有出现"水土不服"或"全盘照搬"的问题。

### 5-Whys根因分析

**Why 1：为什么这次vendor知识融合这么顺畅？**
→ 因为没有全盘照搬vendor设计，而是选择性吸收我们需要的部分。

**Why 2：为什么能做到选择性吸收而不是全盘照搬？**
→ 因为先通过高层文档快速理解了vendor设计的核心，有能力判断哪些适合我们、哪些不适合。

**Why 3：为什么能快速判断适用性？**
→ 因为我们对自己现有的四文件复盘流程的优缺点有清晰认知（前序多次复盘已经验证了我们的优势和不足）。

**Why 4：为什么清晰认知自身体系很重要？**
→ 融合的本质是"优势互补"，不了解自己的优缺点就无法判断该吸收什么、该保留什么。

**Why 5：根本原因是什么？**
→ 跨vendor知识融合的成功关键不是"拿到什么用什么"，而是遵循"理解vendor→认知自我→优势互补融合"的三步流程。

### 模式描述

**模式名称**：跨vendor知识融合三步法

**触发条件（何时使用）**：
- 需要吸收vendor子模块中的优秀设计/工具/流程到自有体系时
- 研究外部开源项目/第三方框架并准备借鉴时
- 有多个可选方案需要决定是"替换"还是"融合"时

**核心步骤/原则**：
1. **理解vendor（高层文档优先）**：先读AGENTS.md/SKILL.md/README等高层文档，快速建立全局认知，不要一上来就读源码
2. **认知自我（优势劣势清单）**：列出自有体系的核心优势（必须保留）和明显不足（需要改进），作为吸收筛选标准
3. **优势互补融合**：只吸收能弥补自身不足的设计，保留自身核心优势；不要全盘照搬，也不要盲目排斥
4. **示例验证**：融合后立即用一个真实任务验证效果，发现问题及时调整

**预期收益**：
- 避免"全盘照搬导致水土不服"
- 避免"盲目排斥错过优秀设计"
- 融合效率提升，不需要漫长的"适配期"
- 产出物同时具备vendor的优秀特性和自身体系的优势

**反模式（绝对不要做）**：
- ❌ 宗教式狂热：看到vendor设计就觉得什么都好，全盘照搬抛弃自有积累
- ❌ NIH综合征（Not Invented Here）：觉得不是自己做的就不好，盲目排斥所有外部设计
- ❌ 不读文档直接用：不理解vendor设计的上下文和初衷，断章取义只抄表面
- ❌ 融合后不验证：没有真实场景验证就宣布融合成功，实际使用时问题百出

### 验证状态
- 本次任务验证：成功（v2.0模板同时具备vendor的P0-P4/ROI和我们的四文件结构优势）
- 前序验证：前序任务中"高层文档优先研究法"已经验证了第一步的有效性
- 成熟度：L1候选
- 后续验证机会：下次需要吸收vendor中其他skill（如skill-creator、archive-folder等）的设计时
