# CLI/工具类分析维度模板

## 核心分析维度

### 1. 命令体系

| 维度 | 分析要点 |
|------|---------|
| 命令结构 | 主命令、子命令层级关系 |
| 参数设计 | 位置参数、可选参数、标志位 |
| 命令分组 | 功能模块划分 |
| 帮助系统 | 自动生成的帮助信息质量 |

### 2. 核心API

| 维度 | 分析要点 |
|------|---------|
| 入口函数 | CLI入口到核心逻辑的调用链 |
| 核心模块 | 业务逻辑核心模块划分 |
| 接口设计 | 模块间接口定义与调用方式 |
| 错误处理 | 异常捕获、错误码定义、用户友好提示 |

### 3. 扩展机制

| 维度 | 分析要点 |
|------|---------|
| 插件系统 | 是否支持插件扩展、插件注册机制 |
| 配置扩展 | 配置项的动态加载机制 |
| 命令扩展 | 子命令的动态注册机制 |

### 4. 配置体系

| 维度 | 分析要点 |
|------|---------|
| 配置来源 | 命令行参数、环境变量、配置文件优先级 |
| 配置格式 | 使用的配置文件格式（TOML/YAML/JSON） |
| 配置验证 | 配置项的校验机制 |
| 敏感配置 | 敏感信息的安全处理方式 |

### 5. 输出体系

| 维度 | 分析要点 |
|------|---------|
| 输出格式 | stdout/stderr分离、结构化输出（JSON） |
| 日志系统 | 日志级别、日志格式、日志输出位置 |
| 用户反馈 | 进度提示、成功/失败提示 |

### 6. 依赖管理

| 维度 | 分析要点 |
|------|---------|
| 依赖声明 | pyproject.toml/requirements.txt |
| 依赖版本 | 固定版本 vs 范围版本 |
| 依赖类型 | 生产依赖 vs 开发依赖 |

## 输出格式要求

```markdown
## CLI/工具类分析报告

### 1. 命令体系分析
- 命令结构：{描述}
- 核心命令列表：{列表}

### 2. 核心API分析
- 入口函数：{路径}
- 核心模块：{模块列表}

### 3. 扩展机制分析
- 插件支持：{是/否，描述}
- 配置扩展：{描述}

### 4. 配置体系分析
- 配置来源优先级：{描述}
- 敏感配置处理：{描述}

### 5. 设计亮点与改进建议
- 亮点：{列表}
- 建议：{列表}
```
