# 示例/Demo类分析维度模板

## 核心分析维度

### 1. 项目结构

| 维度 | 分析要点 |
|------|---------|
| 文件组织 | 目录结构、代码分层 |
| 模块划分 | 核心模块、辅助模块 |
| 资源管理 | 静态资源、配置文件 |

### 2. 集成方式

| 维度 | 分析要点 |
|------|---------|
| API调用 | REST/gRPC/WebSocket集成方式 |
| SDK集成 | 官方SDK使用、版本管理 |
| CLI集成 | CLI工具调用、命令行参数 |

### 3. 演示功能

| 维度 | 分析要点 |
|------|---------|
| 核心场景 | 展示的核心功能场景 |
| 使用流程 | 从启动到完成的完整流程 |
| 交互设计 | 用户交互方式、反馈机制 |

### 4. 测试模式

| 维度 | 分析要点 |
|------|---------|
| 测试类型 | 单元测试、集成测试、E2E测试 |
| 测试覆盖 | 测试覆盖率、测试用例设计 |
| 测试数据 | 测试数据准备、Mock数据 |

### 5. 构建部署

| 维度 | 分析要点 |
|------|---------|
| 构建流程 | 编译、打包、构建工具 |
| 部署方式 | 本地运行、容器化部署 |
| 环境变量 | 部署环境配置、敏感信息 |

### 6. 平台支持

| 维度 | 分析要点 |
|------|---------|
| 多平台 | Android、iOS、Web、桌面端 |
| 平台差异 | 各平台的实现差异 |
| 兼容性 | 版本兼容性、设备兼容性 |

## 关键实体标记

报告末尾必须附「关键实体汇总表」：

```markdown
## 关键实体

| 类型 | 名称 | 说明 |
|------|------|------|
| API | POST /api/v1/test/run | 执行测试任务接口 |
| CONFIG | APP_ID | 应用标识配置 |
| MODULE | demo.app.controller | 应用控制器模块 |
```

**必须标记的实体类型**：

| 标记类型 | 说明 | 示例 |
|---------|------|------|
| API | REST/gRPC/WebSocket接口 | POST /api/v1/test/run |
| CONFIG | 环境变量或配置文件项 | APP_ID |
| MODULE | 代码模块/包名或关键组件 | demo.app.controller |
