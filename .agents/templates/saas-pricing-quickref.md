## 一、六档定价模型速查表

| 套餐 | 价格 | 目标用户 | 核心定位 | 关键权益 |
|---|---|---|---|---|
| **Free** | $0 | 新用户/体验/学生 | 零门槛获客 | 有限配额，能完成1个真实任务，无需信用卡 |
| **Go** | 低价位（$5-$8） | 价格敏感/试水用户 | 第一次付费转化 | 比Free多2-3倍配额，过渡档 |
| **Plus** ⭐ | **$20/月** | 个人主力用户 | **甜点档位（主力收入）** | 完整功能+连接器+多端入口+充足配额 |
| **Pro** | $200/月 | 重度/专业用户 | **价格锚点** + 高ARPU | 5-20倍配额+旗舰模型+不限制使用 |
| **Business** | 年付，2席位起 | 团队/中小公司 | 团队协作 | 管理控制台+安全+共享工作空间+无限Codex |
| **Enterprise** | 定制报价 | 大型企业 | 高ACV | 合规+SLA+定制集成+专属支持 |

## 二、核心定价心理学公式

### 2.1 梯度升级公式

```
Free → 痛点（配额不够）→ Go/Plus → 痛点（重度不够）→ Pro → 痛点（团队管理）→ Business → Enterprise
         ↑ 每次用户自己遇到痛点，不是你推销
```

### 2.2 锚定效应公式

```
主力档价格 × 10 = 锚定档价格
   $20 × 10 = $200（Pro）

效果：用户看到$200觉得贵，再看$20觉得"真便宜"
```

### 2.3 价值展示顺序公式

```
顶部免费CTA → 功能价值（5大模块）→ 信任背书（Logo+证言）→ 定价板块 → 多平台入口 → 转化
     ↑ 价值前置，价格后置，先让用户"想要"再告诉价格
```

### 2.4 PLG企业升级路径公式

```
个人用爽（Plus/Pro）→ 推荐给同事 → 发现团队管理/安全痛点 → 采购Business → 公司全员推广 → Enterprise年框
         ↑ 产品驱动增长，销售在Business/Enterprise阶段介入
```

### 2.5 配额用尽四出口公式

```
配额用尽 → 不锁死 → 给4个选项：
  ① 切mini模型（立即继续，能力略弱）
  ② 等滑动窗口恢复（明确告诉多久）
  ③ 买额外额度（按量付费，不强迫升级）
  ④ 升级套餐（解锁更多配额）
```

## 三、黄金数字速查

| 指标 | 推荐值 | 说明 |
|---|---|---|
| 主力个人档价格 | **$20/月** | SaaS个人订阅黄金价位 |
| 锚定档/主力档价差 | **10倍** | 锚定效应最佳比例 |
| 配额滑动窗口 | **5小时** | 适配集中工作场景，优于每日重置 |
| 免费档体验阈值 | 至少1个完整任务 | 用户必须用出价值才会付费 |
| CTA层数 | **4层** | 导航CTA + Hero CTA + 板块间CTA + 底部CTA |
| 套餐梯度数量 | **4-6档** | 免费→试水→主力→重度→团队→企业 |
| 从免费到主力档跨度 | **≤$20** | 控制在用户不用想太多就能决定的区间 |

## 四、配额用尽用户提示文案设计

### 4.1 文案设计原则

1. **不焦虑**：不说"您的配额已用完"这种终结性语言，说"您当前使用较多"
2. **给掌控感**：告诉用户"您可以选择"，不是"您必须升级"
3. **明确时间**：具体说"约X小时Y分钟后恢复"，不是"稍后"
4. **说清利弊**：每个选项说清楚"您能得到什么"和"代价是什么"
5. **不指责**：永远不说"您已超出限制"这种带有负面情绪的词

---

### 4.2 预警提示（配额用到80%时）

**时机**：用户配额用到80%时提前预警，给准备时间

**网页/桌面端（Toast/横幅）**：
```
💡 温馨提示：您今天的 GPT-5.4 配额已使用约 80%
   还剩约 20 条消息可用，预计可以继续工作 30-40 分钟

   [了解配额]  ·  [切换到 GPT-5.4-mini 继续使用]  ·  [继续使用 GPT-5.4]
```

**IDE/CLI端（状态栏/终端提示）**：
```
[Codex] GPT-5.4 quota: 80% used (~20 remaining). 
Tip: Switch to mini with /model mini to conserve quota, or continue normally.
```

---

- 第三个选项是按量购买，轻量付费不强迫升级整档
- 第四个选项是升级套餐，给重度用户路径
- 底部加一句"不会丢失上下文"消除切换焦虑

### 4.4 IDE 内提示（VS Code/JetBrains）

**时机**：用户在IDE里让Codex干活时触发配额

**内联提示（编辑器右上角/CodeLens位置）**：

```
┌─ Codex ─────────────────────────────────────────────┐
│                                                      │
│  GPT-5.4 quota reached. Continue working:           │
│                                                      │
│  [Switch to mini]  (fast, sufficient for edits)     │
│  [Wait ~3h20m]    (auto-resumes at 5:42 PM)         │
│  [Buy more]       ($5 for 100 messages)             │
│  [Upgrade to Pro] (unlimited for devs)              │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**CLI 终端提示**：
```
⚠️  GPT-5.4 quota temporarily used up.

    Options:
    1) /model mini     → Switch to GPT-5.4-mini and continue (instant)
    2) wait            → Quota resets in ~3h20m (auto-notifies when ready)
    3) /buy 5          → Purchase $5 add-on (100 extra messages, instant)
    4) /upgrade pro    → Upgrade to Pro for 5-20x quota

    Your conversation context will be preserved.
    What would you like to do? [1/2/3/4]: _
```

**CLI设计要点**：
- 数字快捷键，不用输入完整命令
- 提供`wait`后自动通知选项（跑完后桌面通知）
- 明确说"上下文保留"
- 用斜杠命令符合CLI习惯

### 4.6 各选项文案细节（针对不同场景）

**选项①：切换mini模型**

| 场景 | 文案 |
|---|---|
| 正常提示 | "切换到 GPT-5.4-mini，立即继续工作——简单任务和代码补全完全够用，速度更快" |
| 用户犹豫时补充说明 | "不会丢失对话上下文，Codex记得我们之前在做什么。随时可以切回 GPT-5.4" |
| mini不擅长的任务（大型重构/复杂推理） | "mini适合日常编码和简单修改。如果您要做大型重构/复杂架构分析，建议购买额度或升级Pro" |

**选项②：等待配额恢复**

| 场景 | 文案 |
|---|---|
| 显示等待时间 | "配额将在约 X小时Y分钟后恢复（最早 HH:MM）"——必须具体到分钟，不要说"几小时" |
| 等待时可以做什么 | "等待期间您仍然可以：使用mini模型、浏览历史对话、管理项目、审查已有代码" |
| 自动通知选项 | "恢复时通知我"（勾选后，配额恢复自动发推送/桌面通知，不用用户守着） |

**选项③：购买额外额度**

| 场景 | 文案 |
|---|---|
| 轻量购买 | "仅需 $5，立即获得 100 条额外 GPT-5.4 消息，永久不过期" |
| 打消顾虑 | "不需要升级套餐，按量购买，用多少买多少" |
| 多档位选择（可选） | "更多选项：$5/100条 · $15/350条 · $30/800条" |
| 确认按钮 | "立即购买"（点击后走支付流程，成功后额度立即到账） |

**选项④：升级套餐**

| 当前档位 | 推荐升级档位 | 文案重点 |
|---|---|---|
| Free/Go | Plus $20 | "升级Plus解锁完整Codex体验，包含连接器、桌面应用、充足配额" |
| Plus | Pro $200 | "如果您每天重度使用Codex，Pro提供5-20倍配额和旗舰模型，适合专业开发者" |
| Pro/个人 | Business | "团队协作需要统一管理？Business支持多席位、管理员控制、安全共享空间" |

### 4.7 错误文案：绝对不要说的话

❌ **不要说**：
- "您已超出配额限制"——"限制"是负面词，像惩罚
- "您必须升级才能继续"——强迫感，没有给其他选项
- "配额已用尽"——终结感，像游戏结束
- "请购买Pro版"——推销感太强
- "访问被拒绝"——太生硬
- "您今日配额已用完，请明天再来"——用户正干活呢，明天再来等于赶人

✅ **应该说**：
- "GPT-5.4 暂时用满了"（"暂时"说明有解）
- "您有几个选择可以继续"（给用户掌控权）
- "切换到mini可以立即继续"（第一个选项是零成本的）
- "配额约 X 小时后自动恢复"（明确时间，有预期）
- "不会丢失当前上下文"（消除切换焦虑）
