# SaaS定价策略自检Checklist

> **来源**：从[ChatGPT Codex Wiki第十章——定价策略与商业模式](../../docs/knowledge/learning/07-vendor-product-learning/openai/chatgpt-codex-wiki/10-pricing-model.md)中萃取。核心框架基于Codex六档梯度订阅+免费增值模式，涵盖商业模式设计、套餐梯度、配额机制、定价心理学、转化路径、反模式规避六大维度。

---

## 一、商业模式整体框架

### 产品与定价关系

- [ ] **新功能是否包含在现有订阅中，而非单独定价？**（杀手级功能应拉动现有订阅转化，而非独立售卖增加决策负担）
- [ ] **采用Freemium（免费增值）模式而非纯付费或纯免费？**（免费获客→体验价值→付费转化是PLG的标准路径）
- [ ] **是否有清晰的PLG（产品驱动增长）路径？**（个人用爽→推荐团队→团队采购→企业合同自然升级）
- [ ] **开发者/生态入口是否包含在订阅内，不额外收费？**（CLI/IDE扩展/API/多端入口是获客和粘性手段，不是增值项）
- [ ] **是否避免了"再买一个工具"的决策门槛？**（新功能/新产品尽量融入现有套餐，让已有用户"直接能用"）

---

## 二、套餐梯度设计

### 梯度数量与跨度

- [ ] **是否至少有4-6档梯度？**（推荐：Free → 低门槛试水档 → 个人主力档 → 重度/Pro档 → 团队档 → 企业档）
- [ ] **每档之间价格跨度是否合理？**（从免费到主力档控制在$10-$30区间；主力档到Pro档5-10倍形成锚定；团队/企业按需报价）
- [ ] **有没有跨度太大的"空档"？**（不能从Free直接跳到$99，中间必须有过渡档）
- [ ] **免费档和第一付费档之间是否有低门槛试水档？**（Go档类似设计降低第一次付费的心理门槛）

### 锚定效应设计

- [ ] **是否有一个明显贵的"锚定档"？**（Pro档价格是主力档5-10倍，不需要很多人买，但要让主力档显得划算）
- [ ] **主力档和锚定档的价格差是否足够大？**（建议10倍差距，如$20 vs $200，锚定效应才明显）
- [ ] **锚定档是否提供了真实价值，而不是纯摆设？**（重度用户确实会买锚定档，不是虚假选项）

### 每档价值差异化

- [ ] **每一档是否精准解决上一档用户的核心痛点？**（用户自然遇到瓶颈→自然想升级，不是硬推销）
  - Free用户痛点：配额不够 → Go/Plus解决
  - Plus用户痛点：重度使用配额不足 → Pro解决
  - Pro用户痛点：团队无法协作管理 → Business解决
  - Business用户痛点：合规SLA定制需求 → Enterprise解决
- [ ] **每档的权益差异是否清晰可感知？**（用户能一眼看出"我升这一档能多得到什么"）
- [ ] **主力个人档是否定在$10-$30甜点区间？**（$20/月是SaaS个人订阅黄金价位，低到不用犹豫，高到筛选价值用户）

### 免费档设计

- [ ] **免费档是否给够配额让用户体验到核心价值？**（用户必须能真正用出价值，才会想付费）
- [ ] **免费档是否又不够重度日常使用？**（不能无限免费，要让用户用着用着"不得不升级"）
- [ ] **免费档是否不需要信用卡就能开始？**（零门槛，降低注册摩擦）
- [ ] **免费档用户能否完成至少一个完整的真实任务？**（不能做了一半就被卡住连价值都体验不到）

### 团队/企业档设计

- [ ] **团队档是否解决了个人档没有的管理/安全/协作问题？**（统一付费、管理员控制台、数据安全、共享工作空间）
- [ ] **团队档和个人重度档（Pro）定位是否清晰区分？**（Pro解决个人配额问题，Business解决团队问题，不互相抢用户）
- [ ] **是否有清晰的个人→团队→企业升级路径？**（个人推荐→团队使用→痛点出现→采购Business→全员推广→Enterprise合同）
- [ ] **企业档是否支持定制报价、SLA、合规、专属支持？**（大企业决策最关心安全合规，不是价格）

## 三、配额机制设计

### 计量维度

- [ ] **配额是否按操作类型分维度计量？**（推荐至少区分：实时交互/异步任务/特殊操作，不要混在一起一个总配额）
- [ ] **不同模型是否分层计费/计量？**（旗舰模型消耗高配额，mini模型消耗低配额，给用户选择权）
- [ ] **用户能否自主选择模型？**（简单任务用mini省配额，复杂任务用旗舰模型）

### 配额重置机制

- [ ] **是否使用滑动窗口而非每日/每月固定重置？**（5小时滑动窗口比每日0点重置更灵活，适配集中工作场景）
- [ ] **滑动窗口时长是否适配用户使用场景？**（开发者/重度用户可能连续工作3-6小时，窗口不能太短也不能太长）
- [ ] **是否有更长周期（周/月）的软限制防止极端滥用？**（短期灵活+长期管控）

### 配额用尽处理

- [ ] **配额用尽后是否给用户多个出口，而非直接锁死？**（绝对不能在用户干活时直接"请升级"卡死）
  - [ ] 方案1：购买额外额度
  - [ ] 方案2：切换到轻量/mini模型继续使用
  - [ ] 方案3：API按用量计费备选（针对开发者）
  - [ ] 方案4：升级套餐
- [ ] **配额用尽提示是否友好？**（提前预警、告诉用户多久后恢复、给出明确解决方案）

## 四、定价心理学应用

### 价值定价 vs 成本定价

- [ ] **面向普通用户是否用订阅制固定月费，而非按token/按量计费？**（固定费给用户安全感和确定感）
- [ ] **是否避免给C端用户展示"$0.002/1K tokens"这种技术化定价？**（普通用户算不清也不想算）
- [ ] **API/按量计费是否仅作为开发者备选，而非主定价方式？**
- [ ] **定价是否基于"用户获得的价值"而非"你的成本"？**（用户不关心你推理花了多少算力，关心帮他省了多少时间）

### 价值展示顺序

- [ ] **页面/落地页是否先讲价值再展示价格？**（顺序：功能价值→信任背书→定价→行动入口）
- [ ] **是否避免了一打开页面就弹定价/弹付费？**（用户还不知道你是什么就要钱，转化率极低）
- [ ] **页面顶部是否始终可见免费试用/开始使用CTA？**（给急性子用户直接入口，不用翻到最后）
- [ ] **定价板块之后是否紧接着展示"怎么开始用"（多平台入口/下载）？**（用户决定付费/试用后立刻告诉他下一步做什么）

### 信任建立

- [ ] **展示定价之前是否已经建立足够信任？**（客户Logo墙→真实用户证言→可控性/安全承诺）
- [ ] **是否有真实的用户证言（带姓名/职位/公司）？**（stock photo + 假名言没有说服力）
- [ ] **是否有知名企业/品牌的Logo背书？**

### 功能打包策略

- [ ] **一个订阅是否包含所有核心功能和所有端（网页/IDE/CLI/桌面/移动端）？**（"一个订阅全平台全能用"的价值感知极强）
- [ ] **是否避免了把核心功能拆碎单独收费？**（基础+10$买IDE+10$买连接器+10$买桌面=割韭菜感）
- [ ] **开发者工具/生态入口是否全部包含在订阅内？**（这是adoption和留存手段，不是收入来源）

---

## 五、转化路径设计

### CTA节奏与布局

- [ ] **页面是否有多层CTA覆盖不同决策类型用户？**
  - [ ] 导航固定CTA（任何时候想试都能点）
  - [ ] Hero主CTA（冲动型用户首屏转化）
  - [ ] 板块间CTA（兴趣被勾起时转化）
  - [ ] 底部大CTA（理性用户看完所有内容后转化）
- [ ] **主CTA是否用免费开始/零风险试用，而非"立即购买"？**（降低行动门槛）

### 定价页面配合

- [ ] **定价表是否清晰对比各档差异？**（打勾/打叉一目了然）
- [ ] **是否突出推荐档/主力档？**（视觉上标记"最受欢迎"或"推荐"引导用户选择）
- [ ] **价格展示是否简洁？**（不要加隐藏费用、不要小字陷阱）

## 六、反模式规避（绝对不要做）

| 反模式 | 检查 | 危害 |
|---|---|---|
| 只有免费和很贵两档，中间无过渡 | ☐ 避免 | 免费用户不敢跳，卡在免费无法转化 |
| 先展示价格再讲产品价值 | ☐ 避免 | 用户第一反应"又要钱"直接走 |
| 按token/技术单位定价给普通C端用户 | ☐ 避免 | 用户算不清，用着有心理负担 |
| 核心功能拆碎单独收费（基础+IDE+连接器+...） | ☐ 避免 | 用户感觉被割韭菜，价值感知差 |
| 所有档位价格接近（$9/$12/$15），无锚定 | ☐ 避免 | 用户不知道选哪个，对比感知弱 |
| 配额用尽直接锁死"请升级"，无其他出口 | ☐ 避免 | 用户干活干一半被卡死，愤怒卸载 |
| 每日0点重置配额 | ☐ 避免 | 下午高峰配额用完只能等第二天，生产力受损 |
| IDE/CLI/多端入口额外收费 | ☐ 避免 | 开发者直接去用竞品，生态建不起来 |
| 免费版功能砍太狠根本体验不到价值 | ☐ 避免 | 用户用不出好，不会付费 |
| 免费版无限用完全无限制 | ☐ 避免 | 没人付费，商业模式不成立 |
| 个人版和团队版定位模糊功能重叠 | ☐ 避免 | 用户不知道选哪个，决策困难 |
| 一进网站就弹付费弹窗 | ☐ 避免 | 最差体验，转化率极低 |

## 七、定价上线前最终检查

- [ ] **我自己作为用户看完页面，是否能立刻知道"哪个档适合我"？**
- [ ] **我是否能在30秒内说清楚每一档的目标用户和核心差异？**
- [ ] **主力档定价是否让目标用户觉得"这钱花得值，不用想"？**
- [ ] **锚定档是否让主力档看起来"太便宜了"？**
- [ ] **免费用户是否能用出价值但又会自然遇到升级痛点？**
- [ ] **配额用尽场景我自己测试过吗？（会不会在关键时候卡死用户？）**
- [ ] **是否找过3-5个目标用户看过定价，他们的选择是否符合你的预期？**
- [ ] **开发者/重度用户的入口是否都免费包含？（不用再额外花钱买）**
- [ ] **页面转化顺序是否是：价值→信任→定价→行动？**
- [ ] **团队/企业升级路径是否顺畅？（个人用好了自然能带团队进来）**

## 快速参考：Codex六档定价模型参考

| 套餐 | 价格 | 目标用户 | 核心定位 |
|---|---|---|---|
| Free | $0 | 新用户/体验用户 | 零门槛体验价值 |
| Go | 低价位 | 价格敏感试水用户 | 低门槛起步，降低第一次付费门槛 |
| Plus | **$20/月** | 个人主力用户 | 甜点档位，完整功能充足配额 |
| Pro | $200/月 | 重度/专业用户 | 5-20倍配额+旗舰模型，**价格锚点** |
| Business | 年付2席起 | 团队/中小公司 | 管理控制台+安全+协作+无限Codex |
| Enterprise | 定制 | 大型企业 | 合规/SLA/定制/专属支持 |

**核心心理学公式**：Free（获客）→ Go（第一次付费转化）→ $20 Plus（主力收入）+ $200 Pro（价格锚定+高ARPU）→ Business（团队扩张）→ Enterprise（大客户高ACV）

## 辅助参考

- 案例来源：ChatGPT Codex——包含在ChatGPT订阅中，不单独定价，PLG驱动
- 核心洞察：**定价是心理学不是数学**——最好的定价是让不同需求的用户都找到适合自己的档位，并且觉得"太值了"
- 终极检验标准：用户付费后不仅自己不取消，还主动推荐给同事/团队/公司
