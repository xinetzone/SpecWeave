# Tasks

> 主题：docs-restructure（文档体系重组）
> 适用场景：文档拆分/合并/重命名/移动、目录结构调整、去重、命名统一

- [ ] Task 0: 现状调研与方案设计
  - [ ] SubTask 0.1: 盘点当前文档目录结构，列出所有待整理文件完整清单（含文件数、行数统计）
  - [ ] SubTask 0.2: 备份待重组目录（在 .temp/ 或归档位置创建完整备份）
  - [ ] SubTask 0.3: 分析当前问题（重复、混乱、命名不一致、层级错误等），写出具体问题清单
  - [ ] SubTask 0.4: 设计目标目录结构和分类方案，绘制 Mermaid 目录结构图
  - [ ] SubTask 0.5: 识别所有受影响的文件引用（grep 搜索被移动/重命名文件名在所有文件中的引用）
  - [ ] SubTask 0.6: 制定原子化拆分方案（如涉及大文件拆分，规划每个拆分后文件的内容范围）
  - [ ] SubTask 0.7: 规划原子提交策略（将重组拆分为独立的原子步骤，每步可独立回滚）

- [ ] Task 1: 执行重组（严格按原子提交执行）
  - [ ] SubTask 1.1: 创建新目录结构（先创建所有目标目录）
  - [ ] SubTask 1.2: 按原子计划逐个执行变更（每次只做一个操作：如移动一个文件、拆分一个章节）
  - [ ] SubTask 1.3: 每次操作后立即更新所有引用路径（不要攒到最后一次性更新）
  - [ ] SubTask 1.4: 跨目录移动后立即验证路径深度变化（`../` 层数是否需要增减）
  - [ ] SubTask 1.5: 大文件拆分时保持内容完整性（检查拆分后各文件首尾衔接，无内容丢失或重复）
  - [ ] SubTask 1.6: 消除重复内容（识别语义重复段落，合并或移除冗余）
  - [ ] SubTask 1.7: 统一文件命名为 kebab-case，统一标题层级规范

- [ ] Task 2: 验证与修复
  - [ ] SubTask 2.1: 核对文件数量（源文件总数 = 新位置文件数 + 删除文件数，确认无丢失）
  - [ ] SubTask 2.2: 运行 check-links.py 验证无死链
  - [ ] SubTask 2.3: 运行 check-move.py 验证文件迁移完整性
  - [ ] SubTask 2.4: 检查所有文档的相对路径引用正确（特别注意目录深度变化导致的 `../` 层数错误）
  - [ ] SubTask 2.5: 检查文档标题层级正确（H1→H2→H3 渐进，不跳级）
  - [ ] SubTask 2.6: 验证无重复内容（重复内容已合并或移除）
  - [ ] SubTask 2.7: 检查索引文件（README.md）已更新反映新结构

- [ ] Task 3: 收尾与提交
  - [ ] SubTask 3.1: 更新上级索引文档（如 AGENTS.md、docs/README.md、各主题 README.md 等）
  - [ ] SubTask 3.2: 清理空目录（移动后源目录为空时删除）
  - [ ] SubTask 3.3: 清理临时文件和中间产物
  - [ ] SubTask 3.4: 原子提交（遵循 atomic-commit 规范，每次提交一个独立重组变更）
  - [ ] SubTask 3.5: 在对应主题 README.md 的执行看板中登记完成状态

# Task Dependencies

- Task 0 必须最先且最充分地执行（重组类任务风险高，规划不充分会导致大量断链和内容丢失）
- Task 1 严格依赖 Task 0 完成，且必须按原子步骤执行
- Task 2 依赖 Task 1 完成
- Task 3 依赖 Task 2 验证通过后执行
- **重要**：每完成一个原子步骤都应立即验证，不要攒到最后统一验证
