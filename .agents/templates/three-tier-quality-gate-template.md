# 通用三层质量门禁模板

> **L3标准化模式集成**：本模板已应用 [three-tier-governance](../docs/retrospective/patterns/methodology-patterns/governance-strategy/three-tier-governance.md) 三层治理闭环模式
>
> **设计理念**：将质量检查分为三个层级，按优先级递进验证，确保高效且全面的质量保障。

---

## 📊 三层门禁体系概述

| 层级 | 类型 | 验证策略 | 通过标准 | 失败处理 |
|------|------|---------|---------|---------|
| **L0** | 门禁项 | 必须全部通过才能进入下一阶段 | 100%通过 | 立即修复，禁止进入下一阶段 |
| **L1** | 质量项 | 显著影响产出质量，必须逐项验证 | 90%以上通过 | 标记问题，评估影响后决定是否放行 |
| **L2** | 优化项 | 锦上添花，按需验证 | 无强制要求 | 记录建议，后续迭代优化 |

---

## 🔒 L0门禁项（必须全部通过）

### L0-1 基础完整性检查

| # | 检查项 | 检查方法 | 通过标准 |
|---|--------|---------|---------|
| L0-1.1 | 输出文件存在 | 检查文件是否存在 | 文件路径有效，非空 |
| L0-1.2 | 格式正确性 | 验证文件格式（JSON/Markdown/代码） | 无语法错误，可正常解析/渲染 |
| L0-1.3 | 关键字段完整 | 检查必填字段/配置项 | 所有必填项非空 |
| L0-1.4 | 无敏感信息泄露 | Grep搜索密钥/密码/token | 无硬编码敏感信息 |

### L0-2 流程合规性检查

| # | 检查项 | 检查方法 | 通过标准 |
|---|--------|---------|---------|
| L0-2.1 | 前置依赖完成 | 对照依赖清单 | 所有依赖任务已完成 |
| L0-2.2 | Spec文档存在 | 检查spec.md是否存在 | 非平凡任务必须有spec |
| L0-2.3 | Checklist验证 | 对照checklist.md | 所有L0项已勾选 |
| L0-2.4 | 编码规范合规 | 运行lint/format检查 | 无lint错误 |

### L0-3 安全检查

| # | 检查项 | 检查方法 | 通过标准 |
|---|--------|---------|---------|
| L0-3.1 | 文件权限安全 | 检查敏感文件权限 | 凭证文件权限为0o600 |
| L0-3.2 | 环境变量掩码 | 检查日志输出 | 敏感环境变量已掩码显示 |
| L0-3.3 | 无危险操作 | Grep搜索rm -rf/delete等 | 无危险操作或已确认 |

## 🛡️ 失败处理策略

### L0失败处理

```
L0检查失败
    │
    ▼
┌─────────────────────┐
│ 立即修复问题        │
│ 禁止进入下一阶段    │
│                     │
│ 修复后重新运行L0检查│
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│ L0全部通过后        │
│ 才能进入L1检查      │
└─────────────────────┘
```

### L1失败处理

```
L1检查失败（部分项未通过）
    │
    ▼
┌─────────────────────┐
│ 评估失败影响程度    │
│                     │
│ 严重影响核心功能？  │
│    │                │
│    ├── 是 → 修复后  │
│    │         重新检查│
│    │                │
│    └── 否 → 标记问题│
│              进入L2  │
└─────────────────────┘
```

### L2失败处理

```
L2检查发现优化空间
    │
    ▼
┌─────────────────────┐
│ 记录优化建议        │
│ 加入后续迭代计划    │
│                     │
│ 当前任务可正常交付  │
└─────────────────────┘
```

## 📋 使用流程

1. **规划阶段**：根据任务类型，从本模板中裁剪适用的检查项，定制任务专属checklist
2. **执行阶段**：子代理在交付前自行对照checklist自检
3. **验收阶段**：主代理按L0→L1→L2顺序验证
4. **复盘阶段**：记录检查中发现的问题，持续优化检查项
