性能优化 动画性能优化说明

优化清单（11项改造）

1. box-shadow → transform:scale+opacity：导航节点发光、按钮hover光晕、朱砂脉冲环全部改用伪元素 scale+opacity，只触发composite不触发paint
2. left → transform:translateX：按钮流光效果从 left 动画改为 translateX，避免layout thrashing
3. width → transform:scaleX：导航下划线从 width 动画改为 scaleX，同样避免layout
4. translate3d 硬件加速：轨道漂移动画使用 translate3d 提示GPU合成
5. CSS Containment：header元素添加 contain: layout style paint 隔离渲染边界
6. will-change 节制：仅对3种持续动画元素设置will-change，low模式下全部重置为auto释放显存
7. backdrop-filter 移动端降级：@media (max-width:768px) 关闭毛玻璃改为95%不透明实色
8. SVG 滤镜轻量化：feGaussianBlur stdDeviation从3降到2.5，减轻CPU光栅化负担
9. 移动端动画数量削减：768px以下轨道光点从4个减到2个，480px以下全部隐藏；顶部统计项部分隐藏
10. 触摸优化：-webkit-tap-highlight-color: transparent 消除点击高亮，touch-action: manipulation 禁用双击缩放，{passive: true} 滚动监听不阻塞主线程
11. 事件监听节流：scroll事件使用requestAnimationFrame节流，避免每帧触发多次样式计算

三层降级机制（增强版）

L0 系统级：@media (prefers-reduced-motion: reduce) — 自动检测用户系统"减少动态效果"设置，直接low模式
L0.5 硬件能力检测：检测 navigator.hardwareConcurrency（CPU核心数）和 navigator.deviceMemory（内存GB），双核/2GB以下设备直接mid或low模式
L1 JS自动帧率检测：采样6个窗口的平均FPS（而非单帧），移动端阈值更激进（<35降mid，<25降low）；桌面端<45降mid，<30降low；稳定高帧率才升回high
L2 手动切换：右下角"高/中/低"三档动效切换，用户可自主选择；切换到非high模式立即停止FPS检测节省CPU
同时监听 visibilitychange，页面切到后台时暂停rAF循环与FPS检测，节省电量与CPU。

三档动效模式说明

高（High）：全部动画开启，SVG发光滤镜启用，轨道4个光点，呼吸4s周期，脉冲2s周期 — 适合桌面端高性能设备
中（Mid）：关闭SVG发光滤镜，轨道光点减到2个且动画减慢到24s，呼吸减慢到8s，脉冲减慢到3s，按钮hover光晕关闭 — 适合移动端/中端设备
低（Low）：关闭所有持续动画（轨道/呼吸/脉冲/流光/节点发光），重置will-change释放GPU内存，SVG无发光 — 适合低端设备/系统减少动效/用户偏好
