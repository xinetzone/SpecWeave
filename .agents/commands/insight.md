# 洞察指令集

## 触发条件

- 系统监控指标异常
- 任务执行效率下降
- 用户反馈问题集中出现
- 周期性数据分析触发
- 复盘过程中发现需要深入分析的问题

## 输入规范

| 参数 | 类型 | 必选 | 说明 |
|------|------|------|------|
| target | string | 是 | 分析目标：`system`/`task`/`process`/`user` |
| metrics | list | 否 | 关注的指标列表 |
| time_range | string | 否 | 分析时间范围 |
| baseline | string | 否 | 对比基准 |
| threshold | object | 否 | 阈值配置 |

## 执行步骤

### 步骤 1：数据采集

- 收集系统运行指标与日志数据
- 提取任务执行记录与耗时数据
- 汇总用户反馈与问题报告
- 获取代码变更与配置变更记录

### 步骤 2：趋势分析

- 建立时间序列趋势图
- 识别异常波动与变化拐点
- 对比历史基线数据
- 计算增长率与变化率

### 步骤 3：根因分析

- 使用 5-Whys 方法定位根本原因
- 分析因果关系与影响链
- 识别瓶颈与阻塞点
- 评估问题严重程度与影响范围

### 步骤 4：异常检测

- 基于阈值规则识别异常
- 使用统计方法检测离群值
- 分析异常模式与发生频率
- 评估误报与漏报风险

### 步骤 5：生成洞察报告

- 总结关键发现与趋势
- 提出具体优化建议
- 标注优先级与实施成本
- 输出可视化图表与数据

### 步骤 6：预警与通知

- 向相关角色发送预警通知
- 更新系统状态面板
- 同步至自我复盘模块
- 记录洞察到知识库

## 输出规范

| 产出物 | 格式 | 存储位置 |
|--------|------|---------|
| 洞察报告 | Markdown | `docs/retrospective/insights/` |
| 趋势图表 | Mermaid / PNG | 报告内嵌或附件 |
| 异常记录 | JSON | `docs/retrospective/insights/` |
| 优化建议 | Markdown | 报告内嵌 |

## 质量验收

- 数据采集完整，覆盖关键指标
- 趋势分析准确，识别出显著变化
- 根因分析深入，定位到根本原因
- 异常检测有效，误报率 ≤ 5%
- 建议具体可执行，包含实施路径

## 约束条件

- 不负责异常的修复执行（归自我迭代）
- 不负责优化方案的最终决策（归自我进化）
- 异常检测算法需控制误报率在 5% 以内
- 预警通知不得造成通知风暴，需具备去重与分级机制
