# 核心功能

SpecWeave 的核心组件就像一家公司的组织架构，分为规则层和工具层，由统一入口 `AGENTS.md` 驱动。

## 核心模块概览

`.agents/` 目录是 SpecWeave 的核心，包含以下模块：

| 模块 | 路径 | 说明 |
|------|------|------|
| **角色定义** | `.agents/roles/` | 定义 7 种 AI 角色（协调员、架构师、开发者、审查员、测试员等），每种角色有自己的职责说明书，类似公司的岗位职责表 |
| **协作协议** | `.agents/protocols/` | 规定 AI 之间怎么沟通、怎么交接任务、怎么解决冲突，类似公司的工作流程制度 |
| **规则体系** | `.agents/rules/` | 代码怎么写、提交信息怎么标注、文档怎么组织，包含阶段守卫、硬编码治理、数据安全等规则，类似公司的质量标准手册 |
| **脚本工具库** | `.agents/scripts/` | 300+ 自动化脚本，自动检查工作质量，不合规的会被拦住，类似质检流水线 |
| **Skill 技能门面** | `.agents/skills/` | 标准化 Skill 门面（ci-check/docgen/insight/mermaid/forum-posting/link-check 等） |
| **指令集** | `.agents/commands/` | 复盘、洞察、第一性原理、导出报告、原子化、原子提交、Mermaid 管理等标准化命令 |
| **标准工作流** | `.agents/workflows/` | 功能开发、代码审查、测试流程等标准化工作流 |
| **模板库** | `.agents/templates/` | 标准化的文档模板和可复用方案，类似公司的标准表格和经验库 |
| **检查清单** | `.agents/checklists/` | 风险评分等标准化检查清单 |

## 项目亮点

| 亮点 | 说明 |
|------|------|
| **统一入口，不混乱** | AI 一进项目就知道该读什么规则，不会加载无关信息造成混乱 |
| **7 种角色，分工明确** | 像真实团队一样有协调员、开发者、审查员、测试员等角色，各干各的活 |
| **规则 + 工具，双保险** | 不仅告诉 AI "该怎么做"，还有自动检查工具确保"真的做到了" |
| **经过实战验证** | 经过 1300 多次真实使用和迭代优化，不是纸上谈兵 |
| **按需加载，不浪费** | AI 不会一次性读完所有文档，用到什么才读什么，效率高 |
| **开放标准，不锁定** | 基于公开的 AGENTS.md 标准，任何 AI 工具都能用，不绑定特定平台 |
