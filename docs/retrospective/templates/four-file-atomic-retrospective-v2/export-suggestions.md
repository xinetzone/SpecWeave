# {任务名称}复盘—导出建议与行动计划

## P0-P4优先级定义（v2.0新增）

本模板采用五级优先级体系（来自vendor task-execution-summary skill）：

| 优先级 | 名称 | 定义 | 响应时限 | ROI要求 |
|--------|------|------|---------|---------|
| 🔴 **P0** | 紧急阻断 | 必须立即修复的严重问题，影响核心功能/数据安全/流程阻断 | 24小时内 | 不考虑ROI，必须做 |
| 🟠 **P1** | 高价值改进 | 显著提升效率/质量/体验，有明确可量化收益 | 1周内 | ROI ≥ 5（高回报） |
| 🟡 **P2** | 常规优化 | 一般性改进，锦上添花，无明确紧迫性 | 1个月内 | ROI ≥ 2（中等回报） |
| 🟢 **P3** | 低优先级 |  nice-to-have，资源充裕时考虑 | 下个季度 | ROI ≥ 1（低回报） |
| ⚪ **P4** | 暂缓/观察 | 暂不执行，记录在案，等待触发条件或更多信息 | 无限期 | ROI < 1或信息不足 |

### ROI评级说明
- **预期收益**：用1-5分评分（5=重大收益，1=微小收益）
- **实施难度**：用1-5分评分（5=非常困难，1=非常容易）
- **ROI = 预期收益 ÷ 实施难度**
- **ROI ≥ 3 表示高性价比，优先实施**

## 六、完成状态语义规范（保持向后兼容）

| 状态类型 | 标记方式 | 说明 |
|---------|---------|------|
| 已执行 | `[x]` + "已执行" | 动作已经完成，结果已产生 |
| 已制定预案 | `[x]` + "已制定预案" | 计划已制定，待未来触发时执行 |
| 已评估 | `[x]` + "已评估，结论：xxx" | 已完成评估，根据评估结果决定是否执行 |
| 已暂缓 | `[x]` + "已暂缓，原因：xxx" | 评估后决定暂不执行（对应P4） |

**语义使用原则**：
- 状态标记必须精确反映实际情况，禁止模糊表述
- 评估类建议必须附带明确结论，不可仅标记"已完成"而无结论说明
- 暂缓类建议必须说明暂缓原因，便于后续追溯与重新评估
