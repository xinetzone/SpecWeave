# 目录索引 README 模板

适用于模块化文档体系的根目录索引文件。

```markdown
# {目录名称}

> {一句话描述目录用途与定位}

## 目录树

```
{target}/
├── README.md              ← 目录索引与模块说明
├── {subdir-1}/            ← {子目录用途}
│   ├── {file-1}.md        · {文件简述}
│   └── {file-2}.md        · {文件简述}
├── {subdir-2}/            ← {子目录用途}
│   └── {file-3}.md        · {文件简述}
└── {subdir-3}/            ← {子目录用途}
    ├── {file-4}.md        · {文件简述}
    └── {file-5}.md        · {文件简述}
```

## 模块说明

### [{子目录名}]({子目录路径})
{1-2 句话描述该子目录的用途与定位}

- [{文件名}]({文件路径}) — {1 句话描述文件的核心内容}

### [{子目录名}]({子目录路径})
{1-2 句话描述该子目录的用途与定位}

- [{文件名}]({文件路径}) — {1 句话描述文件的核心内容}

## 命名规范

{说明命名规范，如：所有文件名遵循 `kebab-case`（小写字母 + 连字符）命名规范，不含空格、中文、下划线或驼峰命名。}

## 来源说明

{说明文档体系的来源与重构历史，如：本目录体系由 {原文件} 原子化拆分而来，所有内容均来自实际项目中的成功实践。每个模块文件均标注了原始来源与关联模块，便于追溯与导航。}
```

## 使用说明

1. 将 `{target}` 替换为实际目录名
2. 将 `{subdir-N}` 替换为实际子目录名和用途
3. 将 `{file-N}` 替换为实际文件名和简述
4. 确保所有链接指向正确的文件路径（使用相对路径）
5. 模块说明按子目录分组，每个子目录下按文件名排序
