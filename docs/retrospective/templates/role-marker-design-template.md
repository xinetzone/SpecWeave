# 角色视觉标记设计模板

## 一、模板用途

本模板用于为智能体团队中的特殊角色设计视觉标记方案，确保角色在索引层与详情层均具备一致、可识别的视觉与文本标识。适用于新增联合创始、核心贡献者、特邀专家等需要差异化呈现的角色场景。

## 二、设计原则

- **双要素原则**：徽章（emoji）+ 文字前缀，确保多环境兼容。emoji 在富文本环境提供视觉快速识别，文字前缀在纯文本环境（如终端、日志）保持可读性。
- **双点一致原则**：索引层（README.md 角色职责矩阵）与详情层（角色文件标题）同时应用标记，避免"索引可见、详情丢失"或反之的不一致问题。
- **可扩展原则**：tier 字段采用枚举设计（如 `co-founder`、`core`、`contributor`），支持未来新增角色层级而不破坏既有结构。

## 三、设计要素表格

| 要素 | 载体 | 示例 | 作用 |
|------|------|------|------|
| 徽章 | emoji | 🏛️ | 视觉快速识别 |
| 文字前缀 | 方括号标记 | [联合创始] | 纯文本环境识别 |
| tier 字段 | TOML frontmatter | tier = "co-founder" | 机器可读标识 |
| 权限声明 | [permissions] 表 | view/manage | 权限边界声明 |

## 四、应用位置

- **索引层**：README.md 角色职责矩阵"层级标记"列，集中展示所有角色的 tier 标记，便于横向对比。
- **详情层**：角色文件标题前缀，在打开角色定义文件时即可通过标题识别角色层级。

## 五、设计步骤

- **步骤 1**：确定角色 tier 值（如 `co-founder`、`core`、`contributor`），遵循小写连字符命名规范。
- **步骤 2**：选择徽章 emoji（确保常见环境支持渲染，优先选择表意明确、视觉简洁的符号）。
- **步骤 3**：确定文字前缀（简洁、表意明确，建议 2-4 个汉字，使用方括号包裹）。
- **步骤 4**：在角色文件的 TOML frontmatter 声明 `tier` 字段与 `[permissions]` 表，明确权限边界。
- **步骤 5**：在 README.md 角色职责矩阵新增"层级标记"列，填入徽章 + 文字前缀组合。
- **步骤 6**：在角色文件标题应用前缀，格式为 `# {徽章} {文字前缀} {角色名}`。

## 六、示例参考

以联合创始角色为完整示例：

- **tier 值**：`co-founder`
- **徽章**：🏛️
- **文字前缀**：`[联合创始]`
- **frontmatter 声明**：

```toml
tier = "co-founder"

[permissions]
view = ["*"]
manage = ["teams", "roles", "permissions"]
```

- **README.md 矩阵条目**：`🏛️ [联合创始]`
- **角色文件标题**：`# 🏛️ [联合创始] 联合创始者`
