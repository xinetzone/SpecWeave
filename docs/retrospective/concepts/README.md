# 复盘核心概念库

## 🎯 核心概念概述

> **复盘核心概念**是从大量实践复盘萃取而来的、具有跨场景复用价值的底层思维模型。它们不是具体的操作步骤或代码片段，而是指导决策的**元原则**——理解这些概念有助于透过现象看本质，在新场景中做出一致且高质量的判断。

### 概念体系三大支柱

| 支柱 | 核心问题 | 包含概念数 |
|------|---------|-----------|
| 📐 文档治理 | 如何构建可自演化的规范体系？ | 4个 |
| ✅ 验证与检查 | 如何确保检查工具的准确性与可靠性？ | 4个 |
| 📈 模式演进 | 知识如何从单次经验沉淀为可复用资产？ | 2个 |

> **核心洞察**：这10个概念共同构成了 SpecWeave 知识体系的"操作系统内核"——上层的模式库、决策矩阵、自动化工具都建立在这些概念之上。

## 📚 概念分类索引

### 📐 文档治理类

> 解决"规范如何定义自身、如何自我演化、如何产生杠杆效应"的问题。

| 概念 | 一句话定义 | 适用场景 |
|------|-----------|---------|
| [元文档（Meta-Document）](meta-document.md) | 主要目的是描述、分析或评估另一个文档/项目，而非描述自身的文档 | 识别复盘报告、审计报告、迁移方案等特殊文档类型 |
| [元文档杠杆效应（Meta-Document Leverage）](meta-document-leverage.md) | 元文档的战略价值远超其篇幅比例（篇幅<20%，对采纳率贡献>50%） | 资源有限时优先优化README、索引页、导航表等入口文档 |
| [规范自举性（Specification Bootstrapping）](specification-bootstrapping.md) | 规范体系包含"元规范"（定义如何创建新规范的规范），实现自我演化而不破坏一致性 | 构建可扩展的角色体系、协议体系、模板体系时 |
| [自指性（Self-Referentiality）](self-referentiality.md) | 规范体系定义自身的特性——规范文件使用自身定义的格式，变更可自动追踪验证 | 设计TOML frontmatter、source溯源字段等自描述机制时 |

---

### ✅ 验证与检查类

> 解决"自动化检查工具如何减少误报、提高准确率、保持环境独立性"的问题。

| 概念 | 一句话定义 | 适用场景 |
|------|-----------|---------|
| [上下文感知（Context Awareness）](context-awareness.md) | 检查逻辑执行前先感知当前上下文属性（文档类型、路径语义），据此调整检查行为 | 设计链接检查、语义匹配等需要适应不同场景的验证工具 |
| [语义前缀（Semantic Prefix）](semantic-prefix.md) | 路径中隐含的、指示解析基准的前缀（如`.agents/`、`docs/`暗示以根目录为基准） | 实现文档间交叉引用的智能路径解析 |
| [正交验证（Orthogonal Verification）](orthogonal-verification.md) | 多项优化各自独立验证，每项验证的指标变化可精确归因到具体优化 | 性能调优、准确率优化等需要定位单一因素影响的场景 |
| [零依赖原则（Zero-Dependency Principle）](zero-dependency-principle.md) | 工具脚本仅依赖语言标准库，不引入任何第三方包 | CI脚本、验证工具、项目基础设施级脚本需要跨环境运行时 |

---

### 📈 模式演进类

> 解决"经验如何沉淀为模式、模式如何成熟、知识生产如何加速"的问题。

| 概念 | 一句话定义 | 适用场景 |
|------|-----------|---------|
| [模式成熟度分级（Pattern Maturity Levels）](pattern-maturity-levels.md) | 对已萃取模式进行L1/L2/L3三级成熟度评级，标注实践验证轮次 | 面对大量模式时区分"值得直接复用"与"仅供参考"的资产 |
| [临界质量效应（Critical Mass of Methods）](critical-mass-of-methods.md) | 模式数量超过临界点（约6个）后，知识生产从"线性累积"进入"组合爆炸"阶段 | 理解为什么模式库到一定规模后会加速产出新模式 |

## 🔗 概念间关系

### 核心关系图谱（文字描述）

```
文档治理类 ──────────────────────────────────────┐
    │                                            │
    ├─→ 元文档 ←── 元文档杠杆效应（价值放大）      │
    │       │                                    │
    │       └──→ 元文档处理决策矩阵 ──→ 框架层    │
    │                                            │
    ├─→ 规范自举性 ←── 自指性（自描述+自验证）    │
    │                                            │
验证与检查类 ────────────────────────────────────┤
    │                                            │
    ├─→ 上下文感知 ←── 语义前缀（路径解析基础）   │
    │       │                                    │
    │       └──→ 语义匹配阈值决策矩阵 ──→ 框架层  │
    │                                            │
    ├─→ 正交验证（归因精确化）                    │
    │                                            │
    └─→ 零依赖原则（环境独立性）                  │
                                                 │
模式演进类 ──────────────────────────────────────┤
    │                                            │
    ├─→ 模式成熟度分级（质量标签）                │
    │       │                                    │
    │       └──→ 临界质量效应（数量→质变）        │
    │                                            │
    └────────────────────────────────────────────┘
                          │
                          ▼
                   上层模式库/工具链
```

### 关键依赖关系

| 概念对 | 关系类型 | 说明 |
|--------|---------|------|
| 上下文感知 + 语义前缀 | 支撑关系 | 语义前缀是上下文感知在路径解析场景的具体实现 |
| 元文档 + 规范自举性 | 互补关系 | 元文档描述"他者"，自举规范描述"自身如何扩展"，共同构成文档治理的两端 |
| 自指性 + 零依赖原则 | 强化关系 | 自指性要求验证脚本强制执行规范，零依赖确保这些脚本能在任何环境运行 |
| 模式成熟度 + 临界质量 | 递进关系 | 单个模式有成熟度，整个模式库有临界质量——前者是微观质量，后者是宏观效应 |

## 🚀 推荐阅读顺序

### 入门路径（30分钟理解全貌）

1. **Step 1**: [元文档杠杆效应](meta-document-leverage.md)（5分钟）——理解"为什么入口文档如此重要"，建立文档价值认知
2. **Step 2**: [上下文感知](context-awareness.md)（5分钟）——理解"为什么一刀切的检查工具会产生大量误报"
3. **Step 3**: [模式成熟度分级](pattern-maturity-levels.md)（10分钟）——理解"L1/L2/L3"标签含义，学会判断模式可信度
4. **Step 4**: [正交验证](orthogonal-verification.md)（5分钟）——理解优化验证的正确姿势
5. **Step 5**: [临界质量效应](critical-mass-of-methods.md)（5分钟）——理解知识积累的非线性规律

### 深入路径（按兴趣选择）

| 方向 | 推荐阅读 |
|------|---------|
| 📐 文档架构师 | 元文档 → 规范自举性 → 自指性 |
| 🔧 工具开发者 | 上下文感知 → 语义前缀 → 零依赖原则 |
| 🧠 知识管理者 | 模式成熟度分级 → 临界质量效应 → 正交验证 |
