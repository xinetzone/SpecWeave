## 二、可萃取的模式与模板

### 模式候选 1：AI 应用 MVP 最小架构模式

**模式名称**：ai-mvp-minimal-architecture

**模式描述**：面向比赛/黑客松场景的 AI Web 应用最小可行架构。

**核心结构**：
```
project/
├── pyproject.toml    # 依赖 + 启动脚本
├── app.py            # 单文件路由入口
├── modules/          # 职责单一的功能模块
│   ├── feature1.py
│   ├── feature2.py
│   └── feature3.py
├── templates/
│   └── index.html    # 单页面前端（内联 CSS/JS）
└── .env.example      # 环境变量模板
```

**适用场景**：
- 黑客松/比赛快速原型
- AI 功能 Demo 验证
- 概念验证（PoC）项目

**成熟度评估**：L2（已在至少 1 个项目中观察到完整结构）

### 模式候选 2：提示词分层设计模式

**模式名称**：prompt-scene-layered-design

**模式描述**：根据功能场景调整 temperature 和输出结构约束的提示词设计方法。

**设计原则**：
1. 按确定性要求设置 temperature（0.1-0.3 确定性高，0.4-0.6 平衡性，0.7+ 创造性高）
2. 每个场景明确指定输出结构（段落、列表、表格等）
3. 统一的角色设定和语言要求
4. 针对目标用户群体调整表述难度

**成熟度评估**：L2（在 ai-code-assistant 的三个模块中一致应用）

### 模式候选 3：陌生项目理解的信息优先级

**模式名称**：unknown-project-info-priority

**模式描述**：理解陌生代码项目时的信息获取顺序和优先级策略。

**阅读顺序**：
1. P0（5 分钟内）：项目配置文件 → 入口文件 → 路由/API 定义
2. P1（5-10 分钟）：模块接口/类签名 → 关键提示词/配置
3. P2（按需）：前端交互流程 → 业务逻辑细节
4. P3（深入时）：实现细节 → 工具函数 → 样式

**成熟度评估**：L2（本次任务验证有效）
