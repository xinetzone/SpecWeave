# 洞察 F："三层递进流水线"保证对外材料质量

**发现**：竹简悟道采用严格的三层递进顺序——Spec/洞察（设计原则）→ HTML原型（验证假设）→ 报名帖/展示页（对外传播），禁止颠倒。

## 三层流水线详解

| 层级 | 产出物 | 核心原则 | 竹简悟道验证 |
|------|--------|---------|------------|
| 第一层：文档先行 | spec.md + 洞察库 | 先定义原则和边界，再动手做 | 54条洞察写在HTML开发之前 |
| 第二层：原型验证 | HTML完整版 | 用可交互原型验证假设，反向修正设计 | 场景标签三迭代（单行→双行→合并） |
| 第三层：对外包装 | 报名帖 + 展示页 | 只做认知压缩，不做知识增量 | 报名帖四段式从Spec和洞察中提炼，不添加新承诺 |

## 为什么顺序不能颠倒

- 如果先写报名帖（对外声明）再做Spec，容易承诺无法实现的功能
- 如果先写HTML再写洞察，代码决策没有依据，后期重构成本高
- 如果跳过原型直接做展示页，展示的功能可能是假的

## 前台-后台压缩法则

对外文档（前台）不是对内文档（后台）的"简化版"，而是"认知压缩版"——后台每一条洞见到前台变成一句可感知的表述，但这句背后必须有完整的逻辑链支撑。

| 前台亮点（报名帖） | 后台支撑（洞察） |
|-----------------|----------------|
| "陪你慢慢想透" | 洞察9（能让人静下来而非好看）+ 洞察25（微明试错） |
| "不做建议者，只做提问者" | 洞察1（概念解缚）+ 洞察17（对话引擎角色认知）+ 洞察22（Open Questioning） |
| "基于帛书《老子》" | 洞察2（帛书差异化）+ 洞察5（帛书更完整而非更古老）+ 洞察28（核心差异） |

## 可迁移性

**极高**。适用于任何需要路演/参赛/对外发布的项目。
