# 洞察 A：文档体系五层架构是 AI 协作项目的通用骨架

**发现**：竹简悟道 specs 目录天然形成了五层架构（规格→决策→质量→交付→萃取），这不是预先设计的，而是在十四轮迭代中自然演化出来的。

## 五层架构详解

| 层级 | 核心问题 | 文档形态 | AI 协作中的角色 |
|------|---------|---------|----------------|
| L1 规格层 | 我们要做什么？ | Spec 九节结构 | 为所有后续决策提供锚点，防止范围蔓延 |
| L2 决策层 | 为什么这样做？ | 洞察库（分层编号） | 让 AI 在修改代码前能追溯设计意图 |
| L3 质量层 | 现在一致吗？ | 滚动复盘报告 | 防止多轮迭代后文档间出现矛盾 |
| L4 交付层 | 能交出去了吗？ | 交付专项复盘 | 验证对外材料与内部设计的一致性 |
| L5 萃取层 | 下次怎么更好？ | 可迁移资产 | 将项目经验转化为可复用方法论 |

## 为什么有效

这五层对应了 AI 协作中的五个核心风险点——范围模糊（L1解决）、决策无据（L2解决）、版本漂移（L3解决）、交付脱节（L4解决）、经验流失（L5解决）。

## 可迁移性

**极高**。任何与 AI 协作的项目（尤其是产品型项目）都可以直接套用这个五层结构。最小可行版本只需 L1+L2+L3。
