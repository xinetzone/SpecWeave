# md2card 关键方法论萃取

本文档从 md2card 独立开发案例中萃取了 6 个可复用的方法论，覆盖**产品验证、AI开发、产品设计、用户获客、商业模式、技术复用**六个核心维度。

| 序号 | 方法论名称 | 核心价值 | 适用阶段 |
|---|---|---|---|
| 1 | 周末 MVP 验证法 | 快速验证需求，避免闭门造车 | 从0到1阶段 |

## 方法论 1：周末 MVP 验证法

**一句话定义**：用一个周末的极短时间做出最小可用产品，发布到技术社区获取真实用户反馈，快速验证需求而非闭门造车。

**操作步骤**：
1. **想法收敛**：识别一个真实、具体、高频的用户痛点
2. **核心功能裁剪**：只保留解决核心痛点的最小功能集，砍掉一切非必要功能
3. **快速开发**：利用技术复用+AI辅助开发，在一个周末内完成可运行的MVP
4. **技术社区首发**：选择目标用户聚集的技术/创造者社区发布产品
5. **真实反馈收集**：观察用户实际使用行为，收集评论和反馈
6. **迭代决策**：根据真实用户反馈决定是继续迭代、转型还是放弃

**关键要点**：速度优先于完美；真实用户>个人判断；痛点要足够尖锐；发布渠道要精准；做好流量冲击准备。

**md2card 案例**：仅花了一个周末完成MVP；核心功能只做"Markdown→知识卡片"一件事；发布到V2EX获得大量博主推荐；提前解决了API存图和服务器动态扩缩容问题成功应对首发流量。

**适用边界**：适用于工具类产品、单点功能验证；不适用于复杂系统、核心基础设施、高风险产品。

## 方法论 2：Image2Code AI 工作流

**一句话定义**：设计稿先行的渐进式AI编码流程——先生成视觉设计稿建立基准，再基于设计稿生成HTML，最后生成可交互组件，比直接让AI从零写代码效果显著提升。

**操作步骤**：
1. **第一步：生成设计稿**：使用Codex Image2（或类似工具）生成产品界面设计稿，建立视觉基准
2. **第二步：生成静态HTML**：将设计稿作为输入，让AI根据设计稿生成HTML+CSS代码（关键：不要跳过设计稿直接写HTML）
3. **第三步：生成交互组件**：基于已有的静态HTML，让AI生成可交互的组件
4. **第四步：逐步集成**：将生成的组件逐步集成到产品框架中，边集成边调试
5. **第五步：人工校验**：检查AI生成代码的质量、兼容性、性能，进行必要的人工调整

**关键要点**：设计稿先行是核心；每步都有明确输入输出；渐进式生成优于一次性生成；技术栈适配很重要；AI是助手不是替代。

**md2card 案例**：使用Codex作为核心AI开发工具；严格执行三步工作流；选择NextJS + TailwindCSS与AI代码生成适配度高；一个周末完成可用MVP。

**适用边界**：适用于前端UI开发、页面布局实现、快速原型；不适用于后端逻辑、复杂算法、核心业务规则。

## 方法论 3：内容优先产品设计

**一句话定义**：与"模板优先"（如Canva先选模板再填内容）相反的产品设计哲学——用户已经有了内容（如AI输出的Markdown），产品只需一键将内容转化为最终产物，彻底反转创作流程，降低创作门槛。

**操作步骤**：
1. **识别用户已有内容资产**：明确用户已经拥有什么内容——这是起点，不是需要用户创作的部分
2. **反转创作流程**：将传统的"形式→内容"流程反转为"内容→形式"
3. **一键自动处理**：将复杂的排版、适配、拆分等工作全部自动化，用户只需"粘贴→点击"
4. **多平台适配**：自动适配不同目标平台的尺寸、格式要求
5. **零门槛设计**：配合免登录、免费、无配置等特性，消除一切使用摩擦

**关键要点**：差异化是核心；内容是起点而非终点；极致降低使用门槛；精准服务特定人群；自动处理复杂场景。

**md2card 案例**：与Canva的"模板优先"相反，采用"内容优先"定位；流程反转为"粘贴Markdown→一键生成卡片"；免登录、完全免费、不消耗Token、1分钟完成；自动适配小红书/公众号/朋友圈等多平台尺寸；月活达到1.5万。

**适用边界**：适用于AI输出后处理、文件格式转换、已有内容排版；不适用于从零开始的内容创作、专业设计场景。

## 方法论 4：零成本获客策略

**一句话定义**：不依赖任何付费推广，通过"技术社区首发冷启动→自媒体内容输出→用户自发传播→SEO/AI推荐自然流量"的递进飞轮，实现零成本用户增长。

**操作步骤**：
1. **第一步：技术社区冷启动**：选择目标用户聚集的技术/创造者社区发布产品，获得第一批种子用户和博主背书
2. **第二步：自媒体内容获客**：在目标用户聚集的社交平台持续发布使用技巧、教程、演示视频
3. **第三步：用户自发传播**：通过产品体验（免费、好用、零门槛）激发用户自发推荐和分享
4. **第四步：SEO/AI推荐被动流量**：随着品牌知名度提升，搜索引擎和AI助手开始主动推荐产品
5. **持续迭代**：根据用户反馈优化产品，让增长飞轮越转越快

**关键要点**：首发渠道要精准；内容即营销；产品自身带传播属性；零摩擦体验促进推荐；不急于付费推广。

**md2card 案例**：在V2EX发布完成冷启动，获得大量博主推荐；在小红书、公众号发布使用技巧和视频，零付费推广；当前大部分用户为自主访问，形成自增长效应；小部分来自AI推荐和SEO；零成本实现月活1.5万。

**适用边界**：适用于工具类产品、开发者工具、内容创作者工具；不适用于面向大众的消费产品、需要品牌广告的品类。

## 方法论 5：免费+API付费商业模式

**一句话定义**：采用"C端普通用户免费（最大化获客和口碑传播）+ B端/API用户付费（覆盖成本/商业化）"的分层商业模式——先做用户规模和产品价值，再考虑商业化，用户资产的价值大于短期收入。

**操作步骤**：
1. **核心功能全免费**：C端普通用户的所有核心功能完全免费，配合免登录、零配置等特性
2. **设计病毒传播机制**：让免费用户在使用产品的过程中自然带来新用户
3. **识别付费场景**：找到真正有付费意愿和付费能力的用户群体（批量处理/系统集成需求）
4. **开放API付费**：提供API接口，向需要批量处理或集成能力的用户收费
5. **实现自我造血**：用API收入覆盖服务器等运营成本，让产品可以持续稳定运行
6. **积累用户资产**：将免费用户视为宝贵资产——他们是未来新产品的潜在用户基础

**关键要点**：免费层要给足；付费用户画像要清晰；先验证价值再商业化；自我造血是底线；用户资产>短期收入。

**md2card 案例**：普通用户完全免费、免登录、不消耗Token；API付费面向有批量处理/系统集成需求的用户；收入虽少但已能覆盖服务器成本；开发者清醒认识到工具类产品C端ARPU低；更看重1.5万月活作为未来新产品的推广基础。

**适用边界**：适用于工具类产品、开发者工具、有API生态潜力的产品；不适用于内容付费产品、重服务产品。

## 方法论 6：前作技术复用

**一句话定义**：不要从零开始开发新产品——从已有产品中提取可复用的技术模块、核心能力和设计经验，站在前作的肩膀上开发，大幅降低开发成本、缩短MVP周期。

**操作步骤**：
1. **盘点可复用资产**：系统盘点前作中的可复用资产——核心代码模块、技术栈、UI组件、踩坑经验
2. **提取通用能力**：将可复用的模块从原有产品中解耦、提取出来
3. **统一技术栈**：新产品尽量沿用前作的技术栈，避免额外的学习成本和技术风险
4. **专注差异化开发**：有了前作的基础，新产品只需要专注开发差异化的核心功能
5. **经验同步复用**：不仅复用代码，前作积累的产品经验、运营经验、踩坑经验同样要复用
6. **持续积累**：每个新产品都为未来的产品积累更多可复用资产，形成正向循环

**关键要点**：速度来自积累而非冲刺；复用不只是代码；技术栈一致性很重要；核心能力模块化；站在肩膀上而非从零开始。

**md2card 案例**：直接复用了MDXNotes的Markdown编译核心代码能力；技术栈（NextJS + TailwindCSS）与前作完全一致；公众号排版经验为卡片排版提供设计积累；正因为有前作技术积累，才能在一个周末完成MVP开发。

**适用边界**：适用于连续创业者、有产品积累的团队；首次创业者可重点学习"模块化积累"意识。
