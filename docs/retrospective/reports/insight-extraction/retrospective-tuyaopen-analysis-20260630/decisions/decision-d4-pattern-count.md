# 决策 D4：模式萃取数量

**决策背景**：需要确定萃取多少个核心模式。

**备选方案**：

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| 方案A：4个模式 | 硬件抽象层、LLM 适配器、消息总线、配置驱动 | 数量适中，每个模式详细 | 可能遗漏一些次要模式 |
| 方案B：6个模式 | 在4个基础上增加服务层设计、CI/CD 流程 | 更全面 | 每个模式篇幅较短，深度不够 |

**最终选择**：**方案A：4个模式**

**决策依据**：
1. 4个模式覆盖了项目最核心的设计理念
2. 每个模式都有足够的深度和可复用价值
3. 过多模式可能导致深度不足，影响知识质量

**决策影响**：
- **正面影响**：模式深度足够，可复用性强
- **负面影响/代价**：可能遗漏一些次要模式
- **影响范围**：洞察萃取深度

**事后评估**：
- **正确性**：✅ 正确
- **经验启示**：模式萃取应追求深度而非数量，确保每个模式都有足够的可复用价值
