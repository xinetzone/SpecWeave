## 三、洞察部分

### 3.1 根因分析（5-Whys）

**核心问题**：为什么 spec 阶段未发现 `self-cognition.md` 不存在？

| 层级 | 问题 | 答案 | 根因类型 |
|------|------|------|---------|
| Why1 | 为什么 spec 阶段未发现？ | spec 中没有引用验证步骤 | 流程缺失 |
| Why2 | 为什么没有引用验证步骤？ | spec 模板没有要求验证引用 | 模板缺陷 |
| Why3 | 为什么模板没有要求？ | 项目没有 modules 目录文件清单 | 资源缺失 |
| Why4 | 为什么没有文件清单？ | 没有自动化工具检查引用有效性 | 工具缺失 |
| Why5 | 为什么工具缺失？ | 引用验证意识不足，习惯性假设文件存在 | 意识缺失 |

**根本原因**：引用验证意识不足 + spec 模板缺少引用验证检查项（系统性问题）

### 3.2 异常检测

| 异常类型 | 描述 | 严重程度 | 性质 |
|---------|------|---------|------|
| 引用错误 | spec 引用不存在的 self-cognition.md | 中 | 系统性问题 |
| 发现时机 | 验证阶段才发现，非 spec 阶段 | 中 | 流程性问题 |

**判定**：spec 引用验证缺失是一个系统性问题，非偶发噪声。

### 3.3 可复用洞察

#### 洞察1：Spec阶段引用验证模式

- **场景**：创建引用其他文件/模块的文档时
- **问题**：spec 中的引用未经验证，可能在实施阶段才发现目标文件不存在
- **方案**：在 spec 阶段增加"引用验证"步骤，使用 Glob 验证所有引用的文件是否存在
- **价值**：减少实施阶段返工，提高 spec 可靠性

#### 洞察2：并行Sub-Agent执行模式

- **场景**：多个独立任务需要并行执行时
- **问题**：顺序执行独立任务效率低下
- **方案**：将独立任务委托给不同 Sub-Agent 并行执行
- **价值**：提高执行效率，减少总执行时间

### 4.2 模式核心内容

**问题场景**：创建引用其他文件的文档时，引用目标可能不存在。

**解决方案**：在 spec 阶段增加引用验证步骤。

**实施步骤**：
1. 列出所有引用：在 spec 中明确记录所有引用的文件/模块路径
2. 使用 Glob 验证存在性：对每个引用路径执行存在性检查
3. 修正无效引用：对不存在的引用，查找替代文件或标记为"需创建"

**验证标准**：所有引用的文件都存在且路径正确。
