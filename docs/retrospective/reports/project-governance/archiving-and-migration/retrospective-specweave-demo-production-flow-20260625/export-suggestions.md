## 二、HTML 交互增强清单

### 2.1 优先级排序（按投入产出比）

| 优先级 | 增强项 | 实现方式 | 预估代码量 | 投入产出比 |
|--------|--------|---------|-----------|-----------|
| P0 | 侧边栏导航 | 固定位置目录 + 平滑滚动 | CSS 20 行 + JS 15 行 | ⭐⭐⭐⭐⭐ |
| P0 | 量化数字动画 | Dashboard 数字从 0 滚动到目标值 | JS 15 行 | ⭐⭐⭐⭐ |
| P1 | 模式卡片展开 | 34 个模式折叠列表 + 点击展开 | JS 40 行 + 数据结构 | ⭐⭐⭐⭐ |
| P1 | 脚本运行模拟 | CI 检查输入输出示例展示 | JS 30 行 + 示例数据 | ⭐⭐⭐ |
| P2 | 架构图交互 | Mermaid 节点点击展开说明 | 复杂，需 Mermaid 事件绑定 | ⭐⭐ |

### 2.2 实现建议

**侧边栏导航**：
- 固定在页面左侧（`position: fixed`）
- 包含 8 个区块锚点：Hero / Dashboard / 架构 / 问题方案 / 核心资产 / 差异化 / 交叉叙事 / Footer
- 点击平滑滚动（`scrollIntoView({ behavior: 'smooth' })`）
- 当前区块高亮（Intersection Observer API）

**量化数字动画**：
- 页面加载时 Dashboard 数字从 0 滚动到目标值（142/34/23/7/5）
- 使用 `requestAnimationFrame` 实现 60fps 平滑动画
- 持续时间 1.5 秒，缓动函数 `easeOutCubic`

### 3.4 抖音人气通道策略（可选但推荐）

初赛设两条独立晋级通道：专业评审通道 300 席 + 抖音人气通道 50 席。SpecWeave 可同时参与两条通道，多一次晋级机会。

**人气分计算公式**（初赛参赛指南确认）：

```
人气分 = 点赞数 + 评论数 × 2 + 收藏数 + 转发数
```

**关键规则**：
- 最低门槛：单条内容**点赞数 ≥ 500** 才能进入人气榜计分
- 同一作品可发多条，仅取数据最高的一条计分（不累加）
- 统计截止 7 月 15 日 23:59:59
- 进入 TOP 100 的选手，官方通过抖音私信收集校验信息并复核，最终确认 TOP 50

**策略建议**：SpecWeave 的自指涉特性（方法论内容 = 实践过程）天然适合短视频传播——"用 AI 给 AI 建规矩"的叙事有话题性。建议制作 1 条 30 秒短视频，重点展示 142 次对话的量化数字和四层架构图。
