# 三、洞察萃取

## 3.1 新增方法论模式（5 个）

| # | 模式 | 一句话 | 适用场景 |
|---|------|--------|---------|
| 1 | **两栖定位模型** | 通过资产清单+泛化路径图+复用案例，同时定位"具体规范"和"元框架" | 任何积累了大量可复用资产的规范体系 |
| 2 | **结构阅读先行** | 扩展前先完整阅读包结构，同概念域追加、异概念域新建 | 已有清晰包结构的项目扩展 |
| 3 | **差异驱动重构** | 逐段对比→标注重复/相似/独有→提取和参数化→回归验证 | 两个及以上功能重叠的代码文件合并 |
| 4 | **渐进式模板化** | 硬编码验证→模板分离→多类型扩展三阶段 | 模板驱动的代码生成工具开发 |
| 5 | **复盘加速效应** | 复盘频率越高，改进延迟越低，知识转化率越高 | 任何迭代式开发流程 |

## 3.2 核心洞察速览

### 项目级洞察

| 洞察 | 一句话 |
|------|--------|
| 自指性优势 | 规范定义自身，变更全链路可追踪 |
| 临界质量效应 | 模式 ≥6 时从线性累积进入组合爆炸 |
| 工具熵减曲线 | 验证脚本 5-6 个为最优规模 |

### 执行级洞察

| 洞察 | 一句话 |
|------|--------|
| 重构隐藏价值 | 重构 ROI = 消除重复 + 发现 bug + 结构基础 |
| 跨任务加速 | 任务 N 耗时 ≈ 基线/sqrt(N) |
| 包结构杠杆效应 | 分层包让新功能接入成本降为 O(1) |

### 会话级洞察

| 洞察 | 一句话 |
|------|--------|
| 复盘频率效应 | 复盘越频繁，改进延迟越低 |
| 做思交替节奏 | "做→思"交替粒度为最优分块大小 |
| 知识密度递增 | 单次会话中知识转化率随时间递增 |

## 3.3 规律认知

### 3.3.1 工具开发的催化模式

工具不是被"规划"出来的，而是被上一轮工作的摩擦点催生出来的。每个工具都解决了一个在上一轮工作中亲身体验到的痛点。

### 3.3.2 复盘闭环的即时效应

复盘报告不是"交付物"，而是"执行清单"。回顾和优化发生在同一个时间窗口内，没有"规划→排期→等待"的延迟。

### 3.3.3 知识密度的递增规律

在单次会话中，知识转化率随时间递增。"做→思"交替的节奏是最优的分块大小。
