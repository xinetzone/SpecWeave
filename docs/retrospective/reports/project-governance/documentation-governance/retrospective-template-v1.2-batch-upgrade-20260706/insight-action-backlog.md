## 🔔 L2模式跨场景验证触发Checklist（ACT-009）

> **目的**：两个L2模式（phased-rollout-validation、classification-disposition-decision-tree）需完成2次非文档类场景验证方可启动L3升级。遇到以下场景时，**主动触发验证**，不要遗忘。

### 触发前快速自检（启动批量操作前必看）

在开始任何**≥5个对象的批量操作**前，先对照以下清单：

- [ ] **本次批量操作是否属于以下4种非文档类场景之一？** 若是 → 必须应用两个L2模式并记录验证数据
- [ ] **是否已使用分类决策树对所有对象进行五分类？**（轻量升级/按SOP拆分/补全导航/保留原状/保持原状）
- [ ] **是否规划了P0→P1→P1.5→P2四阶段执行？** N≥20时必须包含P1.5集中格式校验
- [ ] **是否准备了验证记录模板？** 验证完成后需在模式文档新增案例

### 四类触发场景与检查点

| 场景 | 触发条件（满足任一即触发） | 必做验证检查点 |
|------|--------------------------|---------------|
| **🔧 ACT-009a 代码重构批量变更** | • ≥5个模块/文件需要同类型重构<br>• API批量重命名、接口批量迁移<br>• 跨模块代码结构调整 | □ 分类决策树能否区分"重构/适配/保留/归档"四类代码对象<br>□ 三阶段推广在代码场景能否控制回归风险<br>□ P1.5集中校验：lint/type check/test全量通过<br>□ 记录问题发现率和返工节省率 |
| **📚 ACT-009b 知识库分类归档** | • ≥5个knowledge条目需要分类/重组/归档<br>• 跨目录知识条目迁移<br>• 知识库标签/分类体系批量调整 | □ 分类决策树对知识条目的分类适用性<br>□ 三阶段推广在知识库操作中的风险控制效果<br>□ P1.5集中校验：标签一致性、链接完整性、索引同步<br>□ 记录误分类率和修正成本 |
| **⚙️ ACT-009c 配置项批量迁移** | • ≥5个配置文件需要修改<br>• 配置格式升级、字段重命名<br>• 路径/环境变量批量迁移 | □ 分类决策树识别配置差异（全新/升级/保留）<br>□ 三阶段能否控制配置变更风险（P0验证回滚能力）<br>□ P1.5集中校验：配置语法检查、引用完整性<br>□ 记录配置错误发现率 |
| **🚦 ACT-009d CI规则批量更新** | • ≥5个仓库或≥5类CI规则需要更新<br>• CI流程/门禁规则批量调整<br>• 跨仓库CI配置统一化 | □ 分类决策树对CI规则的处置分类<br>□ 三阶段在跨仓库场景的适用性（P0单仓库试点）<br>□ P1.5集中校验：规则冲突检测、缺漏检查<br>□ 记录规则冲突率和修复时间 |

### 验证完成标准

完成任一非文档类场景验证后：
1. 在对应复盘报告中记录验证数据（问题发现率、效率提升、与文档类场景差异）
2. 更新两个L2模式文件的`validation_count`（+1）
3. 在模式文档"实际案例"章节新增验证案例
4. 回到本Backlog更新对应子项状态为✅
5. 累计完成2次非文档类验证后，启动L3标准化评审
