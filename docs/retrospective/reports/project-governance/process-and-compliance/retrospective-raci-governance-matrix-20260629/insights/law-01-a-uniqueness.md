# RACI A唯一性强制约束

## 规律陈述

RACI矩阵中每项活动必须有且仅有一个A（Accountable，最终审批者）。双A冲突本质是两个不同审批场景被合并到了同一行，必须按审批层级拆分。

## 触发条件

- RACI行中出现两个单元格含A标记
- 审批活动涉及"常规"和"重大"两种场景
- 审批角色根据条件不同而变化

## 实施步骤

1. 识别双A行中两个A角色的层级关系
2. 按场景拆分为多行（常规场景→低层级A，重大场景→高层级A）
3. 每行保持单一A，场景描述清晰区分
4. 运行A唯一性校验脚本验证

## 正例

```markdown
| 跨模块原子化审批（常规） | C | C | C | **A** | I | I |
| 跨模块原子化审批（重大架构调整） | C | C | I | R | I | **A** |
```

## 反例

```markdown
| 跨模块原子化审批 | C | C | C | **A** | I | **A** |  <!-- 双A，违规 -->
```
