# Markdown 表格安全修改策略（safe-table-edit）

## 模式类型
代码模式

## 成熟度
L1 实验性（仅 1 次成功案例）

## 适用场景
修改 Markdown 文档中的表格结构（插入行、删除行、修改列）。

## 问题背景
Markdown 表格的列分隔符 `|---|---|` 与表头行、数据行在语法上紧耦合。使用 SearchReplace 工具做局部插入时，容易导致分隔符列数与表头不一致，产生渲染错误。

## 规则矩阵

| 操作类型 | 推荐策略 | 原因 | 风险等级 |
|---------|---------|------|---------|
| 表格新增/删除行 | 整表替换 | 避免分隔符列数漂移 | 高（若局部操作） |
| 修改单元格文本 | 局部替换（仅匹配目标行） | 不改变表格结构 | 低 |
| 表格新增/删除列 | 整表替换 | 必须同步更新分隔符 | 高（若局部操作） |

## 正例（安全操作）

```markdown
old_str: 
| 表头 A | 表头 B |
|---|---|
| 数据 1 | 数据 2 |
| 数据 3 | 数据 4 |

new_str:
| 表头 A | 表头 B |
|---|---|
| 数据 1 | 数据 2 |
| 数据 3 | 数据 4 |
| 新增行 5 | 新增行 6 |
```

**说明**：整表替换，分隔符行无需单独处理，单次操作完成。

## 反例（危险操作）

```markdown
old_str: | 数据 3 | 数据 4 |
new_str: | 数据 3 | 数据 4 |
| 新增行 5 | 新增行 6 |
```

**后果**：分隔符行 `|---|---|` 未同步更新，列数可能漂移为 3 列（取决于匹配精度）。

## 分隔符同步原则

表格列分隔符的列数必须与表头一致：

```
| 表头 A | 表头 B | 表头 C |    ← 3 列
|---|---|---|                      ← 必须为 3 列
```

任何列数变化都须同步更新分隔符行。

## 实施检查清单

- [ ] 确认操作是否改变表格行数或列数
- [ ] 若改变，使用整表替换策略
- [ ] 若仅修改文本，可使用局部替换
- [ ] 替换后目视检查分隔符列数与表头一致

## 成功案例

| 任务 | 文件 | 操作 | 结果 |
|------|------|------|------|
| AGENTS.md 路由表新增条目 | AGENTS.md | 局部插入 → 分隔符漂移 → 回退修正 | 最终采用整表替换 |
