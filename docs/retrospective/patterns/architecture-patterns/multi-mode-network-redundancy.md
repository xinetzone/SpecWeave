# 多模网络冗余接入模式（Multi-Mode Network Redundancy Pattern）

## 模式类型

架构模式（网络接入架构）

## 成熟度

L2 已验证（向日葵5款无网远控硬件不同程度验证：控控2有线+WiFi、Q2Pro 4G+有线+WiFi、Q5Pro 5G+4G+有线+WiFi、Q0.5蓝牙近场兜底）

## 适用场景

设备需要在复杂/不稳定网络环境下保证连接可用性：
- 工业控制设备（户外机柜、无人值守站点，有线网络可能中断）
- 医疗设备（需要高可靠连接保障远程协作）
- 移动/车载设备（网络环境频繁切换）
- 高安全等级场景（需要近场物理隔离接入作为最后兜底）
- 跨网络环境设备（有时在内网、有时在外网、有时无网络）

## 问题背景

单模网络接入方案存在单点故障风险：
1. **有线网络故障**：网线松动、交换机故障、网络维护时断连
2. **WiFi不稳定**：信号弱、干扰、切换AP时断连
3. **无外网环境**：涉密场所、物理隔离环境下无法使用任何网络
4. **蜂窝网络盲区**：地下室、偏远地区4G/5G信号弱

对于远程运维、医疗协作等场景，网络断连意味着业务中断，可能造成严重损失。单一网络模式无法覆盖所有异常场景。

## 核心规则

通过同时支持多种网络接入方式，配置优先级策略和自动切换机制，确保极端网络环境下仍可建立远控连接：

**核心公式**：
```
多模冗余 = 多模并存（有线⊕WiFi⊕4G⊕5G⊕蓝牙/USB）⊕ 优先级切换 ⊕ 断网续连 ⊕ 近场兜底
```

### 五大核心要素

| 要素 | 说明 | 设计原则 |
|------|------|---------|
| 多模并存 | 设备同时支持多种网络接入方式（有线以太网、WiFi、4G LTE、5G、蓝牙、USB直连） | 根据产品定位选择2-5种网络模式，高端产品支持更多模式 |
| 优先级切换 | 配置网络优先级（如：有线>WiFi>4G>5G>蓝牙），高优先级故障时自动切换到低优先级 | 切换过程用户无感知，自动恢复远控会话 |
| 链路聚合 | 部分场景可多链路同时传输保证带宽 | 高带宽场景（如高清视频远控）下使用 |
| 断网续连 | 网络切换时自动恢复远控会话，保持远控状态 | 会话保持时间≥30秒，用户操作不丢失 |
| 近场兜底 | 蓝牙/USB直连作为最后兜底手段，完全无外网时仍可近场控制 | 满足物理隔离场景的"最后一米"控制需求 |

### 网络模式梯度设计

产品矩阵中不同定位的产品配置不同的网络模式组合，形成梯度：

| 产品定位 | 推荐网络模式 | 适用场景 |
|---------|------------|---------|
| 近场/物理隔离 | 蓝牙+USB直连 | 涉密环境、近场快速排障 |
| 消费级入门 | 有线+WiFi | 家庭/办公室内网环境 |
| 企业级旗舰 | 有线+WiFi | 企业IT运维、数据中心 |
| 工业级 | 有线+WiFi+4G | 工业控制、户外无人值守 |
| 专业级 | 有线+WiFi+4G+5G+蓝牙 | 医疗协作、移动急救、专业远程支持 |

### 切换决策逻辑

```
当前网络可用?
├── 是 → 保持当前连接
└── 否 → 按优先级尝试下一个网络
    ├── 有线可用? → 切换到有线
    ├── WiFi可用? → 切换到WiFi
    ├── 4G可用? → 切换到4G
    ├── 5G可用? → 切换到5G
    ├── 蓝牙/USB直连? → 近场兜底模式
    └── 均不可用 → 本地缓存指令+等待网络恢复
```

## 反模式

- **反模式：所有产品堆全模式**：不按产品定位配置网络模式，入门款也加5G导致成本过高，违背梯度设计原则
- **反模式：切换断会话**：网络切换时远控会话断开，用户需要重新连接，违背"用户无感知"原则
- **反模式：无近场兜底**：完全依赖外网，物理隔离场景无法使用，失去了无网远控的核心价值
- **反模式：多模但无优先级策略**：多种网络同时存在但无明确优先级，可能导致网络抖动时频繁切换

## 验证案例

| 产品 | 网络模式 | 验证重点 |
|------|---------|---------|
| 向日葵Q0.5 | 蓝牙+USB近场 | 纯近场物理隔离模式，验证近场兜底机制 |
| 向日葵Q1 | 有线+WiFi | 消费级双模式基础验证 |
| 向日葵控控2 | 有线+WiFi | 企业级双模式稳定性验证 |
| 向日葵Q2Pro | 有线+WiFi+4G | 三模冗余+4G蜂窝网络兜底验证（工业场景） |
| 向日葵Q5Pro | 有线+WiFi+4G+5G+蓝牙 | 五模全配置，专业级高可靠连接验证 |
