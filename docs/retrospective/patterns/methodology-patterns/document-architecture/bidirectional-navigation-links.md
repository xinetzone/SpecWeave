# 原子文件双向导航链接：prev/next + 返回索引三链路模式

## 模式类型
领域/方法论模式（文档架构）

## 成熟度
L1 实验性（1次验证：MDI研究报告原子化8个章节文件）

## 适用场景
长文档原子化拆分、多文件教程/指南、系列文档、章节式知识库、分页式阅读体验设计。

## 问题背景

将长文档原子化为多个独立章节文件后，读者的阅读路径容易断裂：
1. **单向链接断裂**：只写了"下一章"链接，没有"上一章"，读者回退时需要依赖浏览器后退按钮
2. **缺少索引返回**：章节文件末尾没有返回目录/索引页的链接，读者读完一章后"迷路"
3. **首末章节缺失边界处理**：第一章没有"上一章"，最后一章没有"下一章"，容易出现空链接或断链
4. **导航位置不一致**：有的章节导航在开头，有的在末尾，读者预期混乱
5. **原子化后形成信息孤岛**：拆分后的文件各自独立，没有链接串联，读者无法按顺序阅读

**典型反模式**：
```markdown
# 第三章：XXX

（正文内容）

→ [下一章](chapter-04.md)
```
读者读到这里，如果想回到第二章，或者想看看目录有哪些章节，没有直接链接可用。

## 解决方案

每个原子化章节文件必须包含**完整的三链路双向导航**：

```mermaid
flowchart LR
    PREV["⬅️ 上一章<br/>(如果不是第一章)"]
    INDEX["📑 返回目录<br/>(必须有)"]
    NEXT["➡️ 下一章<br/>(如果不是最后一章)"]
    PREV --- INDEX
    INDEX --- NEXT
```

### 标准导航模板

在章节文件**末尾**放置导航区块：

```markdown
---

## 章节导航

| [⬅️ 上一章]({prev-file}) | [📑 返回目录]({index-file}) | [➡️ 下一章]({next-file}) |
|:---:|:---:|:---:|

*本文是《{文档标题}》系列的第{N}章*
```

### 三链路设计原则

| 链路 | 必需性 | 首章节处理 | 末章节处理 | 作用 |
|------|--------|-----------|-----------|------|
| ⬅️ 上一章 | 条件必需 | 不显示此链接或显示"已是第一章" | 正常显示上一章 | 支持回退浏览 |
| 📑 返回目录 | **强制必需** | 必须有 | 必须有 | 提供全局导航入口，防止迷路 |
| ➡️ 下一章 | 条件必需 | 正常显示下一章 | 不显示此链接或显示"已是末章" | 支持顺序阅读 |

### 边界处理规则

1. **第一章**：
   - 隐藏"上一章"链接，或显示"（第一章）"占位文本
   - 必须显示"返回目录"和"下一章"

2. **中间章节**：
   - 三个链路必须全部显示
   - 确保上/下一章链接指向正确的相邻文件

3. **最后一章**：
   - 正常显示"上一章"和"返回目录"
   - 隐藏"下一章"链接，或显示"（完）"、"返回目录查看更多"
   - 可以额外添加"回到开头"链接跳转到第一章

### 导航位置规范

- **位置**：统一放在文件**末尾**（正文结束后），与正文用 `---` 分隔线分隔
- **不要放在开头**：读者打开文件首先看到的是内容，不是导航
- **不要头尾都放**：保持简洁，避免重复
- **例外**：超长篇幅章节可以在开头放一个简化的"目录/上/下"导航，但末尾必须有完整导航

## 实际案例

### 案例：MDI研究报告原子化

**场景**：将819行的 `mdi-research-report.md` 原子化为8个独立章节文件 + 1个索引页。

**导航设计**：
- 索引页（原文件改造）：列出所有章节链接
- 每个章节末尾：包含三链路导航
- 章节间顺序：00-executive-summary → 01 → 02 → ... → 07-conclusion

**验证结果**：
- 8个章节文件全部有正确的prev/next链接
- 双向链接验证全部通过（check-links无断链）
- 读者可以从任一章节：顺序阅读、回退、返回目录

## 反模式

### 反模式1：只有单向next链接

```markdown
（正文结束）
下一章：[第四章](04.md)
```

**问题**：读者想回到上一章时只能用浏览器后退，且无法快速跳转到目录。

**修正**：补充prev和index链接。

### 反模式2：导航放在正文开头

```markdown
# 第三章
上一章 | 目录 | 下一章

（正文开始...）
```

**问题**：读者打开文件首先看到的是导航而不是内容，干扰阅读体验；读者滚动到末尾想继续时还得滚回顶部找链接。

**修正**：导航统一放在末尾。

### 反模式3：首末章节出现空链接

```markdown
<!-- 第一章错误示例 -->
| [⬅️ 上一章]() | [📑 目录]() | [➡️ 下一章](02.md) |
```

**问题**：空链接点击无反应或跳转到当前页，造成困惑。

**修正**：第一章不显示上一章链接，或显示灰色不可点击的占位文本。

### 反模式4：没有统一的导航格式

不同章节导航样式不一致：
- 有的用列表
- 有的用表格
- 有的只有文字没有图标

**问题**：读者需要重新适应每个页面的导航样式，增加认知负担。

**修正**：所有章节使用统一的表格模板，图标和文案保持一致。

## 模板生成建议

批量创建原子文件时，导航链接应：
1. **脚本自动生成**：根据文件名序号（00-xx, 01-xx...）自动推导prev/next
2. **在模板中预置占位符**：创建新文件时就有完整的导航框架，只需填入链接
3. **与finalize-atomization工具集成**：原子化收尾时自动验证双向链接完整性

## 检查清单

原子化拆分完成后，逐项检查：
- [ ] 每个章节文件末尾是否有导航区块？
- [ ] 每个章节是否都有"返回目录"链接？
- [ ] 第一章是否正确处理了"无上一章"？
- [ ] 最后一章是否正确处理了"无下一章"？
- [ ] prev链接是否真的指向上一章？（不要搞反方向）
- [ ] next链接是否真的指向下一章？
- [ ] 返回目录链接是否正确指向索引页？
- [ ] 所有导航链接是否通过了check-links验证？
- [ ] 导航格式是否所有章节统一？
