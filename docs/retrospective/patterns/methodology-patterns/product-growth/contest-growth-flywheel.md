# 赛事增长飞轮模型

## 一、核心定义

赛事增长飞轮模型（Contest Growth Flywheel）将赛事运营重新定义为「增长引擎设计」而非「比赛执行」。该模型将赛事中每个用户行为同时映射为产品增长触点，实现「赛产品一体化」的战略目标。

**核心公式**：

```
参赛步骤 = 增长触点
```

传统赛事运营将「参赛」与「使用产品」视为两个独立行为——参赛者先决定参赛，再去学习使用产品。增长飞轮模型则将两者合一：参赛的每一个步骤都内嵌了产品使用行为。

## 二、增长触点映射矩阵

| 参赛行为 | 增长效果 | 产品指标 |
|---------|---------|---------|
| 报名发帖 | 社区内容积累 | UGC 内容量、SEO 索引 |
| 使用工具生成作品 | 产品行为绑定 + 功能验证 | DAU、功能使用率、留存 |
| 领取奖励 | 大赛官网二次触达 | 官网 UV、回访率 |
| 发布抖音作品 | UGC 品牌传播 | 自然流量、品牌认知 |
| 加入社群答疑 | 用户留存 + 关系网络 | 社群活跃、用户忠诚度 |

## 三、飞轮自转机制

```
参赛者产生创意内容（社区帖子）
    → 内容吸引观众和新参赛者
        → 新参赛者注册并使用产品
            → 产品体验驱动口碑传播
                → 更多参赛者加入（循环）
```

飞轮能否持续转动取决于三个执行质量：
1. **社区审核效率**：内容能否快速过审并展示
2. **产品上手体验**：首次使用是否有正反馈
3. **优质内容曝光**：好的作品能否被看见

## 四、与传统赛事运营的区别

| 维度 | 传统赛事运营 | 增长飞轮模型 |
|------|------------|-------------|
| 目标 | 办一场成功的比赛 | 设计一个持续运转的增长引擎 |
| 参赛者角色 | 被动员的参与者 | 增长贡献者 |
| 产品关系 | 赛外使用产品 | 赛中内嵌产品使用 |
| 内容价值 | 赛事资产 | 持续引流的生态资产 |
| 衡量指标 | 参赛人数、奖项分发 | 增长触点转化率、飞轮转速 |

## 五、实施步骤

### 步骤 1：解构参赛路径

将参赛流程分解为原子步骤，每个步骤回答：用户在这个步骤中做了什么？使用了什么工具？产出了什么内容？

### 步骤 2：识别增长触点机会

对每个参赛步骤追问：这个步骤能否同时成为产品增长触点？

- 能否在参赛过程中强制/引导使用产品核心功能？
- 参赛产出能否沉淀为产品相关的内容资产？
- 参赛者能否成为产品的传播节点？

### 步骤 3：设计触点转化路径

为每个增长触点设计转化路径：

```
参赛行为 → 触达产品功能 → 产生产品使用数据 → 转化为产品认知
```

### 步骤 4：建立飞轮闭环

确保增长触点之间存在相互强化的关系：
- 更多参赛者 → 更多内容 → 更多曝光 → 更多参赛者

## 六、适用场景

- AI 产品的社区冷启动
- 工具型产品的用户增长活动
- 需要将市场活动与产品指标绑定的场景
- 以内容生态为核心竞争力的产品

## 七、实施检查清单

- [ ] 参赛流程中的每个步骤是否都有明确的产品增长目标？
- [ ] 参赛产出（内容/数据/关系）是否与产品长期价值挂钩？
- [ ] 是否设计了触点之间的相互强化机制？
- [ ] 飞轮启动阶段是否有足够的初始内容/用户注入？
