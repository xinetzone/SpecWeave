# 三层递进流水线：文档先行→原型验证→对外包装

## 模式概述

对外材料（报名帖、展示页、路演PPT）的质量保障流程，严格按照"文档先行→原型验证→对外包装"三层递进顺序执行，禁止颠倒。确保对外承诺的每一个亮点背后都有完整的设计逻辑和可交互原型支撑。

## 三层流水线详解

| 层级 | 产出物 | 核心原则 | 验证标准 |
|------|--------|---------|---------|
| 第一层：文档先行 | Spec + 洞察库 | 先定义原则和边界，再动手做 | 关键决策都有洞察记录 |
| 第二层：原型验证 | 可交互HTML/原型 | 用原型验证假设，反向修正设计 | 核心场景可走通 |
| 第三层：对外包装 | 报名帖/展示页/PPT | 只做认知压缩，不做知识增量 | 每句承诺都能追溯到文档/原型 |

## 为什么顺序不能颠倒

| 错误顺序 | 后果 |
|---------|------|
| 先写报名帖再做Spec | 承诺无法实现的功能（过度承诺） |
| 先写HTML再写洞察 | 代码决策没有依据，后期重构成本高 |
| 跳过原型直接做展示页 | 展示的功能是假的（演示骗局） |

## 前台-后台压缩法则

对外文档（前台）不是对内文档（后台）的"简化版"，而是"认知压缩版"——后台每一条洞见到前台变成一句可感知的表述，但这句背后必须有完整的逻辑链支撑。

| 前台亮点（对外表述） | 后台支撑（内部洞察） |
|-------------------|-------------------|
| "陪你慢慢想透" | 洞察9（能让人静下来而非好看）+ 洞察25（微明试错） |
| "不做建议者，只做提问者" | 洞察1（概念解缚）+ 洞察17（对话引擎角色认知）+ 洞察22（Open Questioning） |
| "基于帛书《老子》" | 洞察2（帛书差异化）+ 洞察5（帛书更完整而非更古老）+ 洞察28（核心差异） |

## 质量检查清单

- [ ] 第一层完成：所有核心功能点在Spec/洞察中有定义
- [ ] 第二层完成：核心用户场景在原型中可走通
- [ ] 第三层完成：对外材料中的每一个承诺都能追溯到前两层
- [ ] 第三层检查：对外材料没有添加前两层不存在的新承诺

## 适用场景

- 需要路演/参赛/对外发布的项目（极高适用性）
- 产品Demo制作
- 产品官网/落地页
- 融资BP/商业计划书
