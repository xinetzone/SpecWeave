# 用户主权默认模式（User Sovereignty by Default）

## 一、核心定义

用户主权默认（User Sovereignty by Default）是代理操作系统（远程控制、AI Agent、RPA等）的核心信任设计原则：**被代理/被控制方始终拥有最高权限和最终控制权，代理方的所有操作对被代理方可见、可感知、可干预、可终止**。

**核心公式**：

```
信任 = 可见性 + 可控性 + 可终止性
```

传统"最小权限"思路关注的是"控制方不能做什么"，而用户主权思路关注的是"被控制方能感知并阻止什么"。两者结合才构成完整的信任体系。在AI Agent领域，这一原则尤为重要——用户必须始终知道Agent在做什么，并且随时可以接管或终止。

## 二、四大核心特性

| 特性 | 说明 | 具体表现 |
|------|------|---------|
| **最高权限归属** | 被代理方权限 > 代理方权限 | 被控端/用户始终可以一键终止/接管 |
| **操作可见性** | 所有操作对被代理方透明 | 状态提示、操作日志、实时反馈 |
| **风险可感知** | 敏感操作有明确提示 | 隐私屏、水印、操作提醒、风险预警 |
| **随时可干预** | 被代理方可随时中断 | 紧急停止、一键断开、权限回收 |

## 三、适用场景

### 3.1 典型适用场景

- **AI Agent系统**：用户委托AI执行任务，必须保持最终控制权
- **远程控制/远程桌面**：如向日葵、TeamViewer等远控软件
- **RPA机器人流程自动化**：自动化脚本执行时人工可干预
- **API自动化/DevOps工具**：自动化部署/运维操作需要人工审计点
- **自动驾驶/辅助驾驶**：AI驾驶时人类可随时接管
- **智能家居自动化**：自动场景执行时用户可覆盖/终止

### 3.2 反模式（何时不适用/如何避免）

| 反模式 | 风险 | 正确做法 |
|--------|------|---------|
| 代理方默认拥有最高权限 | 用户失去控制，信任崩塌 | 被代理方权限始终高于代理方 |
| 操作"暗箱执行"用户无感知 | 错误操作无法及时发现，造成损失后才发现 | 所有关键操作有明确提示和状态反馈 |
| 一旦开始无法终止 | 错误操作无法止损，小问题变成大事故 | 始终提供"紧急停止"按钮，可随时中断 |
| 权限一次性授予无法回收 | 授权后风险持续存在 | 权限按需申请、临时授权自动过期、可随时回收 |

## 四、实施细则

### 步骤1：权限层级设计

明确权限金字塔：

```
        ┌─────────────────┐
        │  用户/被控制方   │  ← 最高权限：可终止、可接管、可覆盖
        └────────┬────────┘
                 │
        ┌────────▼────────┐
        │  关键操作确认    │  ← 高危操作必须人类明确授权
        └────────┬────────┘
                 │
        ┌────────▼────────┐
        │  常规操作可追溯  │  ← 常规操作自动执行但完整记录
        └────────┬────────┘
                 │
        ┌────────▼────────┐
        │  只读/查询操作   │  ← 低风险操作默认允许
        └─────────────────┘
```

### 步骤2：状态可见性设计

- 始终显示代理/Agent的当前状态（正在做什么）
- 关键操作执行前有明确提示
- 操作过程中有进度反馈
- 敏感操作（如访问隐私数据）有显著标识

### 步骤3：紧急干预机制

- **一键终止**：显眼的"停止"按钮，立即终止所有代理操作
- **即时接管**：用户介入时代理自动暂停，控制权交还给用户
- **权限回收**：可随时撤销已授予的权限
- **操作回滚**：关键操作支持撤销/回滚（如文件变更可恢复）

### 步骤4：审计追溯能力

- 完整记录代理的所有操作（时间、操作内容、参数、结果）
- 操作日志不可篡改
- 支持事后回放审计
- 异常操作自动标记和告警

## 五、案例验证

### 案例1：向日葵远程控制

向日葵远控完整实践了用户主权默认原则：
1. **最高权限归属**：被控端始终可以一键断开远控
2. **操作可见性**：被控端显示远控状态提示，知道何时被连接
3. **风险可感知**：隐私屏让用户知道屏幕被查看，水印提示可能被录屏
4. **随时可干预**：任何时候被控端都可以终止会话、收回权限

### 案例2：AI Agent系统设计映射

将用户主权原则应用到AI Agent：
1. **最高权限**：用户可以随时暂停/终止Agent，Agent不能阻止用户干预
2. **可见性**：Agent的思考过程、工具调用、文件变更都对用户可见
3. **敏感操作确认**：删除文件、执行shell命令、发送消息等关键操作必须用户确认
4. **审计**：完整记录Agent的所有操作，支持事后审计和回滚

## 六、与其他模式的关系

| 关联模式 | 关系 |
|---------|------|
| non-intrusive-security-ux | 用户主权需要"安全不打扰"来平衡——既要可控，又不能过度打断 |
| full-process-defense-depth | 用户主权是事中守护层的核心机制之一 |
| ambient-proactive-agent | 主动式Agent更需要用户主权机制来建立信任 |
