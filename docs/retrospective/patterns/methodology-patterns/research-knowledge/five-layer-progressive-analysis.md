# 五层递进分析框架（Five-Layer Progressive Analysis）

## 模式类型

方法论模式（研究知识/质量审计/分析框架）

## 成熟度

L1 实验级（validation_count=1）：
1. 首次验证：对抗性审查知识库开放性提问任务（10个问题，5层结构，从认识论到边界完整覆盖）

## 适用场景

- 知识库构建完成后，需要进行系统性质量审计
- 需要评估方法论或框架的成熟度和稳健性
- 学术论文或技术方案的同行评审
- 任何需要从"为什么"到"何时失效"的完整追问链的分析任务

## 问题描述

知识库质量审计中，审计者通常只关注"内容是否准确"（内容层）和"格式是否规范"（格式层），遗漏了以下关键维度：

- 知识库的**认识论基础**是否成立？（假设层）
- 方法论**内部是否一致**？（一致性层）
- 质量保证机制**是否引入了新问题**？（偏差层）
- 方法论在**工程上是否可执行**？（实践层）
- 方法论的**适用边界**在哪里？（边界层）

这种维度缺失导致"看起来没问题"的知识库在实际使用中暴露出基础假设错误、内部矛盾、工程不可行等问题。

## 解决方案

### 核心框架

五层递进结构从"为什么"到"怎么做"再到"何时失效"，形成完整的追问链：

```
第1层：认识论基础层（为什么这样做是合理的？）
  ↓ 假设成立后追问
第2层：方法论张力层（方法内部是否一致？）
  ↓ 内部一致后追问
第3层：认知偏差防御层（方法本身是否引入了新偏差？）
  ↓ 偏差受控后追问
第4层：工程实践层（方法在工程上是否可行？）
  ↓ 工程可行后追问
第5层：边界与局限层（方法在什么条件下会失效？）
```

### 每层定义与典型问题

| 层 | 核心追问 | 典型问题示例 | 对应知识库构建阶段 |
|----|---------|-------------|-------------------|
| **认识论基础** | 假设是否成立？ | "自举验证在认识论上是循环论证还是自洽性检验？" | 理论基础 |
| **方法论张力** | 内部是否一致？ | "两大应用分支是否共享深层统一原理？" | 框架设计 |
| **偏差防御** | 是否引入新问题？ | "偏差防御是否会引入'过度否定'的元偏差？" | 质量机制 |
| **工程实践** | 是否可操作？ | "验证成本如何随知识库规模增长？" | 落地执行 |
| **边界局限** | 何时会失效？ | "对抗性审查的'不可审查盲区'如何枚举？" | 适用条件 |

### 操作步骤

```
步骤1：确认分析对象（知识库/方法论/框架）
步骤2：第1层——追问认识论基础：核心假设是什么？这些假设是否成立？在什么条件下成立？
步骤3：第2层——追问方法论一致性：各部分之间是否存在矛盾？表述是否一致？边界是否清晰？
步骤4：第3层——追问偏差防御：质量保证机制本身是否引入了新的偏差？防御措施是否完备？
步骤5：第4层——追问工程可行性：方法论在工程上是否可执行？成本是否可承受？是否有工具支撑？
步骤6：第5层——追问边界条件：方法论在什么条件下会失效？有哪些已知的不可处理问题？
步骤7：汇总各层问题，生成分析报告
```

### 规模适配

| 知识库规模 | 推荐层数 | 说明 |
|-----------|---------|------|
| 小型（<10文件） | 第1-3层 | 重点检查基础假设和方法论一致性 |
| 中型（10-30文件） | 全部5层 | 完整覆盖从假设到边界的全光谱 |
| 大型（>30文件） | 全部5层 + 每层2-3个问题 | 增加每层问题密度，确保覆盖充分 |

## 适用边界

- **适用**：基于明确方法论构建的知识库、框架、协议；有可追溯的理论基础（如引用了哲学/科学理论）
- **不适用**：纯事实性文档（如API参考、配置说明）；无明确方法论的临时笔记
- **与五维验证的关系**：五维验证检查"来源是否可信"（内容质量），五层递进检查"框架是否稳健"（结构质量）——两者互补

## 与其他模式的组合

| 组合模式 | 说明 |
|---------|------|
| + `spec-anchored-questioning` | 用Spec锚定提问法在每层生成具体问题，确保问题可追溯 |
| + `adversarial-review-protocol` | 五维验证（来源可信度）→ 五层递进（框架稳健性），构成完整审计 |
| + `knowledge-system-five-foundations` | 五层递进的第1层（认识论）可直接复用五维根基框架的追问方法 |
