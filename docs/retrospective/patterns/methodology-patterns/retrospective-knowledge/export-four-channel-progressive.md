# 导出四渠道递进模型（Export Four-Channel Progressive Model）

## 模式类型
方法论模式

## 成熟度
L1 实验性（1 次成功案例：methodology-analysis-report.md 综合方法论分析）

## 适用场景
将萃取形成的知识资产外化为具体行为、工具或制度——知识变现的"最后一公里"。

## 问题背景

知识萃取完成后，最常见的失败模式是"知识库是存档室，不是工具箱"——条目写得规范但没有人使用。原因往往是单一导出渠道（仅文档化）导致影响力受限。

四渠道递进模型按照"对人的影响广度"和"制度化程度"两个维度划分了四种导出渠道，并明确其递进关系与适用场景。

## 四渠道分类矩阵

| 渠道 | 影响面 | 制度化程度 | 导出成本 | 影响力持久性 |
|------|-------|-----------|---------|------------|
| 文档化 | 低 | 低 | 最低 | 最低（写了不等于有人看） |
| 模板化 | 中 | 低 | 中 | 中（降低重复决策成本） |
| 工具化 | 中 | 高 | 高 | 高（嵌入系统，强制使用） |
| 制度化 | 高 | 高 | 最高 | 最高（写入规范/绩效） |

## 四渠道详解

### 1. 文档化

**形式**：将知识写入操作手册、Wiki 页面、项目 README
**核心价值**：成本最低、传播最快
**局限性**：影响力最有限——"写了不等于有人看，看了不等于照做"
**适用场景**：
- 知识尚在验证期，频繁迭代的可能性高
- 临时性知识（如特定版本的特定操作）
- 受众已有主动查阅习惯

**反模式警示**：把一切知识都只做文档化——只能影响"主动查阅者"群体。

### 2. 模板化

**形式**：将重复性工作流程固化为可复用模板
**核心价值**：**降低每次重复的决策成本**——使用者不需要每次从头设计结构，只需按模板填充内容
**典型实例**：
- 复盘报告模板
- Spec 规范模板（spec.md / tasks.md / checklist.md）
- 知识条目模板
- Pull Request 模板

**适用场景**：
- 重复性高的工作流程（每周/每月/每项目都要做）
- 受众希望"少做决策"多于"灵活创新"
- 模板可以容纳合理的差异

**反模式警示**：模板字段过少丧失指导性、字段过多降低填写意愿。

### 3. 工具化

**形式**：开发脚本、自动化工具将知识嵌入系统
**核心价值**：**用工具治理工具**——每条烦人的手动检验流程，都可以催生一个新的自动化工具
**典型实例**：
- 索引自动生成脚本（generate_index.py）
- 链接有效性验证脚本（check-links.py）
- Spec 一致性验证脚本（check-spec-consistency.py）
- CI 自动化检查流水线

**适用场景**：
- 知识对应的检查/操作频次高（> 每天一次）
- 人工执行容易遗漏或出错
- 投资开发工具的成本能在合理周期内回收

**反模式警示**：为低频操作开发工具，工具维护成本超过收益。

### 4. 制度化

**形式**：将知识纳入组织的规范体系、角色职责、绩效考核
**核心价值**：**影响力最持久**——通过强制力确保知识被实际应用
**典型实例**：
- "开发者启动前必须查阅知识库"写入角色定义
- "所有 Spec 变更必须通过 checklist 验证"作为合并准入条件
- "复盘覆盖率 ≥ 80%"纳入团队 OKR

**适用场景**：
- 知识已经过反复验证、风险/影响重大
- 需要强制力确保全员执行
- 配套的考核/激励机制就位

**反模式警示**：制度化时机过早（知识尚未成熟）、制度化粒度过细（反而降低自主性）。

## 渐进式导出策略

**核心原则**：一条刚发现的知识不必一步到位制度化，应按成熟度分级推进：

```
[文档化] → [模板化] → [工具化] → [制度化]
   ↑                              ↓
   └──────── 验证失败时回退 ────────┘
```

**推进节奏参考**：
- 文档化 → 模板化：模板需求 ≥ 3 次/月
- 模板化 → 工具化：模板使用 ≥ 5 次/周 或 模板填写耗时占比 > 20%
- 工具化 → 制度化：工具已稳定运行 ≥ 3 个月，违规后果严重

**回退触发条件**：
- 知识本身被证伪
- 受众使用率持续低于预期
- 维护成本超过价值产出

## 决策准则速查

| 知识特征 | 推荐起点渠道 |
|---------|------------|
| 一次性经验、特定版本操作 | 仅文档化 |
| 重复性工作流程（< 3次/月） | 文档化 |
| 重复性工作流程（3-20次/月） | 模板化 |
| 重复性操作（> 20次/月 或 ≥ 5次/周） | 工具化 |
| 已验证的核心原则 + 强制力需求 | 制度化 |
