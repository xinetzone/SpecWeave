# 反模式：20+个Prompt写法陷阱

---

## 1. 章节引言

反模式是那些看起来很有道理、很多人都在用，但实际效果很差的写法——在GPT-3.5时代可能还有点用，到GPT-5.6时代不仅没用，反而有害。

本章收集25个最常见陷阱，每个都有❌错误示例、✅正确写法和危害说明。

> 🟢 **来源标注**：基于GPT-5.6新写法指南和七概念治理层Bad Case沉淀。

---

## 2. 🚫 目标定义类反模式（5个）

---

### 反模式1：形容词陷阱

**识别**：大量用"深入、全面、专业、严谨、高质量"等无法验证的词。

❌ 错误：
```
请深入、全面、专业、严谨地分析2026年新能源汽车市场，给出高质量的深刻洞察，不要泛泛而谈。
```

✅ 正确：
```
请分析2026年新能源汽车市场，回答4个问题：销量增速、TOP5份额、3个技术方向、政策风险。每个回答有数据支撑，标注来源年份，1500字以内。
```

**危害**：模型不知道"专业"是什么意思，只能猜，结果大概率不符合预期，还稀释关键信息。

---

### 反模式2：角色滥用

**识别**：上来写一长串"20年经验资深专家，曾在Google/Meta工作..."，但角色不影响输出。

❌ 错误：
```
你是一位拥有20年经验的资深Python工程师，毕业于顶尖大学，曾在Google工作，主导过千万级系统...请帮我写一个数据清洗函数。
```

✅ 正确：
```
请写一个Python数据清洗函数：输入CSV路径，输出清洗后的DataFrame+报告，处理缺失值/异常值/日期，要有类型注解和docstring。
```

**危害**：模型能力来自训练数据，不来自你贴的标签。只有当角色影响**判断视角**（如"站在新用户角度"）时才有用。

---

### 反模式3：目标模糊

**识别**："帮我优化一下""分析分析""聊聊XX"，不说具体优化什么、分析哪几方面。

❌ 错误：`帮我优化一下这段代码。`

✅ 正确：
```
帮我优化这段Python代码，从3个方面：1.性能（不必要循环→向量化）2.可读性 3.错误处理。给出具体建议，不要直接重写。
```

**危害**：让模型自由发挥，它可能优化了你不在乎的地方，漏掉你真正关心的问题。

---

### 反模式4：矛盾目标

**识别**：同时提两个互相矛盾的要求，让模型无所适从。

❌ 错误：`帮我写项目介绍，既要300字以内简短精炼，又要全面详细覆盖所有功能细节。`

✅ 正确：
```
帮我写两份：1.精简版300字给投资人，讲价值亮点；2.详细版1000字给技术团队，讲功能架构。
```

**危害**：矛盾目标让模型在两个方向摇摆，结果两边都不沾。

---

### 反模式5：没有完成标准

**识别**：只说"做XX"，不说"做成什么样算做完"。

❌ 错误：`帮我分析一下这个竞品。`

✅ 正确：
```
帮我分析竞品，交付报告包含：3个优势/3个劣势、功能对比表、2个可借鉴点+1个避坑点。Markdown格式，800字以内。
```

**危害**：模型不知道什么时候停，可能写50页还在继续，你拿到也没法判断"做完了没有"。

---

## 3. 🚫 结构类反模式（4个）

---

### 反模式6：一逗到底

**识别**：整个Prompt是一大段话，不分段、不用列表。

❌ 错误：`你好我想让你帮我写一份市场调研报告关于AI代码助手我们公司是做开发工具的想了解主要玩家定价策略目标用户核心功能发展趋势新进入者机会2000字要有数据不要瞎编信息不够就问我谢谢`

✅ 正确：
```
# Context
我们是开发工具公司，要做AI代码助手赛道调研给管理层。
# Request
回答：主要玩家/定价/核心功能/趋势/我们的机会。
# Output Format：Markdown，2000字内，数据标来源，对比用表格。
```

**危害**：结构化分段帮模型快速找重点，一大段话理解准确率大幅下降。

---

### 反模式7：信息缺失

**识别**：关键上下文没给，模型不知道你在说什么。

❌ 错误：`帮我看看这个Bug怎么修。`

✅ 正确：
```
帮我修React Bug：点击提交按钮没反应，控制台没报错。React18+TS，用react-hook-form，相关代码是src/components/LoginForm.tsx（见附件）。我已经检查过onClick绑定。
```

**危害**：你知道的背景模型不知道，要么问一堆问题浪费时间，要么按通用情况猜，结果不对。

---

### 反模式8：信息过载

**识别**：堆一堆和当前任务无关的信息，稀释重点。

❌ 错误：（先说2000字公司历史、成立时间、融资情况、人员构成...）...现在想做AI笔记功能，帮我写需求文档。

✅ 正确：
```
我们是笔记App，500万用户，主要是25-35岁白领。想加AI笔记功能：1.语音/文字→结构化笔记 2.支持追问 3.导出Markdown。帮我写PRD大纲，包含核心功能/用户流程/边界情况，1000字以内。其他背景需要再问我。
```

**危害**：无关信息稀释关键信号，甚至误导模型。只给当前任务必需的信息。

---

### 反模式9：重复啰嗦

**识别**：同一条规则说三遍五遍。

❌ 错误：
```
整理会议记录。注意：不要猜测缺失信息→只用记录内容→不确定标[待确认]→不要脑补→信息不足不要瞎猜→不确定就标[待确认]→千万不要编...
```

✅ 正确：`整理会议记录。只用记录中有的内容，缺失/不确定标[待确认]，不要猜测编造。`

**危害**：重复不会让模型更重视，反而浪费Token，甚至因表述差异造成规则冲突。一条规则说一遍说清楚就够了。

## 4. 🚫 过程规定类反模式（4个）

---

### 反模式10：教模型思考

**识别**："请三轮反思""模拟五位专家""先SWOT再PEST""用第一性原理思考"。

❌ 错误：（规定第一步到第八步，三轮反思+三位专家辩论...）

✅ 正确：
```
分析是否进入XX市场，给出：3个机会附依据、3个风险附概率评估（高/中/低）、最终建议（进入/观望/不进入）。乐观和风险视角都要覆盖，结论明确，800字以内。
```

**危害**：GPT-5.6已有成熟自主推理能力，人为规定流程限制最优路径，还会造成"表演式思考"——机械走流程给你看，但不是真的深度分析。

---

### 反模式11：死板规定步骤

**识别**："第一步A第二步B第三步C"，把每一步都规定死。

❌ 错误：`写登录功能严格按步骤：第一步写表单组件，第二步写验证，第三步写API，第四步错误处理，第五步加loading，必须按顺序来。`

✅ 正确：`实现登录功能，包含：邮箱密码表单、表单验证、API调用、错误提示、loading状态。用现有react-hook-form+zod模式，风格和现有页面一致。`

**危害**：你规定的步骤不一定是最优路径，甚至可能是错的。模型比你更清楚怎么高效完成任务。

---

### 反模式12：CoT滥用

**识别**：不管什么任务都加"Let's think step by step"，连1+1也要一步步想。

❌ 错误：`What is 2 + 2? Let's think step by step.`

✅ 正确：`2+2等于几？`

**危害**：链式思考只在复杂推理任务上有用，简单任务用CoT浪费Token，输出还啰嗦。模型自己知道什么时候需要一步步想。

---

### 反模式13：强制多角色辩论

**识别**：不管问题多简单，都要"正方反方辩论""模拟三位专家讨论"。

❌ 错误：`我中午吃什么？模拟营养师、美食家、减肥专家三个人辩论，最后给结论。`

✅ 正确：`推荐3个工作日午餐选择：健康、方便、30分钟内吃完，每个说优缺点。`

**危害**：简单问题不需要辩论。多角色讨论只给复杂、有争议的决策用，否则输出又长又乱，没有清晰结论。

---

## 5. 🚫 输出要求类反模式（4个）

---

### 反模式14：格式不明确

**识别**："结构化输出""条理清晰"，但不说具体什么结构。

❌ 错误：`帮我分析这几个方案优缺点，结构化输出，条理清晰一点。`

✅ 正确：`对比3个方案，用Markdown表格，列包括：名称、开发周期、成本、优点、缺点、推荐指数（1-5星）。表格下写200字以内最终推荐。`

**危害**："结构化"对不同人意思不同——有人觉得分点列，有人觉得用表格，有人觉得要分级标题。不说清楚模型只能猜。

---

### 反模式15：长度不限制

**识别**："不限字数""越长越好""尽量详细"。

❌ 错误：`帮我写产品介绍，不限字数，越详细越好，所有功能都写上。`

✅ 正确：`写产品介绍800字左右，面向新用户：一句话价值主张→3个核心功能（配用户场景）→适用人群→号召用语。`

**危害**："不限字数"几乎必然导致输出冗长，真正重要的信息反而被淹没。给明确长度范围，模型知道怎么取舍。

---

### 反模式16：受众不清晰

**识别**：没说写给谁看，模型不知道该用什么深度、术语、语气。

❌ 错误：`解释一下什么是大语言模型。`

✅ 正确（二选一）：
- `给爷爷奶奶解释LLM，完全不懂技术，不要术语，用生活例子，300字以内。`
- `给有5年经验的ML工程师讲Transformer架构细节，重点讲注意力机制数学原理。`

**危害**：给小白讲和给工程师讲，内容深度完全不同。不说清楚受众，模型取中间值，结果两边都不讨好。

---

### 反模式17：术语/语气矛盾

**识别**："通俗易懂但要专业""口语化一点但要正式"。

❌ 错误：`写公众号文章，要用大白话让所有人看懂，又要专业严谨不能出错，语气轻松活泼但要正式官方适合公司发布。`

✅ 正确：`写公众号文章，面向普通读者，语气轻松像聊天，专业内容用打比方讲，不要术语，不要官话套话。`

**危害**：语气术语要求要统一，不要自相矛盾。有多个要求就说清楚优先级。

---

## 6. 🚫 Codex/Agent专属反模式（5个）

---

### 反模式18：没有文件边界 🔴极高危

**识别**："修个Bug""加个功能"，完全不说哪些文件能动哪些不能动。

❌ 错误：`帮我给这个项目加个登录功能。`

✅ 正确：
```
加登录功能（React+TS）。
✅ 只允许改：src/pages/Login.tsx（新建）、src/api/auth.ts（新建）
❌ 绝对不要改：路由、package.json（不加新包）、schema、现有页面。
开始前列计划，我确认后再动手。
```

**危害**：这是Codex最危险的反模式——没有边界，Agent可能"好心帮你重构"整个项目，改坏核心配置。用**白名单**思维：说清楚哪些可以改，其他都不行。

---

### 反模式19：没有停止条件 🔴极高危

**识别**：不说遇到什么情况该停下来问，出错了就一直重试死循环。

❌ 错误：`帮我升级react版本，解决依赖冲突。`

✅ 正确：
```
把react从18.2.0升到18.3.x最新小版本（不要升19）。
立即停的情况：peer dependency冲突、build失败、需要改源代码兼容、需要升级其他包。
先告诉我打算升哪个版本，确认后再操作。
```

**危害**：没有Checkpoint，Agent遇到问题会自己"想办法解决"——可能乱改代码、升级一堆无关包，小问题变大事故。

---

### 反模式20：允许危险操作 🔴极高危

**识别**：没说禁止什么操作，Agent可能删文件、DROP表、执行危险命令。

❌ 错误：`帮我清理项目里的临时文件和没用的代码。`

✅ 正确：
```
清理临时文件：
✅ 可删：.tmp、本次新建的临时文件、__pycache__
❌ 绝对禁止删：src/下源代码、任何配置文件、不确定能不能删的文件
删除前列清单，我确认后再删。禁止执行rm -rf等批量删除命令。
```

**危害**：Agent没有"谨慎"概念，你没说不能删它就可能删。必须明确禁止：删非新建文件、改核心配置、装包、无WHERE的DELETE/DROP、curl | bash等。

---

### 反模式21：不给项目上下文

**识别**：不说技术栈、项目结构、现有代码模式，Agent自己猜。

❌ 错误：`帮我写用户注册表单。`

✅ 正确：
```
当前项目Next.js14+TS+App Router：shadcn/ui组件、react-hook-form+zod表单、toast()提示、类型在src/types/、API返回{success,data,error}格式。写注册表单，风格和现有Login页一致。
```

**危害**：你知道项目用什么技术栈和约定，模型不知道。不给上下文，它可能用你没装的库，发明新模式，你还要花大量时间改。

---

### 反模式22：不要求先出计划

**识别**：上来就让直接改代码，不先列计划确认。

❌ 错误：`帮我重构用户模块，直接改吧。`

✅ 正确：
```
重构用户模块：1.先读src/modules/user/下所有文件理解结构 2.列出要改哪些文件、每个改什么、重构后结构 3.我确认计划后再动手。约束：只重构user模块，对外API保持兼容，不改其他文件import。
```

**危害**：不先看计划就动手，改完发现方向完全错了，回滚成本极高。先列计划是防返工最后一道防线。

---

## 7. 🚫 多轮对话反模式（3个）

---

### 反模式23：每轮重复完整Prompt

**识别**：每次发消息都把整个Prompt（背景、要求、约束）重新发一遍。

✅ 正确做法：第一轮发完整Prompt，后续只说补充/修改什么即可，模型有上下文记忆。

**危害**：浪费Token，还可能稀释新指令。

---

### 反模式24：纠正不说明原因

**识别**：模型输出不对，只说"不对""重来"，不说哪里错、为什么错、怎么改。

❌ 错误：`不对，重来。`

✅ 正确：`方向不对，两个问题：1.我要的是B端不是C端，目标用户搞错了 2.预算太高，我们只有50万不是200万。其他没问题，按这两点调整。`

**危害**：只说"不对"，模型不知道哪里错，可能猜错问题，改了不该改的地方，还是不对。

---

### 反模式25：上下文丢失不补充

**识别**：对话太长被截断，或开了新对话，还当模型记得之前说过什么。

✅ 正确做法：开新对话或发现模型"失忆"时，快速列一下之前的关键要点再继续。

**危害**：上下文窗口有限，对话太长早期内容会被截断，开新对话是全新上下文。不补充信息，模型完全不知道你在说什么。

---

## 9. 本章小结
