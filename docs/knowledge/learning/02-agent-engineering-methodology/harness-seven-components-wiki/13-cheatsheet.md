> 本页是七大组件的快速参考卡，适合在设计或评审Agent时快速对照。

## 一、七大组件速查表

| # | 组件 | 英文 | 一句话定义 | 核心问题 | 关键原则 | 常见误区 |
|---|------|------|-----------|---------|---------|---------|
| 1 | 模型网关 | Model Gateway | 大脑调度中心 | 这次任务用哪个模型？ | 该省省该花花，规则任务不用模型 | 什么都用最贵模型 |
| 2 | 工具注册表 | Tool Registry | 手脚管理 | 能调用什么工具？怎么调？失败怎么办？ | 工具描述要精确，最小工具集起步 | 工具过多、无失败处理 |
| 3 | 知识库引擎 | Knowledge Base | 业务参考书 | Agent从哪里获得业务判断力？ | 知识库是判断力缓存，质量>数量 | 什么都往里塞、等同于向量库 |
| 4 | 记忆系统 | Memory System | 便签本+档案柜 | 当前状态是什么？长期偏好是什么？ | 不是记得越多越好，要有遗忘机制 | 无限记忆、不做边界、串台 |
| 5 | 策略引擎 | Policy Engine | 规则红线 | 什么绝对不能做？质量标准是什么？ | 硬约束>软提示，安全红线优先 | 策略当Prompt、一刀切 |
| 6 | 可观测性 | Observability | 运营仪表盘 | 干得怎么样？哪里出问题？ | Badcase闭环驱动优化 | 只看结果不看过程 |
| 7 | 配置管理 | Configuration | 调教面板 | 这次任务有什么特殊要求？ | 业务参数配置化，合理默认值 | 硬编码、配置项过多 |

## 二、核心金句速记
- **大模型解决智能问题，Harness解决交付问题**
- **模型提供"能"的潜力，Harness变成"稳"的交付**
- **知识库是你的判断力缓存**
- **该用好模型的地方不要省，该用便宜模型的地方不要浪费，能用规则解决的地方就别麻烦大模型**
- **会做Agent的团队不指望一次写对，而是建立Badcase闭环**
- **Prompt是软请求，Harness组件是硬约束**
- **传统产品经理交付功能，AI产品经理交付效果**
- **没有可观测性就没有Agent运营**
- **你不是每次都重新教Agent怎么写，而是通过配置告诉它这次我要什么**
- **未来有价值的Agent不是会聊天的Agent，而是能在具体业务里稳定交付结果的Agent**

## 三、组件依赖关系速查
```
模型网关（入口）
  ├── 调用 → 工具注册表（能力）
  ├── 检索 → 知识库引擎（知识）
  ├── 读取 → 记忆系统（上下文+偏好）
  ├── 约束 ← 策略引擎（红线，横切所有组件）
  ├── 记录 → 可观测性（全局追踪）
  └── 配置 ← 配置管理（贯穿全程）
```

## 四、落地优先级速查

### 最小可用（MVP）
1. 模型网关（简单路由）
2. 工具注册表（3-5个核心工具）
3. 策略引擎（安全红线）
4. 可观测性（基本日志+Badcase记录）

### 进阶增强
5. 知识库引擎（精选内容入库）
6. 记忆系统（短期+基础长期记忆）

### 成熟运营
7. 配置管理（多场景参数配置）

## 五、Badcase归因速查
当Agent出问题时，快速定位到组件：

| 问题现象 | 最可能的组件 | 排查方向 |
|---------|------------|---------|
| 回答胡说八道/事实错误 | 知识库引擎 | 知识缺失或知识过时 |
| 风格不像/前后矛盾 | 记忆系统 | 长期记忆不准/短期记忆丢失 |
| 泄露隐私/越权操作 | 策略引擎 | 红线规则缺失或未强制执行 |
| 成本太高/响应太慢 | 模型网关 | 路由策略不合理，简单任务用了大模型 |
| 工具调用失败/乱调工具 | 工具注册表 | 工具描述不清/无错误处理 |
| 不知道为什么出错 | 可观测性 | 日志不足，没有追踪链路 |
| 换个场景就要改代码 | 配置管理 | 场景参数硬编码，没有配置化 |

## 六、Prompt vs Harness决策速查
遇到一个需求时，判断应该用Prompt还是Harness组件：

| 如果需求是... | 应该用... |
|-------------|---------|
| "请用友好的语气回复" | Prompt（风格引导） |
| "绝对不能透露客户手机号" | 策略引擎（硬约束） |
| "参考我过去关于XX的观点" | 知识库引擎（知识检索） |
| "记住我不喜欢AI鸡汤" | 记忆系统（长期偏好） |
| "这个分类任务用最便宜的模型" | 模型网关（路由决策） |
| "需要查历史文章再回答" | 工具注册表（工具调用） |
| "这次面向技术受众" | 配置管理（任务配置） |
| "记录每次回答的质量评分" | 可观测性（数据追踪） |
