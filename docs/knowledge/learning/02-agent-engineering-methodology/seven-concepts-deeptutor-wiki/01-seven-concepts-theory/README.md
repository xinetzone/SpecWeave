# 七概念理论 - 索引

---

## 七概念总览

七概念方法论由7个核心概念组成，它们不是孤立的"最佳实践"清单，而是一套有层级、有顺序、有依赖关系的完整思维与执行体系。七个概念分别是：

- **R 复盘**（Retrospective）：把已经发生的事情讲清楚，从时序经验中提炼因果
- **I 洞察**（Insight）：从复盘中发现跨情境可迁移的规律，形式化为四元组
- **E 萃取**（Extraction）：把洞察进一步抽象成可复用的模式，存入知识库
- **C 原子提交**（Atomic Commit）：保证每次变更都是单一职责、可独立回滚的单元
- **A 原子化**（Atomization）：把复杂系统拆分成最优粒度的模块，平衡认知与导航成本
- **F 第一性原理**（First Principles）：从最基础的公理出发，自下而上推导解决方案
- **V 对抗性审查**（Adversarial Review）：主动找反例，防御确认偏误，给每一步认知加防火墙

这七个概念之间的关系可以用一句话概括：**F帮你在未知领域找方向，A帮你把事情拆对，C帮你把事情做对，R帮你做完后想清楚，I帮你从经历中找规律，E帮你把规律变成资产，V在每一步帮你挑错防坑。**

## 五层层级模型

七个概念分布在五个层级上，每个层级有明确的核心职能：

```
感知层 → 认知层 → 验证层（横切） → 执行层 → 沉淀层
```

| 层级 | 核心职能 | 包含的概念/活动 |
|------|---------|----------------|
| **感知层** | 信息采集、现象观察（只记录，不判断） | R的事实采集 |
| **认知层** | 思维推理、本质洞察、方案生成 | F第一性原理、R反事实推演、I洞察、E抽象提升、A粒度判断 |
| **验证层** | 证伪防御、质量保障、偏差修正（横切所有需要判断的环节） | V对抗性审查 |
| **执行层** | 操作落地、变更实施、粒度控制 | A拆分/合并、C原子提交 |
| **沉淀层** | 知识归档、模式复用、资产积累 | R因果转化、E形式化编码、C变更归档、知识库 |

数据流向是从感知层进入，经过认知层加工，通过验证层校验，在执行层落地，最后沉淀到知识库；沉淀的知识又会反过来指导未来的感知和认知，形成闭环。验证层V是横切的——它不单独在某个位置，而是作用在认知层和执行层的每一个关键输出之后，帮你检查是不是错了。

## 七概念公理速查表

| 概念 | 一句话公理 | 4个基础要素 | 层级归属 |
|------|-----------|------------|---------|
| **R 复盘** | 对已发生事件的结构化反事实推理，将时序经验转化为因果知识 | 事实采集、时序结构化、反事实推演、因果转化 | 感知→认知→沉淀 |
| **I 洞察** | 跨情境可迁移规律，最小形式为「C→M→A→B」四元组 | 条件识别、机制揭示、结论生成、迁移验证 | 认知+验证 |
| **E 萃取** | 知识从隐性经验到显性模式的形式化转换，四层漏斗逐级精炼 | 显化转换、抽象提升、漏斗过滤、形式化编码 | 认知→沉淀 |
| **C 原子提交** | 变更集的不可分割单一职责单元，同因同果、可独立回滚、review无认知跳跃 | 职责内聚、因果闭合、独立回滚、认知平滑 | 执行→沉淀 |
| **A 原子化** | 复杂系统向最优信息粒度的收敛，平衡认知负荷与导航成本 | 粒度寻优、单元独立、链接完整、双向收敛 | 认知→执行 |
| **F 第一性原理** | 从不可证伪公理出发，自下而上重构方案的思维方式 | 假设剥离、要素拆解、公理自洽、重构推导 | 认知+验证 |
| **V 对抗性审查** | 主动寻找证伪证据的认知防御机制，构造反例暴露确认偏误 | 证伪导向、多角攻击、偏差防御、审计可溯 | 验证层（横切） |

**洞察四元组格式说明**：`[条件C]` → 因为`[机制M]` → 做`[行动A]` → 导致`[结果B]`。一个合格的洞察必须满足这四个要素都齐全，并且是可证伪的（也就是说，你能说清楚在什么情况下它不成立），还要附上可以迁移到的其他场景。
