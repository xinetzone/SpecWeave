## 映射强度说明

七个概念在DeepTutor中都有**强映射**——这不是巧合，而是DeepTutor作为一个架构设计优秀的系统，自然地体现了优秀设计背后的方法论逻辑。

但它们的体现方式各有侧重：

### 1. 架构层映射（最强）
- **F→底层Agent循环**：这是DeepTutor和其他"套壳LLM"产品最根本的区别。大多数AI学习工具从"做一个聊天机器人"出发，在聊天界面上堆功能；DeepTutor从"终身个性化辅导"这个第一性原理出发，把Agent循环写在底层，六种模式只是同一个引擎的不同视图。这是F最纯粹的体现。
- **A→九大模块设计**：左侧导航九个模块的划分不是随意的——既没有合并成一个大界面导致认知负荷过高，也没有拆成几十个小模块导致导航成本爆炸。这是A的粒度寻优在产品设计上的直接体现。
- **C→模式切换机制**："切模式不换引擎、不丢上下文"这个设计看起来简单，但它要求整个系统从底层就把"上下文共享"作为原子边界——每个模式的切换都是一个"原子提交"，相关的上下文在同一个单元内闭合，可以独立切回而不丢失状态。

### 2. 知识层映射（核心）
- **R→Memory三层记忆**：L1→L2→L3不是简单的"摘要分级"，它完整对应了复盘R的四个要素——L1是客观事实采集（不带判断），L2是时序结构化整理，Memory Graph是反事实推演的证据链（每条结论可追溯），L3是因果转化后的可复用知识。
- **E→Knowledge Center**：从原始文档导入（显化转换），到四引擎RAG索引（抽象提升），到版本管理保留历史（漏斗过滤），到Skills/EduHub技能包（形式化编码）——这完整走了一遍萃取E的四层漏斗。
- **V→Quiz+Mastery Path**：学习中最大的认知偏差是"我以为我懂了"（确认偏误）。Quiz通过出题构造反例来证伪"我懂了"这个假设，Mastery Path的达标gate是质量门——不通过就不能进入下一阶段，这和代码审查不通过不能合并是同一个逻辑。

### 3. 学习层映射（关键）
- **I→Mastery Path追踪**：系统不是线性推完内容就算完成，而是追踪用户在各类题目上的表现，识别"什么条件下用户会做错/做对"的规律，这正是洞察I的四元组结构——它需要找到可证伪的、可迁移的学习规律，而不是简单的"看完了=学会了"。

## 映射关系的启示

从这张总览表可以看出一个有意思的现象：**优秀的系统设计不是刻意套用方法论的结果，而是当你真正解决了正确的问题，自然就会体现出这些方法论原则。**

DeepTutor的作者在设计时可能并没有 consciously 按照七概念来做，但因为他抓住了"AI学习工具到底应该是什么样"这个本质问题，从第一性原理出发设计架构，自然就把R（记忆分层可溯源）、I（掌握规律追踪）、E（知识萃取成可检索形式）、C（模式切换原子性）、A（模块粒度平衡）、V（学习效果验证）这些要素都做对了。

这正是方法论的价值——它不是让你"照本宣科"地套公式，而是帮你在设计时能判断"什么是对的"，在看完优秀系统时能看懂"它为什么这么设计"。

接下来，我们逐个概念深入分析，看看这些映射在细节上是如何体现的。
