# OKR评分方法

> **来源**: [怎么对OKR评分？](https://www.dingteam.com/help/620)

## 评分维度

OKR评分通常考虑两个方面：

| 维度 | 说明 |
|------|------|
| **完成度** | 实际完成情况与目标值的对比 |
| **符合预期程度** | 完成质量是否达到预期 |

---

## 评分标准

评分范围为 **0-1分**，具体标准如下：

| 分数 | 说明 |
|------|------|
| **0分** | 完全没有进展 |
| **0.3分** | 进展很小，远未达到目标 |
| **0.5分** | 完成了一半左右 |
| **0.7分** | 接近目标，略有差距（理想状态） |
| **1.0分** | 完全达成目标 |

---

## 理想得分

**0.6-0.7分**代表不错的表现。

**原因**:
- OKR的要求是激进的
- 0.6-0.7分表示团队在挑战自我
- 即使未完全达成，也推动了团队进步

---

## 低分分析与改进

当OKR评分低于0.4分时，需要从以下两个方面考虑改进：

### 1. 完成度低

**常见原因**:
- 指定目标过于抽象、宏观
- 需要分解目标，思考更多更可靠的途径和方法
- 由于遗忘、不够专注导致

**改进建议**:
- 分解目标，使其更具体
- 提高OKR Review的频率
- 增加跟进和提醒机制

### 2. 符合预期程度低

**常见原因**:
- 结果不符合预期，需要深入分析原因
- 无法得到原因时，需要给出合理假设

**改进建议**:
- 及时复盘，深入分析结果背后的原因
- 给出合理假设，并通过进一步实验验证

---

## 重要提醒

**0.4分以下并不意味着失败！**

它的意义在于：
- 明确重要性及优先级
- 帮助我们发现问题
- 分数永远不是最重要的，除了作为一个直接的引导作用
