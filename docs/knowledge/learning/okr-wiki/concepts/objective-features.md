# Objective（目标）的特征

> **来源**: [最全OKR制定指南（2种思路+7类方法）](https://www.dingteam.com/help/584)

## 五大特征

一个好的Objective应具备以下特征：

| 特征 | 说明 | 示例 |
|------|------|------|
| **鼓舞人心** | 激发团队热情，让人愿意全力以赴 | "打造公司历史上最成功的产品发布" |
| **方向性** | 指明前进方向，而非具体任务 | "提升产品市场影响力" |
| **可理解** | 所有人都能快速理解其含义 | "构建高可用的技术基础设施" |
| **有时限** | 明确时间范围（通常一个季度） | "Q2完成核心功能迭代" |
| **可实现** | 有挑战但不是天方夜谭 | "成为行业前三" |

## 常见误区

| 误区 | 表现 | 问题 |
|------|------|------|
| ❌ **任务导向** | "完成性能优化项目" | 缺少目标和价值，变成待办事项 |
| ❌ **过于宽泛** | "提升用户体验" | 不具体，无法指导行动 |
| ❌ **过于具体** | "降低系统故障率到0.01%" | 应该是KR，不是O |
| ❌ **缺乏挑战** | "保持现有市场份额" | 无法激励团队突破 |

## 正面示例

| 类型 | 示例 |
|------|------|
| ✅ **产品方向** | "打造业界领先的高性能交易系统" |
| ✅ **市场拓展** | "开拓东南亚新兴市场，建立品牌影响力" |
| ✅ **团队建设** | "构建高效协作的研发团队文化" |
| ✅ **技术创新** | "突破核心技术瓶颈，实现自主可控" |

## 制定建议

1. **从价值出发**: 思考这个目标能带来什么价值，而非具体做什么
2. **避免任务化**: 使用"提升"、"打造"、"构建"等动词，而非"完成"、"做"
3. **保持简洁**: 一句话描述，让所有人都能记住
4. **设定时限**: 明确时间范围，通常为一个季度
