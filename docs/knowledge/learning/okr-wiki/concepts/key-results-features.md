# Key Results（关键结果）的特征

> **来源**: [最全OKR制定指南（2种思路+7类方法）](https://www.dingteam.com/help/584)

## 六大特征

好的Key Results必须满足以下标准：

| 特征 | 说明 |
|------|------|
| **可量化** | 有明确的数字指标，能衡量进度 |
| **可衡量** | 能判断是否达成及达成程度 |
| **有时限** | 明确的时间节点 |
| **具挑战** | 60%-70%完成率为最佳平衡点 |
| **强相关** | 直接支撑Objective的达成 |
| **聚焦结果** | 关注产出而非活动 |

## 核心准则

**所有KR全部达成，目标O必须100%实现；缺了任何一个KR，目标就完不成。**

这意味着每个KR都是不可或缺的，它们共同构成了目标达成的必要条件。

## 常见误区

| 误区 | 表现 | 问题 |
|------|------|------|
| ❌ **活动化** | "每周发3篇推文" | 关注活动而非结果 |
| ❌ **不可量化** | "提升客户满意度" | 无法判断完成程度 |
| ❌ **数量过多** | 一个O配8个以上KR | 无法聚焦重点 |
| ❌ **与O无关** | O是"提升产品体验"，KR是"招聘5人" | 无法支撑目标达成 |

## KR量化的七类方法

| 类型 | 说明 | 示例 |
|------|------|------|
| **基准线型** | 建立初始基准指标 | "建立培训满意度衡量标准" |
| **正向增长型** | 基于特定数值的增加 | "销售线索增加5%" |
| **反向控制型** | 对逆向风险的规避 | "客户流失率控制在10%以内" |
| **数量区间型** | 设定数值范围 | "咨询利用率维持在60%-90%" |
| **过程策略型** | 衡量关键策略的数量 | "启动首次登录的三项用研" |
| **里程碑型** | 目标完成的关键节点 | "完成A轮融资" |
| **趣味激励型** | 描述有感情的实现场景 | "让公司50%的小朋友达到年薪30万" |
