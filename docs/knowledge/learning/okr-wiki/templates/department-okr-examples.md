# 部门级OKR示例

> **来源**: [常见OKR模版](https://www.dingteam.com/help/608)

---

## 研发部门

### O1：确定创新方向

| KR | 描述 |
|----|------|
| KR1 | 通过近两年全球学术论文研究，完成技术趋势发展报告 |
| KR2 | 通过销售人员座谈、消费者访问等措施，完成市场需求分析报告 |
| KR3 | 通过团队共创等方法，完成至少5个创意提案 |
| KR4 | 通过创新决策会议，确定创新方向 |

### O2：搭建开放式创新平台

| KR | 描述 |
|----|------|
| KR1 | 完成开放式创新平台方案设计 |
| KR2 | 梳理国内高校、研究所、专业研究公司等机构，完成潜在合作对象清单 |
| KR3 | 拜访至少10家机构，至少与3家机构建立研发合作关系 |

### O2：提升X产品销售

| KR | 描述 |
|----|------|
| KR1 | 完成所有员工X产品培训，确保掌握X产品所有卖点 |

## 销售支持部门

### O1：完善销售分析流程

| KR | 描述 |
|----|------|
| KR1 | 建立销售分析智能平台 |
| KR2 | 回顾销售活动指标，每周向团队发送报告总结 |
| KR3 | 回顾销售渠道指标，每周向团队发送总结报告 |
| KR4 | 回顾销售业绩指标，每周向团队发送总结报告 |

### O2：提高销售支持效率

| KR | 描述 |
|----|------|
| KR1 | 通过风险分析、流程分析、优化销售支持流程 |
| KR2 | 通过每季度对上周期典型资料退回案例分析，实现资料退汇率降低至5% |
| KR3 | 新增3名销售支持人员 |
