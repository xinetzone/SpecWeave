# OKR检查清单

> **来源**: [OKR检查清单](https://www.dingteam.com/help/606)

---

## Objective检查清单

### 基础特征

- [ ] 目标是否鼓舞人心，能激发团队热情？
- [ ] 目标是否方向明确，而非具体任务？
- [ ] 目标是否简洁清晰，所有人都能理解？
- [ ] 目标是否有时限，明确时间范围？
- [ ] 目标是否有挑战，但可实现？

### 战略对齐

- [ ] 目标是否与上级目标对齐？
- [ ] 目标是否符合公司整体战略方向？
- [ ] 目标是否与其他部门目标协同？

### 避免误区

- [ ] ❌ 目标不是任务导向（如"完成xxx项目"）
- [ ] ❌ 目标不是过于宽泛（如"提升用户体验"）
- [ ] ❌ 目标不是过于具体（如"降低故障率到0.01%"）
- [ ] ❌ 目标不是缺乏挑战（如"保持现有市场份额"）

---

## Key Results检查清单

### 基础特征

- [ ] KR是否可量化，有明确的数字指标？
- [ ] KR是否可衡量，能判断完成程度？
- [ ] KR是否与O强相关，直接支撑目标？
- [ ] KR是否有时限，明确完成时间？
- [ ] KR是否有挑战，60%-70%完成率为优？

### 数量控制

- [ ] KR数量是否控制在3-5个？
- [ ] 所有KR达成是否确保O 100%实现？

### 避免误区

- [ ] ❌ KR不是活动化（如"每周发3篇推文"）
- [ ] ❌ KR不是不可量化（如"提升客户满意度"）
- [ ] ❌ KR不是数量过多（超过5个）
- [ ] ❌ KR不是与O无关（如O是"提升产品体验"，KR是"招聘5人"）

---

## 整体检查清单

### 对齐检查

- [ ] 是否完成向上对齐，与上级确认过？
- [ ] 是否完成横向对齐，与相关方沟通？
- [ ] 是否避免了目标脱节，形成"部门墙"？

### 可行性检查

- [ ] 是否有明确的负责人？
- [ ] 是否有必要的资源保障？
- [ ] 是否有潜在风险和应对方案？

### 合规性检查

- [ ] 是否符合公司使命和价值观？
- [ ] 是否符合法律法规要求？
- [ ] 是否涉及敏感信息需要保护？

---

## 制定流程检查清单

### 前期准备

- [ ] 公司使命和战略是否明确？
- [ ] 团队是否了解OKR理念和方法？
- [ ] 是否有指定的OKR负责人？

### 制定过程

- [ ] 是否采用了合适的制定方法（自上而下/自下而上）？
- [ ] 是否进行了充分的团队共创？
- [ ] 是否考虑了不同意见和建议？

### 后期验证

- [ ] 是否与上级进行了充分沟通？
- [ ] 是否进行了必要的调整和优化？
- [ ] 是否在系统中完成了设置和对齐？

---

## 周期管理检查清单

### 跟进检查

- [ ] 是否建立了OKR跟进周会制度？
- [ ] 是否定期更新OKR进度？
- [ ] 是否及时处理影响目标进展的问题？

### 评分检查

- [ ] 是否明确了评分标准和方法？
- [ ] 是否确保评分不与薪资挂钩？
- [ ] 是否对高分和低分进行了分析？

### 复盘检查

- [ ] 是否进行了团队沟通复盘？
- [ ] 是否进行了全员复盘？
- [ ] 是否总结了经验教训？
- [ ] 是否找标杆、树典型？
