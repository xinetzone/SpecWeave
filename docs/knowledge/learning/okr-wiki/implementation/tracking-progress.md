# OKR跟进与复盘

> **来源**: [叮当OKR落地实操详细指南](https://www.dingteam.com/help/582)

## 跟进时间

**OKR周期第2周——OKR周期最后1周**

## 建立OKR跟进周会制度

### 周会频率

**建议**: 每周一或每周五

### 周会前准备

**操作**: 通过叮当OKR的四象限看板，更新以下内容：

| 内容 | 说明 |
|------|------|
| **OKR进度** | 更新当前OKR完成进度 |
| **本周任务** | 列出本周需要完成的任务 |
| **未来计划** | 规划下周的工作计划 |
| **影响目标进展因素** | 记录影响目标进展的问题 |

### 周会内容

**要点**:
- 必要时对OKR进行调整并重新对齐
- 对于需要协调其他员工的任务事项，可以@他人
- 方便其他同事知晓进度

## 管理层跟进

### 跟进频率

**建议**: 以月为维度

### 跟进方式

**建议**: 以会议的方式进行进度追踪

### 跟进内容

**要点**:
- 查看目标完成情况
- 判断风险
- 必要时对OKR进行及时调整并重新对齐

## 更新OKR

### 如何更新OKR？

**操作**: 点击KR的进度按钮，填写当前信心指数、进度值及更新说明

**要点**:
- **信心指数**: 对KR达成的信心程度
- **进度值**: 当前完成进度
- **更新说明**: 进度更新的原因和说明（管理员可设置更新说明是否必填）

### 目标进度更新

**规则**:
- 目标进度默认将按KR进度自动更新
- 若有对齐你的子目标（对齐你的O而非KR），也可选择按子目标进度自动更新

### 更新记录及评论互动

**功能**:
- 点击进入目标详情页，可以看到系统自动记录的每一次更新操作
- 更新后会实时提醒相关人员
- 其他人员可以对该OKR进行点赞、评论，并@相关人员

**好处**: 大大提升协作效率

## 任务管理

### 为什么要创建任务？

**原因**: 在OKR执行过程中，可以将KR再进行拆解，制定短期内要完成的【任务】来支撑目标的完成。

**好处**:
- 将要做的to do事项记录下来
- 确定事情的轻重缓急
- 在复盘时更好地回顾自己的工作

### 如何创建任务？

**操作**: 在OKR目标详情页创建任务

**任务属性**:

| 属性 | 说明 |
|------|------|
| **负责人** | 默认为自己，也可添加任务分配给他人 |
| **起止时间** | 任务逾期后，系统将每日进行通知提醒 |
| **关联关键结果** | 可选择关联或不关联 |
| **优先级** | 不同优先级的任务将以不同颜色显示 |
| **参与人** | 添加任务参与人进行高效协作 |

### 通知机制

**规则**:
- 被分配任务的人员将收到系统通知
- 添加任务参与人时，对方将收到系统通知
- 任务逾期后，系统将每日进行通知提醒
