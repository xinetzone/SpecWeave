# OKR周期设置

> **来源**: [叮当OKR落地实操详细指南](https://www.dingteam.com/help/582)

## 什么是周期？

周期是指OKR的执行时间段。在正式开始制定OKR之前，您需要设置OKR的执行时间段及周期的实施流程。

## 周期类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| **月度** | 每月一个周期 | 快速迭代的业务 |
| **季度** | 每季度一个周期 | 最常见，适合大多数业务 |
| **半年度** | 每半年一个周期 | 长期项目 |
| **年度** | 每年一个周期 | 公司级战略目标 |
| **自定义** | 根据需求自定义时长 | 特殊项目 |

## 如何新建周期？

### 操作步骤

1. **点击右上角的「管理」**
2. **点击「管理周期」**
3. **选择新建周期或对已有的周期进行编辑**

### 关键设置

#### 1. 层级选择

OKR按照层级分为：
- **公司级OKR**
- **部门级OKR**
- **个人级OKR**

新建周期时需选择哪些人员需要制定对应层级的OKR。

#### 2. 执行设置

**自动提醒更新进度**:
- 可开启自动提醒更新进度
- 通知员工及时地更新自己的OKR进度

#### 3. 评分&等级评定

**作用**: OKR周期结束后的回顾复盘评价的一种手段。

**重要提醒**: **回顾比评价更重要！**

---

## 如何设置周期的实施流程？

### 操作入口

在【OKRs】页面，点击页面中心【添加目标】按钮或左上角【+】号，即可开始制定OKR。

### OKR组成

OKR包括了O(目标)和KR(关键结果)两个部分：

| 组成 | 说明 |
|------|------|
| **O(目标)** | 通常是组织希望在近期实现的、有激励性的目标 |
| **KR(关键结果)** | 是实现目标的关键路径，通过量化的KR，我们可以判断目标是否达成 |

### 辅助工具

**填写指引**:
- 点击「灯泡」按钮查看【填写指引】

**模板示例**:
- 直接引用【模板示例】中的OKR进行修改
