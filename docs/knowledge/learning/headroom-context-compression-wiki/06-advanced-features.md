## 七、跨Agent记忆与自动学习

除了核心的上下文压缩能力，Headroom还提供了两个非常有前瞻性的进阶功能：跨Agent共享记忆，以及headroom learn自动学习教训。这两个功能让Headroom从一个"压缩工具"升级为"Agent效率基础设施"。

### 7.1 跨Agent共享记忆

#### 痛点：每个Agent各自为战

很多开发者手上不止用一个Agent：
- 写业务代码用Claude Code
- 快速原型用Codex
- 小修改用Cursor
- 做重构用Aider

问题来了：每个Agent都要自己重新学一遍项目背景。Claude刚读完你的项目结构、搞懂了代码规范，你换Codex做另一个任务，Codex又要从头读一遍——同样的文件、同样的背景、同样的上下文，Token重复消耗了好几遍。

更麻烦的是，Agent之间还没法"交流"：Claude踩过的坑、总结出来的经验，Codex不知道；Codex发现的项目约定，Cursor也得重新踩一遍。

#### 技术实现：本地SQLite + 向量库

Headroom用一个本地SQLite数据库加上向量库，构建了一个跨Agent的共享记忆层：

- **SQLite**：存储结构化的记忆元数据（项目结构、代码规范、踩过的坑等）
- **向量库**：存储语义化的记忆向量，支持语义检索
- **完全本地**：所有记忆数据都存在你的本地机器上，不上传云端

#### 核心价值

1. **多Agent共享同一份记忆**：Claude学过的项目结构，Codex直接拿来用，不用再读一遍
2. **自动去重**：相同的信息不会重复存储，节省空间和检索时间
3. **跨会话持久化**：今天结束的会话，明天打开Agent记忆还在
4. **增量学习**：每次会话积累的经验都沉淀下来，越用越懂你的项目

你可以把这理解为给所有Agent装了一个"共享大脑"——你的项目知识只需要学一次，所有Agent都能受益。

### 7.2 headroom learn：自动学习教训

#### 痛点：同样的错误反复犯

人会"吃一堑长一智"，但传统Agent不会。同样的错误，它可能会反复犯：
- 上次改这个配置文件改错位置了，这次还是错
- 上次跑测试发现缺个依赖，这次又踩同样的坑
- 上次因为没看清楚函数签名参数传错了，这次还是传错

每次都要你重新提醒、重新纠正，重复消耗Token和你的耐心。

#### 工作流程：自动复盘 + 规则沉淀

`headroom learn`是一个非常有创意的功能，它的工作流程是：

1. **扫描失败会话**：Headroom自动扫描你跑失败、出错、需要反复纠正的会话记录
2. **分析翻车原因**：分析哪里出错了、为什么出错、是哪里理解错了
3. **自动调整规则**：把教训提炼成明确的规则
4. **写入规则文件**：把规则自动写进`CLAUDE.md`或者`AGENTS.md`

对，你没看错——它直接修改你的项目规则文件！这样下次Agent启动时，读到这些规则就知道"哦，这个坑我之前踩过，不能这么干"。

#### 价值：Agent自我进化

这个功能的意义在于：**Agent不再是每次都从零开始，它能从过去的失败中学习，越用越聪明，越用越贴合你的项目习惯。**

这跟人类团队的知识沉淀很像：新人进来容易踩坑，踩过之后写进团队Wiki和规范，后面的人就不用再踩了。Headroom把这个过程自动化了。

### 7.3 与SpecWeave项目的关联洞察

值得注意的是，`headroom learn`自动写入`AGENTS.md`的机制，与我们SpecWeave项目的理念高度契合。

在我们的规范体系中：
- `AGENTS.md`是智能体的全局契约入口
- `.agents/rules/`下是各种行为规则和阶段守卫
- `.agents/modules/`是自我演进模块
- 整个体系的核心思想就是：**规则文件是Agent的"记忆"和"行为准则"**

Headroom的`headroom learn`功能，本质上就是在自动做"规则演进"——从失败案例中提取模式，更新规则文件，让未来的执行更顺畅。这与我们自我演进模块的设计思想不谋而合：

1. **感知层**：感知失败会话和异常情况
2. **认知层**：分析原因，提炼教训
3. **执行层**：更新规则文件，落地改进
4. **治理层**：确保规则演进符合规范

这印证了一个方向：**未来的Agent系统一定会有自我演进、规则沉淀的能力，而`AGENTS.md`这类规则文件就是Agent记忆和行为准则的最佳载体。**

### 7.4 应用场景举例

这两个进阶功能在以下场景中价值特别大：

| 场景 | 价值 |
|------|------|
| **大型项目长期开发** | 项目结构、代码规范、踩坑记录自动沉淀，新人（新Agent）上手快 |
| **多Agent切换使用** | 不用每个Agent都重新读一遍项目，Token大幅节省 |
| **团队协作场景** | 一个人踩过的坑，团队所有Agent都能避免 |
| **复杂调试排错** | 调试过程中积累的经验自动记录，下次遇到类似问题直接参考 |

如果说压缩功能是"省当下的Token"，那共享记忆和自动学习就是"省未来的Token"——而且越用越省，越用越顺。
