# 第8章：扩展组件 - 提示框（Admonitions）

提示框（Admonitions）是技术文档中最常用的 UI 组件之一，用于以醒目的视觉样式突出显示重要信息、警告、技巧等内容。

:::{note}
本章配套示例文件：[examples/admonitions-demo.md](examples/admonitions-demo.md)。完整官方示例参见 [../executablebooks-myst-guide/examples/admonitions.md](../executablebooks-myst-guide/examples/admonitions.md)。
:::

## 8.1 概述与基本语法

提示框是带有特殊颜色、图标和边框的内容块，用于在视觉上区分不同语义级别的信息。恰当使用可以引导读者注意力、建立信息层级、提升阅读体验。

**基本语法**：使用三个冒号 `:::` 作为围栏，开头通过 `{type}` 指定类型：

````markdown
:::{note}
这是 note 类型的提示框，用于一般性的备注说明。
:::
````

渲染效果：

:::{note}
这是 note 类型的提示框，用于一般性的备注说明。
:::

:::{tip}
提示框内部支持完整的 Markdown 语法：**粗体**、*斜体*、`代码`、[链接](https://mystmd.org)、列表等。
:::

## 8.2 标准提示框样式

MyST 内置 9 种标准提示框样式，每种有预设的颜色、图标和默认标题。

### 8.2.1 note（备注）

用于一般性补充说明、背景信息、额外注释。

:::{note}
MyST 是 CommonMark 的超集，所有标准 Markdown 语法都可以在 MyST 文档中正常使用。
:::

### 8.2.2 tip（小技巧/最佳实践）

用于实用技巧、快捷键、最佳实践建议、提升效率的方法。

:::{tip}
写作时先列出大纲再填充内容，可以让文档结构更清晰，减少后期重构工作量。
:::

### 8.2.3 hint（提示/暗示）

用于操作指引、小窍门、隐晦问题的提示、调试线索。

:::{hint}
如果命令执行失败，先尝试加上 `--verbose` 参数查看详细日志，多数情况下能快速定位问题。
:::

### 8.2.4 important（重要信息）

用于必须阅读的关键信息、前置条件、核心规则、不可忽略的要求。

:::{important}
升级前务必备份数据库！该版本包含不兼容的数据库 schema 变更，升级后无法回滚到 v1.x。
:::

### 8.2.5 warning（警告/注意事项）

用于可能产生问题的操作、废弃功能警告、常见陷阱、容易出错的地方。

:::{warning}
`delete()` 方法会直接从数据库中永久删除记录，且不会触发确认对话框，请谨慎使用。
:::

### 8.2.6 caution（谨慎操作）

用于需要三思而后行的操作、可能产生副作用的操作、影响范围较大的配置变更。

:::{caution}
修改此配置项将影响集群中所有节点的行为，建议先在测试环境验证后再应用到生产环境。
:::

### 8.2.7 attention（强烈注意）

用于需要用户特别留意的关键事项、截止日期提醒、紧急通知。

:::{attention}
安全补丁将于 7 天后停止支持，请在截止日期前完成升级，否则系统将面临已知安全漏洞风险。
:::

### 8.2.8 danger / error（危险/错误）

用于不可逆的危险操作、数据丢失风险、已发生的错误、破坏性操作。

:::{danger}
危险区域！执行 `rm -rf /` 将删除系统中所有文件，此操作完全不可逆，将导致系统彻底损坏。
:::

error 样式用于展示已发生的错误状态：

:::{error}
连接失败：无法连接到数据库服务器（错误代码：ECONNREFUSED 127.0.0.1:5432）。
:::

### 8.2.9 seealso（另见参考）

用于相关资料链接、延伸阅读、其他章节引用、外部文档参考。

:::{seealso}
- [第7章：注释、脚注与参考文献](07-advanced-notes-citations.md)
- [MyST 官方 Admonitions 文档](https://mystmd.org/guide/admonitions)
:::

## 8.3 自定义标题

默认标题（如"Note"、"Warning"）可以被覆盖，只需在类型后添加自定义标题文本：

````markdown
:::{note} 📝 关于本教程
这是一份面向中文读者的 MyST Markdown 系统教程。
:::
````

渲染效果：

:::{note} 📝 关于本教程
这是一份面向中文读者的 MyST Markdown 系统教程。
:::

自定义标题中可以加入 emoji 增强视觉效果：

:::{tip} 💡 快捷键提示
保存：`Ctrl+S` | 撤销：`Ctrl+Z` | 查找：`Ctrl+F` | 命令面板：`Ctrl+Shift+P`
:::

## 8.4 嵌套提示框

提示框支持嵌套使用，通过增加冒号数量区分层级（外层 4 个冒号，内层 3 个冒号）：

`````markdown
::::{important} 升级操作指南
升级前请仔细阅读：

:::{warning} 数据库备份
执行升级脚本前必须先完成全量备份，并验证备份文件可正常恢复。
:::

:::{tip} 最佳升级时间
建议在业务低峰期（如凌晨 2:00-5:00）执行升级操作。
:::

升级完成后请运行健康检查脚本确认所有服务正常。
::::
`````

渲染效果：

::::{important} 升级操作指南
升级前请仔细阅读：

:::{warning} 数据库备份
执行升级脚本前必须先完成全量备份，并验证备份文件可正常恢复。
:::

:::{tip} 最佳升级时间
建议在业务低峰期（如凌晨 2:00-5:00）执行升级操作。
:::

升级完成后请运行健康检查脚本确认所有服务正常。
::::

:::{caution}
提示框嵌套不宜过深，建议最多嵌套 2 层，过深会影响可读性。
:::

## 8.5 样式自定义

可以通过内联选项为提示框添加自定义 CSS 类或 ID：

**添加 CSS 类**：
```markdown
:::{note}
:class: custom-note
这是带有自定义 CSS 类的提示框。
:::
```

**添加 ID**（用于锚点链接）：
```markdown
:::{important}
:#id: installation-warning
安装前请确保系统满足最低要求...
:::
```

:::{note}
具体 CSS 样式定制需要在项目主题配置或自定义 CSS 文件中完成。
:::

## 8.6 使用场景决策指南

| 类型 | 语义强度 | 颜色基调 | 使用时机 | 典型场景 |
|------|---------|---------|---------|---------|
| **note** | ⚪ 低 | 蓝色/灰色 | 中性补充信息，不影响操作 | 背景介绍、版本说明、额外注释 |
| **tip** | 🟢 积极 | 绿色 | 提供有益的建议或技巧 | 最佳实践、快捷键、效率提升方法 |
| **hint** | 🟢 温和 | 浅绿色 | 隐晦问题的线索或指引 | 调试提示、操作暗示、小窍门 |
| **seealso** | 🔵 参考 | 浅蓝 | 提供相关参考资料 | 延伸阅读、链接到其他章节 |
| **important** | 🟡 高优先级 | 橙色 | 必须知晓的关键信息 | 前置条件、核心规则、不可忽略的要求 |
| **warning** | 🟠 潜在风险 | 黄色/橙色 | 可能出错的地方 | 废弃警告、常见陷阱、易犯错误 |
| **caution** | 🟠 较高风险 | 深橙色 | 需要谨慎决策的操作 | 影响范围大的变更、有副作用的操作 |
| **attention** | 🔴 高优先级警告 | 红色 | 需要立即注意的事项 | 截止日期、安全通知、紧急提醒 |
| **danger/error** | 🔴 最高风险 | 深红色 | 破坏性操作或已发生错误 | 不可逆操作、数据丢失风险、错误展示 |

**选择原则**：
1. **从低到高**：优先用语义级别较低的类型，确需强调时才用高级别
2. **保持一致**：同一类信息始终使用同一种类型
3. **不要滥用**：每页建议不超过 3-5 个，提示框太多等于没有提示框
4. **标题明确**：使用自定义标题让读者一眼明白核心内容

## 8.7 小结

- **基本语法**：`:::{type} 内容 :::`，三个冒号围栏
- **9 种标准样式**：note、tip、hint、important、warning、caution、attention、danger/error、seealso
- **自定义标题**：在 `{type}` 后添加自定义文本覆盖默认标题
- **嵌套**：外层用 4 个冒号，内层用 3 个冒号，不超过 2 层
- **样式自定义**：通过 `:class:` 和 `:#id:` 添加 CSS 类和 ID
- **选择原则**：语义强度从低到高，保持一致，不要滥用
