## 十、常见问题解答

### Q1：无网远控真的不需要被控端有网络吗？

**A**：是的，这是向日葵无网远控硬件的核心特性。"无网远控"指的是**被控端主机不需要联网**——向日葵远控硬件（Q1/Q2Pro/Q5Pro/控控2）自身通过网线/WiFi/4G/5G联网，然后通过HDMI采集视频、通过USB仿真键鼠来控制被控主机。被控主机完全不需要插网线、连WiFi，甚至可以没有网卡。

但需要注意：
- **网络型产品（Q1/Q2Pro/Q5Pro/控控2）**：设备自身需要联网，主控端（你的手机/电脑）也需要联网
- **Q0.5**：完全不需要任何网络，纯线缆直连近场控制，设备和被控端都不联网

### Q2：Q0.5完全不用联网怎么控制？

**A**：Q0.5采用点对点线缆直连方案，不经过任何网络：
1. Q0.5的HDMI-IN连接被控主机的HDMI输出（采集画面）
2. Q0.5的Type-C（被控端）连接被控主机USB（取电+仿真键鼠）
3. Q0.5的Type-C（主控端）通过线缆连接你的笔记本电脑/手机
4. 在你的笔记本/手机上运行向日葵主控端，就可以看到被控主机画面并操作

整个过程完全不涉及WiFi、网线、互联网，数据通过线缆直接传输，相当于一个"带USB透传的HDMI采集卡+KVM"。

### Q3：蓝牙在远控中起什么作用？

**A**：蓝牙5.0在向日葵远控硬件中**不用于传输远控画面和控制信号**，它的作用仅限于：
1. **初始设备绑定**：手机通过蓝牙搜索并绑定设备到你的向日葵账号
2. **WiFi配网**：通过蓝牙向设备下发WiFi的SSID和密码，让设备连接WiFi

绑定配网完成后，蓝牙连接会断开，远控数据通过网线/WiFi/4G/5G传输。蓝牙的优势是近场通信（约10米）、无需同一局域网、配置简便。

支持蓝牙的产品：Q1、Q2Pro、Q5Pro。控控2和Q0.5不支持蓝牙。

### Q4：这些设备支持BIOS级控制吗？系统崩溃了能远控吗？

**A**：是的，五款产品**全部支持BIOS级远程控制**。这是IPKVM硬件相对于软件远控的核心优势。

为什么能控制BIOS？
- 向日葵设备通过USB连接被控主机时，仿真的是标准USB HID键盘鼠标
- 这类标准USB设备不仅操作系统能识别，主板BIOS/UEFI在POST自检阶段就能识别
- HDMI采集是纯硬件层面的信号截取，与操作系统无关

可以实现的操作：
- 开机按Del/F2进入BIOS/UEFI设置
- 修改启动顺序、硬件参数
- 系统崩溃/蓝屏时查看错误画面
- RAID卡配置
- PXE网络引导
- 远程重装系统
- 输入全盘加密密码

软件远控（如TeamViewer、向日葵软件版）在系统崩溃、蓝屏、重启过程中是完全失效的，只有硬件IPKVM才能实现真正的全流程控制。

### Q5：支持哪些操作系统？Mac和Linux能用吗？

**A**：向日葵无网远控硬件对被控端操作系统有广泛的兼容性：

| 操作系统 | 支持情况 |
|---|---|
| Windows | 全面支持（XP/7/8/10/11及服务器版） |
| macOS | 支持 |
| Linux | 支持（主流发行版） |
| Android | 控控2/Q2Pro/Q5Pro支持 |

兼容性好的原因：这些设备不依赖操作系统安装任何驱动或软件，而是通过标准USB HID设备类和标准HDMI接口与被控主机通信——任何支持标准USB键盘鼠标和HDMI输出的设备都能被控制。

Q0.5官方明确被控主机支持：Linux、Windows、macOS。

### Q6：需要安装客户端软件吗？被控端需要装驱动吗？

**A**：**被控端完全不需要安装任何软件或驱动**。这是向日葵无网远控硬件的核心优势：
- 被控端：零安装，即插即用，通过标准USB HID和HDMI工作
- 主控端：需要在你的手机/电脑/平板上安装向日葵远程控制软件（主控端App/客户端），用于查看画面和发送操作指令

为什么被控端不用装驱动？
因为向日葵设备仿真的是操作系统内置驱动支持的标准USB设备类（USB键盘、USB鼠标），操作系统会自动加载标准驱动，就像你插入一个真实的键盘鼠标一样。

### Q7：远控数据安全吗？会被窃听吗？

**A**：向日葵远控硬件采用多层安全机制：

**网络型产品（Q1/Q2Pro/Q5Pro/控控2）**：
- 国际2048位RSA非对称密钥交换
- 基于AES加密机制的自主P2P数据传输协议
- 多因子验证（Q2Pro/Q5Pro明确支持）
- Q5Pro支持葵码登录，无需透露账号密码
- 优先P2P直连传输，数据不经过服务器中转

**Q0.5（物理近场型）**：
- 最彻底的安全方案——物理隔离
- 完全无网络（无网口/WiFi/蓝牙/蜂窝）
- 数据通过线缆点对点传输，不经过任何网络
- 零网络攻击面
- 硬件级防止第三方跳板远控和截屏

安全等级对比：Q0.5（物理隔离）> Q2Pro/Q5Pro（金融级加密）> Q1/控控2（标准加密）。涉密场景请选用Q0.5。

### Q8：可以同时控制多台电脑吗？

**A**：单台向日葵硬件同一时间只能控制一台电脑，但可以通过搭配**KVM切换器**实现多台主机控制：
- 支持KVM切换器的产品：控控2、Q2Pro、Q5Pro（官方明确支持）
- Q1官方未明确提及KVM切换器支持
- Q0.5为一对一近场直连，不支持KVM切换器

使用方式：将KVM切换器的控制台端口连接到向日葵设备的HDMI-IN和USB口，KVM切换器的输入端口连接多台被控主机，通过切换KVM端口即可控制不同主机，将传统KVM升级为IPKVM。

### Q9：Q2Pro和Q2Pro-BLE是什么关系？

**A**：它们本质上是同一款产品。Q2Pro的官方产品页面URL路径是`q2pro-ble`，其中：
- "Q2Pro"是产品型号名称
- "BLE"是Bluetooth Low Energy（蓝牙低功耗）的缩写，指代Q2Pro具备蓝牙5.0绑定功能

本教程和官方页面提到的Q2Pro，就是指这款工业级4G智能远控设备，不存在两个不同的"Q2Pro"和"Q2Pro-BLE"型号。

### Q10：4G/5G版本需要自己办流量卡吗？

**A**：
- **Q2Pro**：需要自行办理标准Mini SIM卡（4G全网通），设备本身不赠送流量
- **Q5Pro**：
  - 内置一张联通贴片卡（可直接使用）
  - 另有一个NANO SIM卡槽可插入自己的流量卡（双卡备份）
  - 贴片卡的流量资费请咨询向日葵官方

建议办理物联网卡（M2M卡），相比普通手机卡更适合设备长期使用，资费更便宜、稳定性更好。普通手机SIM卡也可以使用，但注意PIN码锁定等问题。

> 注意：Q5Pro的TF卡功能需定制，默认TF卡槽不可用于普通存储扩展。

### Q11：支持手机/平板控制吗？

**A**：是的。主控端支持多种设备：
- **电脑**：Windows/macOS/Linux向日葵客户端
- **手机**：iOS/Android向日葵远程控制App
- **平板**：iPad/Android平板向日葵App

绑定设备后，可以在任何安装了向日葵主控端App的设备上发起远控。对于Q0.5，需要用线缆连接主控端（支持手机/电脑直连）。

但需要注意：部分手机因硬件限制或扩展坞兼容性问题，作为被控端时可能无法正常使用（Q1页面特别提示）。

### Q12：分辨率和帧率最多支持多少？

**A**：各产品最高分辨率和帧率如下：

| 产品 | 最高输入分辨率 | 最高输出/远控分辨率 | 最高帧率 |
|---|---|---|---|
| Q1 | 1920×1080 | 1920×1080 | 15fps |
| Q2Pro（V3版） | 3840×2160（4K） | 3840×2160（4K） | 60fps |
| Q2Pro（旧版） | 2560×1600 | 2560×1600 | 60fps |
| Q0.5 | 3840×2160@30Hz | 1920×1080 | 60fps |
| Q5Pro | 3840×2160（4K） | 2560×1600@30fps / 3840×2160@15fps | 15-30fps |
| 控控2 | 官方未详细公开 | 官方未详细公开（推测1080P） | 官方未详细公开 |

注意：
- Q2Pro/Q5Pro的4K分辨率可能需要相关服务支持
- 实际帧率会根据网络带宽动态调整
- Q0.5输入支持4K但输出降为1080P，保证近场操作流畅性
- 医疗设备特殊分辨率（如1280×1024、1920×1200等）都能支持

### Q13：Q0.5为什么不需要外接电源？

**A**：Q0.5通过连接被控主机的Type-C接口从被控主机取电，功耗极低，不需要独立电源适配器。这也是Q0.5能做到极致便携（口袋大小、无需带电源）的关键设计。

而Q1/Q2Pro/Q5Pro/控控2等网络型设备由于需要运行网络模块（WiFi/4G/5G）、视频编码处理器等，功耗较高，必须使用独立电源适配器供电。Q2Pro和Q5Pro还支持9-36V宽电压端子供电，适应工业场景。

### Q14：可以远程传输文件吗？

**A**：
- **Q2Pro**：支持远程文件传输，但需要增购企业服务配件并完成企业实名认证。接入U盘或TF卡后可传输运维日志、安装包等文件。赠送一年远程SSH和远程摄像头功能。
- **Q5Pro**：支持USB远程映射，可将U盘、智能卡、加密狗等USB设备映射到被控主机，实现文件传输等功能。
- **Q1/控控2/Q0.5**：Q0.5的USB-A接口支持本地USB设备映射（近场）；Q1和控控2的远程文件传输能力官方未详细公开。

### Q15：工业级产品（Q2Pro/Q5Pro）的"工业级"体现在哪里？

**A**：与消费级产品（Q1）相比，Q2Pro和Q5Pro的工业级特性主要体现在：

| 特性 | Q1（消费级） | Q2Pro/Q5Pro（工业级） |
|---|---|---|
| 工作温度 | 0-40℃（室内办公环境） | -20~70℃（严苛户外/工业环境） |
| 外壳材质 | 金属铝+工程塑料 | SECC镀锌钢板（抗干扰、防腐蚀） |
| 电源输入 | 单12V DC | 双电源冗余（12V DC + 9-36V端子） |
| 防护设计 | 标准 | 防浪涌（抗雷击）、看门狗 |
| 安装方式 | 桌面放置 | DIN35导轨安装+壁挂蝴蝶扣+桌面 |
| 网络冗余 | 单网口（百兆） | 千兆/2.5G网口+WiFi+4G/5G多模冗余 |
| 以太网 | 100M百兆 | 1000M千兆（Q2Pro）/2500M 2.5G（Q5Pro） |

这些设计使得Q2Pro/Q5Pro可以部署在工厂车间、户外基站、矿山油田等消费级产品无法适应的恶劣环境中。

### Q16：为什么Q5Pro帧率比Q2Pro低（30fps vs 60fps）？

**A**：这是因为Q5Pro定位更高的分辨率和更复杂的功能：
- Q2Pro在2560×1600（旧版）或4K@60fps（V3版）下，专注于视频采集+远控
- Q5Pro在相同分辨率下需要同时处理：4K视频编码+5G网络传输+双向语音+协同远控+USB映射，处理器负载更高，因此帧率策略更保守（2.5K@30fps / 4K@15fps）
- 5G网络虽然带宽大，但实际可用带宽受运营商网络覆盖影响，Q5Pro的帧率设计更侧重稳定性而非极端帧率
- 对于远程运维、医疗示教等场景，15-30fps完全够用，60fps主要针对需要高速动态画面的场景（如游戏、视频播放），这并非IPKVM的典型需求
