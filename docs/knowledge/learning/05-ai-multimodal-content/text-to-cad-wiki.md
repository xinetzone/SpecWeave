## 一、项目概述：解决的痛点与核心价值

> "机械设计又又又卡住了。用AI生成了CAD模型，拿到手里却是STL网格，改不了孔的位置。想要将数据导出为STEP格式给工厂使用，还需要重新构建几何体。机器人描述文件需要手动添加大量的XML标签，很容易出现错误。"

### 1.1 传统AI CAD设计的三大痛点

在机械设计和机器人开发领域，AI辅助设计虽然带来了效率提升，但实际使用中却存在三个难以回避的痛点：

**痛点1：AI生成STL网格模型，无法直接编辑参数**

当前大多数AI CAD工具生成的是STL格式的网格模型——这种模型本质上是由大量三角形面片拼接而成的"死网格"，就像3D扫描出来的模型一样。你无法直接修改孔的位置、调整圆角半径、改变板件厚度，任何参数化修改都需要从头重新生成。

**痛点2：STEP格式导出需要重新构建几何体**

工厂生产、CNC加工、模具制造都需要STEP格式——这是工业界标准的参数化CAD格式。但STL网格无法直接转换为STEP，必须由工程师手动在SolidWorks、Fusion 360等软件中重新构建几何体，这个过程往往需要数小时甚至数天，完全抵消了AI生成带来的效率优势。

**痛点3：机器人URDF文件手动编写XML易出错**

机器人开发需要URDF（统一机器人描述格式）文件，这是一种XML格式，需要定义连杆、关节、碰撞模型、视觉模型等大量标签。手动编写不仅繁琐枯燥，而且一个标签闭合错误、一个坐标数值偏差，都可能导致仿真失败或机器人运动异常。

### 1.2 text-to-cad 项目介绍

正是为了解决这些痛点，**text-to-cad** 项目应运而生。

### 1.3 核心价值

**用AI来写可编辑的CAD源代码，并把它们导出为STEP、URDF等工程文件。**

text-to-cad 不生成"死"的STL网格，而是让AI直接生成**可编辑的CAD源代码**（基于build123d参数化建模库）。这些代码就像SolidWorks的特征树一样，你可以随时修改任何参数，然后一键导出为STEP、STL、URDF等工程格式。

## 二、核心功能详解（5大功能）

text-to-cad 提供了5大核心功能，覆盖从AI建模到制造输出的完整工作流。

### 2.1 参数化CAD源码生成

**技术原理**：与传统AI CAD工具输出神秘的二进制模型或STL网格不同，text-to-cad 让AI直接编写 **Build123d Python源码**。每一个几何特征——无论是底座、孔位、支架还是圆角——都用Python代码明确定义，每个特征还会用 `@cad[feature_name]` 引用标记进行标注，方便后续精准定位和修改。

**实际应用价值**：
- **参数化修改极其方便**：想要调整孔的位置？只需要修改源代码中对应的坐标参数，一键重新运行即可，不需要重绘整个零件
- **完全可审计**：尺寸、公差、装配关系全部写在代码里，一目了然，没有黑盒
- **工业级兼容性**：生成的STEP文件采用标准BREP格式，**可以直接导入SolidWorks、Fusion 360、FreeCAD等专业CAD软件进行二次编辑**，工厂和供应商可以直接使用

### 2.2 机器人描述文件自动生成

**技术原理**：URDF（统一机器人描述格式）是机器人开发的标准格式，但手动编写XML极其繁琐——links、joints、limits、坐标系、惯性参数等每一项都容易出错。text-to-cad 能够根据CAD模型自动生成完整的URDF XML文件，并对关节限位、坐标系变换、惯性参数等进行**合法性校验**，从源头避免数值错误和标签闭合问题。

**实际应用价值**：
- 配合内置的CAD Viewer，可以在本地浏览器中直接查看机器人结构和关节运动
- 生成的URDF文件 **ROS MoveIt2可以直接加载使用**，无需手动调整
- 大幅减少机器人仿真和部署前的调试时间，避免因为一个坐标写错导致的仿真崩溃

### 2.3 标准件库内置

**技术原理**：一般AI生成CAD模型时经常"凭空创造"螺钉尺寸——比如M3螺丝画成3.2mm孔径、螺距不对，或者出现现实中根本不存在的规格。text-to-cad 内置了 **step.parts catalog** 标准件库，包含螺丝、轴承、电机、铝型材等常见机械标准件的精确三维模型，所有尺寸参数都严格遵循工业标准。

**实际应用价值**：
- 装配体中使用的每一个标准件都是**现实中可以实际购买到的型号**
- 从根本上避免了"设计好了但是买不到螺丝"的尴尬问题
- 标准件直接调用，不需要AI重新建模，既准确又高效

### 2.4 本地浏览器预览器

**技术原理**：text-to-cad 内置了基于 WebGL 的本地预览器，直接在浏览器中运行，不需要安装任何专业CAD软件。它支持STEP、STL、URDF等多种格式的加载和渲染，提供轨道旋转、缩放、平移等标准3D交互操作。

**实际应用价值**：
- **零软件依赖**：不需要安装SolidWorks、Fusion 360等大型软件，打开浏览器就能看模型
- **跨平台支持**：甚至**可以在手机上查看生成的零件**，方便和团队、工厂沟通
- **组件树浏览**：可以像CAD软件一样查看装配体的组件层级，单独显示/隐藏某个零件
- **精准引用编辑**：可以直接在预览器中复制 `@cad[...]` 引用标记，基于具体几何位置做后续编辑
- **多种视图模式**：支持正交投影、透视投影两种查看方式，GLSL光线步进渲染模式目前还处于试验阶段

### 2.5 DXF工程图和G-code切片

**技术原理**：text-to-cad 不仅仅停留在3D建模阶段，它还打通了从3D模型到2D制造文件的最后一公里。通过几何投影自动生成DXF格式的2D工程图，还可以集成真实切片器CLI生成3D打印机可用的G-code。

**实际应用价值**：
- **激光切割/钣金加工**：用3D几何投影产生2D DXF文件，激光切割服务商可以直接接收使用
- **自动展开计算**：自动生成钣金展开图、垫片轮廓、切割布局，不需要手动在CAD中展开
- **3D打印直接输出**：支持通过真实切片器CLI生成匹配打印机配置的G-code，**Bambu Labs打印机可以直接对接发起打印**
- **制造前校验**：可以对接SendCutSend等钣金加工服务，在下单前自动进行文件检查
- **全流程自动化**：整个制造流程——从自然语言描述需求→CAD建模→导出工程图→生成打印/切割代码——**都可以自动化完成**

## 三、安装配置指南

text-to-cad 的安装分为4个步骤，覆盖CLI工具、AI插件、Python环境和前端预览器。

### 3.1 Skills CLI 安装

对于支持Skills机制的AI编程工具，使用npx一键安装：

```bash
npx skills install earthtojake/text-to-cad
```

安装完成后，你的AI助手就具备了text-to-cad的CAD生成能力。

### 3.2 Claude Code 插件安装

如果你使用Claude Code作为AI编程环境，可以通过插件市场安装：

```bash
claude plugin marketplace add earthtojake/text-to-cad
claude plugin install cad@text-to-cad
```

安装完成后重启Claude Code即可使用。

### 3.3 Python CAD 环境配置

text-to-cad 依赖Python 3.11和Build123d库，需要先创建虚拟环境并安装依赖：

**Linux/macOS用户：**
```bash
python3.11 -m venv .venv
./.venv/bin/pip install -r requirements-cad.txt
```

**Windows用户：**
```bash
python3.11 -m venv .venv
.\.venv\Scripts\pip install -r requirements-cad.txt
```

激活虚拟环境后，Python CAD运行环境就准备就绪了。

### 3.4 Viewer 前端启动

本地预览器是一个Node.js项目，需要先安装依赖再启动开发服务器：

```bash
npm --prefix viewer install
npm --prefix viewer run dev
```

启动成功后，打开浏览器访问 **http://localhost:4178** 即可查看生成的CAD模型，支持轨道旋转、组件树浏览、@cad引用复制等功能。

## 四、完整使用流程演示

让我们以**设计一个机器人底盘**为例，展示text-to-cad的完整工作流。你会发现，整个过程简单到只需要一句话。

### 4.1 第一步：自然语言描述需求

打开Claude Code（或你安装了text-to-cad插件的AI编程工具），你只需要用自然语言说出你的需求：

> "生成一个矩形底座，有四个安装孔和两个电机支架"

不需要画草图，不需要查参数手册，不需要打开CAD软件——就这么一句话。

### 4.2 第二步：AI自动编写Build123d源码

收到你的需求后，AI立即开始工作：它会自动编写Build123d Python源码。在代码中，每一个几何特征都被精确定义：
- 底座的长度、宽度、厚度
- 四个安装孔的直径、位置坐标、沉头深度
- 两个电机支架的高度、安装孔位、加强筋尺寸

所有尺寸参数都明明白白写在代码里，你随时可以修改。

### 4.3 第三步：自动导出STEP文件

代码编写完成后，text-to-cad会自动运行Python脚本，导出标准STEP格式文件。这个文件**可以直接导入SolidWorks、Fusion 360等专业CAD软件中进行编辑**，工程师可以在此基础上继续细化设计，工厂也可以直接用于加工。

### 4.4 第四步：生成URDF机器人描述

如果你在做机器人开发，text-to-cad还会接着生成URDF机器人描述文件。它会自动把links（连杆）、joints（关节）、坐标系、碰撞模型等全部写入XML中，自动进行关节限位和参数校验。生成的URDF文件 **ROS MoveIt2可以直接加载使用**，不需要任何手动调整。

---

**强调一下整个流程的革命性**：从你说出需求，到CAD建模完成，到STEP文件导出，再到URDF机器人描述生成，**后面所有的工作都是AI自动完成的**。你不需要懂Build123d编程，不需要懂URDF格式规范，不需要操作CAD软件——只需要一句话，就能拿到可以直接用于工程和制造的输出文件。

## 五、局限性与使用边界

作为一个快速发展的开源项目，text-to-cad目前还存在一些局限性，使用前需要了解清楚：

1. **Implicit CAD试验性**：Implicit CAD（隐式建模）功能目前还是试验性质，功能并不完善，不建议在重要项目中使用
2. **Node环境门槛**：Viewer前端预览器需要Node.js环境，对于只熟悉Python的机械工程师来说有一定的环境配置门槛
3. **Git LFS资产**：Git仓库中的benchmark测试资产使用Git LFS存储，默认clone不会拉取，需要手动执行 `git lfs pull --include="benchmarks/**"` 才能获取完整测试文件
4. **无中文文档**：目前项目的README和SKILL.md都是英文文档，对中文用户不够友好
5. **复杂装配体验证不足**：目前项目的benchmark测试主要针对单个零件，**对复杂多零件装配体的生成效果还没有进行充分验证**
6. **许可证问题**：项目核心代码是MIT协议，但依赖的OpenCascade几何内核和step.parts标准件库的商业许可证条款需要自行查阅，商业使用前请确认合规

### 适用场景判断

最后帮你判断一下什么时候该用、什么时候不该用：

- ✅ **适合使用**：需要**工程级可编辑模型**，要导出STEP/URDF/DXF/G-code用于实际制造、机器人开发、3D打印、激光切割等场景
- ❌ **不适合使用**：只需要一张漂亮的3D渲染图做PPT展示或概念可视化——这时候用Meshy、Tripo等AI渲染工具更合适

## 六、核心价值总结

text-to-cad之所以在GitHub上获得7400+ stars，是因为它在AI CAD方向上走对了路。它的核心价值可以总结为5点：

1. **思路正确**：让AI直接生成**CAD源代码**，再导出各种工程文件，而不是生成藏在黑盒里的二进制模型或死网格。这个技术路线选择是项目成功的关键。

2. **参数化修改**：真正实现了参数化设计——修改一个孔的位置、调整一块板的厚度，**只需要修改源代码里的数字，不需要重新画整个零件**，这才是工程师真正需要的AI辅助设计。

3. **代码透明**：所有尺寸、基准、装配关系都明明白白写在代码里，没有黑盒，没有隐藏参数，工程师可以审计每一个尺寸，对设计结果完全可控。

4. **全链路打通**：这是项目最大的价值——它真正把"**自然语言描述→CAD源码生成→STEP工程导出→URDF机器人描述→DXF/G-code制造文件**"这条完整链路打通了，从需求到制造一气呵成。

5. **开源开放**：项目基于MIT协议开源，可以自由使用、修改、分发，社区可以共同改进，不用担心厂商锁定。

## 七、常见问题解答（FAQ）

这里整理了一些常见问题，帮助你快速判断text-to-cad是否适合你的需求。

**Q: text-to-cad适合什么场景使用？**
A: 适合机械设计、机器人开发、3D打印、激光切割等**需要工程级CAD文件**的场景。如果你需要的是可编辑、可制造、可导入专业软件的真实CAD模型，而不仅仅是一张好看的效果图，text-to-cad会很有价值。

**Q: 生成的STEP文件真的可以在SolidWorks中编辑吗？**
A: 是的。因为text-to-cad生成的是基于Build123d的参数化CAD源码，导出的STEP是标准的BREP边界表示格式，不是三角网格，**可以直接在SolidWorks、Fusion 360、FreeCAD等主流CAD软件中编辑特征和参数**。

**Q: 必须使用Claude Code吗？支持其他AI编程工具吗？**
A: 项目主要面向Agent设计，官方提供了Claude Code插件和Skills CLI两种安装方式。理论上任何支持Skill/插件机制的AI编程工具都可以适配，未来可能会支持更多平台。

**Q: 标准件库包含哪些类型的零件？**
A: step.parts catalog包含螺丝、螺母、轴承、电机、铝型材等常见机械标准件，所有型号都是**现实中可以实际采购到的标准规格**，不会出现"AI自创螺钉尺寸"的问题。

**Q: 没有Python基础可以使用吗？**
A: **基本使用不需要懂Python**——你只需要用自然语言描述需求，AI会自动生成所有代码，自动导出文件。但如果要深度定制或手动微调生成的模型，掌握Python和Build123d基础知识会很有帮助。

**Q: 生成的模型可以直接用于3D打印吗？**
A: 可以。text-to-cad支持导出STL格式，还可以直接通过切片器CLI生成匹配你打印机配置的G-code，甚至**可以直接对接Bambu Labs打印机发起打印任务**。

**Q: 项目还在积极维护吗？**
A: 是的。项目在GitHub上有7400+ stars，根据项目介绍作者还在持续更新维护，功能迭代速度很快。
