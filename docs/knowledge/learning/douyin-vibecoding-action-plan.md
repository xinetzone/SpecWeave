## 🗓️ 总体时间规划

假设你已有一个或多个用TRAE开发的作品，按以下时间线执行：

```
Day 0（今天）─── 选题确认 + 形式选择 + 内容策划
Day 1 ───────── 素材拍摄/录制
Day 2 ───────── 后期剪辑/图文制作

Day 4-6 ─────── 等待审核 + 数据观察 + （必要时）优化迭代
```

**总耗时估算**：新手4-6小时 / 有剪辑经验者2-3小时（单条内容）

## 🎬 Day 0：选题·形式·策划（约45分钟）

### Step 1：确定参赛作品（10分钟）

**决策标准**（按优先级排序）：
1. ✅ 作品已在TRAE社区提交Demo帖（**没有这步一切白搭**）
2. ✅ 作品可正常运行，能现场演示
3. ✅ 选题最好命中以下**高传播潜力模式**之一：

| 优先级 | 模式 | 判断标准 | 预期量级 |
|---|---|---|---|
| 🥇 S级 | 文化共鸣+穷尽式 | 击中文化母题（汉字/美食/诗词/历史）+穷尽所有可能性 | 100万+ |
| 🥈 A级 | 实用工具+情绪锚点 | 解决真实痛点+有季节性/情绪共鸣（如高考/减肥/旅行） | 5-20万 |
| 🥉 B级 | 视觉奇观+数据可视化 | 海量数据+美学设计+视觉冲击 | 10-50万 |
| B级 | 真人出镜+真实场景 | 接地气+现场演示+解决真问题 | 5-20万 |
| B级 | 游戏化+历史/IP重构 | "如果"假设+经典IP+交互体验 | 5-15万 |
| C级 | 兴趣垂直工具/个人故事/情感工具 | 圈层共鸣/增长故事/治愈系 | 1-10万 |

> 💡 **如果你有多个作品**：选"最酷最好看最有故事"的那个，不是选"技术最强"的那个。

**一句话定位模板**（写下来，后面到处要用）：
```
我做了一个能【核心功能】的【作品类型】，它可以帮你【解决什么问题/带来什么体验】。
```
例："我做了一个能自动整理旅行照片并生成游记的AI工具，帮你告别旅行回来几百张照片懒得整理的烦恼。"

### Step 2：选择呈现形式（5分钟）

按决策树回答：

```
你的作品是什么类型？
├─ 硬件类 → 📹 实拍为主（展示动态工作状态）+ 代码截图辅助
├─ 游戏类 → 🎮 录屏为主（直接录最精彩片段）
└─ 软件类
   ├─ 重交互？（手势/语音/AR/体感/需要真人操作展示）→ 🧑 第一/三人称拍摄
   ├─ 不重交互，会视频剪辑？→ 🖥️ 电脑录屏+旁白+剪辑
   └─ 不擅长视频剪辑？→ 🖼️ 图文形式（推荐新手！门槛最低）
```

> 💡 **新手强烈推荐图文形式**：制作快、不会翻车、信息密度高。等图文熟练了再尝试视频。

## 📹 Day 1：素材拍摄/录制（约30-60分钟）

### 通用标准（任何形式都要遵守）

| 维度 | 标准 | 常见错误 |
|---|---|---|
| **画面稳定** | 三脚架固定，绝对不要手持 | 手持晃动=直接低质 |
| **光线** | 窗边自然光45度侧光最佳，避免逆光/暗光 | 昏暗房间拍=看不清 |
| **背景** | 白墙/干净桌面/书架，不要在床上/杂物堆前 | 脏乱背景=不专业 |
| **声音** | 安静环境，离麦克风近一点 | 杂音/爆音=致命问题 |
| **录屏准备** | 开专注模式、关通知、隐藏敏感信息、清理桌面 | 弹通知/露API Key=事故 |

### 分类型拍摄要点

**📹 硬件类实拍**：
- [ ] 展示产品**动态工作状态**，不要只拍静止摆放
- [ ] 多拍几个角度（正面/侧面/特写/使用中）
- [ ] 展示灯光/运动/交互效果（这些是硬件的核心卖点）
- [ ] 可以穿插屏幕录屏展示软件界面

**🎮 游戏类录屏**：
- [ ] **不要从启动游戏开始录**——直接进入最精彩的画面
- [ ] 录高光时刻（神操作/有趣瞬间/精彩关卡）
- [ ] 剪掉加载/菜单/等待等无聊片段
- [ ] 可以叠加自己的真实反应（惊喜/崩溃/笑）增强感染力

**🖥️ 软件类录屏**：
- [ ] 分辨率1080p以上，帧率30fps+
- [ ] 鼠标不要乱晃，操作要干脆
- [ ] 提前过一遍演示流程，不要在录屏时犹豫
- [ ] 多录2-3遍备用

**🧑 真人出镜**：
- [ ] 三分法构图：人在画面1/3处，产品操作在2/3处
- [ ] 穿着得体整洁自然（不用正装，干净就行）
- [ ] 外接领夹麦收音
- [ ] 操作自然，可以有小失误（更真实）
- [ ] 表情自然有变化（不要全程面无表情念稿）

**🖼️ 图文截图**：
- [ ] 用产品最酷的界面截图
- [ ] 关键处可以加红圈/箭头标注
- [ ] 尺寸统一（3:4竖版最佳）
- [ ] 首图用AI生图或最惊艳的产品截图
- [ ] 代码截图只截关键片段（3-5行），不要全屏代码

## ✂️ Day 2：后期剪辑/制作（新手1-2h/熟练30-60min）

### 视频剪辑流程（用剪映）

1. **粗剪**：导入所有素材，按脚本顺序排列，大刀阔斧剪掉所有"3秒没新信息"的片段
2. **加旁白**：
   - 可以自己录音（安静环境，慢速清晰）
   - 也可以用剪映AI配音（选一个自然的声音，不要用机械感太重的）
   - **关键**：讲价值（"这个功能可以帮你自动整理..."），不讲操作（"我现在点击这个按钮..."）
3. **加字幕**：自动识别字幕，检查错别字，字幕放屏幕下方不挡内容
4. **局部放大**：展示关键操作/亮点时，放大对应区域（1.5-2倍，停留2-3秒）
5. **BGM**：抖音音乐库选轻节奏纯音乐，音量调到-20db左右（不盖人声）
6. **转场**：用简单淡入淡出或硬切，**不要加花哨转场**
7. **前3秒检查**：第一帧是不是最惊艳的画面？第一句话是不是说清了产品是什么？

### 图文制作流程

1. 图片按顺序导入
2. 裁剪统一尺寸（3:4竖版）
3. 关键处加标注（箭头/红圈/简短文字）
4. 按模板写好简介文案
5. 准备好标题和封面

### ❌ 剪辑避坑清单

- [ ] ❌ 没有满屏文字/代码（密密麻麻=观众划走）
- [ ] ❌ BGM没有盖过人声
- [ ] ❌ 没有花哨片尾（"关注我看更多"之类的不要加）
- [ ] ❌ 没有从"大家好我是XXX"开头（浪费前3秒）
- [ ] ❌ 每个镜头不超过5秒（除非是在展示连续操作）

## 📊 Day 4-6：等待·观察·迭代

### 正常节奏
- 每天看1-2次抖音站内信（消息→通知）
- 审核周期：**1-3个工作日**
- 审核通过：流量自动到账，站内信通知
- 审核未通过：看原因，🟡类问题修改后重发

### 数据不好怎么办？

**发布24小时后播放量<500？按顺序自查**：

```
第一步：投稿规范都做对了吗？
  → 漏了话题/@/问卷？补上，必要时修改重发

第二步：有没有收到违规通知？
  → 收到了→看是哪类：
     🔴 抄袭/低俗/灰产→删除，做新内容
     🟡 引流/重复/广告→删掉违规内容，修改后重发

第三步：内容质量检查（90%的问题在这）
  → 无声音/画面极差/完全看不懂？→硬伤，修改后重发
  → 开头无聊/没亮点/流水账？→优化前3秒，加故事/情感后重发
  → 内容平庸但无大错？→不要重复发！总结经验，做下一个更好的
```

> 💡 **核心认知**：抖音违规一定会发站内信，**不存在"偷偷限流"**。没收到通知就不是限流，90%是内容质量问题。

## ⚡ 五条核心心法

1. **前3秒决定生死**——用户划走的成本是零，第一帧就要给惊艳
2. **真诚是最高级的算法**——卡壳、试错、真实感受比完美展示更打动人
3. **普通人视角**——讲"这东西能帮你做什么"，不讲"我用了什么技术"
4. **不要堆量要质量**——认真做好一条比敷衍发十条强
5. **做完不是终点，讲好故事才是**——开发是和机器对话，做内容是和人对话
