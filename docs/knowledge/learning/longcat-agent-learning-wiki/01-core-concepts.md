## 二、LongCat-2.0核心概念解析

### 2.1 模型基本信息

LongCat-2.0是美团推出的大语言模型，在OpenRouter平台上月调用量排名第三（仅次于DeepSeek-V4-Flash和小米MiMo-V2.5）。其核心参数如下：

| 指标 | 数值 | 说明 |
|------|------|------|
| 总参数量 | 1.6万亿（1.6T） | 超大参数规模，提供丰富的知识表示能力 |
| 激活参数 | 约480亿（48B）/token | 每次推理只激活部分参数，显著降低计算成本 |
| 架构类型 | MoE（混合专家） | 多个专家子网络，每次推理选择最相关的专家激活 |
| 最大上下文 | 1M tokens | 支持超长上下文，适合大型项目代码库 |
| 训练算力 | 5万余国产算力芯片 | 完整训练流程基于国产算力集群完成 |
| 训练数据 | 超过35万亿tokens | 大规模预训练数据 |

### 2.2 MoE（混合专家）架构

MoE（Mixture of Experts，混合专家）是LongCat-2.0的核心架构设计。与传统的稠密模型（每个token激活所有参数）不同，MoE模型包含多个专家子网络，每次推理时仅激活与当前任务最相关的少数专家。

这种设计的优势在于：

- **成本效率**：1.6T总参数，但每次只激活约480亿参数，在保持强大能力的同时大幅降低推理成本
- **专业化分工**：不同专家可以专注于不同类型的任务（代码、推理、文本生成等）
- **可扩展性**：通过增加专家数量来扩展模型能力，而非简单地增加单个模型的深度

### 2.3 稀疏注意力机制

LongCat-2.0采用了稀疏注意力（Sparse Attention）设计，这是其能够支持1M超长上下文的关键技术。传统注意力机制的计算复杂度随上下文长度呈平方级增长，而稀疏注意力只关注最相关的部分，大幅降低了长上下文场景下的计算开销。

在Agent编程场景中，这意味着模型可以高效处理整个项目目录的代码文件、终端输出、错误日志等大量上下文信息，而不会因为上下文过长而丢失关键信息。

### 2.4 完全基于国产算力训练

LongCat-2.0的一个显著特点是其完整训练流程和大规模部署都基于国产算力集群完成。预训练使用了5万余国产算力芯片，消耗超过35万亿tokens。这在当前国际芯片供应链受限的背景下，具有重要的战略意义和技术示范价值。

### 2.5 Agent原生设计

资料显示，LongCat-2.0是Agent原生设计，代码能力强，已经适配了主流的编程工具：

- **Claude Code**：本次实测使用的编程Agent框架
- **OpenClaw**：开源AI编程工具
- **Hermes**：主流编程工具之一

所谓"Agent原生"，意味着模型在设计阶段就考虑了Agent场景的特殊需求：不是简单地回答问题，而是需要**读需求、看文件、改代码、跑命令、看报错，再继续修**——这是一个连续的、多轮的工具调用和反馈循环。

### 2.6 术语对照表

| 术语 | 英文 | 说明 |
|------|------|------|
| 混合专家 | MoE (Mixture of Experts) | 包含多个专家子网络的模型架构，每次推理只激活部分专家 |
| 稀疏注意力 | Sparse Attention | 只关注上下文中最重要的部分，降低长文本计算开销 |
| Agent原生 | Agent-Native | 模型设计时即考虑多轮工具调用和自主任务执行的需求 |
| 循环工程 | Loop Engineering | Agent在执行任务时形成"编码→测试→报错→修复"的迭代循环 |
| 上下文窗口 | Context Window | 模型一次能处理的最大文本长度 |
