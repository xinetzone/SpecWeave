# 常见问题

## Q: 翻译质量如何？和人工翻译比？

A: Rainman Translate Book 的翻译质量取决于 Claude 模型本身的能力。对于技术书籍、学术论文等非文学类文本，翻译质量较好，可以达到"可读译本"水平。但对于文学作品（诗歌、小说），文学修辞和韵味难以被 AI 准确传达。

和人工翻译相比：AI 翻译在速度上远胜（一本书几小时 vs 数周），但在质量上仍有差距，尤其是文化特定表达、一词多义、修辞手法等方面。建议将 AI 翻译作为"初稿"，再进行人工校对和润色。

---

## Q: 需要多少费用？

A: 费用主要来自 Claude API 的 token 消耗，取决于书的长度和子代理数量。以一本中等厚度的技术书（约 300 页）为例：

- 原文约 10 万单词
- 8 个子代理并行翻译
- 预计消耗 token 量在百万级别
- 按 Claude API 当前价格估算，费用约在 **几美元到十几美元** 之间

时间方面，完整翻译一本 300 页的书大约需要 **1-3 小时**（取决于 API 响应速度和网络状况）。

> **注意**：以上为估算值，实际费用取决于具体书的长度、Claude API 定价策略和你的 Anthropic 账户类型。建议先用短文本测试，了解实际消耗后再翻译整本书。

## Q: 支持哪些语言？

A: Rainman Translate Book 理论上支持 Claude 模型支持的所有语言对。目前已知的主要使用场景是**英文→中文**翻译。

由于 glossay 提取和术语锁定机制是语言无关的，该工具也可以用于其他语言对的翻译，只需在 Claude Code 中指定目标语言即可（例如"翻译成日文""翻译成法文"）。

---

## Q: 翻译中断了怎么办？

A: 直接重新运行同一个翻译命令即可。Rainman Translate Book 内置断点续传功能，通过 `manifest.json` 和 `run_state.json` 中的 SHA-256 哈希比对，自动识别哪些块已完成、哪些块需要翻译，从断点继续，不会从头重跑。

如果中断后希望完全重来，可以删除 `{book_name}_temp` 目录，再重新运行翻译命令。

---

## Q: 术语翻错了怎么改？

A: 有两种方式：

**方式一（推荐）：修改 glossary.json 后增量重译。**
1. 在 `{book_name}_temp` 目录下找到 `glossary.json` 文件
2. 手动修改需要修正的术语译文
3. 重新运行翻译命令
4. 系统自动识别受影响的块，只重新翻译这些块

**方式二：翻译完成后统一替换。**
在 `output.md` 中使用全局查找替换功能修正术语。这种方式简单直接，但需要手动确认替换是否准确。

---

## Q: 能翻译图片/图表里的文字吗？

A: 不能。Rainman Translate Book 的翻译流程依赖 Calibre 进行文字提取，图片中的文字（如截图中的 UI 文字、嵌入图片中的图表标注）无法被提取和翻译。

如果书中有大量包含文字的图片，建议：
- 翻译完成后，手动处理图片中的文字翻译
- 使用 OCR 工具先提取图片中的文字，再整合到翻译流程中

---

## Q: 和直接让 Claude 网页版翻译有什么区别？

A: 核心区别在于以下几点：

| 维度 | Rainman Translate Book | Claude 网页版直接翻译 |
|---|---|---|
| **上下文窗口** | 8 个独立窗口，互不污染 | 单个窗口，翻译到后面术语容易混乱 |
| **术语一致性** | glossary.json 硬约束锁定 | 无约束，靠模型记忆 |
| **断点续传** | 支持，SHA-256 哈希比对 | 不支持 |
| **多格式输出** | 自动输出 5 种格式 | 需手动复制粘贴 |
| **翻译速度** | 8 并行，速度快 | 单会话，逐段翻译 |
| **使用门槛** | 需要配置工具链 | 打开网页即可 |
| **适用场景** | 整本书翻译 | 短文、片段翻译 |

简单来说：如果只是翻译一段话或一篇文章，Claude 网页版更方便。如果要翻译整本书，Rainman Translate Book 在术语一致性、速度、格式输出方面有显著优势。
