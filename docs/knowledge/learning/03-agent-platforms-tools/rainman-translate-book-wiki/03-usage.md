# 使用流程

Rainman Translate Book 的使用极为简洁，大多数情况下只需一条命令。本章从快速上手开始，逐步介绍进阶操作。

---

## 快速上手：翻译一本书

### 基本命令

在 Claude Code 终端中，直接输入自然语言指令：

```
把 /path/to/book.pdf 翻译成中文
```

Claude Code 会自动走完整个流水线：

1. **转换**：调用 Calibre 将输入文件（PDF/DOCX/EPUB）转为 HTMLZ
2. **分块**：将内容拆分为每块约 6000 字符的 Markdown 小块
3. **提取术语**：采样 5 块，生成 `glossary.json`
4. **并行翻译**：启动 8 个子代理，每个子代理翻译一块
5. **验证**：SHA-256 哈希比对，确保无丢块、无错位
6. **合并**：Pandoc 合并所有 Markdown 块
7. **输出**：Calibre 输出 HTML、DOCX、EPUB、PDF 四种格式

整个过程不需要手动干预，运行完成后在当前目录下即可找到 `{book_name}_temp` 输出文件夹。

### 支持的输入格式

| 格式 | 说明 |
|---|---|
| PDF | 最常用的电子书格式，注意扫描版 PDF 可能无法正确提取文字 |
| DOCX | Word 文档，格式保留较好 |
| EPUB | 电子书标准格式，转换效果最佳 |

### 输出文件

翻译完成后，`{book_name}_temp` 文件夹中包含：

| 文件 | 用途 |
|---|---|
| `output.md` | 完整 Markdown 源文件，适合进一步编辑 |
| `book.html` | 带浮动目录的网页版，适合在线阅读 |
| `book.docx` | Word 文档，适合人工校对和排版 |
| `book.epub` | 电子书格式，适合 Kindle 等电子阅读器 |
| `book.pdf` | 打印版 PDF，适合纸质阅读 |

## 进阶操作

### 指定 EPUB 封面

如果需要为 EPUB 指定自定义封面，可以在合并阶段使用 `--cover` 参数：

```bash
python3 scripts/merge_and_build.py --temp-dir book_temp --title "《译名》" --cover cover.jpg
```

参数说明：
- `--temp-dir`：翻译过程中生成的临时目录路径
- `--title`：输出 EPUB 的书名
- `--cover`：封面图片路径（支持 JPG、PNG 格式）

### 修改术语表后重新翻译

翻译完成后，如果发现某个术语翻译不准确，可以按以下步骤修正：

1. **编辑 glossary.json**：在 `{book_name}_temp` 目录下找到 `glossary.json` 文件，手动修改需要修正的术语译文
2. **重新运行翻译命令**：在 Claude Code 中再次输入相同的翻译指令
3. **自动增量重译**：系统会自动比对，识别出哪些块使用了被修改的术语，只重新翻译这些受影响的块

例如，如果发现 "Manhattan" 被翻译成了"曼哈坦"，只需在 `glossary.json` 中将对应条目的 `translation` 字段改为"曼哈顿"，重新运行翻译即可。全书不会重跑，只重译涉及 "Manhattan" 的那些块。

## 使用建议

1. **首次使用建议先用短文本测试**：拿一篇几十页的论文或手册试跑，熟悉流程后再翻译整本书
2. **翻译前先检查输入文件质量**：扫描版 PDF 可能无法正确提取文字，建议使用文字版 PDF 或 EPUB
3. **翻译完成后人工校对**：AI 翻译虽然质量不错，但建议对关键术语和专有名词进行人工校对
4. **善用 glossary.json**：如果对初次翻译的术语不满意，修改 glossary 后增量重译，成本很低
5. **注意 API 费用**：翻译整本书会消耗大量 API token，建议先估算费用
