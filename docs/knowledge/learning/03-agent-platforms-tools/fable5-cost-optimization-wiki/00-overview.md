# Fable 5成本优化技巧Wiki - 概述

## 一、主题概述

2026年7月，Anthropic宣布Fable 5即将从订阅套餐中退出，转向按量计费模式。原定于7月7日退出，后延期至7月12日（北京时间7月13日15:00）。这一变化标志着"月卡无限刷"时代的结束，按量计费成为唯一使用路径。

Fable 5作为当前最强的AI编程模型，具备无人值守自动编程、复杂需求一把过、糊图精准识别等不可替代的能力，在GPT-5.6尚未发布的情况下暂无平替。然而，其高昂的按量计费价格——输入$10/百万token，输出$50/百万token（是Opus 4.8的两倍）——使得成本优化成为所有用户的刚需。

## 二、核心问题

按量计费带来了根本性的心理差异：
- **订阅模式**：有限的痛苦，额度用完即停，可以放心大胆地让模型跑任务
- **按量计费**：每一次回车都是在刷卡，通宵跑任务一晚可能产生几百美元的账单

这种"成本焦虑"催生了开发者社区对降本方法的热烈讨论，GitHub和推特上涌现出大量开源方案和使用技巧。

## 三、五大方法速览

| 类型 | 方法 | 核心思路 | 节省效果 | 适用场景 |
|------|------|----------|----------|----------|
| 社区开源 | Skill蒸馏法 | 趁订阅期用Fable 5将项目知识沉淀为Skills，传承给更便宜的模型 | 长期复用，边际成本递减 | 固定流程、批量重复任务 |
| 社区开源 | pxpipe文字转图片 | 利用图片与文字的计费价差，将上下文渲染成PNG输入 | 账单直降59%~70% | 上下文特别长的场景（整个项目、大段文档） |
| 社区开源 | 包工头调度模式 | Fable 5只做策略制定和质量把关，具体编码交给便宜模型 | 显著降低高价值token消耗 | 团队使用、任务难度参差不齐 |
| 官方技巧 | 缓存经济学 | 利用prompt cache机制，保持对话活跃让缓存续命，输入价可砍90% | 最高省90%输入成本 | 多轮对话、长会话场景 |
| 官方技巧 | 批量接口 | 非实时任务走批量通道，输入输出全部半价 | 直接省50% | 不着急要结果的批量任务 |
