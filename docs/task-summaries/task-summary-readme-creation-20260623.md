### 5.2 问题详情

#### 问题I1：PowerShell 不支持 bash heredoc 语法

**问题描述**：使用 `cat <<'EOF'` 语法在 PowerShell 中报错 "Missing file specification after redirection operator"

**根本原因**：PowerShell 不支持 bash heredoc 语法，`<<` 是 PowerShell 的输入重定向操作符

**解决方案**：改用多个 `-m` 参数传递 commit message

```bash
# 变更前（失败）
git commit -m "$(cat <<'EOF'
标题
正文
EOF
)"

# 变更后（成功）
git commit -m "标题" -m "正文"
```

**经验教训**：
- ✅ Windows 环境下避免使用 bash 特有语法
- 📌 PowerShell 与 bash 的语法差异需注意
- ⚠️ Git commit message 使用多个 `-m` 参数可创建多段

#### 问题I2：vendor/flexloop/README.md 的徽章被误复制

**问题描述**：用户将 vendor/flexloop/README.md 的徽章（针对 xinetzone/tao）复制到了 d:\AI\README.md

**根本原因**：用户操作时直接复制了参考文件的内容，未替换为项目实际信息

**解决方案**：替换为适配 atomgit 的 8 个静态徽章 + 引用链接格式

**经验教训**：
- ✅ 参考其他文件时需替换为项目实际信息
- 📌 使用引用链接格式集中管理 URL
- ⚠️ shields.io 不支持所有代码托管平台，需使用静态徽章
