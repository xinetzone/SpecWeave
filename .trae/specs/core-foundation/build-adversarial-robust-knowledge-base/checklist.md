# 对抗式健壮知识库系统 - Verification Checklist

## 功能完整性检查
- [ ] 分级加密存储功能完整：public/internal/confidential三级安全级别正常工作
- [ ] 完整性自校验：读取时自动验证SHA-256校验和，损坏立即检测
- [ ] 自动修复：Git版本存在时损坏文件自动恢复
- [ ] 第一性原理分类增强：knowledge_type多维标签体系（factual/procedural/conditional/metacognitive）
- [ ] 异常输入防御：超大文件、路径遍历、畸形YAML、特殊字符均被正确处理
- [ ] 对抗式审查工作流：多Agent并发从攻击者视角测试，生成结构化报告
- [ ] Git集成与备份：本地冗余备份、知识时光机、批量恢复功能正常
- [ ] 检索增强：按多维标签查询，结果自动过滤损坏条目
- [ ] Agent体系集成：AGENTS.md更新、角色职责更新、命令集封装完成
- [ ] 向后兼容：现有知识条目无需修改即可正常读取，迁移脚本可用

## 安全性检查
- [ ] AES-256-GCM加密正确实现，无已知漏洞
- [ ] confidential级条目无密钥完全无法读取
- [ ] 路径遍历攻击（../等）100%被拦截
- [ ] 文件大小限制生效，超大文件被拒绝
- [ ] 所有外部输入经过sanitize处理
- [ ] 错误信息不泄露敏感路径或密钥信息
- [ ] 加密密钥不硬编码在代码中
- [ ] 审查测试不修改/破坏真实知识条目

## 健壮性检查
- [ ] 处理10MB超大文件不OOM、不挂死
- [ ] 畸形/截断/空文件不导致崩溃
- [ ] 无限循环/深度嵌套场景有防护机制
- [ ] 单文件损坏不影响系统整体运行
- [ ] 所有错误场景都有清晰错误信息而非裸露栈追踪
- [ ] 并发操作无文件损坏风险

## 性能检查
- [ ] 1000条目索引生成<30秒
- [ ] 单条目完整性校验<100ms
- [ ] 加密/解密单条目<500ms
- [ ] 检索查询响应<2秒

## 兼容性检查
- [ ] 所有现有知识条目能正常读取（默认security_level=public）
- [ ] public条目Git diff正常工作，无加密噪声
- [ ] Python 3.8+兼容（Windows/macOS/Linux）
- [ ] 不引入非必要重量级第三方依赖
- [ ] 现有generate_index.py脚本继续正常工作或正确增强

## 测试覆盖检查
- [ ] 单元测试覆盖率≥80%
- [ ] 所有核心模块有对应单元测试
- [ ] 集成测试覆盖端到端流程
- [ ] 对抗测试用例覆盖所有威胁模型场景
- [ ] 所有测试通过无回归
