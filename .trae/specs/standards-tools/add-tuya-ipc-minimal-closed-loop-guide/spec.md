# 涂鸦 Tuya IPC 最小闭环跑通路径 Spec

<!-- changelog -->
## Changelog
- 2026-06-30 | added | 定义 Tuya IPC（网络摄像机）最小闭环跑通路径的交付范围、最小功能闭环节点、全流程步骤结构、可观测验收标准、常见问题排查方向与依赖关系图
<!-- changelog -->

## Why

当前团队缺少一份“可落地执行、可独立验证”的涂鸦（Tuya）IPC（网络摄像机）最小闭环跑通路径文档，导致环境搭建、SDK 集成、设备端联调与云端验证过程高度依赖经验传递，难以复用且排查成本高。本 Spec 旨在沉淀一条可执行、可观测、覆盖关键功能节点的最小闭环路径，用于快速把 IPC 从“编译可过”推进到“端云手机可验证”的闭环状态。

## What Changes

- **新增知识库文档 `docs/knowledge/operations/tuya-ipc-minimal-closed-loop.md`**：提供 Tuya IPC 最小闭环跑通路径（步骤、依赖、验收、排查）
- **更新 `docs/knowledge/README.md` 索引**：在 operations 分类中登记新文档（若知识库索引为脚本生成，则在任务中包含重新生成索引步骤）

## Impact

- Affected specs: 无
- Affected code: 无（仅新增/更新文档）
- Affected docs:
  - `docs/knowledge/operations/tuya-ipc-minimal-closed-loop.md`
  - `docs/knowledge/README.md`

## ADDED Requirements

### Requirement: 提供端到端最小闭环路径文档

系统 SHALL 提供一份可落地执行的 Tuya IPC 最小闭环跑通路径文档，覆盖从账号与产品准备、开发环境搭建、SDK 集成、固件编译烧录、设备配网、云端绑定、音视频采集与传输、移动端控制、基础事件上报到闭环验收的全流程步骤。文档必须以“步骤”为主线组织，每一步都包含：可执行操作节点、依赖条件、可观测验证标准、常见问题排查方向。

#### Scenario: 用户按文档跑通闭环

- **WHEN** 用户按照文档逐步完成环境搭建、SDK 集成与设备端联调
- **AND** 完成设备配网、云端绑定、移动端控制、事件上报与音视频能力验证
- **THEN** 能在移动端 App 中看到设备在线、可预览实时视频/语音对讲、可执行至少一种基础控制
- **AND** 能观测到至少一种基础事件在云端/移动端产生可验证记录

### Requirement: 最小闭环覆盖核心功能节点

文档 SHALL 明确最小闭环必须覆盖的核心功能节点，并为每个节点提供最小实现建议与验证方法：

- 设备初始化与配网
- 音视频采集与传输（至少支持实时预览）
- 云端设备绑定（设备出现在云端/账号下）
- 移动端设备控制（至少一种控制可生效）
- 基础事件上报（至少一种事件可在云端/移动端观测）

#### Scenario: 核心节点可逐一验证

- **WHEN** 用户逐一完成上述 5 个节点的实施
- **THEN** 对每个节点均能在文档定义的观测点获得“成功/失败”可判定结果（日志、App 交互、云端控制台状态或 API 响应）

### Requirement: 明确前置准备与依赖版本要求

文档 SHALL 在正式步骤前给出可直接执行的前置准备清单，并明确每项依赖的来源与满足条件，包括但不限于：

- 开发者账号申请与权限开通（Tuya 平台账号、云项目权限、App/SDK 权限）
- 产品创建流程（IPC 产品、PID 获取、必要云服务开通项）
- 固件编译环境配置要求（推荐 OS、编译工具链、基础依赖库）
- 依赖工具版本要求（编译器/CMake/Python/OpenSSL 等最小版本线）

#### Scenario: 用户能在开始前完成全部准备

- **WHEN** 用户按前置清单准备账号、产品、工具链与依赖库
- **THEN** 可以在进入 SDK 编译/集成步骤前完成可检查的准备项自检（如：登录成功、PID/授权文件齐备、工具链版本满足要求）

### Requirement: 每一步给出可观测验收标准与排查方向

文档 SHALL 为流程中的每一步提供：

- 可观测的验收标准（如：设备日志出现关键关键字、控制台状态变更、App 侧 UI 状态、接口返回码等）
- 常见问题的排查方向（至少包含“现象 → 可能原因 → 优先排查点/命令/日志位置”）

#### Scenario: 任一步失败可定位方向

- **WHEN** 用户在任意步骤未达到验收标准
- **THEN** 可在同一步的“排查”章节找到不依赖额外背景知识的排查入口（例如：日志开关、关键配置项、常见误配清单）

### Requirement: 提供依赖关系与最小闭环验收总表

文档 SHALL 给出流程依赖关系图（推荐 Mermaid），并在末尾提供“最小闭环验收总表”，用于独立复核闭环是否真正完成。

#### Scenario: 流程无关键节点遗漏

- **WHEN** 用户查看依赖关系图与验收总表
- **THEN** 能清晰识别每个节点的前置依赖、后继步骤与闭环出口
- **AND** 不存在“前置未定义但后续依赖”的断裂节点

## Document Outline（交付物结构约束）

交付文档 `docs/knowledge/operations/tuya-ipc-minimal-closed-loop.md` SHALL 包含以下章节（可在不改变语义的前提下微调标题，但信息不可缺失）：

1. 范围与目标（最小闭环定义、非目标）
2. 总览：端-云-手机闭环架构与数据流（含依赖关系图）
3. 前置准备清单（账号/产品/授权/工具链/依赖版本）
4. 全流程步骤（每步必须含：操作节点、依赖条件、验收标准、排查方向）
5. 最小闭环验收总表（勾选式）
6. 附录：日志开关、常用命令/工具、问题定位索引
