# README 零基础用户友好全面重写 Spec

## Why

当前 `README.md` 面向有编程和AI技术背景的开发者撰写，使用大量专业技术术语（如"渐进式披露""Core/Tools双层治理""阶段守卫运行时""L0/L1/L2三层架构"等），零编程/AI背景的普通用户完全无法理解项目目的、价值与使用方法。需要将 README 全面重写为面向零基础用户的友好版本，使任何人均能独立阅读并理解。

## What Changes

- **重写全文**：将所有技术概念翻译为通俗易懂的非技术语言
- **新增「这是什么」章节**：用类比和生活化语言解释项目是什么、解决什么问题
- **新增目录**：文档顶部添加完整目录和关键内容索引，方便快速定位
- **新增术语解释**：对不可避免的技术术语（如 AI 智能体、Git、Markdown 等）提供通俗解释
- **新增概念示意图**：用 Mermaid 流程图/示意图可视化展示核心概念和工作流程
- **优化快速开始**：扩展操作步骤，每一步配有详细说明和预期结果
- **重写项目亮点**：用普通人能理解的成就描述替代技术指标
- **保留核心链接**：保留所有指向 docs/ 的内部链接，确保文档导航完整性
- **遵循文档边界**：README 面向人类读者，不混入 `.agents/` 机器可读规范内容

## Impact

- Affected specs: 无（全新 spec）
- Affected code: `README.md`（全面重写）
- Affected docs: 无其他文档变更

## ADDED Requirements

### Requirement: 零基础可理解性

README 全文 SHALL 使用通俗易懂的语言撰写，一个没有编程或AI背景的普通成年人应能够独立阅读并理解项目目的、价值和使用方法。

#### Scenario: 项目简介可理解
- **WHEN** 零基础用户阅读项目简介段落
- **THEN** 能够在 30 秒内理解"这个项目是做什么的"
- **AND** 简介使用生活化类比（如"给AI写员工手册"）而非技术定义

#### Scenario: 核心概念有解释
- **WHEN** 用户遇到不熟悉的概念
- **THEN** 同一段落中有括号内的简短通俗解释，或链接到术语解释区
- **AND** 不使用"L0/L1/L2""渐进式披露""双层治理"等未解释的内部术语

#### Scenario: 操作步骤可跟随
- **WHEN** 零基础用户按快速开始章节操作
- **THEN** 每一步都能独立执行，无需额外背景知识
- **AND** 每步标注"预期看到什么"帮助用户确认操作正确

### Requirement: 结构化与可导航性

README SHALL 包含完整目录和合理的信息层级，使用户能快速找到所需内容。

#### Scenario: 目录完整
- **WHEN** 用户打开 README
- **THEN** 顶部可见完整目录，列出所有一级和二级标题
- **AND** 目录支持 GitHub 锚点跳转

#### Scenario: 标题层级合理
- **WHEN** 用户浏览 README
- **THEN** 标题层级不超过 3 级（H1→H2→H3）
- **AND** 每个章节内容长度适中，不超过约 30 行

### Requirement: 概念可视化

README SHALL 使用至少 2 个 Mermaid 图表辅助解释核心概念，帮助非技术用户直观理解。

#### Scenario: 工作流程图
- **WHEN** 用户阅读"如何使用"章节
- **THEN** 能看到一张简单的 Mermaid 流程图展示从获取到使用的完整流程
- **AND** 图表使用中文标签，节点不超过 8 个

#### Scenario: 概念关系图
- **WHEN** 用户阅读"项目包含什么"章节
- **THEN** 能看到一张展示各组成部分关系的示意图
- **AND** 图表标注清晰，适合非技术人员理解

### Requirement: 术语通俗化

所有不可避免的技术术语 SHALL 在首次出现时提供通俗解释。

#### Scenario: 术语首次出现有解释
- **WHEN** 文档中首次出现"AI 智能体""Git""GitHub""Markdown""克隆"等技术术语
- **THEN** 紧接术语后有括号内的简短通俗解释
- **AND** 解释使用生活化类比，避免用另一个术语解释术语

#### Scenario: 术语区可供查阅
- **WHEN** 用户需要查阅多个术语
- **THEN** 文档末尾有「常见术语解释」汇总区
- **AND** 术语按拼音首字母排序

## MODIFIED Requirements

无。本次为全面重写，非修改现有内容。

## REMOVED Requirements

### Requirement: 技术密集型内容展示
**Reason**: 当前 README 中大量技术细节（L0/L1/L2 架构、Core/Tools 治理、量化数据表格等）对零基础用户构成信息过载。这些内容仍保留在 `docs/` 下的专业技术文档中供技术人员查阅。
**Migration**: 技术细节迁移至 `.agents/docs/project-overview.md` 和 `.agents/docs/project-highlights.md`（已存在），README 改为通俗摘要并链接到详细文档。
