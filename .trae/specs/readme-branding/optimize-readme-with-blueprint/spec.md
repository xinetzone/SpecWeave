# README 内容补充与项目蓝图规划 Spec

## Why

当前 `README.md` 仅 90 行，聚焦于快速开始与文档导航，未能充分展示项目的核心价值、技术创新点与发展前景。作为面向人类读者的项目入口，README 缺少对项目亮点、量化成果与未来蓝图的呈现，不利于项目传播、社区贡献与生态拓展。需要在保留现有结构的基础上，补充项目亮点与蓝图规划两大核心章节，使 README 成为项目价值的完整展示窗口。

## What Changes

- **新增「项目亮点」章节**：系统梳理核心优势、技术创新点、功能特色与量化成果，以清晰、有说服力的方式呈现项目价值
- **新增「项目蓝图」章节**：制定短期目标、中长期战略、技术路线演进、功能迭代计划与市场拓展策略
- **优化现有章节**：在快速开始前增加项目简介段落，提升首屏可读性；调整文档导航顺序，使亮点与蓝图前置
- **保持结构简洁**：README 仍作为入口文件，详细内容通过链接指向 `docs/` 下的原子文档，避免文件膨胀
- **遵循文档边界**：README 面向人类读者，不混入机器可读规范内容；所有补充内容均基于已有项目资产提炼，不虚构未实现的功能

## Impact

- **Affected specs**: 无直接影响其他 spec，但与 `create-agents-md-and-config`（规范体系创建）和 `retrospective-agents-spec-system`（项目复盘）的成果强相关
- **Affected code**: 主要修改 `README.md`；可能微调 `.agents/docs/related-links.md`（补充蓝图相关外部链接）；不涉及 `.agents/` 与代码逻辑
- **Affected docs**: `README.md`（主要变更）、`.agents/docs/related-links.md`（可选补充）

## ADDED Requirements

### Requirement: 项目亮点章节

README SHALL 包含「项目亮点」章节，系统呈现项目的核心价值与技术特色。

#### Scenario: 展示核心优势
- **WHEN** 读者浏览 README 的项目亮点章节
- **THEN** 能够通过分组列表或表格清晰看到 4-6 项核心优势（如单一入口路由、5 角色分工、机器可读定义、完整协作协议等）
- **AND** 每项优势配有简短说明，解释其价值

#### Scenario: 展示技术创新点
- **WHEN** 读者浏览技术创新点小节
- **THEN** 能够看到 3-5 项技术创新（如入口+容器二元架构、三层递进提示词体系、TOML frontmatter 绑定、元工具体系、三层治理模型等）
- **AND** 每项创新标注来源（如来自复盘报告或实践验证）

#### Scenario: 展示量化成果
- **WHEN** 读者浏览量化成果小节
- **THEN** 能够通过表格或徽章看到关键数据（如 42 个交付物、5 个验证脚本、7 份复盘报告、6 个方法论模式、被 30+ 工具识别等）
- **AND** 数据准确反映项目实际规模

### Requirement: 项目蓝图章节

README SHALL 包含「项目蓝图」章节，呈现项目的发展规划与战略方向。

#### Scenario: 展示短期目标
- **WHEN** 读者浏览短期发展目标小节
- **THEN** 能够看到 3-6 个月内可交付的具体目标（如完善角色体系、扩展工具链、增强跨工具兼容性等）
- **AND** 每项目标有明确的交付标志

#### Scenario: 展示中长期战略
- **WHEN** 读者浏览中长期战略方向小节
- **THEN** 能够看到 6-18 个月的战略方向（如建立社区生态、形成行业标准、跨领域适配等）
- **AND** 战略方向与项目定位一致

#### Scenario: 展示技术路线演进
- **WHEN** 读者浏览技术路线演进小节
- **THEN** 能够通过 Mermaid 时间线或路线图看到技术演进的阶段划分
- **AND** 每个阶段标注关键技术里程碑

#### Scenario: 展示功能迭代计划
- **WHEN** 读者浏览功能迭代计划小节
- **THEN** 能够看到按优先级排序的功能迭代清单
- **AND** 每项功能标注预期版本或阶段

#### Scenario: 展示市场拓展策略
- **WHEN** 读者浏览市场拓展策略小节
- **THEN** 能够看到目标用户群体、推广渠道与合作模式
- **AND** 策略具体可行，不空泛

### Requirement: Mermaid 可视化

蓝图章节 SHALL 使用 Mermaid 表达技术路线与演进关系，符合项目"Mermaid 优先"原则。

#### Scenario: 技术路线图可视化
- **WHEN** 读者查看技术路线演进
- **THEN** 能够看到可渲染的 Mermaid 图表（如 timeline、flowchart 或 roadmap）
- **AND** 图表在 GitHub/AtomGit 等 Markdown 渲染器中可正常显示

## MODIFIED Requirements

### Requirement: README 整体结构

README 在保留现有快速开始、文档导航、许可证、联系方式的基础上，调整章节顺序，使项目亮点与蓝图规划在首屏可见范围内呈现。

#### Scenario: 章节顺序优化
- **WHEN** 读者打开 README
- **THEN** 首屏能够看到项目简介、核心亮点摘要
- **AND** 滚动后依次看到快速开始、项目亮点（详细）、项目蓝图、文档导航、许可证与联系方式

#### Scenario: 保留现有功能
- **WHEN** 读者使用快速开始或文档导航
- **THEN** 现有的克隆、验证、文档导航表功能保持不变
- **AND** 折叠的规范体系文档索引保留

## REMOVED Requirements

无移除项。本次为纯增量补充与顺序优化，不删除任何现有内容。
