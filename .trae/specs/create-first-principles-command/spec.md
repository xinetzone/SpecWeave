# 第一性原理指令集创建 Spec

## Why

当前 `.agents/commands/` 指令集库缺少面向"第一性原理"思维方法的标准化指令。第一性原理是一种从最基础、不可再分的公理出发，推导问题解决方案的思维方式，对于复杂问题拆解、根因定位、创新方案设计具有不可替代的价值。引入该指令集可使智能体在面对复杂决策时，从经验类比思维升级为公理化推导思维，提升分析的深度与方案的创新性。

## What Changes

- 新增 `first-principles.md` 指令集文件，位于 `.agents/commands/` 目录
- 文件内容覆盖：第一性原理定义、触发条件、应用场景、实施步骤、注意事项、输出规范与质量验收
- 在 `.agents/commands/README.md` 的指令集清单表格中追加"第一性原理"条目，关联自我认知模块
- 遵循现有指令集格式约定（YAML frontmatter + 标准章节结构 + RACI 矩阵）

## Impact

- Affected specs: 无既有 spec 受影响，属新增能力
- Affected code:
  - 新增文件：`.agents/commands/first-principles.md`
  - 修改文件：`.agents/commands/README.md`（清单追加一行）
- 不涉及代码实现，仅为指令集文档的新增与索引维护

## ADDED Requirements

### Requirement: 第一性原理指令集定义

系统 SHALL 提供 `first-principles` 指令集，作为智能体应用第一性原理思维方法进行问题分析与解决的标准化流程入口。

#### Scenario: 指令集元数据规范

- **WHEN** 读取 `first-principles.md` 文件
- **THEN** 文件头部 YAML frontmatter 包含 `id`、`title`、`source`、`x-toml-ref` 四个字段
- **AND** `id` 值为 `first-principles`
- **AND** `source` 值为 `AGENTS.md#指令集`

#### Scenario: 指令集核心章节完整

- **WHEN** 浏览 `first-principles.md` 正文
- **THEN** 包含以下章节：触发条件、输入规范、RACI 责任分配矩阵、核心原则、执行步骤、输出规范、质量验收、约束条件、关联资源
- **AND** 章节顺序与现有指令集（retrospective / insight / atomization）保持一致

### Requirement: 第一性原理定义与应用场景

指令集 SHALL 明确第一性原理的定义、思维特征与适用场景，便于智能体判断何时启用该指令集。

#### Scenario: 定义清晰可辨

- **WHEN** 智能体阅读指令集的"核心原则"或"定义"章节
- **THEN** 能识别第一性原理的核心特征：从不可再分的基础公理出发、剥离类比与经验假设、自下而上重构认知
- **AND** 能区分第一性原理思维与类比思维、归纳思维的差异

#### Scenario: 触发条件可识别

- **WHEN** 出现以下情形之一时，智能体应触发该指令集
  - 面对复杂问题，常规经验方案失效或效果有限
  - 需要从根本上理解事物本质，而非套用现有模式
  - 创新性方案设计，需突破既有路径依赖
  - 技术选型或架构决策中存在多重权衡，需归零思考
  - 复盘或洞察过程中识别到"知其然而不知其所以然"的问题
- **THEN** 指令集明确列出上述触发条件，供智能体自动判定

### Requirement: 实施步骤标准化

指令集 SHALL 提供分阶段的可执行步骤，将第一性原理思维方法转化为可操作的流程。

#### Scenario: 步骤覆盖完整思维链路

- **WHEN** 智能体按指令集步骤执行
- **THEN** 步骤覆盖以下环节：问题识别与边界界定、假设剥离与归零、基础要素识别、公理提炼、自下而上重构、方案验证与决策
- **AND** 每个步骤包含具体的操作动作与产出物

#### Scenario: 步骤可验证

- **WHEN** 完成每个步骤后
- **THEN** 具备明确的中间产出物（如假设清单、要素清单、公理列表、重构方案），可用于质量验收

### Requirement: 注意事项与边界约束

指令集 SHALL 明确第一性原理应用的注意事项、适用边界与常见误区，防止滥用或误用。

#### Scenario: 误区警示明确

- **WHEN** 智能体应用第一性原理
- **THEN** 指令集提示以下常见误区：过度拆解导致成本失控、忽视经验价值的合理部分、为创新而创新脱离实际约束、在简单问题上滥用复杂方法
- **AND** 提供适用边界判断准则（如成本收益阈值、问题复杂度阈值）

#### Scenario: 与其他指令集协同关系明确

- **WHEN** 智能体同时面临复盘/洞察/原子化等需求
- **THEN** 指令集明确第一性原理与现有指令集的协同与边界划分，避免职责重叠
- **AND** 在"关联资源"章节列出协同关系

### Requirement: 输出规范与质量验收

指令集 SHALL 定义标准化产出物格式与质量验收标准，确保第一性原理分析的成果可追溯、可审计。

#### Scenario: 产出物规范

- **WHEN** 第一性原理分析完成
- **THEN** 输出包含：第一性原理分析报告（Markdown）、要素拆解清单、重构方案、验证记录
- **AND** 产出物归档至指定目录

#### Scenario: 质量验收可执行

- **WHEN** 对第一性原理分析成果进行验收
- **THEN** 验收项包括：基础要素是否真正不可再分、公理是否自洽、重构方案是否覆盖原问题、验证是否基于事实而非假设
- **AND** 验收结果可记录为通过/不通过

### Requirement: README 索引同步更新

`.agents/commands/README.md` 的指令集清单表格 SHALL 同步追加"第一性原理"条目。

#### Scenario: 清单条目新增

- **WHEN** `first-principles.md` 创建完成
- **THEN** README 表格新增一行，包含：指令集名称（第一性原理）、ID（first-principles）、用途（应用第一性原理思维方法进行复杂问题分析与创新方案设计）、关联模块（自我认知）
- **AND** 表格其余行内容保持不变
