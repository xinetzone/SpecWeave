# 竹简悟道洞察库重组 - 产品需求文档

## Overview
- **Summary**: 对 `apps/zhujian-wudao/.agents/docs/insights/` 目录下的洞察库文件进行结构化重组。修复文件名错误、标题层级错乱、标准结构缺失、临时统计块残留等问题，将原有的2个大小失衡的文件（701行 vs 3265行）拆分为3个按层级组织的文件，并更新全项目所有交叉引用，确保重组后的洞察库结构清晰、格式统一、易于维护。
- **Purpose**: 当前洞察库存在文件名与内容不匹配（`insights-31-65.md`实际含31-68）、Markdown标题层级错乱（洞察53-68的子节误用`##`而非`###`）、临时统计块散落、文件大小严重失衡（1:4.7）、部分洞察标题模糊、标准结构（来源/核心内容）缺失等问题，影响可读性和维护效率。
- **Target Users**: 竹简悟道项目的AI Agent开发者、设计文档维护者、内容创作者。

## Goals
- 修复文件名编号与内容不匹配问题（`insights-31-65.md` → 正确的编号范围）
- 按项目规范的四层结构（产品/架构/哲学/元层）将洞察库拆分为3个大小均衡的文件
- 统一所有洞察的标准结构（`## 洞察N：标题` → `**来源**` → `**核心内容**` → `### 一、...`）
- 修复所有Markdown标题层级错误（子节从`##`改为`###`）
- 清理散落的临时统计块，仅保留文件头部的结构性声明
- 优化模糊的洞察标题（洞察16、18、20）
- 移除头部动态行数统计（遵循洞察55"上策"）
- 更新全项目所有交叉引用，确保链接有效
- 每个文件头部包含完整的文件说明、关联文件、层级结构声明

## Non-Goals (Out of Scope)
- 不修改任何洞察的实质内容（不重写、不删减、不合并洞察）
- 不改变洞察编号顺序或重新编号
- 不拆分体道四法系统化手册（洞察49/51/52/53/66）
- 不新增洞察或补充缺失内容
- 不调整非insights目录下的文档结构（product/reviews/knowledge-transfer保持不变）
- 不精简洞察49第七节"三宝现代社会体现"的扩展案例内容
- 不改变HTML原型或代码文件

## Background & Context
- 洞察库当前包含68条洞察，分存于2个文件
- 第一个文件（01-30）701行，覆盖产品层（1-15）和架构层（16-30）
- 第二个文件（31-65）3265行，实际包含洞察31-68（哲学层31-53 + 元层54-68），文件名错误
- 第二个文件在迭代过程中逐步增长，产生了多个结构问题：
  - 洞察53-68的子节标题使用`##`（应使用`###`），导致Markdown层级断裂
  - 洞察54及之后多条缺少`**来源**`和`**核心内容**`标准标记
  - 文件中散落4处临时统计块（58条/59条/60条/68条），仅最后一处准确
  - 洞察16/18/20标题过于模糊（"架构"/"架构"/"命名"）
- 项目规范（conventions.md）明确定义了洞察标准结构和四层分层
- 洞察55（文档声明的熵增定律）已指出头部动态统计数据必然过时，建议移除
- 全项目有44+处对insights文件名的引用需要更新

## Functional Requirements
- **FR-1**: 文件名必须准确反映内容范围，格式为`{YYYY-MM-DD}-insights-{起始}-{结束}.md`
- **FR-2**: 洞察库按三层分组为3个文件：01-30（产品+架构）、31-53（哲学）、54-68（元层+场景拓展）
- **FR-3**: 每条洞察必须遵循标准结构：`## 洞察N：标题` → `**来源**：[§xxx]` → `**核心内容**：` → 可选`---` → `### 一、...`至`### 七、...`
- **FR-4**: 所有洞察子节使用`###`（三级标题），不得与洞察标题（`##`）同级
- **FR-5**: 文件头部仅保留结构性声明（编号范围、洞察数量、层级说明、关联文件），不包含动态行数/字节数
- **FR-6**: 模糊标题必须优化为描述性标题
- **FR-7**: 所有跨文件引用使用正确的相对路径和文件名
- **FR-8**: 每个文件头部声明关联的其他洞察库文件

## Non-Functional Requirements
- **NFR-1**: 重组后3个文件的行数应大致均衡，单个文件不超过2000行
- **NFR-2**: 所有现有洞察内容必须100%保留，不得丢失文字
- **NFR-3**: 交叉引用的行号锚点应更新为新文件中的正确位置（无法精确到行号时至少确保文件名正确）
- **NFR-4**: 文件编码保持UTF-8，换行符保持一致

## Constraints
- **Technical**: 必须遵循 `conventions.md` 定义的文件命名规范和洞察标准结构
- **Business**: 洞察编号全局递增（1-68），不得跳号、重号
- **Dependencies**: 重组会影响AGENTS.md、conventions.md、project.md、README.md、product-spec.md、review文件、role文件、skill文件等12+个文件中的引用

## Assumptions
- 洞察内容的实质不需要修改，仅做结构化调整
- 拆分点（洞察53/54之间）是自然边界，因为文件头已声明"哲学层（31-53）+ 元层（54-68）"
- 洞察66（柔弱不争系统化手册）虽与洞察49/51/52/53同属系统化手册系列，但因编号顺序必须放在54-68文件中，通过交叉引用关联
- 跨文件引用的行号锚点更新以"文件名正确+指向正确洞察"为最低标准，精确行号由子agent在实施时校准
