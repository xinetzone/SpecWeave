# 支持 50 多种协议，终于开源了。

---

## 第一章：引言

我的工控下载站，终于上线了。

工厂能耗监测一直是个很麻烦的事。

电表、水表、光伏、空压机等数据分别存在不同的系统中。

想要得到一份总表的话，就需要花很多时间来整理。

传统的方法要么是自己定制开发成本很高，要么是买现成软件。

最近在 Gitee 上刷到了一个开源的EMS系统。

它叫 energy-management。

它是一套完整的开源能源管理系统，从数据采集到可视化展示都有。

---

## 第二章：项目简介

### 01 项目简介

它有大模块：
数据采集转发、接收、计算引擎、分布式数据Sharding、能源管理、设备预警、报表大屏、一次图系统。

适用于生产制造型企业、工厂、园区、写字楼等高耗能企业的能源管理场景。
可以对水、电、汽、热、油、空压机、中央空调、光伏逆变器等数据进行采集和分析。
使用Vue3、SpringBoot、SpringCloud和Alibaba微服务架构，代码注释率大于40%。

## 第三章：核心亮点

### 02 几个核心亮点

**① 支持海量并发，秒级处理数据可达5万条**

它用了ShardingSphere刀片技术和分布式数据库方案。
数据分片、读写分离、分布式事务等可以大大提高数据存储和计算的效率。
8核32G的云服务器配置，在PTS压测的情况下，每秒钟可以处理的数据量达到五万条。
支持虚拟电表创建，D = d1+d2+d3，配置虚拟点位计算楼层、车间、班组汇总数据。

**② 支持50多种工业协议，包括Modbus、IEC104**

硬件网关支持Modbus、IEC101/102/100/104、61850、DL/T645、MQTT、OPC等50多种协议。
数据采集可以对电表、水表、热用表、光伏、逆变器、空压机、配电箱等设备进行采集。
后台可视化配置参数之后，直接订阅硬件网关上传的数据，基本上不需要做开发。
支持MQTT订阅模式、WebSocket订阅模式、IEC104协议、Modbus RTU/TCP协议直接连接。

**③ 数据库分片技术，分表+分库+分片+分区**

用 Sharding 刀片技术之前，秒级 1000 个点位都吃力，内存干到 90%，CPU 100%。
用 Sharding 之后，秒级数据 5 万条轻轻松松，分布式数据刀片技术采用数据分片。
读写分离、分布式事务、数据加密、联邦查询、影子库、弹性扩展，海量数据处理能力大幅提升。

**④ 可视化配置，自定义报表+一次图绘制+页面编辑**

支持数据看板可视化组件库配置、拖拽选择展示方式、全部可视化配置。
可以自定义报表模板，并且可以关联数据进行一键式可视化配置。
一个图绘制软件、页面可视化编辑功能、自定义大屏配置、自定义网管配置。

## 第四章：使用教程

### 03 怎么用

部署环境基于 Ubuntu 22.04，需要 MySQL、Redis、Nacos。

**第一步，更新系统并安装 JDK1.8：**

```bash
sudo apt update
sudo apt upgrade
sudo apt install openjdk-8-jdk
```

**第二步是安装Maven和Node.js：**

```bash
sudo apt install maven
curl -sL https://deb.nodesource.com/setup_18.x | sudo bash -
sudo apt-get install -y nodejs
```

**第三步安装Nacos 2.2.2：**

下载Nacos并启动单机模式，设置好注册中心和配置中心。

**第四步，配置数据库和 Redis：**

创建MySQL数据库，并把初始数据导入进去，配置 Redis 权限认证。

**第五步，克隆项目并启动：**

克隆项目代码，启动后端微服务，启动前端 Vue3 应用。
打开http://127.0.0.1:8888就可以看到系统的界面了。

## 第五章：注意事项

### 04 注意事项也提前说一下

项目的技术栈比较重，微服务架构的部署和运维有一定的门槛。
适合有一定微服务开发经验的企业或者团队进行二次开发。
不适用于小团队或者个人快速上手，部署成本比较高。
要有一定的运维能力，才能使微服务集群稳定运行。

## 第六章：写在最后

### 05 写在最后

能耗数据其实挺值钱的。
通过对数据进行分析可以发现哪些设备存在能耗异常、哪个时间段电费成本较高、哪条生产线效率低下。
这些分析，直接关系到工厂的运营成本。
EMS 系统把数据采集、计算、可视化全部打通，并且可以自定义报表和看板。
对高耗能的工厂而言，这是一个可以尝试的工具。
有需要的朋友可以部署试试。
