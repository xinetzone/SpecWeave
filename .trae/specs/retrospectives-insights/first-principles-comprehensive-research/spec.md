# 第一性原理全面资料搜集与系统化档案建立 - Product Requirement Document

## Overview
- **Summary**: 本项目旨在全面搜集第一性原理（First Principles）相关的学术资料、理论文献、应用案例及权威解读，建立系统化、结构化的资料档案体系，覆盖哲学起源、物理学应用、商业创新实践、知名学者论述等多个维度，为后续研究、方法论应用及知识沉淀提供权威、准确、深入的信息支持。
- **Purpose**: 第一性原理作为一种根本性的思维方法论，在哲学、物理学、工程学、商业创新等领域具有重要价值。建立系统化的资料档案有助于：（1）追溯第一性原理的理论渊源与发展脉络；（2）系统梳理跨领域的应用实践与典型案例；（3）提炼可复用的方法论框架与思维工具；（4）为SpecWeave项目的方法论体系建设提供理论基础。
- **Target Users**: 项目研究人员、方法论实践者、AI智能体开发者、产品创新团队、学术研究者

## Goals
- G1: 全面搜集第一性原理哲学起源与发展历程资料，建立从亚里士多德到当代的完整理论脉络
- G2: 系统整理物理学领域第一性原理的应用实例，包括经典力学、量子力学、计算物理等方向
- G3: 深度搜集商业与创新领域第一性原理的实践案例，涵盖埃隆·马斯克、查理·芒格等知名实践者
- G4: 整理理查德·费曼、亚里士多德等知名学者关于第一性原理的核心论述与原著片段
- G5: 搜集学术期刊论文、专业著作、行业报告等权威来源资料，确保学术严谨性
- G6: 建立分类清晰、结构合理的系统化资料档案，便于检索、引用与后续扩展
- G7: 提炼第一性原理的核心概念、方法论框架与思维步骤，形成可操作的实践指南
- G8: 建立全面的对抗性审查机制，通过多维度来源验证确保所有资料的准确性、客观性和可靠性

## Non-Goals (Out of Scope)
- 不进行第一性原理的原创性学术研究或理论创新
- 不开发基于第一性原理的软件工具或产品
- 不涉及特定行业的深度咨询或解决方案设计
- 不进行第一性原理与其他思维方法论（如类比思维、系统思维）的对比研究（除非作为背景资料）
- 不翻译整本学术著作，仅选取关键章节或核心观点
- 不涉及实时动态追踪，聚焦于已确立的经典资料与权威论述
- 不覆盖中国古代哲学类似思想的系统对比（v1.x版本范围外）

## Background & Context
- **理论背景**: 第一性原理最早由古希腊哲学家亚里士多德提出，指"每个系统中存在一个最基本的命题，它不能被违背或删除"。在物理学中，第一性原理计算（First-principles calculation）指从基本的物理定律出发，不依赖经验参数的计算方法。在商业领域，埃隆·马斯克将其推广为一种创新方法论，强调"从基本原理出发进行推理，而不是通过类比进行推理"。
- **项目关联**: SpecWeave项目本身强调从第一性原理出发构建智能体协作体系，本资料搜集将为项目方法论建设提供理论支撑。此前项目已积累了提示词工程、Agent工程等相关方法论资料（如vibe-coding-prompts学习分析报告）。
- **技术背景**: 利用deep-research深度研究技能进行多源资料搜集与整合，结合WebSearch、WebFetch等工具进行权威来源验证。

## Functional Requirements
- **FR-1**: 哲学起源与发展历程资料搜集
  - 亚里士多德《形而上学》中关于第一性原理的原始论述
  - 第一性原理在西方哲学史上的发展脉络（笛卡尔、康德等哲学家的相关论述）
  - 第一性原理与其他哲学概念（如公理、公设、本体论）的关系
  - 当代哲学界对第一性原理的重新阐释与讨论

- **FR-2**: 物理学中的第一性原理应用实例
  - 经典物理学中的第一性原理思想（牛顿力学、麦克斯韦方程组）
  - 量子力学与第一性原理计算（密度泛函理论DFT、从头算方法ab initio）
  - 计算材料科学中的第一性原理应用（材料设计、药物发现）
  - 理查德·费曼关于第一性原理的论述与物理学方法论
  - 物理学诺贝尔奖相关工作中体现的第一性原理思维

- **FR-3**: 商业与创新领域第一性原理实践案例
  - 埃隆·马斯克的第一性原理实践（SpaceX火箭成本降低、Tesla电池设计、Hyperloop）
  - 查理·芒格的多元思维模型与第一性原理关联
  - 亚马逊贝索斯的"Day 1"哲学与第一性原理思维
  - 科技行业创新案例（Google搜索算法、iPhone设计、云计算架构）
  - 制造业、能源、生物科技等行业的第一性原理创新案例
  - 创业与产品设计中的第一性原理应用方法论

- **FR-4**: 知名学者与实践者核心论述
  - 亚里士多德：《形而上学》《物理学》相关原文与权威解读
  - 理查德·费曼：费曼学习法、物理学讲义中关于第一性原理的内容
  - 埃隆·马斯克：公开演讲、访谈、文章中关于第一性原理的论述汇编
  - 查理·芒格：《穷查理宝典》中关于基本原理、多元思维模型的论述
  - 其他学者：笛卡尔、康德、库恩等哲学家/科学家的相关论述
  - 当代商业思想家（如彼得·蒂尔、里德·霍夫曼）对第一性原理的应用阐述

- **FR-5**: 学术文献与权威资料搜集
  - 哲学领域关于第一性原理的核心学术论文与著作
  - 物理学/计算科学领域第一性原理方法的权威综述论文
  - 管理学/创新领域关于第一性原理思维的学术研究
  - 权威媒体（如《哈佛商业评论》《MIT斯隆管理评论》）的深度分析文章
  - 知名大学（如斯坦福、MIT）公开课/讲座资料
  - 行业研究报告与白皮书

- **FR-6**: 系统化资料档案建立
  - 按领域分类的资料目录结构（哲学/物理学/商业/方法论/案例库）
  - 每份资料包含：来源信息、作者/发布方、发布时间、核心内容摘要、关键引用片段、权威性评级、相关资源链接
  - 核心概念术语表（Glossary）
  - 学者/实践者索引目录
  - 时间线发展脉络图
  - 参考资料引用规范统一

- **FR-7**: 方法论提炼与框架总结
  - 第一性原理思维的核心步骤与操作流程
  - 第一性原理与类比思维的区别与适用场景
  - 第一性原理应用的常见误区与挑战
  - 不同领域应用第一性原理的共性与差异
  - 可操作的第一性原理思维工具/检查清单

- **FR-8**: 对抗性审查与来源验证机制（新增）
  - 建立多维度来源验证流程：交叉验证不同信息渠道、核查发布机构资质信誉、评估信息时效性、分析逻辑一致性与数据支持
  - 制定明确的资料筛选与可信度评级标准
  - 系统性识别与排除信息偏差、利益冲突、虚假内容
  - 建立来源评估档案，记录每份资料的可信度评分及验证过程
  - 关键事实与引用必须经过至少2个独立权威来源交叉验证
  - 对争议性观点标注不同立场与证据强度
  - 建立异常信息标记与存疑资料追溯机制

- **FR-9**: 第一性原理指令集创建（后续迭代）
  - 在 `.agents/commands/` 下创建 `first-principles.md` 指令集
  - 包含明确的触发条件、输入规范、RACI责任矩阵
  - 定义6步执行流程（问题识别→本质拆解→方案重构→验证迭代→模式沉淀→归档关联）
  - 关联知识库6个关键文件作为执行参考资源
  - 遵循项目指令集编写规范，与已有指令集风格一致

- **FR-10**: 指令集↔知识库双向关联与模式沉淀（后续迭代）
  - 指令集侧添加知识库关键文件链接（关联资源章节）
  - 知识库README添加指令集反向引用（交叉引用章节）
  - 萃取并沉淀7个可复用方法论模式至模式库
  - 完成复盘报告全面更新（v1.1），反映指令集创建与双向关联进展
  - 验证"关联对应性前提"模式：建立关联前必须验证知识库存在系统性资料档案

## Non-Functional Requirements
- **NFR-1**: 资料来源权威性 - 优先使用学术期刊、权威出版社、知名大学、官方发布等一级来源；网络博客、自媒体等二级来源需交叉验证
- **NFR-2**: 内容准确性 - 所有引用内容需标注准确来源，重要观点需至少2个独立来源交叉验证
- **NFR-3**: 结构完整性 - 资料档案需覆盖FR-1至FR-8所有功能需求，分类体系清晰一致
- **NFR-4**: 可扩展性 - 目录结构与档案格式设计需支持后续资料追加与更新
- **NFR-5**: 可检索性 - 提供清晰的索引、目录、术语表，支持快速定位所需资料
- **NFR-6**: 学术严谨性 - 引用格式统一，区分原始论述与二手解读，标注观点的适用边界
- **NFR-7**: 文件规范 - 所有文件遵循项目命名规范（kebab-case英文命名），使用标准Markdown格式，YAML frontmatter符合MDI v1.0规范
- **NFR-8**: 对抗性审查透明度 - 每份资料的验证过程、可信度评分、来源交叉验证情况必须可追溯、可审计
- **NFR-9**: 偏差防控 - 主动识别并标注可能存在的确认偏差、幸存者偏差、商业利益冲突等认知偏差来源
- **NFR-10**: 存疑标记 - 无法通过验证或存在争议的内容必须明确标记为"待验证"或"存疑"，不得作为确定事实呈现

## Constraints
- **Technical**: 
  - 基于现有工具集（WebSearch、WebFetch、deep-research skill等）进行资料搜集
  - 无法直接访问付费学术数据库（如IEEE Xplore、ScienceDirect全文），优先使用开放获取资源
  - 文件存储使用Markdown格式，最大单文件不超过500行
- **Business**: 
  - 资料搜集聚焦于公开可获取的权威资料，不涉及付费内容购买
  - 尊重知识产权，合理引用并明确标注来源
- **Dependencies**: 
  - deep-research技能提供多源资料整合能力
  - WebSearch/WebFetch工具进行网络资料获取
  - 项目现有docs/knowledge/目录结构作为最终归档位置

## Assumptions
- 假设通过公开网络资源（学术开放获取平台、大学公开课、权威媒体、官方发布等）可获取足够的高质量资料
- 假设英文资料在第一性原理学术研究领域占主导地位，需要进行准确的中文译介与整理
- 假设用户期望建立一个基础性的资料档案，而非穷尽所有相关文献
- 假设资料整理完成后将归档至docs/knowledge/learning/目录下的专门子目录
- 假设过程中可以使用子代理并行搜集不同领域的资料

## Acceptance Criteria

### AC-1: 哲学起源资料完整性
- **Given**: 项目启动，资料搜集工作开始
- **When**: 完成哲学领域资料搜集
- **Then**: 包含亚里士多德原始论述、关键哲学家发展脉络、哲学概念辨析三个核心部分，至少覆盖5位关键哲学家，引用至少3本哲学经典著作相关章节
- **Verification**: `programmatic`
- **Notes**: 需区分第一性原理在不同哲学语境下的含义差异

### AC-2: 物理学应用资料覆盖度
- **Given**: 物理学领域资料搜集完成
- **When**: 检查资料内容
- **Then**: 包含经典物理、量子力学/DFT计算、费曼方法论三个核心模块，至少3个具体应用案例，引用费曼相关著作或讲座内容
- **Verification**: `programmatic`
- **Notes**: 物理学第一性原理计算部分需兼顾概念解释与典型应用，避免过度数学化

### AC-3: 商业创新案例代表性
- **Given**: 商业领域资料搜集完成
- **When**: 审核案例库
- **Then**: 包含埃隆·马斯克至少3个典型案例（SpaceX、Tesla、其他），覆盖科技、制造、互联网至少3个行业，每个案例包含问题背景、第一性原理应用过程、成果数据
- **Verification**: `human-judgment`
- **Notes**: 案例需体现从基本原理出发推理的具体过程，而非仅仅是成果展示

### AC-4: 知名学者论述准确性
- **Given**: 学者论述整理完成
- **When**: 检查核心引用
- **Then**: 每位核心学者（亚里士多德、费曼、马斯克、芒格）至少3条关键论述，标注准确来源（著作名/演讲场合/时间），引用内容忠实原文，解读部分与原文明确区分
- **Verification**: `human-judgment`

### AC-5: 资料来源权威性
- **Given**: 所有资料搜集完成
- **When**: 统计来源分布
- **Then**: 一级来源（学术著作、期刊论文、官方发布、权威媒体）占比不低于70%，所有资料标注来源URL或出版信息，无明确来源的网络传闻不纳入核心档案
- **Verification**: `programmatic`

### AC-6: 资料档案结构合理性
- **Given**: 资料档案建立完成
- **When**: 检查目录结构
- **Then**: 目录层级不超过3层，分类符合FR-6要求，包含README索引、术语表、时间线，跨文件引用使用file:///绝对路径格式
- **Verification**: `programmatic`

### AC-7: 方法论框架实用性
- **Given**: 方法论提炼完成
- **When**: 评估方法论内容
- **Then**: 包含明确的步骤流程（至少5个可操作步骤）、常见误区提示（至少3条）、应用检查清单（至少10项），框架可直接用于指导实际问题分析
- **Verification**: `human-judgment`

### AC-8: 文件规范符合性
- **Given**: 所有文件生成完毕
- **When**: 运行项目规范检查
- **Then**: 所有文件名符合kebab-case英文命名，无中文文件名；YAML frontmatter格式正确；单文件不超过500行；无文件放置在项目根目录
- **Verification**: `programmatic`
- **Notes**: 通过运行`python .agents/scripts/check-filename-convention.py`验证

### AC-9: 对抗性审查机制有效性
- **Given**: 对抗性审查流程应用于所有资料
- **When**: 检查资料验证档案与质量控制记录
- **Then**: （1）所有引用的关键事实至少经过2个独立来源交叉验证；（2）每份资料具备可信度评分（A/B/C/D级）与验证过程记录；（3）建立明确的来源分级标准（一级/二级/三级来源）；（4）识别并标注出存在利益冲突或潜在偏差的来源；（5）存疑或无法验证的内容明确标记；（6）争议性观点呈现多方立场与证据强度；（7）建立来源评估档案文件（source-validation-log.md）记录完整验证过程
- **Verification**: `human-judgment`
- **Notes**: 对抗性审查不是一次性检查，而是贯穿资料搜集全过程的质量控制机制

### AC-10: 第一性原理指令集质量
- **Given**: 知识档案完成后，创建指令集
- **When**: 检查 `.agents/commands/first-principles.md`
- **Then**: （1）包含触发条件、输入规范、RACI责任矩阵；（2）定义6步可执行流程；（3）关联资源章节引用至少6个知识库关键文件；（4）遵循项目指令集规范，与已有指令集风格一致
- **Verification**: `programmatic`
- **Notes**: 指令集是知识档案从"静态资料"到"可执行方法论"的关键转化

### AC-11: 双向关联闭环
- **Given**: 指令集创建完成
- **When**: 检查双向链接
- **Then**: （1）指令集侧有知识库反向链接（关联资源章节）；（2）知识库README有指令集反向引用（交叉引用章节）；（3）所有链接通过check-links验证无断链；（4）路径风格遵循"入乡随俗"原则（.agents/用相对路径，docs/用.agents/前缀）
- **Verification**: `programmatic`

### AC-12: 模式沉淀与复盘完成
- **Given**: 所有产出完成
- **When**: 检查模式库与复盘报告
- **Then**: （1）至少7个可复用模式归档至模式库（含L2级模式至少2个）；（2）复盘报告完成v1.1更新，包含执行复盘、洞察提取（5条原子化）、导出建议；（3）关键数据准确（87来源/4869行/77.3%一级来源/78.5%A级可信度）；（4）所有本地链接验证通过
