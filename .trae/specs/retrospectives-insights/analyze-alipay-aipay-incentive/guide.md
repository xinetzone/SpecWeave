## 3. 报名步骤（FR-2）

### 3.1 详细报名流程

> ⚠️重要提示：**报名是用户数计算起点，报名前接入产生的交易用户不计入活动统计！** 请务必先报名再接入。

1. **访问活动页面**
   - 打开支付宝AI支付开发者激励计划活动页（链接见附录）
   - 建议使用电脑端浏览器访问，方便后续开发对接

2. **登录支付宝开放平台账号**
   - 点击页面右上角"登录"
   - 使用支付宝扫码或账号密码登录
   - 确认登录账号即为后续接入应用、关联阿里云账号的主账号

3. **填写报名表单**
   - 点击页面上的"立即报名"或"参与活动"按钮
   - 按要求填写表单字段（字段说明见3.2）
   - 仔细阅读活动协议

4. **勾选协议并提交**
   - 勾选《活动参与协议》《支付宝开放平台服务协议》等相关协议
   - 确认信息无误后点击"提交报名"
   - 等待系统审核（通常即时通过，最晚不超过1个工作日⚠️待确认）

5. **报名成功确认**
   - 收到报名成功的页面提示
   - 可在活动页面查看个人进度（此时用户数计数已开始）

### 3.2 表单字段填写说明

| 字段 | 填写建议 |
|------|---------|
| 手机号 | 填写常用手机号，用于接收重要通知（审核结果、奖励发放等） |
| 功能描述 | 简述你的AI应用类型、使用场景、预计用户规模，例如："基于GPT的智能写作工具，面向内容创作者，提供网页版付费使用"；真实描述即可，不需要夸大 |
| 选择接入产品 | 可多选，但建议先选1个最适合的产品快速接入（产品选择建议见第4章） |
| 其他信息 | 按实际情况填写即可，无特殊要求 |

### 3.3 报名后注意事项

- ⏰ **用户数开始计算时间点**：提交报名并通过审核后立即开始计算，在此之前的所有交易用户均不计入活动
- 📧 **审核等待**：一般报名后即时审核通过；如果超过24小时未通过，可联系支付宝开放平台客服
- 🎯 **不要重复报名**：同一账号报名一次即可，重复报名可能导致统计异常
- 🔍 **进度查看**：可在活动页面实时查看当前有效交易用户数、已解锁档位等信息
- ⚠️待确认：用户数统计是否有延迟（建议以活动页面显示为准）

### 4.5 沙箱→生产的环境切换流程

> ⚠️关键提醒：不要在本地开发时直接用生产环境参数——可能产生真实交易、回调地址不通导致问题！

**正确流程：**

1. **沙箱环境开发测试**
   - 使用支付宝沙箱环境的APPID、网关、密钥
   - 使用沙箱买家账号测试支付流程
   - 验证回调通知、订单查询等逻辑正确性
   - 沙箱测试金额建议设为0.01元（即使走账也不心疼）

2. **沙箱自测通过**
   - 支付流程可正常走通
   - 支付成功后回调能正确接收和处理
   - 订单状态查询正常
   - 异常场景（取消支付、支付失败）处理正确

3. **切换到生产环境**
   - 替换为生产环境APPID
   - 替换为生产环境应用私钥/支付宝公钥
   - 网关地址切换为生产网关（https://openapi.alipay.com/gateway.do）
   - 回调地址配置为生产环境HTTPS地址（必须是备案域名）

4. **生产环境小额验证**
   - 先用自己的支付宝账号支付1分钱做验证
   - 确认回调正常、订单状态更新正确
   - 确认资金能正常到账

5. **正式上线**
   - 验证通过后可以正式对外开放给用户使用

## 5. 时间规划与关键节点（FR-5）

### 5.1 T+7机制详解

**什么是T+7？**
- "T"指用户完成有效交易、达到某档位标准的日期
- "+7"指需要等待7个自然日的确认期
- 只有过了7天确认期，该档位的奖励才会正式解锁发放
- 确认期内如果发现交易违规、无效，会被扣除

**为什么T+7很重要？**
- 活动8月21日结束，意味着你的最后一笔有效交易必须在 **8月14日** 之前完成
- 8月14日达标 → 8月21日T+7期满 → 刚好赶在活动结束前解锁
- 8月15日及以后达标，T+7期满时活动已经结束，无法解锁！

### 5.2 关键时间节点倒推表

以8月21日活动结束为终点倒推：

| 日期 | 节点 | 说明 |
|------|------|------|
| 7月21日 | 活动开始 | 报名通道开启，可开始接入 |
| **最晚7月31日** | ICP备案截止 | 如果还没备案域名，今天必须提交备案申请（保守按20天算） |
| **最晚8月3日** | 开始开发 | 如果从零开始，这天必须动手写代码 |
| **8月7日（保守安全线）** | 完成上线+审核通过 | ⭐强烈建议在这天前完成所有接入、测试、上线工作，留出7天运营时间 |
| 8月10日 | 中等风险上线日 | 如果8月10日上线，只有4天时间拉用户，时间紧张 |
| **8月14日** | 达标截止日 | ⚠️硬性截止：必须在今天23:59:59前达成目标档位的有效交易用户数 |
| 8月15日-8月20日 | T+7确认期 | 等待解锁，期间不要作弊、不要出问题 |
| 8月21日 | 活动结束 | T+7期满，奖励解锁 |

## 6. 常见问题FAQ（FR-6）

### 6.1 官方FAQ

**Q1：活动奖励什么时候发放？以什么形式发放？**
A：用户达标后需等待T+7天确认期，确认期结束且无违规情况，阿里云Token资源包将发放至与报名支付宝账号关联的阿里云账号（⚠️具体发放方式以官方通知为准，可能需要绑定阿里云账号）。Token包用于抵扣阿里云AI/云服务费用，不可提现。

**Q2：三款产品可以同时接入吗？用户数是否累加？**
A：可以同时接入，三款产品可在**同一认证的个人账号名下**汇总统计有效交易用户数（企业认证账号是否可汇总待确认）；建议在同一认证账号下接入多款产品。

### 6.2 补充FAQ（关键问题）

**Q3：报名前接入的用户算吗？**
A：**不算！** 报名是用户数计算的唯一起点，报名前产生的所有交易用户都不计入活动统计。务必先报名再接入。

**Q4：什么是"有效交易用户"？定义是什么？**
A：⚠️官方未公开完整定义。通常行业惯例是：完成首次真实支付、非退款、非测试、非作弊的独立用户。建议预留20-30%缓冲，不要刚好卡着线。

**Q5：我可以找亲戚朋友帮忙刷量吗？**
A：**绝对不要！** 支付宝有严格的风控系统，熟人互刷、IP集中、设备异常、小额高频等行为都会被识别为无效交易，严重的可能取消活动资格甚至限制账号。

**Q6：个人开发者可以参加吗？**
A：可以。AI网页应用收款、AI移动应用收款个人认证账号大概率可以接入；AI按量付费可能需要企业资质，请以实际签约页面提示为准。

**Q7：npx命令是唯一接入方式吗？我不是Node.js开发者怎么办？**
A：npx只是快速初始化方式，不是必须的。支付宝提供Java/Python/Go/PHP/.NET等所有主流语言的SDK，用你熟悉的语言开发后端即可。

**Q8：我有多个支付宝账号，可以分别报名吗？**
A：⚠️待确认官方规则。通常同一身份/同一主体下的多个账号可能被识别为关联账号，有违规风险；建议用主力账号报名参与。

**Q9：本地开发可以用localhost/IP吗？**
A：沙箱环境可以用本地地址测试回调，但生产环境的回调地址、网页支付域名必须是已ICP备案的HTTPS域名。

**Q10：如果我8月14日刚好达标，但是T+7期间用户退款了怎么办？**
A：如果确认期内有效交易用户数低于档位标准，奖励可能无法解锁。建议达标后再多拉20-30%用户留作缓冲。

**Q11：奖励是阿里云Token包，需要交税吗？能提现吗？**
A：奖励形式为阿里云Token资源包（用于抵扣阿里云AI/云服务费用），非现金，无法直接提现。Token包的税务处理以阿里云官方规则为准，通常云资源兑换券类奖励不涉及个人所得税代扣代缴，具体以官方通知为准。

**Q12：第1期参加过还能参加第2期吗？**
A：第1期参与者需重新报名，可再次参与第2期活动。

**Q13：接入后必须用支付宝的AI能力吗？还是我自己的AI也可以？**
A：⚠️待确认。从产品名称看是"支付宝AI支付"——即用于AI应用的收款工具，你的AI应用可以用任意模型/能力（OpenAI、文心一言、通义千问、自研等均可）。

**Q14：用户支付后资金多久到账？奖励和交易资金是一回事吗？**
A：用户支付的交易资金按支付宝常规结算周期到账（通常T+1，具体以签约的产品费率和结算规则为准）。活动奖励是独立的阿里云Token资源包，与交易资金完全分开，Token包不进入支付宝余额。

**Q15：我可以在接入过程中更换产品吗？**
A：报名时可以多选产品，接入过程中也可以切换/新增产品，但建议先选一个最快接入的跑通，再考虑其他产品。

| 8 | 以为达标了立刻发奖，Token包马上能用 | T+7要等7天确认期，不是即时发放 | 达标后耐心等待确认期结束，Token包将发放至关联阿里云账号 |

### 9.2 不同场景开发者建议

#### 👤 个人开发者（无企业资质）
- **推荐产品**：AI网页应用收款（最友好）→ AI移动应用收款
- **目标档位**：STARTER/GROWTH/PRO，不建议冲LEGEND
- **关键提醒**：有备案域名直接上，没备案今天就去提交
- **快速路径**：用npx初始化一个简单Demo，接一个简单AI功能（比如AI文案润色），快速上线

#### 🚀 已有现成产品（网站/App）
- **推荐产品**：直接匹配你现有产品形态（网页选网页收款，App选移动收款）
- **目标档位**：可以冲ELITE，看现有用户规模
- **关键提醒**：先报名！然后把支付功能加进去，推给现有用户即可
- **快速路径**：半天就能接入完，主要是走通测试上线流程

#### 🌱 从零开始（没有产品、没有备案）
- **推荐产品**：先做一个极简AI网页应用，同时立即提交备案
- **目标档位**：STARTER/GROWTH，不要期望太高，参与为主
- **关键提醒**：备案是第一优先级，同时开发MVP，备案下来立刻上线
- **备选方案**：如果备案下不来，试试做个移动应用（但上架也需要时间）

#### 🔧 非Node.js开发者（Java/Python/Go/PHP等）
- **推荐产品**：都可以，选你擅长语言的SDK
- **目标档位**：按你开发速度评估
- **关键提醒**：不用纠结npx，直接下载对应语言的服务端SDK，按文档接入即可
- **快速路径**：前端用简单的HTML/JS调起支付，后端用你最熟悉的语言写接口

#### 🏆 第1期参与者
- **推荐产品**：可以尝试接之前没接过的产品，三款产品用户数可在同一账号下汇总
- **目标档位**：根据第1期经验设定
- **关键提醒**：需重新报名参与第2期，同一账号有效
- **快速路径**：轻车熟路，直接报名→复用之前的代码→快速上线
