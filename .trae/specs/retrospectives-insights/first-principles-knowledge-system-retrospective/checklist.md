# 第一性原理知识体系构建系统性复盘 - 验证清单

## 事实收集与数据验证
- [ ] Git提交历史已完整提取，关键commit hash已记录
- [ ] 文件数统计通过PowerShell实际验证（不是估算）
- [ ] 行数统计通过Get-Content | Measure-Object实际验证
- [ ] 来源可信度数据从10-source-validation-log.md验证
- [ ] 6份已有阶段性复盘报告均已阅读并整合
- [ ] 核心决策点清单≥10个，每个都有时间和上下文记录
- [ ] 事实收集阶段严格只记录事实，未混入主观判断

## 核心决策第一性原理分析
- [ ] 每个决策都有完整的5-Whys根因分析链
- [ ] 每个决策都列出≥2个备选方案及放弃原因
- [ ] 决策依据明确分类：第一性原理推导/类比惯例/权宜之计
- [ ] 每个决策都有效果评估（预期结果+意外结果）
- [ ] 分析深度足够——追溯到问题本质，不停留在表面
- [ ] 反直觉决策和争议决策得到重点分析

## 问题与挑战复盘
- [ ] 识别的重大挑战/问题/瓶颈≥8个
- [ ] 每个问题都有"表象→根因"的双层分析
- [ ] 每个问题都记录了当时的解决过程和实际效果
- [ ] 包含事后复盘（站在v1.7视角的反思）
- [ ] 问题分类清晰（工具/流程/认知/技术等）
- [ ] 重复出现的问题和"解决问题带来新问题"的案例得到重点关注

## 方法论应用分析
- [ ] 识别的应用方法论≥5个
- [ ] 每个方法论都有具体的应用场景和适配调整说明
- [ ] 每个方法论都有客观的效果评估（正面+负面）
- [ ] 明确区分了领域特定方法和普适性方法
- [ ] 识别出≥1个涌现的新方法论或方法论改进
- [ ] 分析了"方法论为什么在这个场景有效"的本质原因

## 关键洞察萃取
- [ ] 提炼的关键洞察≥8条
- [ ] 每条洞察都有具体的事实/案例/文件引用支撑
- [ ] 洞察明确分类：项目特定vs普适性
- [ ] 洞察有深度，具备"啊哈效应"（揭示表面看不到的本质）
- [ ] 高价值普适性洞察有明确的模式沉淀建议
- [ ] 洞察不是空泛的"我们要重视XX"，而是具体的本质发现

## 复盘报告质量
- [ ] 报告归档至正确目录：docs/retrospective/reports/project-reports/
- [ ] 报告包含完整8个章节：执行摘要→事实还原→过程分析→决策复盘→问题与解法→洞察提炼→改进建议→附录
- [ ] 执行摘要≤1页篇幅，核心信息完整
- [ ] 改进建议≥5条，每条都有优先级、具体措施、验收标准
- [ ] YAML frontmatter格式正确，包含source字段
- [ ] 文件名符合kebab-case规范
- [ ] 所有本地链接通过check-links.py验证100%有效
- [ ] 所有章节通过Grep验证完整存在
- [ ] 所有统计数据通过工具实际验证（三查法完成）
- [ ] 报告逻辑清晰，未参与项目的人也能看懂

## 知识体系构建模板质量
- [ ] 模板作为独立方法论模式存储在docs/retrospective/patterns/methodology-patterns/research-knowledge/
- [ ] 模板包含完整7个必备部分：结构框架、内容组织、质量控制、实施流程、工具链、反模式清单、适配指南
- [ ] 结构框架提供了可直接套用的目录结构示例、命名规范、frontmatter标准
- [ ] 实施流程分阶段定义，每阶段有明确的输入/输出/关键活动/验收标准/常见陷阱
- [ ] 质量控制标准包含可操作的检查清单（不是空泛原则）
- [ ] 反模式清单≥8个常见陷阱，每个都有识别信号、后果、预防措施
- [ ] 工具链配置提供了具体的脚本/工具推荐和配置示例
- [ ] 适配指南说明了不同规模/类型知识项目如何调整模板
- [ ] 模板具备"拿过来就能用"的可操作性
- [ ] TOML frontmatter符合模式文件规范，maturity初始设为L2
