脑子里有个半成型的需求，想让 AI 帮忙实现，但要把背景、目标、约束、参考资料一一说明白，往往还没开始写代码，先写了几百字的 prompt。

更烦人的是，有时候还没组织好语言，思路就已经跑偏了。

后来发现了一个有意思的现象：很多开发者其实更习惯用画布来梳理思路。把想法写成卡片，随手画几条线理清关系，这种空间化的思维方式远比线性的文字描述更直观。

但问题是，画布上的东西怎么传给 AI 呢？总不能截图发过去吧。

直到遇到 **BonsAI** ，这个问题才算真正解决了。

### 什么是 BonsAI？

**BonsAI** 是一款 macOS 原生的小应用，它的核心思路很简单：给你一块无限画布，让你把想法写成卡片，随手画图理清关系。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/rfBHhQhezUg9YSIWHxylkoJNQxumQOUia2lEJsEm0OwrwGPqumSGichaeDkPfhicaGqYCBZmRQM4mwIo5TQVjXSy8LfOsWUmoQY2KiaRz7hNCdI/640?wx_fmt=png&from=appmsg)

然后，画布上挂着一个 Agent，用的是你自己装的 Claude Code、Codex 或 OpenCode，能实时读取和改写整块画布。

这个想法看似简单，但用起来真的很舒服。

更妙的是，想法整理得差不多了，可以一键复制出一份自带上下文的完整 prompt，直接喂给编程 Agent。

### 核心亮点

#### 1. 可视化思考，告别文字地狱

传统方式：脑子里的想法 → 组织语言 → 打字描述 → AI 理解

BonsAI 方式：脑子里的想法 → 画布卡片/画图 → AI 直接读取

这个转变带来的效率提升是巨大的。以前用文字描述一个中等复杂度的需求大概需要 5-10 分钟，现在在画布上拖拖拽拽、写写画画，2-3 分钟就能搞定，而且 AI 理解得更准确。

#### 2. 实时 Agent 交互，边想边做

画布上的 Agent 不是摆设，它能实时读取你的画布内容。你画一条线，它能理解这是两个模块之间的连接；你写一段文字，它能分析这段文字的含义。

你在画布上列出几个待办事项，然后让 Agent 帮你逐个实现。实现过程中，Agent 还会把代码片段、进度更新写回到画布上，形成一个完整的工作流闭环。

#### 3. 一键导出完整 Prompt，无缝对接编程 Agent

这应该是 BonsAI 最贴心的功能了。当你觉得想法整理得差不多了，点击一下就能生成一份完整的、自带上下文的 prompt。这份 prompt 会包含：

- 画布上的所有卡片内容
- 卡片之间的关系和连接
- 你的任何备注和说明
- 甚至包括你画的图形描述

直接复制到 Claude Code、Codex 或 OpenCode 里，AI 就能立刻理解你的完整需求，不需要你再补充任何信息。

### 连接器功能

BonsAI 还有一个很强大的连接器功能，支持连接 Notion、GitHub、Figma 等平台。

你可以：

- 直接把 Notion 里的文档放到画布上
- 把 GitHub 的 issue 拉进来作为待办卡片
- 把 Figma 的设计稿贴到画布上做参考
- 甚至可以导入本地文件

整个流程一气呵成，中间不需要切换任何应用。

### 这个思路很重要

在 AI 编程工具层出不穷的今天，很多工具都在比拼模型能力、代码生成质量，但很少有人关注"人如何把想法传递给 AI"这个环节。

实际上，这个环节才是效率瓶颈所在。一个好的 prompt 决定了 AI 的输出质量，但写好 prompt 本身就是一项技能。

很多开发者花了大量时间学习如何跟 AI 沟通，却忽略了更根本的问题——我们的大脑本来就不是为线性文字输入设计的。

BonsAI 的聪明之处在于，它没有试图教会用户如何更好地打字，而是提供了一个更自然的表达方式：画布。

画布是一种空间化的媒介，它允许我们：

- 并行展示多个想法
- 直观表达想法之间的关系
- 快速迭代和调整
- 可视化复杂的结构

这些都是线性文字无法做到的。

### 写在最后

BonsAI 不是一个复杂的工具，它做的事情很简单：让你用更自然的方式表达想法，然后让 AI 更好地理解你。

它不跟 Claude Code、Cursor 这些工具竞争，而是成为它们的「前置处理器」，帮你把想法整理好，再交给专业的编程工具去执行。
