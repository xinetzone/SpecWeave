# 火山方舟协作奖励计划学习分析 - Product Requirement Document

## Overview
- **Summary**: 本文分析火山引擎方舟（Volcengine Ark）平台的"协作奖励计划"（rewardPlan）产品功能。协作奖励计划是火山方舟推出的"数据换免费额度"商业模式：用户授权火山引擎采集其在指定模型推理接入点的调用数据（经匿名化加密处理后用于模型优化），即可获得冷启动资源包和每日等量免费Tokens奖励。核心功能包括：参与授权、撤回授权（取消链接）、权益分级（个人/企业）、奖励自动发放、数据安全保障等。活动合作期延长至2026年7月31日，每日每模型最高采集500万Tokens并返还等量免费资源包。
- **Purpose**: 通过系统性学习与深度洞察分析，全面理解协作奖励计划页面的功能结构、业务逻辑、交互流程（特别关注"授权/撤回授权/取消链接"操作）、商业模式设计及数据合规机制，提取关键信息并形成结构化学习笔记与洞察报告，为理解大模型平台的用户增长策略、数据飞轮机制、合规设计、增长黑客模式提供参考。
- **Target Users**: 产品经理、增长运营、大模型平台设计者、数据合规人员、商业模式研究者、火山方舟用户、AI应用开发者

## Goals
- 完整梳理协作奖励计划的产品定位与商业模式
- 系统解析页面功能结构与交互流程
- 重点分析"授权"与"撤回授权（取消链接）"的完整操作链路
- 解析奖励机制设计（冷启动包+每日奖励包）
- 分析个人/企业权益分级体系
- 解析数据安全保障与合规设计
- 分析支持的模型范围与API调用方式
- 洞察"数据飞轮"商业模式的设计逻辑
- 梳理活动规则与法律协议要点
- 总结对大模型平台用户增长与数据获取的启示

## Non-Goals (Out of Scope)
- 不进行实际参与/授权/撤回授权操作测试
- 不对奖励额度具体数值进行分析（以页面公示为准）
- 不进行法律合规性的专业判断
- 不与其他云厂商类似计划做深度对比
- 不分析活动ROI或商业效益
- 不涉及具体接入点配置的技术细节
- 不分析反作弊系统的技术实现

## Background & Context
- **产品来源**: 火山引擎方舟大模型服务平台 - 开通管理模块
- **页面定位**: 用户增长活动 + 数据采集授权入口
- **活动状态**: 第二期（合作期延长至2026年7月31日）
- **入口路径**: 
  1. 控制台-开通管理页 → "立即参与"按钮
  2. 模型广场 → 模型详情页 → "火山方舟模型奖励计划"入口（当前支持Doubao-Seed-Evolving）
- **行业背景**:
  - 大模型训练需要大量高质量真实场景数据
  - 开发者对API调用成本敏感，免费额度是强吸引力
  - "数据换服务"是AI平台常见的数据获取模式
  - 数据合规与用户信任是此类计划的核心挑战
  - 数据飞轮效应：更多用户→更多数据→更好模型→更多用户
