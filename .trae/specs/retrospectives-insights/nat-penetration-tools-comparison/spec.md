# 神卓互联、cpolar、花生壳三款内网穿透工具对比分析 - Product Requirement Document

## Overview
- **Summary**: 对神卓互联（shenzhuohl.com）、cpolar（cpolar.com）、花生壳/贝锐（hsk.oray.com）三款主流国内内网穿透工具进行全面对比分析，从功能特性、性能表现、易用性、安全性、价格策略、技术支持服务质量六个核心维度展开，基于2026年最新官方信息和公开资料，形成客观公正的对比分析报告Wiki，帮助个人用户、小型企业、大型企业等不同场景用户选择最适合的内网穿透解决方案。
- **Purpose**: 通过系统性的产品横向对比，深入理解国内内网穿透SaaS市场的产品分层、定价策略、技术路线差异，为远程访问、内网服务发布、NAS外网访问、企业ERP远程办公等场景的技术选型提供专业、客观、数据驱动的决策参考。
- **Target Users**: 个人开发者、NAS用户、中小企业IT管理员、企业IT决策者、系统运维工程师、远程办公方案选型人员。

## Goals
- 收集并核实三款工具2026年最新的官方定价、功能特性、带宽限制、并发数等核心参数
- 从六个关键维度（功能特性、性能表现、易用性、安全性、价格策略、技术支持）进行结构化对比
- 分析各产品的目标用户定位、核心优势与局限性
- 针对个人使用、小型企业、大型企业/政企三类典型场景提供明确的选型建议
- 形成符合项目知识库规范的对比分析Wiki文档，包含YAML frontmatter
- 在相关产品系列索引中更新关联（如适用）

## Non-Goals (Out of Scope)
- 不涉及实际产品的安装配置或代码开发
- 不进行三款产品的实际性能测试（基于公开资料和官方参数分析）
- 不包含frp、ngrok、rathole、ZeroTier、Tailscale等其他内网穿透/组网工具的深度对比（仅在必要时作为参照提及）
- 不提供具体的商业采购建议或议价指导
- 不涉及三款产品的非公开内部数据或商业机密
- 不开发可运行的软件产品或自动化工具

## Background & Context
- 项目知识库中已有向日葵远程控制系列产品Wiki（Sunlogin，贝锐旗下另一产品线），贝锐花生壳与向日葵同属上海贝锐科技，产品定位有协同性
- 用户近期持续关注远程连接/内网穿透领域产品，先询问了内网穿透工具推荐，后追问神卓互联详情，最终提出三款工具对比需求
- 内网穿透（NAT Traversal）是远程办公、IoT设备管理、私有云访问的核心基础能力，国内市场已形成从免费个人版到企业级专线的完整产品分层
- 三款产品定位差异明显：
  - **花生壳（贝锐）**：国内老牌厂商（2006年成立），DDNS+内网穿透双核心，个人/企业全覆盖，产品线最完整（含硬件盒子、SD-WAN组网蒲公英、远程控制向日葵三大产品矩阵）
  - **cpolar**：国内团队开发的轻量化穿透工具，开发者友好，支持Docker/容器化部署，P2P+云中继混合架构，定价亲民，开发调试场景口碑好
  - **神卓互联**：专注企业级市场，硬件盒子+SD-WAN组网+内网穿透全方案，强调稳定性和SLA保障，NAS和视频监控场景有专门优化
- 已通过WebSearch收集到三款产品2026年最新的官方定价页面、功能说明文档等公开信息

## Functional Requirements
- **FR-1**: 整理三款工具的基本信息（公司背景、成立时间、产品定位、核心技术路线）
- **FR-2**: 功能特性对比：端口映射类型（HTTP/HTTPS/TCP/UDP/Socks5等）、带宽限制（各套餐带宽/流量）、并发连接数、隧道/映射数量、协议支持、DDNS支持、P2P直连支持、自定义域名、固定域名/端口、HTTPS证书自动部署等
- **FR-3**: 性能表现对比：连接稳定性（SLA保障、容灾链路、服务器节点分布）、数据传输速度（带宽上限、实际传输速率参考、BGP线路质量）、延迟表现、重连等待时长、服务器集群规模
- **FR-4**: 易用性对比：客户端支持平台（Windows/Mac/Linux/NAS/路由器/Docker/硬件盒子）、界面友好度（Web UI/桌面客户端/移动端APP）、配置复杂度（零配置/图形化/命令行）、文档完善度、上手门槛
- **FR-5**: 安全性对比：加密方式（TLS版本、AES加密强度）、访问控制机制（IP白名单/黑名单、访问密码、时间/区域/浏览器/系统限制）、认证机制、数据隐私保护、安全防护配件、等保合规支持
- **FR-6**: 价格策略对比：免费版限制（带宽/隧道数/流量/域名稳定性）、各付费套餐价格（个人版/基础版/专业版/商业版/企业版/旗舰版/臻享版等）、硬件产品定价、续费政策、优惠活动、增值服务价格
- **FR-7**: 技术支持对比：客服渠道（工单/在线客服/电话/400专线）、支持时间（5×8/7×24）、SLA故障响应时间、专属技术专家、文档/教程/社区资源、1V1顾问服务
- **FR-8**: 产品矩阵与生态对比：配套硬件（穿透盒子/组网盒子）、关联产品（SD-WAN组网/远程控制）、NAS专属套件（群晖/威联通/飞牛）、场景化解决方案（视频监控/ERP远程办公/游戏联机）
- **FR-9**: 优劣势总结：各产品核心优势、主要不足、适用人群
- **FR-10**: 场景化选型建议：个人开发调试、家庭NAS外网访问、小微企业远程办公、中型企业OA/ERP发布、大型企业/政企高安全场景、视频监控集中传输、游戏联机等场景的明确推荐
- **FR-11**: 生成符合docs/knowledge/learning/规范的Markdown格式Wiki文档，包含正确YAML frontmatter
- **FR-12**: 更新知识库相关索引文件（如07-vendor-product-learning下的comparison目录索引或产品系列索引）

## Non-Functional Requirements
- **NFR-1**: 数据准确，所有核心参数来自2026年官方公开信息，标注来源或时间节点，避免过时信息
- **NFR-2**: 对比客观中立，不偏向任何厂商，优势和不足都如实呈现
- **NFR-3**: 结构清晰，使用大量表格进行多产品横向对比，便于快速查阅
- **NFR-4**: 维度全面，覆盖用户明确要求的六个核心维度，补充产品生态和场景建议
- **NFR-5**: 语言使用标准现代汉语，专业术语准确，表述专业易懂
- **NFR-6**: 文档格式符合项目现有知识库规范（参考现有Wiki的frontmatter格式、章节结构、表格风格、链接格式）
- **NFR-7**: 文件命名使用kebab-case纯英文，符合项目文件名规范
- **NFR-8**: 选型建议明确可落地，不同场景直接给出推荐产品和版本，不模棱两可

## Constraints
- **Technical**: 基于公开可访问的官网信息、帮助文档、定价页面进行分析，不进行实际性能测试或压力测试；无法获取非公开的SLA细节或内部运营数据
- **Business**: 分析客观中立，不涉及商业诋毁或虚假宣传；不提供投资建议
- **Dependencies**: web搜索工具、现有知识库中的贝锐/向日葵系列Wiki作为参考、consulting-analysis技能（如适用）

## Assumptions
- 三款产品的官方网站可公开访问，定价页面和功能说明为最新有效信息
- 公开渠道（官网、帮助中心、官方文档、代理商页面）提供的信息足够进行全面对比
- 输出为Markdown格式Wiki文档，放置在docs/knowledge/learning/07-vendor-product-learning/目录下，可考虑新建comparison/子目录或沿用现有结构
- 用户期望得到一份实用的选型参考，而非纯学术性比较

## Acceptance Criteria

### AC-1: 三款产品核心参数收集完整准确
- **Given**: 已通过WebSearch获取三款产品官方信息
- **When**: 整理产品基本信息和核心参数
- **Then**: 覆盖所有套餐的定价、带宽、映射数、并发数、主要功能点，数据与官网一致，标注信息来源时间
- **Verification**: `human-judgment`
- **Notes**: 重点核对2026年最新价格调整（如花生壳2026年5月带宽免费升级活动）

### AC-2: 功能特性对比维度全面
- **Given**: 已收集功能参数
- **When**: 进行功能特性对比
- **Then**: 对比表格涵盖协议支持、带宽/流量限制、隧道/映射数量、并发连接数、域名策略、P2P支持、HTTPS支持、DDNS、访问控制等至少10项核心功能点
- **Verification**: `human-judgment`

### AC-3: 性能表现对比数据具体
- **Given**: 已收集性能相关参数
- **When**: 进行性能对比
- **Then**: 包含SLA等级、服务器节点地域分布、带宽上限、重连时长、服务器集群规模等可量化指标的对比
- **Verification**: `human-judgment`

### AC-4: 易用性对比贴近实际使用体验
- **Given**: 已收集平台支持和配置相关信息
- **When**: 进行易用性评估
- **Then**: 清晰列出各平台支持情况，客观评估配置复杂度和上手门槛，对新手友好度给出明确评价
- **Verification**: `human-judgment`

### AC-5: 安全性对比机制清晰
- **Given**: 已收集安全相关功能信息
- **When**: 进行安全性对比
- **Then**: 涵盖加密方式、访问控制粒度、认证机制、安全配件、合规支持等维度，明确各产品安全能力差异
- **Verification**: `human-judgment`

### AC-6: 价格策略对比完整透明
- **Given**: 已收集定价信息
- **When**: 进行价格对比
- **Then**: 从免费版到最高级企业版的全系列套餐对比表格，包含价格、带宽、映射数、并发数、核心权益；同时列出主要硬件产品价格
- **Verification**: `programmatic`
- **Notes**: 价格单位统一为人民币元/年，标注活动优惠

### AC-7: 技术支持对比服务等级明确
- **Given**: 已收集服务支持信息
- **When**: 进行技术支持对比
- **Then**: 清晰列出各套餐对应的客服渠道、支持时间、响应时间、专属服务等差异
- **Verification**: `human-judgment`

### AC-8: 优劣势分析客观公正
- **Given**: 已完成六大维度对比
- **When**: 总结优劣势
- **Then**: 每款产品至少列出3项核心优势和2项主要不足，评价客观有依据，不夸大不贬低
- **Verification**: `human-judgment`

### AC-9: 场景选型建议清晰可落地
- **Given**: 已完成优劣势分析
- **When**: 提供选型建议
- **Then**: 针对至少6种典型场景（个人调试、NAS访问、小微企业、中型企业、大型政企、视频监控）给出明确推荐产品和推荐版本，并说明理由
- **Verification**: `human-judgment`

### AC-10: Wiki文档格式合规
- **Given**: 分析内容已完成
- **When**: 生成Wiki文档
- **Then**: 文档包含正确YAML frontmatter（id、title、source、日期、标签等），章节结构清晰，表格格式规范，链接格式正确，文件命名为kebab-case纯英文，放置在正确目录
- **Verification**: `programmatic`
- **Notes**: 文件名建议：nat-penetration-tools-comparison-wiki.md；需运行文件名规范检查脚本

### AC-11: 知识库索引更新
- **Given**: Wiki文档已创建
- **When**: 更新相关索引
- **Then**: 在合适的索引文件（如07-vendor-product-learning目录下的索引或CATEGORIES.md）中添加新Wiki的条目
- **Verification**: `programmatic`

### AC-12: 报告整体质量专业
- **Given**: 完整Wiki文档已生成
- **When**: 审阅最终报告
- **Then**: 报告逻辑清晰、数据翔实、对比直观、建议实用，达到专业IT选型参考报告水准
