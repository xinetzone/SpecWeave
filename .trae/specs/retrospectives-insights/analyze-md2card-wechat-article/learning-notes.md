## 四、12 个 Q&A 要点整理

### Q1：自我介绍
- 本职是前端工程师，在 AI 协助下成为全栈工程师
- 主要技术栈：NextJS + TailwindCSS
- 前作：MDXNotes——跨平台笔记软件 + 公众号排版编辑器，这是 md2card 的开发基础
- 核心复用：Markdown 编译部分的代码能力
- 开发路径：有基础产品 + 新想法 → 尝试将 Markdown 转图片做成产品

### Q2：md2card 是什么？一句话介绍
- 定位：把 Markdown 一键变成小红书 / 公众号 / 抖音可用的知识卡片工具
- 定价：普通用户完全免费，API 付费
- 核心功能：一键将 Markdown 转小红书、公众号、朋友圈等可用的知识卡片，支持自动拆分成多张卡片，整个过程只需 1 分钟
- 核心优势：不消耗 Token、产品无需登录、完全免费
- 差异化定位：相比于 Canva，是一款**以内容优先**的知识卡片工具——用户根据内容来做社交图片，而不是先选图片模板再想内容
- 网址：md2card.cn

### Q3：当初为什么想做这个产品？
- 背景：AI 大部分输出都是 Markdown 格式
- 痛点：部分 AI 的回答质量很高，用户需要分享到小红书、朋友圈等平台
- 机会：帮助用户快速分享 AI 产出的优质内容

### Q4：开发过程是怎样的？有遇到什么困难吗？
- MVP 开发周期：仅花了一个周末
- 冷启动：发布到 V2EX 后，迎来大量博主推荐
- 技术困难：
  - API 存图问题
  - 服务器资源动态扩缩容部署
- 现状：并发量再大也已解决，不是问题

### Q5：目前的数据表现怎么样？
- 月活：约 1.5 万
- 用户来源：
  - 大部分：用户自主访问
  - 小部分：AI 推荐、SEO 搜索
- 产品状态：稳定运行中

### Q6：你认为这款产品能成功的核心原因是什么？
- 时代契机：AI 的大爆发
- 成本痛点：当时 Token 比较贵
- 技术痛点：模型生图的文字输出不稳定
- 产品价值：正好帮助用户不花钱，并且能快速转成想要的图文卡片

### Q7：你是怎么获客的？有哪些增长方法？
- 获客方式：仅通过自媒体方式
- 渠道：小红书和公众号等社交平台
- 内容形式：发布一些使用技巧和视频
- 推广投入：没有做过其他付费类型的推广

### Q8：收入情况怎么样？
- 产品定位：大部分功能免费
- 收入现状：收入很少
- 收入来源：API 用户付费
- 收支情况：收入仅能 Cover 服务器成本

### Q9：分享一下你的开发工作流？
- 核心工具：使用 Codex 实现产品原型
- **Codex Image2Code 三步工作流**：
  1. **第一步**：通过 Codex Image2 实现一张设计稿
  2. **第二步**：根据设计稿生成 HTML（比纯让大模型直接写 HTML 效果好很多）
  3. **第三步**：让 Codex 根据 HTML 产生可交互组件，逐步应用到产品中

### Q10：接下来有什么计划？
- 未来方向：可能尝试开发其他产品
- 产品判断：md2card 可能不是一款成功的商业化产品
- 策略：可以基于 md2card 的用户基础快速推广一款新的产品

### Q11：给其他独立开发者一些建议？
- 核心建议：需要有真正的、真实的一批用户来体验产品
- 方法论：通过解决用户的痛点和需求来逐渐完善自己的产品
- 反面警示：不要完全独立一个人去判断方向，一旦方向错了，可能白白浪费几个月甚至更久的时间
- 关键词：**真实用户验证，避免闭门造车**

### Q12：给 md2card 一句广告词吧
- 广告词：**免费的 Markdown 转图文卡片工具**

## 六、关键细节清单

1. **开发速度**：MVP 仅用一个周末完成开发
2. **冷启动渠道**：V2EX 发布后获得大量博主推荐
3. **技术挑战**：API 存图问题 + 服务器资源动态扩缩容部署（已解决）
4. **AI 开发工作流**：Codex Image2Code 三步法——Image2 生成设计稿 → 生成 HTML → 生成可交互组件（设计稿先行比直接写 HTML 效果更好）
5. **产品零门槛**：免登录、完全免费、不消耗 Token
6. **技术栈**：NextJS + TailwindCSS
7. **技术复用**：基于前作 MDXNotes 的 Markdown 编译能力，不是从零开始
8. **获客策略**：自媒体内容营销（小红书/公众号发布使用技巧视频），零付费推广
9. **商业模式**：C 端免费获客 + API 付费，当前仅覆盖服务器成本
10. **独立开发者建议**：真实用户验证优先，避免闭门造车判断方向
11. **产品差异化**：内容优先（用户已有 Markdown 内容→一键生成卡片）vs Canva 模板优先（先选模板再想内容）
12. **产品网址**：md2card.cn
13. **用户来源构成**：用户自主访问为主，AI 推荐/SEO 为辅
14. **后续规划**：不深入迭代 md2card 商业化，而是基于其用户基础推广新产品
