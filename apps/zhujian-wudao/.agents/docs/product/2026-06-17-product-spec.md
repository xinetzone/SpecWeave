# §一、产品定位

## 1.1 产品定义

竹简悟道是一款基于帛书《老子》的智能问答工具，帮助用户在经典文本与当代生活之间建立深度连接。

产品不是知识库，不是问答机器人，而是一座桥梁——让两千年前的智慧在当代管理者的决策场景中重新发出光芒。

## 1.2 概念解缚

《老子》五千言，本质是一份**概念解缚手册**。

"道可道，非常道"——第一个"道"是概念，第二个"道"是命名。整部《老子》都在说一件事：**概念是工具，人是主人**。

绝大多数人读书，是被概念推着走——读到"无为"，就想去"无所事事"；读到"柔弱"，就以为要"懦弱退缩"。不是书错了，是读书的方式错了。

**竹简悟道的核心价值主张：帮人"看见"概念，而不是被概念支配。**

## 1.3 核心承诺

产品给用户的承诺，是四个字：**玄同**。

> 不可得而亲，不可得而疏，不可得而利，不可得而害，不可得而贵，不可得而贱。——《老子》第56章

这是《老子》最核心的修炼境界：超越亲疏、利害、贵贱的对立，回到事情的本然状态。

玄同不是平等主义，不是相对主义，不是"怎么都行"。玄同是**看清全局之后的精准回应**——该亲时亲，该疏时疏，但内心不受牵绊。

# §二、核心功能

## 2.1 竹简目录

产品入口是一份完整的**81章帛书《老子》原文**。

- 按帛书本顺序排列，不做人为分类
- 每章配有场景标签（可选）
- 支持快速跳转和检索

用户看到的是一份"干净的"原典，没有翻译，没有评注，只有点击后才展开的解读层。

## 2.2 问道对话

产品核心交互是**AI引导的问答对话**。

对话的设计原则：

1. **不做建议者**：不告诉用户"你应该怎么做"
2. **只做提问者**：用问题帮用户澄清自己的处境
3. **回到原典**：每个问题都与帛书原文形成对话关系

问道对话提供四条**体道路径**：

| 路径 | 入口标签 | 核心问题 |
|------|----------|----------|
| 虚静内观 | 收 | 我内心的真实状态是什么？ |
| 自然无为 | 守 | 我在强求什么？ |
| 柔弱不争 | 柔 | 我在争什么？ |
| 生活实践 | 行 | 这个决定如何落在当下？ |

## 2.3 原文赏读

每章提供三层解读：

1. **原文朗读**：竖排显示，配合古典背景音乐
2. **文字解读**：逐句疏通，兼顾字源与语境
3. **版本差异**：帛书版vs传世版的核心差异点

版本差异是产品的重要内容资产，帮助用户理解为什么帛书版比传世版更接近老子原意。

## 2.4 场景书签

产品内置**9个场景标签**：

- 且问且行（默认入口标签）
- 决策前
- 迷茫时
- 冲突中
- 变革期
- 日常修养
- 团队管理
- 个人成长
- 行无行（知行合一入口标签）

场景标签是**弱关联**：用户选择标签后，系统会优先推荐相关的章节和问题，但不强制绑定。用户始终可以自由探索。

# §三、交互与视觉设计

## 3.1 视觉风格

产品视觉遵循**古典雅致**的调性：

- **背景**：竹简卷轴肌理，淡淡的宣纸质感
- **字体**：竖排显示，使用衬线字体（正文）或楷体（标题）
- **印章标签**：使用红色印章风格标注场景标签
- **间距**：宽松留白，模拟竹简的呼吸感

视觉不是复古展览，而是**让古典氛围成为思考的容器**。

## 3.2 对话设计

问道对话的AI交互原则：

```
不给予答案
不做出评价
不引导特定方向

只提出问题
只澄清事实
只回到原典
```

具体实现为**苏格拉底式提问法**：

- 当用户表达一个判断时，AI追问"这个判断的依据是什么？"
- 当用户表达一个目标时，AI追问"这个目标是你的需要，还是别人的期待？"
- 当用户陷入两难时，AI追问"如果你不受任何限制，你会怎么选？"

## 3.3 用户旅程

用户进入产品的典型路径：

```
打开应用 → 看见81章竹简目录 → 点击"且问且行"标签 
→ 进入问道对话 → 描述当下处境 → AI提问 
→ 用户反思回应 → AI继续追问或回到原典 
→ 用户产生新的觉察 → 对话结束 → 用户自行决定行动
```

# §四、帛书内容体系

## 4.1 体道四法

产品将帛书《老子》的核心修炼方法归纳为**四法**：

### 虚静内观

> 致虚极，守静笃。万物并作，吾以观复。——第16章

核心动作：**收敛外求** → **澄明内心** → **体证本然**

适用于：焦虑弥漫、思绪纷乱、需要回到根本的时刻

### 自然无为

> 道法自然。——第25章  
> 为学日益，为道日损。——第48章

核心动作：**识别强求** → **放下控制** → **让事物自行演化**

适用于：过度用力、试图改变他人、执着于特定结果的时刻

### 柔弱不争

> 柔弱胜刚强。——第36章  
> 夫唯不争，故天下莫能与之争。——第22章

核心动作：**辨认争心** → **守柔处下** → **以退为进**

适用于：陷入竞争、感到被冒犯、想要证明自己的时刻

> 📌 柔弱不争已系统化为独立洞察：[洞察66：柔弱不争——体道四法"柔"与"退"的系统化操作手册](../insights/meta-layer/2026-06-17-insights-66-68.md)。洞察66整合洞察42（守柔处下基础概念）和洞察48（冲突场景操作步骤），补全三层递进（看见争心→放下争心→不争而胜）和体道链映射。至此体道四法全部完成七节操作手册。

### 生活实践

> 上士闻道，勤而行之。——第41章

核心动作：**将领悟落实在具体决定** → **在做事中检验** → **在反馈中深化**

适用于：需要做决策、想要将道理化为行动的日常时刻

## 4.2 体道链

产品提出**体道链**概念，作为衡量用户对《老子》理解深度的标尺：

```
名 → 反 → 有无 → 正言若反 → 无为 → 玄同
```

- **名**：能说出《老子》的概念（虚静、无为、柔弱、玄同）
- **反**：理解"反者道之动"——事物的对立面是动力来源
- **有无**：理解"有无相生"——看得见与看不见相互生成
- **正言若反**：理解真正的智慧表达像是反话（"柔弱胜刚强"）
- **无为**：不是不做事，而是不强求、不控制、让事物按其本性发生
- **玄同**：超越亲疏、利害、贵贱的对立，回到事情本然

体道链的每一层都不是"学会"的知识，而是需要反复在生活中验证的修炼。

## 4.3 虚静内观三层

产品对"虚静内观"进行了细化，提出**三层递进结构**：

### 第一层：收敛外求

- 标志信号：思绪纷乱、外求不止、停不下来
- AI回应策略：帮助用户识别"我在抓取什么"
- 典型问题："你现在的焦虑是从哪里来的？"

### 第二层：澄明内心

- 标志信号：外求的冲动减弱、内心的声音开始显现
- AI回应策略：陪伴用户倾听，等待清晰
- 典型问题："在这个纷乱的思绪之下，你真正想要的是什么？"

### 第三层：体证本然

- 标志信号：内心平静、洞察浮现、不再挣扎
- AI回应策略：将洞察带回原典，确认与老子教导的呼应
- 典型问题："这个看见，和老子说的哪个章节相呼应？"

三层不是线性过程，可以随时折返，但整体方向是从外到内、从动到静。

# §五、用户留存设计

## 5.1 留存核心逻辑

产品留存不靠功能堆砌，而靠**问题本身的生命力**。

《老子》提出的不是一次性问题，而是人一辈子反复面对的永恒课题：

- 我在强求什么？
- 我在争什么？
- 我内心的真实状态是什么？
- 这个决定如何落在当下？

用户会回来，因为他们自己会回来——不是被推送拉回，而是被自己的问题召唤回来。

## 5.2 留存机制

产品设计了三种轻量留存机制：

### 每日一问

每天早晨推送一条来自帛书原典的简短问题，用户可以在当天的任意时刻回答，也可以忽略。

> 📌 每日一问已从设计说明系统化为独立洞察：[洞察53：每日一问——从"有困惑才打开"到"每天都要用"的习惯核心机制](../insights/philosophy-layer/2026-06-17-insights-53.md)。洞察53涵盖哲学根基（"为道日损"的每日实践）、当前七道题目的四法覆盖分析（揭示生活实践的结构性缺失）、三阶段习惯养成机制（推送依赖→自驱而来→"成为自己"）、推送触达与Freemium分层设计、演进路线（补全生活实践入口→周主题轮转→AI回复智能化）以及战略位置分析。

问题示例：
- "今天，你在哪里看到了'柔弱胜刚强'的迹象？"
- "此刻，你内心有没有在争什么？"

### 场景触发

当用户在生活中遇到以下场景时，系统会推送相关章节提醒（弱提醒，可关闭）：

- 决策前（决策相关章节）
- 迷茫时（虚静内观相关章节）
- 冲突中（柔弱不争相关章节）

### 定期回顾

用户可以设置"周回顾"——每周固定时间，系统展示用户过去一周的问道路径（不展示内容，只展示路径），邀请用户做一次简短反思。

> 📌 定期回顾已系统化为独立洞察：[洞察67：定期回顾与流失预警——"为道日损"的留存哲学](../insights/meta-layer/2026-06-17-insights-66-68.md)。洞察67解决了反效率产品的留存悖论，设计了三层回顾（路径回顾→模式辨认→简短反思）和"不争之争"召回策略。

## 5.3 流失预警

产品通过对话深度判断用户状态：

- **活跃但浅层**：用户只浏览目录，从不进入对话 → 推送"今天想探索哪一章？"
- **沉默期**：用户超过7天未登录 → 推送"上周的问题，你现在有答案了吗？"
- **流失风险**：用户超过30天未登录 → 推送"竹简一直在这里，等你回来"

> 📌 流失预警的哲学论证和系统化设计见[洞察67](../insights/meta-layer/2026-06-17-insights-66-68.md)。核心原则：召回不是"争用户"，是"不争之争"——在用户需要时出现，不需要时沉默。成功标准是"召回后对话深度"而非"召回率"。

### 睡前静心

当用户在夜间时段（22:00-02:00）打开问道对话时，AI自动切换为"睡前静心"模式——第一句话从场景标签选择变为"今天有什么事还在脑子里转？"。

三层递进：归日（回顾一天的纷扰）→ 归根（找到纷扰的根）→ 归静（帛书原文+5秒留白+屏幕渐暗）。

> 📌 睡前静心已系统化为独立洞察：[洞察68：睡前静心——"归根曰静"的场景化设计](../insights/meta-layer/2026-06-17-insights-66-68.md)。与每日一问形成"早问晚归"的呼吸节奏——早问是"吸"（早服·第59章），晚归是"呼"（归根·第16章）。

# §六、版权与合规

## 6.1 帛书原文版权

帛书《老子》（马王堆汉墓出土）属于**文物文献**，不存在现代版权问题。

产品使用帛书原文，无需获取授权。

## 6.2 解读内容版权

产品的解读内容（原创解读、版本差异分析、体道链等）属于**原创内容**，受著作权法保护。

解读内容仅授权用户在个人学习场景下使用，不得用于商业目的或二次传播。

## 6.3 AI生成内容边界

产品中AI回答的内容，边界如下：

- **可生成**：基于原典的解读、个人理解层面的引导性问题
- **不可生成**：声称是"老子的原意"（无法考证）、带有确定性结论的判断（"你应该XXX"）

## 6.4 合规要求

产品需遵守：

- 《生成式人工智能服务管理暂行办法》
- 《互联网信息服务深度合成管理规定》
- 《老子》第56章"不可得而贵，不可得而贱"——不以权威自居，不做价值判断

# §七、商业模式

## 7.1 商业模式框架

产品采用**免费+增值**模式：

### 基础层（免费）

- 81章帛书原典浏览
- 每章有限的原文解读
- 基础的问道对话（有限次数）

### 增值层（订阅）

- 无限问道对话
- 完整版本差异分析
- 每日一问完整版
- 定期回顾功能
- 深度体道链追踪

### 定价策略

订阅定价参考知识付费类产品中等价位：

- 月度订阅：低于一杯咖啡的价格
- 年度订阅：约等于一顿饭的价格
- 强调：这不是成本定价，而是价值定价——用户获得的是决策质量的提升

## 7.2 用户价值主张

产品对用户的核心价值主张：

> **在关键决策时刻，给你一个不被概念绑架的空间。**

不是告诉你答案，而是帮你问出更好的问题。

## 7.3 竞争格局

市场上存在多种《老子》相关产品：

| 类型 | 代表产品 | 核心模式 | 与竹简悟道的差异 |
|------|----------|----------|------------------|
| 有声书/解读课 | 喜马拉雅等 | 内容消费 | 我们是问答工具，不是内容 |
| 知识库 | 各AI问答产品 | 检索式回答 | 我们不做答案，只做提问 |
| 冥想/正念 | Calm等 | 音频引导 | 我们的静心有文本依托 |

**差异化定位**：市场上没有真正以"概念解缚"为核心定位的《老子》产品。

## 7.4 市场风险与对冲

### 风险1：AI问答产品竞争

通用AI可能具备类似能力。

**对冲**：
- 我们专注《老子》原典，深度不可替代
- 我们的对话框架（不做建议者，只做提问者）是产品护城河
- 体道链等原创概念体系形成认知壁垒

### 风险2：用户认为"无用"

《老子》的道理太抽象，用户不知道怎么用。

**对冲**：
- 场景书签降低使用门槛
- 问道对话直接切入用户当下处境
- 强调"决策前""迷茫时""冲突中"等实用场景

# §八、技术架构

## 8.1 整体架构

产品采用**前台-后台分离**架构：

```
┌─────────────────────────────────────────────┐
│                   前台                        │
│  用户端应用（移动端/网页）                    │
│  - 竹简目录展示                              │
│  - 问道对话界面                              │
│  - 个人设置                                  │
└─────────────────────────────────────────────┘
                    ↓ API
┌─────────────────────────────────────────────┐
│                   后台                        │
│  - LLM调用层（对话引擎）                      │
│  - 知识库（《老子》原典+解读）                │
│  - 用户状态管理                              │
│  - 内容推荐                                  │
└─────────────────────────────────────────────┘
```

## 8.2 前台设计原则

前台采用**单点承载**原则：

- 核心交互只有一个入口：问道对话
- 目录浏览是辅助，不是重心
- 场景书签是弱关联，不做强制分流

## 8.3 后台LLM设计

对话引擎的设计要点：

### 系统提示词（System Prompt）

```
你是一位《老子》研究者，陪伴用户探索帛书《老子》的智慧。

你的角色不是给出答案，而是提出问题。
你不说"你应该怎么做"，而是说"你看到的这个情况，让我想到《老子》说的……你怎么看？"
你不做价值判断，而是帮助用户澄清自己的处境。

你始终回到原典，但不让原典成为新的权威。
你用提问推动反思，而不是用道理说服用户。
```

### 对话策略

- **第一轮**：了解用户处境（开放式问题）
- **第N轮**：追问核心（苏格拉底式提问）
- **必要时**：引回原典（"这让我想到第X章……"）
- **结束时**：不强留用户，让用户自己决定行动

### 知识库结构

```
原典层：81章帛书原文
解读层：每章的原创解读
差异层：帛书vs传世版的差异点
洞察层：66条产品洞察（体道链、玄同等）
场景层：9个场景标签的关联映射
```

## 8.4 数据安全

- 用户对话内容不用于模型训练（明确声明）
- 用户可随时删除自己的对话记录
- 对话内容加密存储
- 符合《个人信息保护法》要求

# §九、社会价值与公益

## 9.1 社会价值定位

竹简悟道的存在，不仅是一款产品，更是一种**文化实践**：

- 让《老子》从"知道"变成"用到"
- 让古典智慧在当代管理者的决策中发挥作用
- 让"概念解缚"成为更多人的思维习惯

### 面向群体

- 高校哲学/中文专业学生（免费年度订阅）
