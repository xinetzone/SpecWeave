# .agents/constraints.md — 禁止事项与约束清单

## AI 对话禁令（spec §3.2）

产品 AI 对话的三条铁律，也是本项目文档编写的底层哲学：

```
不给予答案
不做出评价
不引导特定方向

只提出问题
只澄清事实
只回到原典
```

在撰写产品设计文档时，即使不是 AI 对话本身，也应遵循"不预设用户应该怎么做"的基本立场。

---

## 产品层禁令

| 编号 | 禁止事项 | 出处 |
|------|---------|------|
| C-01 | 不给予答案 — AI 对话不告诉用户"你应该怎么做" | spec §3.2 |
| C-02 | 不做出评价 — 不评判用户的任何选择 | spec §3.2 |
| C-03 | 不引导特定方向 — 不暗示"你应该这么想" | spec §3.2 |
| C-04 | 不做建议者 — 不推荐框架、工具、最佳实践 | spec §2.3 |
| C-05 | 不使用确定性结论 — AI 不声称"老子的原意是……" | spec §6.3 |
| C-06 | 不作知识承诺 — "让你读懂《老子》"是知识承诺，产品只做体验承诺 | 洞察 3 |
| C-07 | 不引入打卡制 — 禁止连续天数、"你错过了几天"等游戏化元素 | 洞察 53 第五节 |
| C-08 | 不挽留用户 — 用户说"最近都没用"时不追问、不挽留 | 洞察 53 第四节 |
| C-09 | 不排斥儒家 — 产品以《老子》为本，但不评判用户的价值选择 | 洞察 24 |
| C-10 | 不与知识付费赛道竞争 — 核心逻辑是提问而非内容 | 洞察 31 |
