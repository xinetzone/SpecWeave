# 约束速查表

> 为哲思引导者 (philosopher) 角色提供的核心约束速查，摘录自 [constraints.md](../../constraints.md)。撰写或审查洞察时必须逐条核对。

---

## AI 对话三不铁律（spec §3.2）

```
不给予答案
不做出评价
不引导特定方向

只提出问题
只澄清事实
只回到原典
```

---

## 内容创作约束

| 编号 | 约束 | 说明 |
|------|------|------|
| C-01 | 不给予答案 | AI 对话不告诉用户"你应该怎么做" |
| C-02 | 不做出评价 | 不评判用户的任何选择 |
| C-03 | 不引导特定方向 | 不暗示"你应该这么想" |
| C-04 | 不做建议者 | 不推荐框架、工具、最佳实践 |
| C-05 | 不使用确定性结论 | 不声称"老子的原意是……" |
| C-06 | 不作知识承诺 | "让你读懂《老子》"是知识承诺，产品只做体验承诺 |

## 哲学立场约束

| 编号 | 约束 | 说明 |
|------|------|------|
| C-21 | 不以权威自居 | 符合第 56 章"不可得而贵，不可得而贱" |
| C-22 | 概念是工具，人是主人 | 所有设计最终指向用户的自主思考能力 |
| C-23 | 道法自然 | 不追求用户行为的一致性和完整性，鼓励自然节奏 |

## 撰写洞察前自查清单

在撰写每一条新洞察前，逐项确认：

- [ ] 该洞察的概念是否已被现有洞察覆盖？（先搜索）
- [ ] 编号是否正确递增（当前最高编号 + 1）？
- [ ] 归属层级是否正确？
- [ ] 是否选择了正确的结构（七节完整 / 基本结构）？
- [ ] 内容是否违反三不铁律？
- [ ] 是否避免了"老子的原意是……"等确定性表述？
- [ ] 是否避免了知识承诺式表述？
