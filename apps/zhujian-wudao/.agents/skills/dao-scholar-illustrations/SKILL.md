# 道德经学者正文配图

## 核心定位

为竹简悟道（及其他道家哲学）文章设计和生成 16:9 横版正文配图。目标不是做国风插画、国学课堂配图或传统文化海报，而是把文章里的哲学判断、过程、结构、状态或隐喻，变成一张极简、安静、深远、可读但不说明书的手绘哲学图。

默认视觉 IP 是「道德经学者」：极简手绘的古代学者轮廓，宽袖长袍，面部留白或仅有垂目墨点，安静、沉默、专注。他不是一个具体的历史人物，而是人在天地之间面对大道保持沉默追问的姿态凝结。学者必须参与画面的核心哲学隐喻——观道、解缚、微明、体道、无为——不能只是站在旁边当装饰。

## 先读这些参考

按任务需要读取，不要一次塞满上下文：

- `references/style-dna.md`：风格 DNA——极简手绘、墨色线条、朱砂红与石青、留白美学、禁忌。
- `references/scholar-ip.md`：道德经学者 IP 的形象、性格、常见姿态和禁忌。
- `references/composition-patterns.md`：九种构图类型（观道/解缚/微明/四法/体道链/三层递进/无为/玄同/行无行）、原创隐喻方法。
- `references/prompt-template.md`：单张生图提示词模板。
- `references/qa-checklist.md`：生成后检查和迭代规则。
- `assets/examples/`：只作低频视觉校准，不进入默认生成路径。不要照抄这些案例的构图、物件或标注。

## 工作流

### 1. 消化正文

先读用户给的正文、链接、Markdown 文件或截图内容。提炼：

- 核心哲学观点是什么
- 哪些段落承担认知转折
- 哪些内容适合用图表达——抽象的哲学概念、结构、过程
- 哪些地方只适合文字，不需要图

不要平均配图。优先选择「哲学锚点」，例如：核心判断、概念解缚过程、体道四法的路径分岔、三层递进、微明过程、玄同状态。

### 2. 先出配图策略

如果用户只是说"分析怎么配图 / 思考哪些地方需要配图"，先给 shot list。每张图写清楚：

- 放在哪个段落后
- 图的主题
- 核心哲学意味
- 结构类型（九种构图类型择一）
- 学者在图里做什么（具体姿态）
- 建议自然意象（水/山/云/竹/石/溪/树/谷）
- 建议中文标注词（3-5 个）

默认 4-8 张。文章很短时 1-3 张；长文也不要轻易超过 9 张。够用就好，留白比填满重要。

### 3. 单张生成

如果用户明确要求"生成 / 输出 / 做图 / 帮我生成"，不要停下来等确认；用内置 `image_gen` 每张单独生成。不要把多张图拼在一张里。

每张图只讲一个核心哲学结构。提示词必须包含：

- 16:9 横版中文哲学正文配图
- 纯白背景
- 极简黑色手绘线稿（竹简刻写质感的线条）
- 少量朱砂红/石青中文手写批注
- 大量留白（至少 40%）
- 道德经学者作为核心哲学动作主体
- 禁止国画、PPT、宗教符号、具体历史人物、商业插画

不要复刻过往案例。案例只提供风格密度和学者参与方式，不能直接复用已有构图，除非用户明确要求复刻某张图。每次都要从当前文章重新发明一个基于自然意象的隐喻。

### 4. 检查与迭代

生成后检查 `references/qa-checklist.md`。如果出现以下问题，优先重生成或局部编辑：

- 学者只是装饰
- 画面太满（空白不足 40%）
- 太像国画/PPT/国学课堂
- 出现了道教符号
- 中文太多或错字严重
- 左上角出现标题
- 画风太商业、太精致、太空洞

### 5. 保存交付

如果用户在 workspace 内工作，把最终图复制到：

```text
assets/<article-slug>-illustrations/
```

按顺序命名：

```text
01-topic-name.png
02-topic-name.png
```

保留原始生成文件，不要覆盖已有资产，除非用户明确要求替换。

## 输出口径

生成前的策略输出要短而准。生成后的交付要包含：

- 生成了几张
- 每张图的用途
- 保存路径
- 哪些图最稳，哪些图是可选

不要长篇解释风格理论；让图自己说话。好图的标准：空白很多，学者很小，线条很简，看完后心里有一个东西停在那里。
