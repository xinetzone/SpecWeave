# 约束守护

> 洞察撰写时的必检约束清单。摘录自 [constraints.md](../../../constraints.md)。撰写和审查每条洞察时，必须逐条核对。

---

## 三不铁律（spec §3.2）

```
不给予答案
不做出评价
不引导特定方向

只提出问题
只澄清事实
只回到原典
```

## 必检约束项

### 内容创作约束

| 编号 | 约束 | 违规示例 |
|------|------|---------|
| C-01 | 不给予答案 | "所以你应该放下执念" |
| C-02 | 不做出评价 | "这种想法是不对的" |
| C-03 | 不引导特定方向 | "你可以试着从另一个角度想想" |
| C-04 | 不做建议者 | "建议每天冥想 10 分钟" |
| C-05 | 不使用确定性结论 | "老子的原意是不要执着" |
| C-06 | 不作知识承诺 | "读完这条洞察，你就能明白无为的真谛" |

### 文档编写约束

| 编号 | 约束 | 说明 |
|------|------|------|
| C-11 | 洞察不能跳号或插号 | 编号必须全局递增 |
| C-12 | 洞察标题不能同名 | 已有标题不可复用 |
| C-13 | 洞察内容不能高度重复 | 搜索现有洞察确认概念未被覆盖 |

### 哲学立场约束

| 编号 | 约束 | 说明 |
|------|------|------|
| C-21 | 不以权威自居 | 符合第 56 章"不可得而贵，不可得而贱" |
| C-22 | 概念是工具，人是主人 | 所有设计最终指向用户的自主思考能力 |
| C-23 | 道法自然 | 不追求用户行为的一致性和完整性 |

## 撰写前自查清单

- [ ] 该洞察的概念是否已被现有洞察覆盖？（先搜索）
- [ ] 编号是否正确递增（当前最高编号 + 1）？
- [ ] 归属层级是否正确？
- [ ] 结构选择是否适当（七节完整 / 基本结构）？
- [ ] 内容是否违反三不铁律？
- [ ] 是否避免了"老子的原意是……"等确定性表述？
- [ ] 是否避免了知识承诺式表述？
- [ ] 标题是否与已有洞察同名？
- [ ] 帛书章号和原文引用是否准确？

## 撰写后自查清单

- [ ] 统计声明是否已更新（洞察数 + 1）？
- [ ] 相关洞察是否已添加反向交叉引用？
- [ ] 复盘报告是否已同步更新？
- [ ] 所有引用路径是否使用相对路径？
- [ ] project.md 的洞察数表格是否已更新？

## 常见违规模式

| 模式 | 说明 | 修正方式 |
|------|------|---------|
| "这意味着" | 暗示确定性解释 | 改为"一种可能的理解是"或直接陈述原文 |
| "用户应该" | 给予答案/引导方向 | 改为"当用户面对……时，可自问……" |
| "正确的做法是" | 做出评价 | 改为"帛书第 X 章说……" |
| "建议" | 做建议者 | 改为引用原文或提问 |
| "老子告诉我们" | 确定性结论 | 改为"帛书第 X 章记载" |
