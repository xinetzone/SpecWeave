# 洞察结构参考

> 洞察撰写的两种标准结构、编号规则与层级归属判断。摘录自 [conventions.md](../../../conventions.md) 和 [project.md](../../../project.md)。

---

## 两种标准结构

### 七节完整操作手册（体道四法系统化洞察）

适用于体道四法（虚静内观、自然无为、柔弱不争、生活实践）相关洞察，必须为七节完整结构：

```markdown
## 洞察{N}：{标题}

**来源**：[§{章节}]

**核心内容**：

{简短摘要——用什么法门、解决什么概念陷阱、达到什么效果}

---

### 一、{第一子节}
### 二、{第二子节}
### 三、{第三子节}
### 四、{第四子节}
### 五、{第五子节}
### 六、{第六子节}
### 七、{第七子节}
```

### 基本结构（非系统化洞察）

适用于产品定义、架构决策、元层反思等非体道四法的洞察：

```markdown
## 洞察{N}：{标题}

**来源**：[§{章节}]

**核心内容**：

{简短摘要——这个洞察揭示什么、为什么重要}
```

## 编号规则

- 全局递增，从 1 开始，当前最高 **65**
- **不能跳号、插号**（C-11）
- 每条洞察的 Markdown 标题：`## 洞察{N}：{标题}`
- **标题不能同名**（C-12）
- 新增洞察前先搜索确认编号区间内无空缺

## 层级归属判断

| 层级 | 编号范围 | 说明 | 典型内容 |
|------|---------|------|---------|
| 产品层 | 1-15 | 直接指导产品设计 | 概念解缚、场景标签、每日一问 |
| 架构层 | 16-30 | 支撑产品体系的底层设计 | 双层架构、对话引擎、知识库结构 |
| 哲学层 | 31-54 | 帛书哲学概念的系统化展开 | 玄同、恒德、体道四法系统化 |
| 元层 | 55-65 | 跳出产品内容，分析洞察过程 | 洞察方法论、可迁移模式 |

## 交叉引用格式

所有跨文件引用必须使用**相对路径** + **行号定位**：

| 引用目标 | 格式 |
|----------|------|
| Spec 某节 | `[spec §5.2](../../docs/superpowers/specs/2026-06-17-zhujian-wudao-spec.md#L245-L263)` |
| 某条洞察 | `[洞察53](../../docs/superpowers/specs/2026-06-17-zhujian-wudao-insights-31-65.md#L1366-L1492)` |
| 约束条款 | `[C-05](../../constraints.md#L27)` |

## 统计更新

每次新增洞察后必须更新：

1. **洞察库头部**：更新「统计」行（洞察数）
2. **洞察库末尾**：如末尾有统计声明，同步更新
3. **project.md**：更新洞察库三层结构表格的洞察数
4. **复盘报告**：将新洞察添加到洞察列表
