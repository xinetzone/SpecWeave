# 提示词模板

> 不同场景的洞察撰写提示词模板。根据用户任务选择对应模板并替换变量。

---

## 模板一：撰写体道四法系统化洞察

适用场景：用户要求系统化展开某个体道四法概念。

```text
为竹简悟道项目撰写一条体道四法系统化洞察。

核心概念：{概念名称}
所属法门：{虚静内观/自然无为/柔弱不争/生活实践}
帛书根源：第 {章号} 章 "{原文}"

要求：
1. 使用七节完整操作手册结构
2. 编号为 {当前最高编号+1}，归属哲学层（31-54）
3. 每个子节聚焦「三层递进」中的一层：入门(第一层) → 深入(第二层) → 本然(第三层)
4. 以帛书原文为唯一权威来源，标明章号
5. 遵循三不铁律：不给予答案、不做出评价、不引导特定方向
6. 避免确定性结论（"老子的原意是……"）
7. 避免知识承诺式表述（"让你读懂……"）
8. 所有跨文件引用使用相对路径

相关洞察参考：{现有相关洞察编号列表}
```

## 模板二：撰写非系统化洞察

适用场景：产品定义、架构决策、元层反思等。

```text
为竹简悟道项目撰写一条非系统化洞察。

核心概念：{概念名称}
归属层级：{产品层1-15/架构层16-30/哲学层31-54/元层55-65}
来源：spec §{章节}

要求：
1. 使用「来源 + 核心内容」基本结构
2. 编号为 {当前最高编号+1}
3. 核心内容简洁明了，不超过 3 段
4. 遵循三不铁律
5. 避免确定性结论和知识承诺
6. 标明帛书章号来源（如适用）

相关洞察参考：{现有相关洞察编号列表}
```

## 模板三：分析概念是否需要新洞察

适用场景：用户提出一个概念，不确定是否需要撰写新洞察。

```text
分析这个概念是否需要在竹简悟道洞察库中新增一条洞察。

概念：{概念名称}
可能的归属层级：{层级}

请执行：
1. 搜索现有洞察库（洞察 1-65），确认该概念是否已被覆盖
2. 如已覆盖：指出被哪条洞察覆盖，解释为什么不需要重复
3. 如未覆盖：给出建议编号、建议结构、核心观点摘要
4. 如部分覆盖：指出已有洞察的缺口，建议新洞察的定位和切入点
```

## 模板四：审查已有洞察

适用场景：审查一条已有洞察是否符合规范。

```text
审查以下洞察条目是否符合竹简悟道规范。

洞察编号：{N}
所在文件：{路径}

请逐项检查：
1. 编号是否正确（无跳号、插号）
2. 标题是否与已有洞察同名
3. 内容是否违反三不铁律
4. 是否存在确定性结论（"老子的原意是……"）
5. 是否存在知识承诺式表述
6. 交叉引用是否正确（路径可达、行号准确）
7. 结构选择是否适当
8. 帛书引用是否准确

给出逐项检查结果和修正建议。
```
